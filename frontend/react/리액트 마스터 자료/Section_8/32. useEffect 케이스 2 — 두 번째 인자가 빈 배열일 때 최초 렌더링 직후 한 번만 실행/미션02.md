# 🟦 창 크기 감지기 만들기 (useEffect 빈 배열 + cleanup)

## 📝 설명

이번 미션에서는 `useEffect(() => {...}, [])` 패턴을 활용해
컴포넌트가 처음 화면에 나타날 때 한 번만 실행되고,
사라질 때 정리(cleanup) 되는 구조를 직접 구현합니다.

React에서는 DOM이 준비된 뒤 한 번만 동작해야 하는 로직이 종종 필요합니다.
대표적인 예로 브라우저 창 크기를 감지하는 이벤트 등록이 있습니다.
이벤트 리스너는 컴포넌트가 살아 있는 동안만 유지해야 하므로,
컴포넌트가 사라질 때 반드시 정리(cleanup) 단계에서 제거해야 합니다.

이번 미션에서는 다음을 실습합니다:

- `useEffect`의 두 번째 인자를 빈 배열(`[]`) 로 설정해 최초 한 번만 실행하기
- `window.addEventListener`로 이벤트 등록하기
- cleanup 함수에서 `removeEventListener`로 이벤트 제거하기

## 🖥️ 출력 결과 예시

```text
# 초기 로딩 시
📏 resize 이벤트 등록됨
현재 창 너비: 1536px

# 창 크기를 조절할 때마다
현재 창 너비: 1280px
현재 창 너비: 1024px
...

# 컴포넌트가 언마운트될 때 (예: 다른 페이지로 이동)
🧹 resize 이벤트 제거됨
```

## ✅ 요구사항

- `useState`를 사용해 `width` 상태를 관리해야 한다.
- `useEffect`의 두 번째 인자는 반드시 빈 배열(`[]`) 이어야 한다.
- `useEffect` 안에서 `window.addEventListener("resize", handler)`를 등록해야 한다.
- cleanup 함수에서 반드시 `removeEventListener`를 호출해야 한다.
- `setWidth(window.innerWidth)`를 통해 상태를 갱신해야 한다.
- 화면에는 항상 현재 창 너비(px) 가 표시되어야 한다.

## 🧱 Base Code

```jsx
import { useState, useEffect } from "react";

export default function WindowSizeTracker() {
  const [width, setWidth] = useState(window.innerWidth);

  useEffect(() => {
    // TODO 1: handleResize 함수 정의
    // TODO 2: window resize 이벤트 등록
    // TODO 3: cleanup에서 removeEventListener
  }, []);

  return <p>현재 창 너비: {width}px</p>;
}
```

## 📌 미션의 의의

이 미션은 `useEffect`의 빈 배열(`[]`) 패턴을 몸에 익히기 위한 실습입니다.
이 패턴은 "컴포넌트가 처음 나타날 때 딱 한 번 실행되고, 사라질 때 정리(cleanup)되는 구조"를 만듭니다.

이 방식은 다음과 같은 실제 상황에서 자주 사용됩니다.

- 화면 진입 시 한 번만 API 데이터를 불러오기
- 외부 라이브러리 초기화 및 인스턴스 생성
- `window`, `document` 이벤트 등록 및 해제
- `setInterval`, `setTimeout` 등 반복 타이머 관리

즉, "컴포넌트의 생명주기와 함께 시작하고 끝나는 동작" 을 다룰 때 필수 패턴입니다.
