# 🟦 렌더링마다 실행되는 useEffect + Cleanup 시각화

## 📝 설명

두 번째 인자(의존성 배열)를 생략한 `useEffect`를 실험하며,

모든 렌더링 직후 effect가 실행되고,
다음 effect가 실행되기 직전 cleanup이 먼저 호출되어 이전 동작(타이머 등)을 정리한다는 걸 콘솔 + 화면으로 시각화합니다.

## 🖥️ 출력 결과 예시

```text
초기 마운트 직후 콘솔에: Effect 실행: count = 0
1초마다 콘솔에: ⏱ 타이머 동작 중... count = 0
버튼 클릭으로 count가 1이 되면, 콘솔 순서:
🧹 Cleanup 실행 (count = 0)
Effect 실행: count = 1
이후 타이머 메시지도 count = 1 기준으로 갱신
```

화면 예시:

```text
현재 카운트: 1
최근 로그: 🟢 Effect 실행 (count = 1)
```

## 💡 구현에 필요한 개념

- `useEffect(() => { ... })` (no deps): 최초 마운트 + 모든 리렌더 이후 실행
- cleanup 함수: effect 콜백이 반환하는 함수. 다음 effect 실행 직전/언마운트 시 호출
- 타이머 관리 패턴: `setInterval` 시작 → cleanup에서 `clearInterval`로 정리
- 렌더 흐름: state 변경 → 렌더 → DOM 반영 → effect → (다시 렌더) → cleanup → 새 effect
