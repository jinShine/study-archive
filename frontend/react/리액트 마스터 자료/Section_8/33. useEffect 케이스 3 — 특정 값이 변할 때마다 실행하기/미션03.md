# 🟦 실시간 검색창 만들기: 의존성 배열 + 디바운스 + 요청 취소 [고난도]

## 📝 설명

검색창에 글자를 입력할 때마다 서버에 목록을 요청하면 네트워크가 과도하게 사용되고, 느린 응답이 뒤늦게 도착하면 오래된 결과가 최신 UI를 덮어쓰기도 합니다.
이번 미션에서는 다음을 구현해 안정적인 실시간 검색 UX를 완성합니다.

- `useEffect(..., [keyword, category, sort])` : 세 값 중 하나라도 바뀌면 재실행
- 디바운스(400ms) : 타이핑 도중엔 대기, 멈추면 한 번만 요청
- 요청 취소(`AbortController`) : 새 검색이 시작되면 이전 요청 취소 + cleanup
- 로딩/에러/결과 표시 상태를 예측 가능하게 관리

실제 `fetch` 대신 모의 API(`mockFetch`)로 동작을 재현합니다. 구조만 교체하면 실서비스에도 그대로 적용 가능합니다.

## ✅ 요구사항

- `keyword`, `category`, `sort`를 `useState`로 관리한다.
- `useEffect`의 의존성 배열에 `[keyword, category, sort]`를 넣는다.
- effect 내부에서 디바운스 타이머를 설정하고, cleanup에서 `clearTimeout` 한다.
- 네트워크 호출에 `AbortController`를 사용하고, cleanup에서 `controller.abort()` 한다.
- 로딩 중에는 `"로딩 중..."`, 실패 시에는 에러 메시지, 성공 시에는 결과 목록을 렌더링한다.
- 빈 검색어일 때는 요청하지 않고 `"검색어를 입력하세요"` 문구를 띄운다.

import { useEffect, useState } from "react";

// 모의 API: 400~700ms 후 결과 반환, Abort 신호를 존중
function mockFetch(query, { signal }) {
  return new Promise((resolve, reject) => {
    const delay = 400 + Math.floor(Math.random() * 300);
    const timer = setTimeout(() => {
      if (signal?.aborted) return; // 이미 abort된 경우 아무 것도 하지 않음
      const sample = Array.from({ length: 5 }, (_, i) => ({
        id: `${query.kw}-${query.cat}-${query.sort}-${i + 1}`,
        title: `[${query.cat}/${query.sort}] ${query.kw} 결과 ${i + 1}`,
      }));
      resolve(sample);
    }, delay);
    signal?.addEventListener("abort", () => {
      clearTimeout(timer);
      const err = new DOMException("Aborted", "AbortError");
      reject(err);
    });
  });
}

## 📌 미션의 의의

- 의존성 배열은 "이 값들이 바뀌면 effect를 다시 실행"이라는 명시적 계약입니다.
- cleanup 순서: 항상 "이전 effect의 cleanup → 새 effect 실행". 중복 요청/리스너 누적을 원천 차단합니다.
- 디바운스 + Abort는 실무 필수 콤보입니다. 타이핑 폭주에 대한 네트워크 보호, 오래된 응답의 레이스 컨디션 방지를 동시에 해결합니다.
