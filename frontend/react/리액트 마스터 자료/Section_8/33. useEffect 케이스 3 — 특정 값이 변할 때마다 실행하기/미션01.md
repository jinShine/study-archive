# 🟦 카운터 로그 기록기

## 📝 설명

이 미션에서는 버튼을 눌러 카운트를 증가시킬 때마다, 콘솔에 "현재 카운트"가 출력되도록 만들어 봅니다.
즉, `count` 값이 바뀔 때마다 `useEffect`가 다시 실행되어야 합니다.
이 코드를 통해 `useEffect`의 의존성 배열 `[count]` 이 어떻게 작동하는지를 체험합니다.

## ✅ 요구사항

- `count` 상태를 `useState(0)`으로 만든다.
- 버튼 클릭 시 `setCount(count + 1)`로 카운트를 증가시킨다.
- `useEffect`의 의존성 배열에 `[count]`를 넣고, `console.log`로 `"현재 카운트: {count}"`를 출력한다.
- cleanup 함수는 `"Effect 정리됨"`을 콘솔에 출력한다 (다음 effect 실행 직전 실행됨).

## 🧱 Base Code

```jsx
import { useState, useEffect } from "react";

export default function CounterLogger() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    // TODO: count가 바뀔 때마다 콘솔에 "현재 카운트: ..." 출력
    // TODO: cleanup에서 "Effect 정리됨" 출력
  }, [count]);

  return (
    <div>
      <h2>카운터 로그 연습</h2>
      <p>현재 카운트: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

## 📌 미션의 의의

- `useEffect(..., [count])`는 `count`가 바뀔 때마다 다시 실행됩니다.
- cleanup 함수는 다음 effect 실행 직전 실행됩니다.
- 이 구조를 이해하면 API 요청, 이벤트 등록 등 "상태 변화 반응형 로직"의 기본을 익힐 수 있습니다.
