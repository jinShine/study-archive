# 🟨 map과 filter로 리스트를 효율적으로 렌더링하기

## 📝 설명

배열을 화면에 그릴 때 React에서는 `map`으로 항목을 JSX 요소로 변환하고, 필요하면 `filter`로 먼저 추린 뒤 `map`을 적용합니다. 이 미션은 세 가지 축으로 연습합니다.

- **배열 → JSX 표준 패턴**
  - `{array.map(item => (<li key=...>{...}</li>))}`처럼 표현식을 즉시 반환합니다.
  - JSX의 `{}`는 "값을 만드는 표현식"만 허용합니다. 화살표 함수에서 블록(`{}`)을 쓰면 반드시 `return`을 명시해야 합니다.
- **필터링 + 변환 체이닝**
  - `filter`로 조건에 맞는 데이터만 새 배열로 만든 뒤, 이어서 `map`으로 JSX로 변환합니다.
  - 다중 조건(예: 가격 상한 + 이름 포함 검색)도 `filter` 콜백에서 논리식을 조합해 한 번에 처리할 수 있습니다.
- **안정적인 `key`와 빈 배열 처리**
  - 각 항목에는 고유하고 안정적인 `key`(가능하면 `id`)를 부여합니다.
  - 데이터가 비었을 때는 Empty UI(예: "표시할 항목이 없습니다")를 보여 줍니다.
  - `products?.filter(...)`처럼 옵셔널 체이닝 또는 `const safe = products ?? []`로 무결성을 확보한 뒤 순회합니다.

## 🖥️ 출력 결과 예시

```text
A. 과일 목록
사과
바나나
체리

(빈 배열일 때)
표시할 과일이 없습니다.

B. 2만원 미만 상품
Mouse — 15,000원

(조건 불일치 시)
조건에 맞는 상품이 없습니다.

C. 반환 규칙 & key 점검
"즉시 반환" 목록과 "블록 + return" 목록이 동일하게 렌더링됨
콘솔에 key 경고 없음

D. 다중 조건 필터(이름 포함 + 최대 가격)
예: query="o", maxPrice=20000 →
Mouse — 15,000원
조건 불일치 시 Empty UI
```

## 💡 구현에 필요한 개념

- JSX 중괄호 `{}`와 표현식
- `map`으로 형태 변환, `filter`로 조건 선별
- 고유한 `key` (가능하면 `id`, 불가피할 때만 `index`)
- 빈 배열/무결 데이터 처리: `?? []`, Empty UI
- 다중 조건 필터링: 문자열 포함(대소문자 무시), 숫자 범위, 논리식 결합
- 숫자 포맷팅: `toLocaleString()`
- 의미론적 마크업: `<ul>`/`<ol>` + `<li>` (필요 시 `aria-*`)
