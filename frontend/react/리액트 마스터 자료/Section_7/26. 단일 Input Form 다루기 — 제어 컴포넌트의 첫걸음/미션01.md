# 🟫 React가 직접 값을 관리한다! — 제어 컴포넌트로 단일 입력 제어하기

## 📝 설명

이번 미션에서는 React에서 단일 input을 제어 컴포넌트로 만드는 방법을 직접 구현해 봅니다.
HTML에서는 `<input>`의 값이 DOM이 스스로 관리하지만, React에서는 state가 값의 주인입니다.
즉 사용자의 입력이 state로 들어가고, 그 state가 다시 input의 `value`로 표시되는 순환 구조를 통해
UI와 데이터가 완전히 동기화됩니다.

이 구조를 Controlled Component(제어 컴포넌트)라고 부르며, React Form 처리의 출발점입니다.

## 🖥️ 출력 결과 예시

```text
# 초기 상태
[    ]  ← 이름 입력창 (비어 있음)
현재 입력값:

# "영희" 입력 후
[영희]
현재 입력값: 영희

# 제출 버튼 클릭 시
📢 alert: "제출된 이름: 영희"
```

## 💡 구현에 필요한 개념

- **제어 컴포넌트(Controlled Component)**
  - input의 `value`가 React의 state에 의해 완전히 제어됩니다.
  - 사용자의 입력 → `onChange` → `setState` → 새로운 `value` → UI 반영

- **양방향 바인딩(one-way + reflection)**
  - 사용자가 입력하면 state가 바뀌고, state가 다시 input의 `value`에 반영됩니다.
  - React는 이 순환을 통해 항상 최신 값을 유지합니다.

- **폼의 기본 동작 제어 (`preventDefault`)**
  - HTML의 form은 기본적으로 제출 시 새로고침을 유발합니다.
  - React에서는 SPA 환경이므로 `e.preventDefault()`로 이를 차단하고
  - JavaScript 로직으로 제출을 처리해야 합니다.

- **state를 통한 유효성 검사 및 제한**
  - 입력값을 React state에서 바로 확인 가능하므로
  - 글자 수 제한, 실시간 검증 등 다양한 로직을 손쉽게 적용할 수 있습니다.

## 🎯 미션 목표

- `useState`로 입력값을 관리하고, `onChange` 이벤트로 state를 갱신합니다.
- `value={state}`로 input을 React가 제어하도록 만듭니다.
- `onSubmit`에서 `preventDefault()`로 기본 동작을 막고, `alert`로 제출 결과를 표시합니다.
