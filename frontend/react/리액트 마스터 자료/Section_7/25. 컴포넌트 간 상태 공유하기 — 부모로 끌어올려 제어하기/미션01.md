# 🟫 형제 컴포넌트가 서로 통하는 비결 — 상태를 부모로 끌어올려 동기화시키기

## 📝 설명

이번 미션에서는 컴포넌트 간에 같은 데이터를 공유하는 구조를 직접 만들어 봅니다.
React의 데이터 흐름은 한 방향(부모 → 자식)으로만 흐르기 때문에, 형제 컴포넌트끼리 상태를 공유하려면 반드시 공통 부모로 끌어올려야 합니다.
이 패턴을 "Lifting State Up"이라고 부르며, React 상태 관리의 기초이자 확장성 있는 설계의 핵심입니다.

## 🖥️ 출력 결과 예시

```text
# 초기 상태
☐ 알림 받기
→ 알림을 받지 않습니다.

# 체크박스 클릭
☑ 알림 받기
→ 알림을 받습니다.
```

## 💡 구현에 필요한 개념

- **단방향 데이터 흐름**
  - React의 상태는 부모에서 자식으로만 내려갑니다.
  - 자식끼리는 직접 데이터를 주고받을 수 없습니다.

- **상태 끌어올리기 (Lifting State Up)**
  - 형제들이 같은 데이터를 써야 한다면, 그 데이터를 공통 부모로 이동시킵니다.
  - 부모가 상태를 "단일 소유자(single source of truth)"로 관리합니다.

- **제어 컴포넌트 (Controlled Component)**
  - 입력 요소(`<input>`)는 항상 `value`(또는 `checked`)와 `onChange`가 세트로 작동해야 합니다.
  - 부모의 state를 기반으로 값을 제어하면, 입력과 화면 표시가 정확히 일치합니다.

- **Props를 통한 상태 전달**
  - 부모 → 자식 : 현재 값 전달 (`checked={checked}`)
  - 자식 → 부모 : setter 함수 전달 (`onChange={() => setChecked(...)}`)

## 🎯 미션 목표

- 두 형제 컴포넌트(`Checkbox`, `Summary`)가 같은 상태를 공유해야 합니다.
- `checked` 상태를 부모로 끌어올리고, 두 자식은 props를 통해 동기화되어야 합니다.
- `Checkbox`를 클릭하면 `Summary`의 문구가 함께 변경되어야 합니다.
