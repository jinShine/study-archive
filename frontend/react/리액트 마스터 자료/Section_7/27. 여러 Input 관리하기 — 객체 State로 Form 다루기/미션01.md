# 🟫 회원가입 Form 만들기 — 여러 Input을 객체 State로 제어하기

## 📝 설명

이번 미션에서는 여러 개의 input을 하나의 객체 state로 묶어 관리하는 제어 컴포넌트를 직접 만들어 봅니다.
이 패턴은 회원가입, 로그인, 배송지 입력 등에서 매우 자주 등장합니다.

핵심 아이디어는 모든 필드를 하나의 객체(`user`)로 관리하고, 각 input에 `name` 속성을 부여하여
`handleChange`에서 어떤 필드가 바뀌었는지를 식별하는 것입니다.
그런 다음 전개 연산자(`...user`)로 기존 객체를 복사하고, `[name]: value` 구문으로 특정 필드만 덮어써서
항상 새로운 객체를 생성(불변성 유지) 하는 방식으로 state를 갱신합니다.

## 🖥️ 출력 결과 예시

```text
# 초기 화면
이름: [    ]
이메일: [    ]
나이: [    ]
[제출]
> 제출된 정보: , ,

# 입력 후
이름: [영희]
이메일: [young@example.com]
나이: [25]
[제출]
> 제출된 정보: 영희, young@example.com, 25
```

## 💡 구현에 필요한 개념

- **객체 기반 state 관리**

  ```jsx
  const [user, setUser] = useState({ name: "", email: "", age: "" });
  ```

  - 여러 필드를 하나로 묶은 state 구조

- **입력 구분용 `name` 속성**
  - 각 input에 `name="name"`, `name="email"`, `name="age"`를 지정하면
  - `e.target.name`으로 어떤 input이 바뀌었는지 알 수 있음.

- **전개 연산자(Spread Operator)**

  ```jsx
  setUser({ ...user, [name]: value });
  ```

  - 기존 객체 복사 + 특정 필드만 덮어쓰기
  - 불변성 유지(React가 참조 변경을 감지해 리렌더링 가능)

- **제어 컴포넌트 패턴**
  - 모든 input의 `value`는 `user` state에서 가져오고,
  - `onChange`로 state를 갱신하여 UI와 데이터가 항상 동기화됨.
