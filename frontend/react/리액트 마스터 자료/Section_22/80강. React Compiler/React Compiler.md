# 80강. React Compiler

## 도입

React Compiler는 React 팀이 개발 중인 차세대 빌드 타임 최적화 도구입니다. 기존에 개발자가 수동으로 `useMemo`, `useCallback`, `React.memo`를 사용하여 최적화하던 작업을 **컴파일러가 자동으로 처리**해주는 것을 목표로 합니다.

## 개념 설명

### React Compiler란?

- 기존 React는 **런타임**에서 Virtual DOM 비교를 통해 리렌더링을 결정
- React Compiler는 **빌드 타임**에 코드를 분석하여 불필요한 리렌더링을 자동으로 방지
- `useMemo`, `useCallback` 없이도 컴파일러가 자동으로 메모이제이션 삽입

### 핵심 변화

| 기존 (수동 최적화) | React Compiler (자동 최적화) |
|-------------------|---------------------------|
| `useMemo(() => compute(a), [a])` | `const result = compute(a)` — 컴파일러가 자동 메모이제이션 |
| `useCallback((x) => fn(x), [dep])` | `const handler = (x) => fn(x)` — 컴파일러가 참조 안정성 보장 |
| `React.memo(Component)` | 불필요한 리렌더링을 컴파일 단계에서 차단 |

### 작동 원리

1. **정적 분석**: 컴포넌트의 의존성 그래프를 빌드 타임에 분석
2. **자동 메모이제이션**: 변경되지 않는 값/함수에 대해 자동으로 캐싱 코드 삽입
3. **세밀한 리렌더링**: 실제로 변경된 부분만 업데이트하도록 코드 변환

## 핵심 포인트

- React Compiler는 **수동 메모이제이션의 종말**을 예고하는 차세대 도구
- 빌드 타임에 최적화하므로 런타임 오버헤드가 없음
- 기존 코드를 수정하지 않아도 자동으로 최적화 적용 가능
- 아직 실험적 단계이므로 프로덕션 도입 시 주의 필요
