# 🟩 Mission X — 요청 취소 & 에러 처리 고도화 + axios 인스턴스 실무 패턴

## 📝 설명

쇼핑몰 대시보드 상품 목록 패널을 React로 구현합니다. 사용자는 페이지 이동/탭 전환/필터 변경을 빠르게 반복할 수 있으며, 네트워크가 불안정하거나 서버가 일시 에러(404/500...)를 낼 수 있습니다.

목표는 다음을 모두 만족하는 안정적인 데이터 패칭 컴포넌트를 만드는 것입니다.

- **요청 취소**: 컴포넌트가 언마운트되거나 새 요청이 필요할 때, 진행 중인 요청을 `AbortController`로 안전하게 취소한다.
- **에러 고도화**: 실패 원인을 구체적 메시지로 구분 노출(취소/타임아웃/네트워크 단절/404/5xx/기타).
- **axios 인스턴스화**: `baseURL`/`timeout`/`headers` 공통설정 및(선택) 인터셉터로 중복 제거 & 일관성 확보.
- **React 흐름 정합성**: 로딩/성공/에러 상태 전이를 무조건 state 기반으로 처리하여 예측 가능성 확보.

## ✅ 요구사항 (체크리스트)

### 핵심 기능

- `apiClient`(axios 인스턴스)를 생성: `baseURL=https://fakestoreapi.com`, `timeout=8000`, 공통 headers.
- `ProductList` 컴포넌트 마운트 시 `/products` 요청 후 목록 렌더.
- 로딩/성공/에러 UI 분기 (로딩: "불러오는 중...", 에러: 원인별 메시지).
- `AbortController`로 취소: 언마운트 or 재실행 직전 `abort()` 호출.
- 에러 세분화 처리: 취소(`ERR_CANCELED`), 타임아웃(`ECONNABORTED`), 네트워크 단절(응답 없음), 404, 5xx, 기타.
- (선택) axios 인터셉터로 공통 에러 가공/토큰 주입/로깅.

### 기술/안전

- `useEffect` 안에서 `async` 함수 정의/호출(effect 자체를 `async`로 만들지 않기).
- state 업데이트는 요청 성공 이후에만; 취소 시 `setState` 금지(불필요 렌더/경고 방지).
- DevTools(Network/Console)로 취소/재시도/에러 분기 확인 가능.

## 🧱 Base Code

의도: 요청 취소와 에러 세분화가 비어 있고, axios 인스턴스가 없어 중복 설정이 발생합니다. 여기에 안정화 패턴을 채워 넣으세요.

```jsx
import { useEffect, useState } from "react";
import axios from "axios";

export default function ProductListBase() {
  const [products, setProducts] = useState([]);
  const [loading, setLoading]   = useState(true);
  const [error, setError]       = useState(null);

  useEffect(() => {
    // TODO:
    // 1) AbortController 생성 및 axios 요청에 signal 연결
    // 2) 에러 원인별 분기 처리
    // 3) cleanup에서 controller.abort() 호출
    // 4) axios 인스턴스 도입 전이라 요청 설정이 중복됨

    (async () => {
      try {
        const res = await axios.get("https://fakestoreapi.com/products");
        setProducts(res.data);
        setLoading(false);
      } catch (err) {
        // TODO: 세분화 X (모든 에러 동일 처리 중)
        setError("데이터를 불러오지 못했습니다.");
        setLoading(false);
      }
    })();

    return () => {
      // TODO: 요청 취소 미구현
    };
  }, []);

  if (loading) return <p>불러오는 중...</p>;
  if (error)   return <p style={{ color: "red" }}>{error}</p>;

  return (
    <ul>
      {products.map(p => <li key={p.id}>{p.title}</li>)}
    </ul>
  );
}
```

## 📌 미션의 의의 (DevTools 관찰 포인트 · 안정성 포인트)

- **AbortController + cleanup**: 언마운트/재실행 직전 반드시 `abort()` → Network 탭 `(canceled)` 확인.
- **에러 세분화 UX**: "실패했습니다" 대신 원인 중심(취소/타임아웃/네트워크/404/5xx/기타).
- **렌더 흐름의 투명성**: `loading` → (`success` | `error`)로 단방향 전이. 취소 시 state 업데이트 없음.
- **인스턴스화 효과**: 반복설정 제거, 토큰/로깅/재시도 정책 등 확장성 확보.
- **실서비스 적합성**: 빠른 탭 전환/필터 변경/느린 네트워크에서도 메모리 누수·레이스 컨디션 방지.

### DevTools 체크

- Network: 컴포넌트 언마운트 시 요청 `(canceled)` 표시.
- Console: 취소/분기 로그 확인, React 경고(언마운트 후 `setState`) 없음이어야 함.
