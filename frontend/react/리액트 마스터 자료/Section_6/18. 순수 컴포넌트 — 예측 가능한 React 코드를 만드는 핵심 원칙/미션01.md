# 🟪 순수 컴포넌트 실습 — 외부 변수 없이 동일 입력 → 동일 출력 유지하기

## 📝 설명

이번 미션에서는 **순수 컴포넌트(Pure Component)** 의 핵심 원칙을 직접 체험합니다.
React에서 순수한 컴포넌트란,

> 동일한 props와 state가 주어졌을 때 항상 동일한 UI를 반환하는 컴포넌트
> (외부 변수를 수정하거나, props를 직접 변경하지 않음)

을 의미합니다.

## 🎯 미션 목표

- 비순수 컴포넌트(`ProductListImpure`)를 먼저 구현해 봅니다.
- 전역 변수 `discountRate`를 선언하고,
- 컴포넌트 안에서 그 값을 직접 변경(`discountRate += 0.1`)합니다.
- 결과적으로, 같은 props로 렌더링할 때마다 다른 UI가 나타나게 됩니다.
- 이 컴포넌트는 "순수하지 않은" 동작을 보여줘야 합니다.
- 그 다음, 순수 컴포넌트(`ProductListPure`)를 구현합니다.
- 전역 변수 대신 props로 `discountRate`를 전달받습니다.
- props로 받은 값만 이용해 JSX를 렌더링합니다.
- 이렇게 하면 항상 동일한 입력 → 동일한 출력이 보장됩니다.
- 두 컴포넌트를 한 화면에 나란히 렌더링하여 차이를 직접 확인합니다.
- `ProductListImpure`는 렌더링할 때마다 결과가 바뀌고,
- `ProductListPure`는 렌더링해도 항상 동일한 결과를 유지합니다.

## 🖥️ 출력 결과 예시

```text
# 첫 번째 렌더링
[비순수 컴포넌트]
셔츠: 900원
청바지: 1800원
현재 할인율: 0.2

[순수 컴포넌트]
셔츠: 900원
청바지: 1800원
할인율: 0.1

# 두 번째 렌더링 (props 동일)
[비순수 컴포넌트]
셔츠: 800원
청바지: 1600원
현재 할인율: 0.3  ← 결과가 바뀜 ❌

[순수 컴포넌트]
셔츠: 900원
청바지: 1800원
할인율: 0.1  ← 동일 결과 ✅
```

같은 props(`products`, `discountRate`)를 넘겼는데
비순수 컴포넌트는 외부 변수를 변경하여 매번 다른 결과가 나오고,
순수 컴포넌트는 props에만 의존하므로 항상 같은 UI가 나옵니다.

## 💡 구현에 필요한 개념

- **순수 함수(Pure Function)**
  - 동일 입력 → 동일 출력
  - 외부 상태(side effect) 변경 ❌

```jsx
function add(a, b) { return a + b; } // ✅ 순수
function getTime() { return new Date(); } // ❌ 비순수
```

- **React 순수 컴포넌트 원칙**
  - JSX는 오직 props와 state에만 의존해야 함
  - 컴포넌트 안에서 전역 변수 수정 금지
  - props 직접 수정 금지

- **불변성 유지(Immutability)**
  - props나 외부 데이터를 변경하지 말고 복사본을 만들어 수정
  - 예: `products.concat(newItem)` 또는 `products.map(...)`

- **Strict Mode**
  - React 개발 모드에서 두 번 렌더링을 실행하여,
  - "같은 props인데 다른 결과가 나오는 컴포넌트"를 자동 감지
