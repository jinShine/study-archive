# 🟪 왜 이름이 바뀌었는데 화면은 그대로죠? — 배열 속 중첩 객체까지 안전하게 업데이트하기

## 📝 설명

이번 미션에서는 배열 안의 객체가 또 다른 객체를 포함할 때, 불변성을 유지하며 안전하게 업데이트하는 방법을 실습합니다.
이 구조는 실무에서 자주 등장합니다. 예를 들어 게시글 목록에서 `author` 정보를 수정하거나, 장바구니의 상품 옵션을 변경하는 경우가 모두 여기에 해당합니다.

React는 "값의 내용"이 아니라 "참조(주소)가 달라졌는가"만 확인합니다.
즉, 내부 속성만 직접 바꾸면 화면은 갱신되지 않습니다.
그래서 중첩 구조에서는 단계별로 새로운 객체를 만들어야 React가 변화를 감지하고 컴포넌트를 다시 렌더링합니다.

## 🖥️ 출력 결과 예시

```text
# 초기 화면
첫 번째 글 — 철수 (chul@example.com)
두 번째 글 — 영희 (young@example.com)

# [작성자 이름 바꾸기] 클릭
첫 번째 글 — 민수 (chul@example.com)
두 번째 글 — 영희 (young@example.com)
```

## 💡 구현에 필요한 개념

- **배열의 불변성 유지**
  - 기존 배열을 직접 수정하지 않고 `map()`을 사용해 새로운 배열을 생성해야 합니다.

```jsx
setPosts(posts.map(...));
```

- **객체의 불변성 유지**
  - 수정할 객체를 새로 만들어 교체해야 합니다.

```jsx
{ ...post, author: { ...post.author, name: "민수" } }
```

- **React의 렌더링 원리**
  - React는 참조가 달라졌을 때만 다시 그립니다.
  - 내부 속성만 바꾸면 같은 참조로 판단하여 UI를 업데이트하지 않습니다.

- **얕은 복사(Shallow Copy)**
  - 전개 연산자(`...`)는 한 단계까지만 복사합니다.
  - 따라서 중첩된 객체를 바꿀 때는 두 번 이상 복사가 필요합니다.
  - 예: `...post`, `...post.author`

## 🎯 미션 목표

아래의 구조에서 특정 post의 `author.name`을 변경했을 때, 화면이 즉시 갱신되도록 코드를 작성하세요.

```jsx
const [posts, setPosts] = useState([
  {
    id: 1,
    title: "첫 번째 글",
    author: { name: "철수", email: "chul@example.com" }
  },
  {
    id: 2,
    title: "두 번째 글",
    author: { name: "영희", email: "young@example.com" }
  }
]);
```
