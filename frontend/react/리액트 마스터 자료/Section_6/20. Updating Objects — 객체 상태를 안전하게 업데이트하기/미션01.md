# 🟪 객체 상태 업데이트 실습 — 불변성을 지켜 React 렌더링을 정확하게 유지하기

## 📝 설명

이번 미션에서는 객체 형태의 state를 올바르게 업데이트하는 방법을 실습합니다.
React에서 state는 단순한 값뿐만 아니라 **객체(Object)** 로도 자주 사용됩니다.
하지만 객체를 다룰 때는 반드시 **불변성(immutability)** 을 유지해야 합니다.

자바스크립트에서 객체는 참조(reference) 타입이기 때문에,
객체 내부의 속성을 직접 바꾸면 React는 "값이 변했다"고 인식하지 못합니다.
이 경우 화면이 즉시 갱신되지 않거나, 두 번에 걸쳐 갑자기 바뀌는 버그가 생길 수 있습니다.

이 미션의 목표는 **직접 수정(❌)** 과 **새 객체 복사 후 교체(✅)** 의 차이를 눈으로 확인하고,
React가 왜 "새로운 객체를 만들어 교체해야만" UI를 정상적으로 업데이트하는지 체험하는 것입니다.

## 🖥️ 출력 결과 예시

```text
# ❌ 잘못된 코드 (user.age += 1)
철수 — 20살
[버튼 클릭]
→ 변화 없음
[버튼 다시 클릭]
→ 철수 — 22살 (두 칸 올라감 ❌)

# ✅ 올바른 코드 (불변성 유지)
철수 — 20살
[버튼 클릭]
→ 철수 — 21살
[버튼 다시 클릭]
→ 철수 — 22살
```

## 💡 구현에 필요한 개념

- **객체는 참조(reference) 타입이다**

```jsx
const a = { age: 20 };
const b = a;
b.age = 21;
console.log(a.age); // 21 — 같은 객체 참조
```

- **React는 참조 변경으로 state 변화를 감지한다**
  - React는 "객체 내부의 속성이 바뀌었는가?"가 아니라
  - "객체의 참조(주소)가 달라졌는가?"를 기준으로 렌더링 여부를 판단합니다.

- **불변성(immutability)을 지키는 방법**
  - 기존 객체를 직접 수정하지 않고 복사본을 만들어 교체해야 합니다.
  - 스프레드 연산자(`...`)를 이용해 기존 속성을 펼친 뒤, 필요한 값만 덮어씌웁니다.

```jsx
setUser({
  ...user,
  age: user.age + 1
});
```

- **불변성을 지키면 얻는 이점**
  - React가 state 변화를 정확히 감지하여 UI를 즉시 갱신
  - Virtual DOM diff 과정이 효율적으로 수행
  - `React.memo`, Redux DevTools 등의 기능이 정상 동작
