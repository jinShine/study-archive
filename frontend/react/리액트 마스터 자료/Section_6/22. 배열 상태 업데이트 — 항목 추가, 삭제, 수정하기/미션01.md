# 🟪 할 일 추가했는데 화면이 안 바뀌어요! — 배열 업데이트에서 불변성을 지키는 세 가지 기술

## 📝 설명

이 미션에서는 배열 state를 추가·삭제·수정할 때 불변성을 유지하는 방법을 직접 구현합니다.
React는 배열의 "내용"이 아니라 **배열 참조(주소)** 가 바뀌었는지를 기준으로 렌더링 여부를 판단합니다.
따라서 `push()`, `splice()`처럼 원본을 직접 바꾸면 React가 변화를 감지하지 못합니다.
새로운 배열을 만들어 교체해야만 화면이 즉시 갱신됩니다.

## 🖥️ 출력 결과 예시

```text
# 초기 상태
공부하기
운동하기

# [할 일 추가] 클릭 후
공부하기
운동하기
청소하기

# [삭제] 클릭 후
운동하기
청소하기

# 체크박스 클릭 시
공부하기 (✅)
운동하기 (✅)
```

## 💡 구현에 필요한 개념

- **불변성 유지 (Immutability)**
  - React는 배열의 "참조가 바뀌었는가"로만 변화를 판단합니다.
  - → 반드시 `push`, `splice` 대신 `concat`, `filter`, `map`, 또는 `[...배열]`을 사용하세요.

- **항목 추가: 전개 연산자(`...`)**

```jsx
setTodos([...todos, "청소하기"]);
```

  - 기존 배열을 복사한 뒤 새 항목을 추가해 새로운 배열을 생성합니다.

- **항목 삭제: `filter()`**

```jsx
setTodos(todos.filter(todo => todo.id !== id));
```

  - 조건을 만족하지 않는 항목만 남겨 새로운 배열을 반환합니다.

- **항목 수정: `map()` + 전개 연산자**

```jsx
setTodos(
  todos.map(todo =>
    todo.id === id ? { ...todo, done: !todo.done } : todo
  )
);
```

  - 수정 대상만 새로운 객체로 교체하고 나머지는 그대로 유지합니다.
