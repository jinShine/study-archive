# 🟪 도시만 바꿨는데 화면이 안 바뀐다고? — 중첩 객체 state를 '복사로' 안전하게 업데이트하기

## 📝 설명

중첩 객체 상태(예: `user.address.city`)를 직접 수정하지 말고, 단계별 새 객체를 만들어 교체하여 불변성을 지키는 미션입니다.

## ✅ 요구사항

- `user`를 아래 형태로 관리합니다.

```jsx
{ name: "영희", age: 25, address: { city: "서울", zipcode: "12345" } }
```

- "부산으로 이사하기" 버튼 클릭 시 `city`만 정확히 "부산"으로 바뀌어 화면이 즉시 갱신되어야 합니다.
- "우편번호 54321로 변경" 버튼 클릭 시 `zipcode`만 바뀌어야 합니다.
- (옵션) "초기화" 버튼으로 원래 값으로 복원하세요.

## 🖥️ 출력 결과 예시

```text
# 초기
영희 — 서울 (12345)

# [부산으로 이사하기] 클릭
영희 — 부산 (12345)

# [우편번호 54321로 변경] 클릭
영희 — 부산 (54321)

# [초기화] 클릭
영희 — 서울 (12345)
```

## 💡 구현에 필요한 개념

- **얕은 비교(Shallow Compare)**: React는 "객체 내용"이 아니라 참조가 바뀌었는지를 본다.

- **불변성 유지**: 중첩 경로의 각 단계마다 새 객체를 만든다.

```jsx
setUser({
  ...user,
  address: {
    ...user.address,
    city: "부산",
  },
});
```

- **직접 수정 금지**: `user.address.city = "부산"` 같은 코드는 참조가 그대로라 리렌더가 건너뛰어질 수 있다.

- **함수형 업데이트(권장 상황)**: 이전 값 기준으로 계산할 때 안전하다.

```jsx
setUser(prev => ({ ...prev, address: { ...prev.address, zipcode: "54321" } }));
```
