# State의 이해와 활용

---

## 1. State 활용 패턴 3종 -- Counter, Toggle, SelectableList

실무에서 반복적으로 등장하는 세 가지 상태 패턴이다. 데이터 타입(숫자, 불리언, 인덱스)에 따라 상태 설계가 달라진다.

### 패턴 한눈에 비교

| 패턴 | State 타입 | 초기값 | 변경 방식 | 대표 UI |
|------|-----------|--------|-----------|---------|
| **Counter** | `number` | `0` | `count + 1` | 좋아요, 장바구니 수량 |
| **Toggle** | `boolean` | `false` | `!isOn` | 다크모드, 사이드바 열기/닫기 |
| **SelectableList** | `number \| null` | `null` | `index` 저장 | 탭 메뉴, 리스트 선택 |

### Counter -- 숫자 상태

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>현재 카운트: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

- `setCount(count + 1)` 호출 -> React가 리렌더링 -> 새 `count` 반영
- 숫자를 더하거나 빼는 모든 UI에 적용 가능

### Toggle -- 불리언 상태와 조건부 렌더링

```jsx
function ToggleButton() {
  const [isOn, setIsOn] = useState(false);

  return (
    <div>
      <p>{isOn ? "켜짐 상태입니다." : "꺼짐 상태입니다."}</p>
      <button onClick={() => setIsOn(!isOn)}>
        {isOn ? "끄기" : "켜기"}
      </button>
    </div>
  );
}
```

**핵심: `!isOn`으로 상태 반전**

- `true` -> `false`, `false` -> `true`를 한 줄로 처리
- 삼항 연산자(`? :`)로 상태에 따라 **텍스트, 버튼 라벨, 스타일** 등을 분기

### SelectableList -- 선택 인덱스 상태

```jsx
function SelectableList() {
  const items = ["사과", "바나나", "체리"];
  const [selected, setSelected] = useState(null);

  return (
    <ul>
      {items.map((item, index) => (
        <li
          key={index}
          onClick={() => setSelected(index)}
          style={{
            cursor: "pointer",
            fontWeight: selected === index ? "bold" : "normal",
          }}
        >
          {item}
        </li>
      ))}
    </ul>
  );
}
```

**동작 흐름:**

```
1. 초기 상태: selected = null → 모든 항목 normal
2. "바나나" 클릭 → setSelected(1)
3. 리렌더링 → selected === 1인 항목만 bold
4. "체리" 클릭 → setSelected(2)
5. 리렌더링 → 이전 선택 해제, 새 항목만 bold
```

**패턴 포인트:**
- 배열 데이터 자체는 변하지 않는다 -- State가 아님
- **"어떤 항목이 선택되었는가"**만 State로 관리
- `selected === index` 비교로 스타일을 조건부 적용

### 세 패턴의 공통 원리

```
[이벤트 발생] → [setState 호출] → [React 리렌더링] → [새 State로 UI 갱신]
```

State 타입만 다를 뿐, **이벤트 -> 상태 변경 -> UI 반영**이라는 흐름은 동일하다.

## 2. 다중 State 관리 -- 여러 useState를 한 컴포넌트에서

하나의 컴포넌트에서 서로 독립적인 여러 상태를 동시에 관리할 수 있다. `useState`를 여러 번 호출하면 된다.

### 기본 예제: 텍스트 + 카운터

```jsx
function MultiStateExample() {
  const [text, setText] = useState("");
  const [count, setCount] = useState(0);

  return (
    <div>
      <input
        value={text}
        onChange={(e) => setText(e.target.value)}
        placeholder="텍스트를 입력하세요"
      />
      <p>입력한 텍스트: {text}</p>

      <p>현재 카운트: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
    </div>
  );
}
```

- `text`가 바뀌어도 `count`에 영향 없음
- `count`가 바뀌어도 `text`에 영향 없음
- 각 `useState`는 **독립적으로** 상태를 추적한다

### 실전 예제: 필터 패널 (boolean + string + number)

```jsx
function FilterPanel() {
  const [isSale, setIsSale] = useState(false);
  const [category, setCategory] = useState("all");
  const [minPrice, setMinPrice] = useState(0);

  return (
    <div>
      {/* 체크박스: boolean */}
      <label>
        <input
          type="checkbox"
          checked={isSale}
          onChange={(e) => setIsSale(e.target.checked)}
        />
        세일 상품 보기
      </label>

      {/* 셀렉트: string */}
      <select value={category} onChange={(e) => setCategory(e.target.value)}>
        <option value="all">전체</option>
        <option value="clothes">의류</option>
        <option value="electronics">전자제품</option>
      </select>

      {/* 숫자 입력: number */}
      <input
        type="number"
        value={minPrice}
        onChange={(e) => setMinPrice(Number(e.target.value))}
        placeholder="최소 금액"
      />

      <p>
        현재 필터: {isSale ? "세일 상품" : "모든 상품"},
        카테고리 {category}, 최소 금액 {minPrice}원
      </p>
    </div>
  );
}
```

### 입력 타입별 값 추출 정리

| 입력 요소 | 값 추출 | State 타입 |
|-----------|---------|-----------|
| `<input type="text">` | `e.target.value` | `string` |
| `<input type="number">` | `Number(e.target.value)` | `number` |
| `<input type="checkbox">` | `e.target.checked` | `boolean` |
| `<select>` | `e.target.value` | `string` |

> `type="number"`에서도 `e.target.value`는 **문자열**이다. `Number()`로 명시적 변환이 필요하다.

## 3. State 설계 원칙

### 분리 vs 통합 판단 기준

| 기준 | 분리 (`useState` 여러 번) | 통합 (`useState` 한 번, 객체) |
|------|--------------------------|------------------------------|
| **상태 간 관계** | 서로 독립적 | 항상 함께 변경됨 |
| **변경 시점** | 각각 다른 이벤트에서 변경 | 하나의 이벤트에서 동시 변경 |
| **예시** | 필터 패널의 체크박스 + 셀렉트 | 좌표 `{ x, y }` |
| **복잡도** | 단순, 추적 쉬움 | 상태가 많아지면 한 곳에서 관리 |

### 분리가 적합한 경우

```jsx
// 각 필터는 독립적으로 변경됨 → 분리
const [isSale, setIsSale] = useState(false);
const [category, setCategory] = useState("all");
const [minPrice, setMinPrice] = useState(0);
```

### 통합이 적합한 경우

```jsx
// x, y는 항상 함께 변경됨 → 통합
const [position, setPosition] = useState({ x: 0, y: 0 });

function handleMouseMove(e) {
  setPosition({ x: e.clientX, y: e.clientY });
}
```

### 설계 원칙 요약

1. **독립적인 값은 분리한다** -- 서로 다른 이벤트에서 변경되면 별도의 `useState`
2. **항상 함께 변하는 값은 통합한다** -- 하나의 객체로 묶으면 일관성 유지가 쉽다
3. **State를 최소화한다** -- 계산 가능한 값은 State로 만들지 않는다
4. **데이터 자체와 "선택/상태"를 구분한다** -- 배열은 고정, 선택 인덱스만 State

```jsx
// ❌ 불필요한 State
const [items] = useState(["사과", "바나나"]);  // 변하지 않는 데이터
const [count, setCount] = useState(0);
const [doubled, setDoubled] = useState(0);     // count * 2로 계산 가능

// ✅ 최소한의 State
const items = ["사과", "바나나"];               // 일반 변수
const [count, setCount] = useState(0);
const doubled = count * 2;                     // 파생값은 렌더링 중 계산
```

## 면접 핵심 Q&A

**Q. 하나의 컴포넌트에서 여러 상태를 관리하는 방법?**
> `useState`를 여러 번 호출하면 된다. 각 호출은 **독립적인 상태**를 생성하며, 하나를 변경해도 다른 상태에 영향을 주지 않는다. React는 Hook 호출 순서로 각 상태를 구분한다.

**Q. State를 분리할지 합칠지 판단 기준?**
> **함께 변하는가**가 핵심 기준이다. 항상 동시에 업데이트되는 값(좌표의 x, y 등)은 하나의 객체로 합치고, 독립적으로 변하는 값(체크박스와 셀렉트 등)은 별도의 `useState`로 분리한다.

**Q. 조건부 렌더링과 State의 관계?**
> 불리언 State와 삼항 연산자를 결합하면 조건부 렌더링이 된다. `isOn ? "켜짐" : "꺼짐"`처럼 State 값에 따라 **다른 텍스트, 스타일, 컴포넌트**를 렌더링할 수 있다. State가 변하면 리렌더링이 일어나면서 조건 분기 결과도 바뀐다.

**Q. 리스트에서 선택 상태를 관리하는 패턴?**
> 배열 데이터는 일반 변수로, **선택된 항목의 인덱스(또는 id)**만 State로 관리한다. `selected === index` 비교로 선택된 항목에만 스타일을 적용한다. 데이터 자체를 State로 복사하지 않는 것이 핵심이다.

**Q. `e.target.value`와 `e.target.checked`는 언제 구분하는가?**
> `<input type="text">`, `<input type="number">`, `<select>`는 `e.target.value`로 값을 가져온다. `<input type="checkbox">`만 `e.target.checked`로 **불리언 값**을 가져온다. number 타입은 value가 문자열이므로 `Number()`로 변환이 필요하다.

**Q. 변하지 않는 데이터도 State로 관리해야 하는가?**
> 아니다. 변하지 않는 데이터는 **일반 변수나 상수**로 선언한다. State는 "변경 시 리렌더링이 필요한 값"에만 사용한다. 불필요한 State는 성능 낭비이자 코드 복잡도를 높인다.

## 용어 글로서리

| 용어 | 정의 |
|------|------|
| **Toggle** | 불리언 상태를 `!value`로 반전시키는 패턴 |
| **조건부 렌더링** | State 값에 따라 다른 UI를 보여주는 기법. 삼항 연산자, `&&` 등 사용 |
| **선택 상태 (Selection State)** | 리스트에서 현재 선택된 항목을 인덱스나 id로 추적하는 State |
| **다중 State** | 하나의 컴포넌트에서 `useState`를 여러 번 호출하여 독립적 상태를 관리하는 것 |
| **파생값 (Derived Value)** | 기존 State에서 계산할 수 있는 값. State로 만들지 않고 렌더링 중 계산 |
| **제어 컴포넌트** | `value`와 `onChange`를 통해 React State가 입력 요소의 값을 제어하는 패턴 |
| **State 최소화** | 꼭 필요한 값만 State로 두고, 나머지는 계산하거나 상수로 처리하는 원칙 |

## 자가 점검 체크리스트

- [ ] Counter, Toggle, SelectableList 세 패턴의 State 타입과 변경 방식을 구분할 수 있다
- [ ] 불리언 State와 삼항 연산자로 조건부 렌더링을 구현할 수 있다
- [ ] 리스트에서 선택 상태를 인덱스로 관리하고 스타일을 조건부 적용할 수 있다
- [ ] 하나의 컴포넌트에서 여러 `useState`를 사용하여 독립적 상태를 관리할 수 있다
- [ ] checkbox, select, number input 각각에서 올바른 값 추출 방법을 안다
- [ ] State를 분리할지 합칠지 판단 기준을 설명할 수 있다
- [ ] 변하지 않는 데이터와 파생값을 State로 만들지 않아야 하는 이유를 설명할 수 있다
- [ ] `Number(e.target.value)` 변환이 필요한 이유를 안다
