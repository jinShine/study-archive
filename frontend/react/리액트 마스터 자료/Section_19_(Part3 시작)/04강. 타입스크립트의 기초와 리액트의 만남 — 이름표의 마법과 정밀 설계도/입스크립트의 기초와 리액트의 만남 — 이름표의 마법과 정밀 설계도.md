📖 상세 개념 가이드 (The Core Manual)
1. 이름표 붙이기의 시작: 기본 타입
타입스크립트의 가장 밑바닥 기초는 변수나 속성 뒤에 콜론(:)을 찍고 정체를 밝히는 것입니다.

/* [Example]: 기본 타입 이름표 붙이기
   [Copyright]: © nhcodingstudio 소유 */

let age: number = 25;       // 숫자만 들어갈 수 있는 상자
let name: string = "Gemini"; // 글자만 들어갈 수 있는 상자
let isVIP: boolean = true;  // 참/거짓만 들어갈 수 있는 상자

// 리액트 Props 인터페이스에 적용
interface UserCardProps {
  userName: string;
  userAge: number;
  isAdmin?: boolean; // '?'는 선택적 속성(Optional)을 의미합니다.
}

string: 글자 데이터.
number: 숫자 데이터.
boolean: 참(true) 또는 거짓(false).
? (선택적 속성): "있을 수도 있고 없을 수도 있다"는 의미입니다. 이를 통해 컴포넌트 사용 시 필수 입력의 부담을 줄이고 유연성을 확보합니다.
2. 구별된 공용체(Discriminated Unions)
단순히 isLoading: boolean으로 상태를 관리하면, 로딩 중이면서 동시에 에러가 발생하는 모순된 상황을 막을 수 없습니다. '이름표(state)' 하나로 데이터의 유무를 결정짓는 이 기법은 논리적 버그를 0%로 줄여줍니다.

/* [Example]: 논리적 무결성을 위한 구별된 공용체
   [Copyright]: © nhcodingstudio 소유 */

type Status =
  | { state: 'loading' }                     // 로딩 중일 땐 데이터가 없음
  | { state: 'success'; data: string }       // 성공했을 때만 data가 존재함
  | { state: 'error'; error: Error };        // 에러일 때만 error 객체가 존재함

3. 리액트 전용 만능 보자기: ReactNode
ReactNode는 리액트가 화면에 그릴 수 있는 **'모든 것'**을 의미합니다. 텍스트, 숫자, JSX 요소, 혹은 그들의 배열까지 담을 수 있는 만능 보자기입니다.

/* [Example]: 만능 보자기 ReactNode 사용법
   [Copyright]: © nhcodingstudio 소유 */

interface LayoutProps {
  children: React.ReactNode; // 배 속에 무엇이든 담을 수 있게 약속합니다.
}
4. 유전자의 복제: ComponentPropsWithRef
우리가 만든 버튼이 진짜 HTML 버튼의 모든 기능(클릭, 비활성화, 포커스 등)을 갖게 하려면 ComponentPropsWithRef가 필요합니다. 여기에 **Omit**을 더하면 특정 유전자만 정밀하게 교체할 수 있습니다.

/* [Example]: 유전자 복제와 Omit 조작
   [Copyright]: © nhcodingstudio 소유 */

// 기존 button의 'color' 속성 유전자를 제거(Omit)하고 커스텀 타입을 주입합니다.
interface MyButtonProps extends Omit<React.ComponentPropsWithRef<'button'>, 'color'> {
  customColor: 'red' | 'blue';
}

import React from 'react';

// 설계도: 컴포넌트가 받는 재료의 이름표를 붙입니다.

export function Welcome({ name, age, isVIP }: WelcomeProps) {

        {name}님, {age}살이 되신 것을 축하합니다! {isVIP && '👑'}

🔍 1단계 코드 상세 해설
interface WelcomeProps: 컴포넌트에 들어오는 props 객체의 정밀 설계도입니다.
name: string: name 속성에는 문자열만 할당 가능하도록 강제합니다.

{ name, age, isVIP }: WelcomeProps 규격을 따르는 객체에서 필요한 값들을 구조 분해 할당으로 추출합니다.
💻 실습 2단계: 구별된 공용체로 상태 관리 (StatusDisplay.tsx)
/* [File Path]: src/components/StatusDisplay.tsx
  [Copyright]: © nhcodingstudio 소유
  [Test Process]:
  1. src/components/StatusDisplay.tsx를 생성합니다.
  2. state가 'loading'일 때 data를 호출해보세요. 타입 에러가 발생하는 것이 정상입니다.
  3. 성공(success) 상태에서만 데이터가 안전하게 출력되는지 확인합니다.
*/

import React from 'react';

// 구별된 공용체: 상태(state)에 따라 사용 가능한 데이터가 유기적으로 변합니다.
type FetchStatus =
  | { state: 'loading' }
  | { state: 'success'; data: string }
  | { state: 'error'; error: Error };

interface StatusDisplayProps {
  status: FetchStatus;
}

export function StatusDisplay({ status }: StatusDisplayProps) {
  // state 이름표를 확인하는 순간(Narrowing), 타입스크립트는 데이터의 존재를 확신합니다.
  if (status.state === 'success') {
    return (
      <div style={{ color: '#2ecc71', padding: '10px', backgroundColor: '#eafaf1' }}>
        <strong>성공:</strong> {status.data}
      </div>
    );
  }

  if (status.state === 'error') {
    return (
      <div style={{ color: '#e74c3c', padding: '10px', backgroundColor: '#fdedec' }}>
        <strong>에러 발생:</strong> {status.error.message}
      </div>
    );
  }

  return (
    <div style={{ color: '#3498db', padding: '10px' }}>
      ⏳ 데이터를 불러오는 중입니다...
    </div>
  );
}

🔍 2단계 코드 상세 해설
type FetchStatus: 여러 가지 가능한 객체 모양을 하나로 묶은 유니온 타입입니다.
state: 'loading' | 'success' | 'error': 공통된 속성인 state가 식별자(Discriminant) 역할을 수행하여 객체의 종류를 구별합니다.
타입 좁히기(Type Narrowing): if문을 통해 state가 무엇인지 확인되면, 그 안에서는 해당 상태에 존재하는 속성(data 등)에만 접근할 수 있도록 타입스크립트가 보호해줍니다.
💻 실습 3단계: 유전자 복제 및 커스텀 버튼 (CustomButton.tsx)
/* [File Path]: src/components/CustomButton.tsx
  [Copyright]: © nhcodingstudio 소유
  [Test Process]:
  1. src/components/CustomButton.tsx를 생성합니다.
  2. onClick, disabled 등 버튼의 기본 속성을 주입했을 때 잘 작동하는지 확인합니다.
  3. customColor 속성이 'primary' 혹은 'secondary'만 허용되는지 확인합니다.
*/

import React from 'react';

/**
 * ComponentPropsWithRef: HTML <button>의 모든 표준 속성을 복제합니다.
 * Omit: 그 중 기존의 'color' 속성은 제거하고, 우리의 customColor로 대체합니다.
 */
interface CustomButtonProps extends Omit<React.ComponentPropsWithRef<'button'>, 'color'> {
  customColor: 'primary' | 'secondary';
  children: React.ReactNode; // 무엇이든 담는 만능 보자기
}

export function CustomButton({ customColor, children, style, ...rest }: CustomButtonProps) {
  // React.CSSProperties를 사용해 스타일 객체에 이름표를 붙입니다.
  const buttonStyle: React.CSSProperties = {
    backgroundColor: customColor === 'primary' ? '#646cff' : '#2f3640',
    color: 'white',
    padding: '12px 24px',
    border: 'none',
    borderRadius: '8px',
    cursor: 'pointer',
    fontWeight: 'bold',
    ...style // 외부 스타일과 합성
  };

  return (
    <button style={buttonStyle} {...rest}>
      {children}
    </button>
  );
}

🔍 3단계 코드 상세 해설
extends Omit<...>: 기존 태그의 기능을 모두 상속받으면서 충돌하는 속성만 골라내는 고난도 테크닉입니다.
React.ReactNode: 버튼 내부에 글자뿐만 아니라 아이콘, 다른 컴포넌트 등 무엇이든 들어올 수 있음을 보장합니다.
...rest: customColor와 children을 제외한 모든 HTML 표준 속성들을 실제 버튼 태그로 전달합니다.
💻 실습 4단계: 이벤트와 스타일 정밀 조작 (InputField.tsx)
/* [File Path]: src/components/InputField.tsx
  [Copyright]: © nhcodingstudio 소유
  [Test Process]:
  1. src/components/InputField.tsx를 생성합니다.
  2. e.target.value에서 에러가 없는지, 자동 완성이 잘 되는지 확인합니다.
  3. React.CSSProperties 덕분에 스타일 오타가 방지되는지 확인합니다.
*/

import React, { useState } from 'react';

export function InputField() {
  // 상태값의 타입을 string으로 명시합니다.
  const [text, setText] = useState<string>("");

  /**
   * React.ChangeEvent<HTMLInputElement>:
   * 입력창 전용 이벤트 이름표입니다. e.target이 무엇인지 명확히 알려줍니다.
   */
  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    setText(e.target.value);
  };

  const containerStyle: React.CSSProperties = {
    display: 'flex',
    flexDirection: 'column',
    gap: '10px',
    alignItems: 'center',
    margin: '20px 0'
  };

  return (
    <div style={containerStyle}>
      <label style={{ fontWeight: 'bold' }}>실시간 설계도 검증:</label>
      <input
        type="text"
        value={text}
        onChange={handleChange}
        placeholder="내용을 입력하세요"
        style={{ padding: '10px', borderRadius: '5px', border: '1px solid #ccc', width: '250px' }}
      />
      <p>입력된 값: <span style={{ color: '#646cff' }}>{text}</span></p>
    </div>
  );
}

🔍 4단계 코드 상세 해설
useState<string>: 제네릭을 사용해 이 상태는 오직 문자열만 다룬다고 선언합니다.
React.ChangeEvent<HTMLInputElement>: 리액트가 제공하는 정밀한 이벤트 타입입니다. 이를 통해 e.target이 HTML input 요소임을 확정하고 value 속성을 안전하게 읽어옵니다.
React.CSSProperties: 스타일 객체를 작성할 때 marginTop 대신 margin-top을 쓰는 등의 실수를 방지해주는 도우미입니다.
