최적화된 가져오기: import type 문법을 통해 빌드 결과물의 효율성을 극대화합니다.

📖 상세 개념 가이드 (The Core Manual)
1. Interface: 데이터의 '법적 계약서'
인터페이스는 "이 데이터 객체는 반드시 A, B 속성을 가져야 한다"는 계약입니다. 자바스크립트는 이 계약이 없어서 실행 중에 건물이 무너지지만, 타입스크립트는 코드를 치는 순간 계약 위반 여부를 판별합니다.

2. import type: 시니어의 디테일
우리가 import { User }라고 쓰면 컴파일러는 이것이 실제 값(Variable)인지 타입(Type)인지 헷갈려 할 수 있습니다. 하지만 import type을 명시하면 "이건 100% 설계도일 뿐이니 실행 파일(JS)에는 포함하지 마!"라고 명령하는 것입니다. 이는 최종 번들 용량 감소와 순환 참조 에러 방지에 탁월합니다.

3. 런타임 에러의 종말
사용자가 앱을 쓰다가 갑자기 멈추는 이유는 대부분 "값이 있을 줄 알았는데 없어서"입니다. 타입스크립트 설계도는 그 '불확실성'을 제거하여, 코드가 실행되기도 전에 모든 시나리오를 검증합니다.

💻 실습 1단계: 데이터 규격 정의 (Interface)
가장 먼저 우리 프로젝트의 '기준'이 될 유저 데이터 규격서를 작성합니다.

/* [File Path]: src/types/user.ts
  [Copyright]: © nhcodingstudio
  [Test Process]:
  1. src 폴더 내부에 types 폴더를 생성합니다.
  2. user.ts 파일을 생성하고 아래 코드를 작성합니다.
  3. 이 파일은 다른 파일들이 데이터의 형태를 확인할 때 사용하는 '표준 설계도'입니다.
*/

export interface User {
  id: number;          // 사용자의 고유 번호 (정수 또는 실수)
  displayName: string; // 화면에 노출할 유저의 이름 (문자열)
}

🔍 1단계 코드 상세 해설
export: 이 인터페이스를 다른 파일(.tsx)에서 import하여 사용할 수 있도록 '공개' 상태로 만듭니다.
interface User: User라는 이름의 새로운 데이터 타입을 정의합니다. 객체의 구조를 정의할 때 가장 널리 쓰이는 방식입니다.
id: number;: 객체 내의 id라는 키(Key)는 반드시 숫자(Number) 타입의 값만 가질 수 있다는 제약 조건입니다.
displayName: string;: 유저의 이름 정보는 반드시 문자열(String)이어야 합니다. 세미콜론(;)은 각 속성의 정의가 끝났음을 알리는 구분자입니다.
💻 실습 2단계: import type 기반 UI 컴포넌트 제작
설계도를 참고하여 유저 정보를 안전하게 렌더링하는 UI 조각을 만듭니다.

/* [File Path]: src/components/UserProfile.tsx
  [Copyright]: © nhcodingstudio
  [Test Process]:
  1. src/components 폴더에 UserProfile.tsx 파일을 생성합니다.
  2. import type 문법이 정확히 적용되었는지 확인하세요.
  3. user.displayName.toUpperCase() 코드에 마우스를 올려 TS의 타입 추론을 확인합니다.
*/

import React from 'react';
// 중요: 실제 실행 로직이 아닌 '설계도(타입)' 정보만 가져옵니다.
import type { User } from '../types/user';

/**
 * UserProfileProps: 컴포넌트가 부모로부터 받을 데이터(Props)의 규격입니다.
 */
interface UserProfileProps {
  user: User; // 위에서 가져온 User 인터페이스를 속성 타입으로 재사용합니다.
}

export function UserProfile({ user }: UserProfileProps) {
  return (
    <div style={{
      padding: '20px',
      border: '2px solid #646cff',
      borderRadius: '12px',
      backgroundColor: '#f9f9f9',
      boxShadow: '0 4px 10px rgba(0,0,0,0.05)'
    }}>
      <h3 style={{ margin: 0, color: '#333' }}>👤 엔지니어 프로필</h3>
      <p style={{ fontSize: '1.2rem', margin: '15px 0' }}>
        성함: <strong>{user.displayName.toUpperCase()}</strong>
      </p>
      <code style={{ color: '#666', backgroundColor: '#eee', padding: '2px 5px' }}>
        ID Tag: {user.id}
      </code>
    </div>
  );
}

🔍 2단계 코드 상세 해설
import type { User }: 타입스크립트 3.8부터 도입된 문법으로, 오직 타입 정보만을 위해 가져온다는 것을 명시합니다. 이는 컴파일 시 실제 자바스크립트 코드에서 완전히 제거되어 실행 성능을 높입니다.
interface UserProfileProps: 리액트 컴포넌트의 입력값인 props의 구조를 정의합니다. 여기서는 user라는 이름으로 User 타입 데이터를 받겠다고 계약합니다.
({ user }: UserProfileProps): 함수의 매개변수 부분에서 구조 분해 할당을 사용함과 동시에, 전체 인자의 타입이 UserProfileProps임을 명시합니다.
user.displayName.toUpperCase(): 타입스크립트 덕분에 displayName이 문자열임이 100% 보장됩니다. 따라서 문자열 메서드인 toUpperCase()를 안심하고 호출할 수 있으며, 자동 완성 기능이 완벽하게 지원됩니다.
💻 실습 3단계: 중앙 제어 및 데이터 주입 (App.tsx)
실제 데이터를 생성하고 컴포넌트를 조립하는 리액트의 핵심 로직입니다.

/* [File Path]: src/App.tsx
  [Copyright]: © nhcodingstudio
  [Test Process]:
  1. src/App.tsx의 기존 내용을 모두 지우고 아래 코드를 입력합니다.

import React from 'react';
import { UserProfile } from './components/UserProfile';
// 파일 간의 의존성을 명확히 하기 위해 import type을 사용합니다.
import type { User } from './types/user';

function App() {
  // 설계도(User)를 완벽하게 준수하는 가상 데이터입니다.
  const currentUser: User = {
    id: 1004,
    displayName: 'React with TypeScript'
  };

  return (
    <div style={{ maxWidth: '800px', margin: '0 auto', padding: '60px', textAlign: 'center' }}>
      <header>
        <h1 style={{ color: '#2c3e50', fontSize: '2.5rem', marginBottom: '10px' }}>
          03강. 정밀 설계도 시스템
        </h1>
        <p style={{ color: '#7f8c8d', fontSize: '1.1rem' }}>
          TypeScript Interface & import type 문법 최적화 실습
        </p>
      </header>

      <hr style={{ margin: '2.5rem 0', border: '0', borderTop: '1px solid #eee' }} />

      <main style={{ display: 'flex', justifyContent: 'center' }}>
        {/* UserProfile 컴포넌트에 'user'라는 이름으로 currentUser를 전달합니다. */}
        <UserProfile user={currentUser} />
      </main>
    </div>
  );
}

export default App;

🔍 3단계 코드 상세 해설
const currentUser: User: 변수를 선언할 때 : User를 붙여 타입을 고정했습니다. 만약 객체 내부에 오타가 있거나 필수 속성을 빠뜨리면 이 시점에서 코드가 멈춥니다.
<UserProfile user={currentUser} />: UserProfile 컴포넌트가 요구하는 User 타입의 규격과 currentUser 데이터의 타입이 일치하기 때문에 완벽한 결합이 이루어집니다.
JSX (HTML-like syntax): 리액트의 선언적 UI 방식으로, 데이터가 어떻게 화면에 그려질지를 직관적으로 묘사합니다.
💻 실습 4단계: 브라우저 렌더링 엔진 장착 (main.tsx)
모든 설계 조각들을 모아 실제 브라우저에 뿌려주는 최종 엔트리 포인트입니다.

/* [File Path]: src/main.tsx
  [Copyright]: © nhcodingstudio
  [Test Process]:
  1. src/main.tsx 파일이 아래와 같이 작성되어 있는지 확인합니다.
  2. 'npm run dev' 실행 후 브라우저에 "REACT WITH TYPESCRIPT"가 크게 나오는지 확인합니다.

import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';

/**
 * as HTMLElement:
 * 타입스크립트에게 id="root"인 요소가 반드시 존재함을 확신시키는 '타입 단언' 문법입니다.
 */
const rootElement = document.getElementById('root') as HTMLElement;

ReactDOM.createRoot(rootElement).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);

🔍 4단계 코드 상세 해설
import ReactDOM: 리액트 컴포넌트를 브라우저의 실제 DOM 요소에 렌더링하기 위한 라이브러리입니다.
document.getElementById('root'): index.html 파일에 정의된 리액트 앱의 부착 지점을 찾습니다.
as HTMLElement: 타입스크립트는 HTML 파일을 직접 읽지 못해 getElementById의 결과가 null일 수도 있다고 의심합니다. 우리는 개발자로서 이 요소가 반드시 존재함을 알기에 as 키워드로 타입을 강제 확정해 줍니다.
<React.StrictMode>: 개발 환경에서 코드가 엄격한 규칙을 따르는지 한 번 더 검사해주는 도우미 컴포넌트입니다.
