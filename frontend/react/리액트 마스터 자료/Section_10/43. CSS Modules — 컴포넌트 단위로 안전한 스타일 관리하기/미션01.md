# ⬜ CSS Modules로 버튼 충돌 문제 해결하기 — 클래스 이름의 고유화 체험하기

## 📝 설명

이번 미션에서는 Vanilla CSS의 단점을 직접 체험한 뒤, CSS Modules을 적용해
클래스 이름 충돌 없이 각 컴포넌트의 스타일이 독립적으로 작동하는 모습을 실습해 보겠습니다.

여러분은 Header 버튼과 Footer 버튼이 각각 다른 색으로 표시되도록 구현하게 됩니다.
이 과정에서 `.module.css` 파일이 하는 역할과, React에서 `import styles from ...` 문법이 어떻게 작동하는지도 직접 확인하실 수 있습니다.

## 🖥️ 출력 결과 예시

```text
HEADER 영역
[빨간 버튼]

FOOTER 영역
[파란 버튼]
```

각 영역의 버튼이 서로 다른 스타일을 가짐에도 불구하고, CSS 클래스 이름 충돌이 발생하지 않는 것을 확인하시면 성공입니다.

## 💡 구현에 필요한 개념

- **전역 충돌(Global Conflict)**
  Vanilla CSS에서는 같은 클래스 이름을 쓰면 마지막에 불러온 파일이 덮어씌워집니다.
- **CSS Modules의 원리**
  `.module.css` 파일은 클래스 이름을 자동으로 고유화시켜 빌드 시
  `button___3Gh7x`처럼 변환합니다.
  따라서 다른 컴포넌트의 `.button`과 절대 겹치지 않습니다.
- **불러오기 방식 차이점**
  - 전역 CSS: `import "./App.css";`
  - CSS Modules: `import styles from "./Button.module.css";`
- **사용 방식**
  `className={styles.button}`
  → `styles` 객체 안의 속성명(`button`)을 꺼내 실제 HTML 클래스 이름으로 변환됩니다.

## 🧱 Base Code

### `src/Header.module.css`

```css
.button {
  background-color: crimson;
  color: white;
  padding: 10px 14px;
  border: none;
  border-radius: 6px;
  cursor: pointer;
}
```

### `src/Footer.module.css`

```css
.button {
  background-color: royalblue;
  color: white;
  padding: 10px 14px;
  border: none;
  border-radius: 6px;
  cursor: pointer;
}
```

### `src/Header.js`

```jsx
import styles from "./Header.module.css";

export default function Header() {
  return (
    <header>
      <h2>HEADER 영역</h2>
      <button className={styles.button}>Header Button</button>
    </header>
  );
}
```

### `src/Footer.js`

```jsx
import styles from "./Footer.module.css";

export default function Footer() {
  return (
    <footer>
      <h2>FOOTER 영역</h2>
      <button className={styles.button}>Footer Button</button>
    </footer>
  );
}
```

### `src/App.js`

```jsx
import Header from "./Header";
import Footer from "./Footer";

export default function App() {
  return (
    <div style={{ textAlign: "center", marginTop: "40px" }}>
      <Header />
      <hr style={{ margin: "30px 0" }} />
      <Footer />
    </div>
  );
}
```
