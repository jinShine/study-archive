# 🟥 이벤트와 State 결합 실습 — 클릭으로 토글 상태 바꾸기

## 📝 설명

이번 미션에서는 이벤트(`onClick`) 와 `state`(`useState`) 를 결합해 클릭 시 UI가 즉시 바뀌는 경험을 직접 만들어 봅니다.
지난 시간까지 배운 이벤트와 `state`의 관계를 정리하면 다음과 같습니다.

> 사용자가 버튼을 클릭한다 → 이벤트 발생(`onClick`) → `state` 변경(`setState`) → React가 컴포넌트를 다시 렌더링 → 화면(UI) 갱신

React의 진짜 강점은 이 과정을 우리가 DOM을 직접 조작하지 않고도 자연스럽게 처리한다는 점입니다.
이번 미션에서는 버튼을 클릭할 때마다 켜짐/꺼짐 상태가 번갈아 표시되는 토글 버튼을 구현해 보세요.

## 🖥️ 출력 결과 예시

```text
# 초기 화면
꺼짐 상태입니다.
[켜기]

# 버튼 클릭 1회 후
켜짐 상태입니다.
[끄기]

# 버튼 클릭 2회 후
꺼짐 상태입니다.
[켜기]
```

## 💡 구현에 필요한 개념

- **`useState`로 상태 선언하기**

  ```jsx
  const [isOn, setIsOn] = useState(false);
  ```

  - `isOn`: 현재 상태(boolean)
  - `setIsOn`: 상태를 바꾸는 함수
  - `false`: 초기 상태 (꺼짐)

- **이벤트 핸들러로 `state` 변경하기**
  - JSX 안에서 `onClick` 이벤트를 사용해 상태 변경 함수를 호출합니다.

  ```jsx
  <button onClick={() => setIsOn(!isOn)}>...</button>
  ```

- **조건부 렌더링(Conditional Rendering)**
  - `state` 값에 따라 다른 문구나 버튼을 표시합니다.

  ```jsx
  {isOn ? "켜짐 상태입니다." : "꺼짐 상태입니다."}
  ```

- **React의 선언적 UI 갱신 원리**
  - `state`가 바뀌면 React가 Virtual DOM을 새로 만들고
  - 이전 DOM과 비교(diffing)해 바뀐 부분만 실제 화면에 반영합니다.
