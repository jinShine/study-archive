# 🟥 React에서 State의 힘 느껴보기 — 버튼 클릭으로 화면 숫자 업데이트하기

## 📝 설명

이번 미션에서는 React의 핵심 개념인 `state`를 직접 실습합니다.
지난 시간에 배운 이벤트(`onClick`, `onChange`)는 "사용자의 행동을 감지"하는 역할이었지만, `state`는 그 행동을 "화면 변화로 연결"하는 역할을 합니다.
`Props`는 외부에서 주어진 값이라 바꿀 수 없지만, `state`는 컴포넌트 내부에서 직접 변경 가능한 데이터입니다.

이번 미션에서는 버튼을 클릭할 때마다 숫자가 증가하는 간단한 카운터 컴포넌트를 만들어 보세요.
로컬 변수(`let count = 0`) 대신 `useState`를 사용해야 실제 화면이 자동으로 업데이트됩니다.

## 🖥️ 출력 결과 예시

```text
# 초기 화면
현재 카운트: 0

# 버튼을 한 번 클릭
현재 카운트: 1

# 버튼을 두 번 클릭
현재 카운트: 2
```

## 💡 구현에 필요한 개념

- **`state` 선언과 구조 분해**

  ```jsx
  const [count, setCount] = useState(0);
  ```

  - `count`: 현재 상태 값
  - `setCount`: 상태를 바꾸는 함수
  - `0`: 초깃값

- **렌더링과 화면 갱신 원리**
  - `setCount()`가 호출되면 React가 자동으로 Virtual DOM을 갱신하고
  - 이전 상태와 비교(diffing)하여 바뀐 부분만 실제 DOM에 반영합니다.

- **Props vs State**
  - `Props` → 외부에서 전달, 읽기 전용
  - `State` → 내부에서 선언, 변경 가능

- **비동기 업데이트(batch update)**
  - 여러 `state` 변경을 모아 한 번에 처리하므로, `setCount` 직후 `console.log` 값은 바로 바뀌지 않을 수 있습니다.
  - 하지만 화면(UI)은 최신 값으로 자동 갱신됩니다.
