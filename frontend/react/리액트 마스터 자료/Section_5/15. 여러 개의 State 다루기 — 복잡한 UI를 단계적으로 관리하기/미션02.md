# 🟥 이전 값으로 안전하게 업데이트하기 — `useState`의 함수형 업데이트 실습

## 📝 설명

이번 미션에서는 함수형 업데이트(Functional Update) 패턴을 실습합니다.
이 패턴은 `setState` 호출 시 이전 `state` 값을 기반으로 새로운 `state`를 계산하는 방식으로,
여러 번의 클릭이나 비동기 상황에서도 항상 최신 값을 보장해 줍니다.

일반적으로 다음과 같이 작성하면 문제가 생길 수 있습니다.

```jsx
setCount(count + 1);
setCount(count + 1);
```

이렇게 하면 두 번 더하려 해도 React가 같은 렌더 기준값을 사용하기 때문에 실제로는 한 번만 증가합니다.
이 문제를 해결하려면 이전 값을 인자로 받는 콜백을 전달해야 합니다.

```jsx
setCount(prev => prev + 1);
setCount(prev => prev + 1);
```

이 미션에서는 버튼을 클릭하면 `count`가 정확히 2씩 증가하는 "두 번 올리기 카운터"를 만들어 보세요.
그리고 `setTimeout` 안에서도 동일한 동작이 유지되도록 해보세요.

## 🖥️ 출력 결과 예시

```text
# 초기 상태
count: 0

# 버튼 클릭 1회 후
count: 2

# 버튼 클릭 2회 후
count: 4

# "1초 후 +3" 버튼 클릭 시 1초 뒤에
count: 7
```

## 💡 구현에 필요한 개념

- **함수형 업데이트 문법**

  ```jsx
  setState(prev => prev + 1);
  ```

  - `prev`는 React가 보관한 "가장 최신의 `state` 값"입니다.
  - 여러 번의 업데이트가 동시에 일어나도 꼬이지 않고 순서대로 반영됩니다.

- **비동기 상황에서의 안정성**

  ```jsx
  setTimeout(() => {
    setCount(prev => prev + 3);
  }, 1000);
  ```

  - 나중에 실행되는 코드(`setTimeout`, API 응답 등)에서도 항상 최신 `state`를 사용합니다.

- **순수 함수 원칙**
  - 전달받은 `prev`를 직접 수정하지 않고, 새로운 값을 반환해야 합니다.
