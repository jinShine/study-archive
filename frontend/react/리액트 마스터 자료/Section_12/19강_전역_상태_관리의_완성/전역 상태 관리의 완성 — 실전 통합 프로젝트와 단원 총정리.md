# 19강. 전역 상태 관리의 완성 — 실전 통합 프로젝트와 단원 총정리

## 도입

10강부터 18강까지 배운 모든 기법을 하나로 합쳐 **실전 통합 프로젝트**를 완성한다. Props Drilling의 문제 인식에서 출발해, Context 생성, Reducer 결합, 채널 분리, 다중 엔진 구축, 커스텀 훅 추상화까지 — 이 모든 것이 유기적으로 결합된 최종 아키텍처를 구현하고, 단원 전체를 총정리한다.

## 개념 설명

최종 아키텍처의 구성:

1. **AuthProvider** — 인증 상태(user, isLoading)를 관리. State/Dispatch 채널 분리 + 커스텀 훅(`useAuthState`, `useAuthDispatch`)
2. **TodoProvider** — 업무 상태(todos)를 관리. 동일한 패턴으로 State/Dispatch 분리 + 커스텀 훅(`useTodoState`, `useTodoDispatch`)
3. **Provider 중첩** — Auth가 Todo를 감싸는 의존성 순서 유지
4. **컴포넌트** — 커스텀 훅만으로 필요한 데이터와 기능에 접근

설계 원칙:
- App은 복잡한 `useState`나 `useEffect`가 전혀 없다. 오직 **엔진 배치**에만 집중하는 순수한 관제탑이다.
- 각 컴포넌트는 커스텀 훅을 통해 필요한 엔진에만 접속한다.
- 서로 다른 엔진의 데이터를 조합하여 비즈니스 로직(권한 체크 등)을 구현한다.

## 코드 예제

### 인증 엔진 (AuthContext)

```jsx
import { createContext, useReducer, useContext, useMemo } from "react";

const AuthStateContext = createContext();
const AuthDispatchContext = createContext();

const authReducer = (state, action) => {
  if (action.type === "LOGIN") return { user: { name: "정용수" }, isLoading: false };
  if (action.type === "LOGOUT") return { user: null, isLoading: false };
  return state;
};

export function AuthProvider({ children }) {
  const [state, dispatch] = useReducer(authReducer, { user: null, isLoading: false });
  const memoizedState = useMemo(() => state, [state]);
  return (
    <AuthStateContext.Provider value={memoizedState}>
      <AuthDispatchContext.Provider value={dispatch}>
        {children}
      </AuthDispatchContext.Provider>
    </AuthStateContext.Provider>
  );
}

export const useAuthState = () => useContext(AuthStateContext);
export const useAuthDispatch = () => useContext(AuthDispatchContext);
```

### 업무 엔진 (TodoContext)

```jsx
import { createContext, useReducer, useContext, useMemo } from "react";

const TodoStateContext = createContext();
const TodoDispatchContext = createContext();

const todoReducer = (state, action) => {
  switch (action.type) {
    case "ADD_TODO":
      return {
        ...state,
        todos: [...state.todos, { id: Date.now(), text: action.payload, completed: false }]
      };
    case "TOGGLE_TODO":
      return {
        ...state,
        todos: state.todos.map(t =>
          t.id === action.payload ? { ...t, completed: !t.completed } : t
        )
      };
    default:
      return state;
  }
};

export function TodoProvider({ children }) {
  const [state, dispatch] = useReducer(todoReducer, { todos: [] });
  const memoizedState = useMemo(() => state, [state]);
  return (
    <TodoStateContext.Provider value={memoizedState}>
      <TodoDispatchContext.Provider value={dispatch}>
        {children}
      </TodoDispatchContext.Provider>
    </TodoStateContext.Provider>
  );
}

export const useTodoState = () => useContext(TodoStateContext);
export const useTodoDispatch = () => useContext(TodoDispatchContext);
```

### 통합 관제탑 (App)

```jsx
import { AuthProvider } from "./contexts/AuthContext";
import { TodoProvider } from "./contexts/TodoContext";
import Header from "./components/Header";
import TodoDashboard from "./components/TodoDashboard";

export default function App() {
  return (
    // Auth가 Todo를 감싸는 이유: 업무 로직(Todo)이 사용자(Auth) 정보를 참조해야 하기 때문
    <AuthProvider>
      <TodoProvider>
        <div style={{ padding: '20px', backgroundColor: '#f5f7fa', minHeight: '100vh' }}>
          <Header />
          <hr />
          {/* 두 엔진의 에너지를 동시에 수신하는 대시보드 */}
          <TodoDashboard />
        </div>
      </TodoProvider>
    </AuthProvider>
  );
}
```

### 헤더 — 인증 상태 표시 (Header)

```jsx
import { useAuthState, useAuthDispatch } from "../contexts/AuthContext";

export default function Header() {
  const { user } = useAuthState();
  const dispatch = useAuthDispatch();

  return (
    <div style={{ display: 'flex', justifyContent: 'space-between', padding: '10px' }}>
      <span>{user ? `환영합니다, ${user.name}님!` : '로그인이 필요합니다.'}</span>
      <button onClick={() => dispatch({ type: user ? "LOGOUT" : "LOGIN" })}>
        {user ? "로그아웃" : "로그인"}
      </button>
    </div>
  );
}
```

### 업무 대시보드 — 엔진 융합 화면 (TodoDashboard)

```jsx
import { useAuthState } from "../contexts/AuthContext";
import { useTodoState, useTodoDispatch } from "../contexts/TodoContext";

export default function TodoDashboard() {
  // 인증 엔진 접속: 현재 누가 로그인했는지 확인
  const { user } = useAuthState();

  // 업무 엔진 접속: 데이터(State)와 명령(Dispatch) 채널을 각각 수신
  const { todos } = useTodoState();
  const dispatch = useTodoDispatch();

  // 가드 로직: 유저 정보가 없으면 로직 실행을 원천 차단
  if (!user) {
    return (
      <div style={{ textAlign: 'center', padding: '50px' }}>
        <h3>로그인이 필요한 서비스입니다.</h3>
        <p>상단 버튼을 눌러 로그인을 먼저 진행해 주세요.</p>
      </div>
    );
  }

  return (
    <div style={{ padding: '20px', background: 'white', borderRadius: '15px', boxShadow: '0 4px 6px rgba(0,0,0,0.1)' }}>
      <h2>{user.name}님의 업무 리스트</h2>
      <ul>
        {todos.map(todo => (
          <li key={todo.id} style={{ marginBottom: '10px' }}>
            <span style={{ textDecoration: todo.completed ? 'line-through' : 'none' }}>
              {todo.text}
            </span>
            <button
              onClick={() => dispatch({ type: "TOGGLE_TODO", payload: todo.id })}
              style={{ marginLeft: '10px' }}
            >
              {todo.completed ? '취소' : '완료'}
            </button>
          </li>
        ))}
      </ul>
      <button
        onClick={() => dispatch({ type: "ADD_TODO", payload: "새로운 업무" })}
        style={{ marginTop: '10px', width: '100%', padding: '10px', backgroundColor: '#4f46e5', color: 'white', border: 'none', borderRadius: '8px' }}
      >
        + 할 일 추가하기
      </button>
    </div>
  );
}
```

## 코드 해설

- **`useAuthState`, `useTodoState`:** `useContext(UserContext)` 방식보다 훨씬 직관적이다. 코드를 읽는 사람도 "이 컴포넌트는 인증 정보와 업무 정보를 쓰는구나"라고 단번에 파악할 수 있다.
- **복합 제어:** `if (!user)` 구문을 통해 서로 다른 엔진의 데이터를 조합하여 사용자 경험(UX)을 제어한다.
- **App의 단순화:** App은 `useState`나 `useEffect`가 전혀 없다. 오직 엔진 배치에만 집중하는 순수한 관제탑이다.

### 통합 시스템의 데이터 순환

1. **의도(Intent):** 사용자가 "할 일 추가" 버튼을 누른다.
2. **보안 체크:** 컴포넌트 내부에서 인증 엔진의 `user` 상태를 확인한다. 유효하지 않다면 명령을 중단한다.
3. **명령(Dispatch):** 유효하다면 업무 엔진의 `TodoDispatchContext`를 통해 `ADD_TODO` 액션을 발송한다.
4. **최적화(Optimization):** 업무 목록이 업데이트되더라도, `AuthDispatchContext`를 사용 중인 로그아웃 버튼이나 Header는 절대 리렌더링되지 않는다. 채널이 분리되어 있기 때문이다.
5. **반영(Update):** 오직 업무 목록을 구독 중인 `TodoDashboard`만 화면을 다시 그린다.

## 실무 비유

최종 아키텍처는 **스마트 빌딩의 통합 관제 시스템**과 같다:
- **보안 시스템(Auth)** — 출입 카드를 발급하고 권한을 관리한다.
- **업무 관리 시스템(Todo)** — 각 입주사의 업무를 독립적으로 관리한다.
- **통합 대시보드(TodoDashboard)** — 보안 시스템으로 신원을 확인한 후, 업무 시스템에 접근을 허용한다.
- **관제탑(App)** — 각 시스템의 전원만 관리하고, 실제 운영은 각 시스템에 위임한다.
