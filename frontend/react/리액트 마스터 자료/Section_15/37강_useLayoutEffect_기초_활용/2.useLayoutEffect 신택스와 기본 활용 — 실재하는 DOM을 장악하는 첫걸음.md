# 37강-2. useLayoutEffect 신택스와 기본 활용 -- DOM을 장악하는 첫걸음

## 도입

지난 시간 우리는 리액트의 렌더링 파이프라인을 통해 `useLayoutEffect`가 왜 탄생했는지 그 배경을 공부했다. 이번에는 그 강력한 무기를 실제로 어떻게 손에 쥐고 휘두르는지 신택스(Syntax)와 기본 활용법을 마스터해본다. 겉모습은 `useEffect`와 거울처럼 닮아있지만, 그 내면의 동기적 차단의 힘을 이해하면 실재하는 DOM을 완벽히 장악할 수 있다.

## 개념 설명

### useLayoutEffect의 표준 신택스

리액트 엔진은 이 훅을 만나는 즉시 브라우저에게 "기다려! 내가 이 함수를 다 실행할 때까지 화면에 아무것도 그리지 마!"라는 강력한 명령을 내린다.

```jsx
import { useLayoutEffect } from 'react';

useLayoutEffect(() => {
  // 1. 실행 로직: 실재하는 DOM 수치를 측정하거나 스타일을 '동기적'으로 수정한다.

  return () => {
    // 2. 클린업 로직: 다음 실행 전이나 컴포넌트가 사라지기 전 정리 작업을 수행한다.
  };
}, [/* 3. 의존성 배열 */]);
```

## 코드 예제

### 실제 DOM 너비 측정 시스템 (BasicLayoutEffect.jsx)

리액트의 가상 세계가 아닌, 브라우저가 계산한 실제 물리적 수치를 측정하여 화면에 표시하는 기초 예제이다.

```jsx
import { useState, useLayoutEffect, useRef } from 'react';

export default function BasicLayoutEffect() {
  const elementRef = useRef(null);
  const [width, setWidth] = useState(0);

  useLayoutEffect(() => {
    if (elementRef.current) {
      // 브라우저 레이아웃 엔진에게 실제 수치를 직접 묻는다.
      const rect = elementRef.current.getBoundingClientRect();
      setWidth(rect.width);
    }
  }, []); // 처음 나타날 때 딱 한 번 측정

  return (
    <div
      ref={elementRef}
      style={{ width: '50%', background: 'lightgrey', padding: '20px', borderRadius: '10px' }}
    >
      내 실제 너비는 <b>{width}px</b> 입니다.
    </div>
  );
}
```

### 팝업 정중앙 위치 보정 시스템 (PopupCenterSystem.jsx)

팝업이 나타나는 찰나의 순간에 높이를 측정하여, 사용자가 '점프' 현상을 전혀 느끼지 못하게 정중앙에 배치하는 실무형 로직이다.

```jsx
import { useState, useLayoutEffect, useRef } from 'react';

export default function PopupCenterSystem() {
  const [isOpen, setIsOpen] = useState(false);
  const [offset, setOffset] = useState(0);
  const popupRef = useRef(null);

  useLayoutEffect(() => {
    if (isOpen && popupRef.current) {
      // 화면에 그려지기 직전, 팝업의 실제 픽셀 높이를 가져온다.
      const height = popupRef.current.offsetHeight;
      setOffset(height / 2); // 높이의 절반만큼 위로 올리기 위한 보정값 저장
    }
  }, [isOpen]);

  return (
    <div style={{ textAlign: 'center', padding: '50px' }}>
      <button onClick={() => setIsOpen(!isOpen)}>팝업 토글</button>
      {isOpen && (
        <div
          ref={popupRef}
          style={{
            position: 'fixed',
            top: `calc(50% - ${offset}px)`,
            left: '50%',
            transform: 'translateX(-50%)',
            padding: '40px',
            backgroundColor: 'white',
            boxShadow: '0 10px 30px rgba(0,0,0,0.3)',
            borderRadius: '20px'
          }}
        >
          <p>사용자는 0.1초의 깜빡임도 보지 못하는</p>
          <p>완벽한 정중앙 팝업이다.</p>
        </div>
      )}
    </div>
  );
}
```

## 코드 해설

### 이펙트 안의 상태 업데이트는 안전한가?

보통 이펙트 안에서 `setState`를 하면 리렌더링이 발생하므로 비효율적이라고 생각할 수 있다. 하지만 `useLayoutEffect`는 특별하다.

- **렌더링 일괄 처리**: 리액트는 `useLayoutEffect`에서 발생한 상태 업데이트를 브라우저가 화면을 그리기 전에 하나의 작업 단위로 묶어버린다.
- **결과**: 사용자는 업데이트 전의 '불완전한 화면'을 보지 못하고, 보정이 끝난 '최종 결과물'만 보게 된다.

## 실무 비유

`useLayoutEffect` 내부에서 `setState`를 호출하는 것은 연극 무대에서 관객에게 커튼을 열기 전 소품 배치를 한 번 더 조정하는 것과 같다. 관객은 최종 완성된 무대만 보게 된다.

## 핵심 포인트

- `useLayoutEffect`의 신택스는 `useEffect`와 동일하나, 실행 시점이 Paint 이전이라는 결정적 차이가 있다.
- `useRef`와 결합하여 실제 DOM의 물리적 수치(너비, 높이, 좌표)를 측정할 수 있다.
- 내부의 `setState` 호출은 브라우저 페인팅 전에 일괄 처리되어 사용자에게 불완전한 화면이 노출되지 않는다.
