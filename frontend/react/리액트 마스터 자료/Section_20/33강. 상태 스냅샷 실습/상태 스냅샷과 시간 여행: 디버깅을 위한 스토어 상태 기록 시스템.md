📖 심층 배경지식 가이드: 상태 사진첩과 불변성
지난 시간 우리는 useShallow와 커스텀 동등성 함수를 통해 리액트의 눈을 밝혀주고, 불필요한 리렌더링을 완벽하게 차단하는 성능 최적화의 정점을 찍었습니다. 이제 우리의 앱은 초당 60프레임의 부드러움을 유지하는 아주 견고한 시스템이 되었죠.

하지만 시스템이 정교해질수록 개발자인 우리는 "방금 전의 상태로 되돌릴 수는 없을까?" 혹은 "어떤 과정을 거쳐서 현재의 에러 상태에 도달했지?"와 같은 의문에 빠지게 됩니다. 오늘은 우리가 관리해 온 상태들의 궤적을 하나하나 기록하고, 마치 타임머신을 타듯 자유롭게 과거와 미래를 오갈 수 있는 상태 스냅샷(State Snapshot)과 시간 여행(Time Travel) 시스템을 설계해 보겠습니다.

상태 스냅샷이라는 개념은 우리가 매 순간 정성스럽게 찍어두는 '디지털 사진첩'과 같습니다. 현재의 상태를 하나의 장면으로 캡처하여 저장해두면 나중에 언제든지 그 사진을 꺼내 보며 당시의 상황을 재현할 수 있게 됩니다.

1. 시간 여행의 핵심 원리: Past & Future Stack
과거로 돌아가기 위해서는 단순히 값을 바꾸는 것이 아니라, 과거의 상태 객체 자체를 어딘가에 보관하고 있어야 합니다. 이를 위해 우리는 두 개의 스택(Stack) 구조를 운영합니다.

Past Stack (과거 스택): 사용자가 새로운 행동을 할 때마다 현재 상태가 밀려 들어가는 저장소입니다.
Future Stack (미래 스택): 실행 취소(Undo)를 했을 때, 다시 앞으로 가기 위해 현재 상태를 임시 보관하는 저장소입니다.
2. Zustand와 불변성 (Immutability)
Zustand에서 시간 여행을 구현할 때 가장 중요한 것은 불변성입니다. Zustand의 set 함수는 기존 상태를 직접 수정하지 않고 새로운 객체를 반환해야 리액트가 변화를 감지합니다. 이 특성 덕분에 우리는 과거의 상태 객체를 배열에 담아두는 것만으로도 완벽한 기록 시스템을 만들 수 있습니다.

💻 실습 1단계: 타임머신 기능을 담은 스토어 (useHistoryStore.ts)
우리는 상태를 정의할 때 현재의 값뿐만 아니라 지나온 길인 past와 앞으로 갈 길인 future 배열을 함께 관리합니다.

/* [File Path]: src/store/useHistoryStore.ts */
import { create } from 'zustand';

interface HistoryState {
  count: number;
  past: number[];
  future: number[];
  increment: () => void;
  undo: () => void;
  redo: () => void;
}

export const useHistoryStore = create<HistoryState>((set) => ({
  count: 0,
  past: [],
  future: [],

  // [1] 새로운 행동 기록 및 값 증가
  increment: () => set((state) => {
    // 1. 현재의 count를 포함한 새로운 과거 배열을 생성
    // 2. slice(-50)을 통해 최근 50개의 스냅샷만 유지 (메모리 최적화)
    const newPast = [...state.past, state.count].slice(-50);

    return {
      past: newPast,
      count: state.count + 1,
      // 3. 새로운 액션이 발생하면 기존의 미래 분기는 끊기므로 초기화
      future: [],
    };
  }),

  // [2] 실행 취소 (Undo): 과거로 돌아가기
  undo: () => set((state) => {
    // 방어 코드: 되돌릴 과거가 없으면 상태를 그대로 유지
    if (state.past.length === 0) return state;

    // 1. 과거 스택의 마지막 값(가장 최근)을 현재 값으로 복구
    const previous = state.past[state.past.length - 1];

    // 2. 마지막 값을 제외한 새로운 과거 배열 생성 (불변성 유지)
    const newPast = state.past.slice(0, state.past.length - 1);

    return {
      past: newPast,
      count: previous,
      // 3. 현재의 count는 나중에 Redo 할 수 있도록 미래 스택의 맨 앞에 추가
      future: [state.count, ...state.future],
    };
  }),

  // [3] 다시 실행 (Redo): 미래로 복구하기
  redo: () => set((state) => {
    // 방어 코드: 앞으로 갈 미래가 없으면 상태 유지
    if (state.future.length === 0) return state;

    // 1. 미래 스택의 첫 번째 값(가장 가까운 미래)을 꺼내기
    const next = state.future[0];

    // 2. 사용한 첫 번째 값을 제외한 새로운 미래 배열 생성
    const newFuture = state.future.slice(1);

    return {
      // 3. 현재의 count를 다시 과거 기록으로 편입
      past: [...state.past, state.count],
      count: next,
      future: newFuture,
    };
  }),
}));

🔍 상세 코드 분석 (Zustand Deep Dive)
increment 로직의 핵심:
역사 보존:[...state.past, state.count]는 현재 상태를 과거 배열에 Push하는 행위입니다.
역사 절단:future: []는 새로운 사용자 액션이 일어나는 순간, 이전에 Undo로 만들어진 '미래의 가능성'들을 모두 지우는 것입니다. 이는 타임라인이 다시 선형적으로 흐르게 만듭니다.
undo 로직의 핵심:slice(0, length - 1): 원본 배열을 건드리지 않고 마지막 원소가 빠진 새로운 배열 참조를 만듭니다. Zustand는 이 새로운 참조를 보고 렌더링을 결정합니다.
상태 전이: 과거(past) → 현재(count) → 미래(future)로 데이터가 한 칸씩 뒤로 밀려나는 흐름입니다.
redo 로직의 핵심: * future.slice(1): 배열의 첫 번째 요소를 제거한 새 배열을 만듭니다.
상태 복원: 미래(future) → 현재(count) → 과거(past)로 데이터가 다시 앞으로 당겨지는 흐름입니다.
💻 실습 2단계: 시간 여행 UI 구현 (App.tsx)
/* [File Path]: src/App.tsx */
import React from 'react';
import { useHistoryStore } from './store/useHistoryStore';

export default function App() {
  // 스토어에서 상태와 제어 함수들을 구조 분해 할당으로 가져옵니다.
  const { count, past, future, increment, undo, redo } = useHistoryStore();

  return (
    <div style={{ padding: '30px', fontFamily: 'sans-serif', maxWidth: '800px', margin: '0 auto' }}>
      <h1>🕒 Zustand 타임머신 실습</h1>

      <div style={{ textAlign: 'center', backgroundColor: '#f9f9f9', padding: '30px', borderRadius: '15px', marginBottom: '30px', border: '1px solid #eee' }}>
        <h2 style={{ fontSize: '80px', margin: '10px 0', color: '#333' }}>{count}</h2>

        <div style={{ display: 'flex', justifyContent: 'center', gap: '10px' }}>
          <button onClick={increment} style={activeBtnStyle}>숫자 증가 (+)</button>

          {/* past가 비어있으면 Undo 불가 */}
          <button onClick={undo} disabled={past.length === 0} style={past.length === 0 ? disabledBtnStyle : activeBtnStyle}>
            Undo (뒤로)
          </button>

          {/* future가 비어있으면 Redo 불가 */}
          <button onClick={redo} disabled={future.length === 0} style={future.length === 0 ? disabledBtnStyle : activeBtnStyle}>
            Redo (앞으로)
          </button>
        </div>
      </div>

      <div style={{ display: 'flex', gap: '30px' }}>
        <section style={{ flex: 1 }}>
          <h3 style={{ color: '#666' }}>📜 Past Stack ({past.length})</h3>
          <ul style={listStyle}>
            {past.length === 0 && <p style={{ color: '#ccc' }}>과거 기록이 없습니다.</p>}
            {past.map((val, i) => (
              <li key={`past-${i}`} style={itemStyle}>Snap: {val}</li>
            ))}
          </ul>
        </section>

        <section style={{ flex: 1 }}>
          <h3 style={{ color: '#666' }}>🔮 Future Stack ({future.length})</h3>
          <ul style={listStyle}>
            {future.length === 0 && <p style={{ color: '#ccc' }}>미래 기록이 없습니다.</p>}
            {future.map((val, i) => (
              <li key={`future-${i}`} style={itemStyle}>Snap: {val}</li>
            ))}
          </ul>
        </section>
      </div>
    </div>
  );
}

// 스타일 정의 (재사용 가능하게 구성)
const activeBtnStyle = { padding: '12px 24px', fontSize: '16px', cursor: 'pointer', borderRadius: '8px', border: '1px solid #4CAF50', backgroundColor: '#fff', color: '#4CAF50', fontWeight: 'bold' };
const disabledBtnStyle = { ...activeBtnStyle, border: '1px solid #ccc', color: '#ccc', cursor: 'not-allowed' };
const listStyle = { backgroundColor: '#f5f5f5', padding: '15px', borderRadius: '8px', minHeight: '200px', listStyle: 'none', margin: 0 };
const itemStyle = { padding: '8px 0', borderBottom: '1px solid #ddd', fontSize: '14px', color: '#444' };

🔍 상세 코드 분석 (UI & Interaction)
조건부 렌더링 (disabled): 기록 스택의 length를 기반으로 버튼 활성화 여부를 결정합니다. 이는 사용자가 불가능한 동작을 시도하지 않도록 막는 방어적 UI 설계의 기본입니다.
실시간 동기화: 리액트는 Zustand 스토어의 past나 future 배열 참조가 바뀌는 즉시 map 함수를 재실행하여 리스트를 갱신합니다. 이를 통해 데이터가 스택 사이를 이동하는 과정을 시각적으로 완벽하게 추적할 수 있습니다.
