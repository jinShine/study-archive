## 1. Truthy와 Falsy — JavaScript의 진실과 거짓 🔴

> 💡 **비유: 출석 체크**
> JavaScript에서는 `boolean`이 아닌 값도 "참처럼(Truthy)" 또는 "거짓처럼(Falsy)" 취급된다.
> `if (value)`라고 쓰면, value가 boolean이 아니어도 Truthy/Falsy로 판별된다.

### Falsy 값 — 딱 8개만 외워라!

```js
// 이 8개만 false로 취급. 나머지는 전부 Truthy!
false
0
-0
0n          // BigInt 0
""          // 빈 문자열
null
undefined
NaN
```

### Truthy — Falsy가 아닌 모든 것

```js
// 주의! 이것들은 전부 Truthy (true로 취급)
"0"         // 비어있지 않은 문자열 → Truthy! ⚠️
"false"     // 문자열 "false" → Truthy! ⚠️
[]          // 빈 배열 → Truthy! ⚠️⚠️⚠️ (React에서 함정!)
{}          // 빈 객체 → Truthy!
-1          // 0이 아닌 숫자 → Truthy
```

### React에서의 Falsy 함정 🔴

```jsx
// ❌ 위험: 0이 화면에 그대로 렌더링됨!
const count = 0;
return <div>{count && <span>알림 {count}개</span>}</div>;
// 결과: <div>0</div>  😱 (0은 Falsy지만 React는 숫자 0을 렌더링함!)

// ✅ 안전: 명시적으로 boolean 변환
return <div>{count > 0 && <span>알림 {count}개</span>}</div>;

// ❌ 위험: 빈 배열은 Truthy!
const items = [];
if (items) { console.log("있다!"); }  // 실행됨! 빈 배열도 Truthy!

// ✅ 안전: 길이로 체크
if (items.length > 0) { console.log("진짜 있다!"); }
```

## 2. 단축 평가(Short-circuit Evaluation) 🔴

> 💡 **비유: 면접관의 판단**
> AND(&&): "첫 번째 조건부터 틀리면 두 번째는 볼 것도 없다"
> OR(||): "첫 번째 조건이 맞으면 두 번째는 볼 필요 없다"

### && (논리 AND)

```js
// 왼쪽이 Falsy → 왼쪽 값 반환 (오른쪽 실행 안 함)
// 왼쪽이 Truthy → 오른쪽 값 반환

false && "hello"     // false
null && "hello"      // null
0 && "hello"         // 0

"world" && "hello"   // "hello" (왼쪽 Truthy → 오른쪽 반환)
1 && 2 && 3          // 3 (전부 Truthy → 마지막 값)
1 && 0 && 3          // 0 (첫 Falsy에서 멈춤)
```

```jsx
// React 조건부 렌더링의 원리!
{isLoggedIn && <UserProfile />}
// isLoggedIn이 true → <UserProfile /> 반환 (렌더링!)
// isLoggedIn이 false → false 반환 (렌더링 안 됨)
```

### || (논리 OR)

```js
// 왼쪽이 Truthy → 왼쪽 값 반환 (오른쪽 실행 안 함)
// 왼쪽이 Falsy → 오른쪽 값 반환

"hello" || "world"   // "hello"
null || "기본값"     // "기본값"
0 || 42              // 42
"" || "대체 텍스트"  // "대체 텍스트"

// 기본값 설정 패턴
const name = userName || "익명";  // userName이 Falsy면 "익명"
```

> ⚠️ **||의 함정**: `0`과 `""`도 Falsy이므로 의도치 않게 기본값으로 대체될 수 있다!

## 3. ?? (Nullish Coalescing) — null/undefined만 잡는 명사수 🔴

```js
// || 의 문제점: 0, "", false도 Falsy로 취급
const count = 0;
const result1 = count || 10;   // 10 😱 (0이 유효한 값인데 대체됨!)

// ?? 의 해결: null과 undefined만 잡음!
const result2 = count ?? 10;   // 0 ✅ (0은 null/undefined가 아니므로 유지!)

// 비교
null ?? "기본값"       // "기본값" ✅
undefined ?? "기본값"  // "기본값" ✅
0 ?? "기본값"          // 0 ✅ (0은 유효한 값!)
"" ?? "기본값"         // "" ✅ (빈 문자열도 유효한 값!)
false ?? "기본값"      // false ✅
```

> 💡 **실무 원칙**: 기본값 설정 시 `||` 대신 `??`를 써라.
> `||`은 Falsy 전체를 잡지만, `??`는 null/undefined만 정밀 타격한다.

## 4. ?. (Optional Chaining) — 안전한 접근 🔴

> 💡 **비유: 미리 확인하고 문 열기**
> 방 안에 들어가기 전에 "이 방이 존재하긴 해?" 먼저 확인한다.
> 없으면 에러 대신 조용히 `undefined`를 반환한다.

```js
const user = {
  name: "철수",
  address: {
    city: "서울"
  }
};

// ❌ 위험: user.company가 없으면 TypeError!
console.log(user.company.name);  // TypeError: Cannot read properties of undefined

// ✅ 안전: ?.으로 체크
console.log(user.company?.name);  // undefined (에러 없이 조용히 종료)
console.log(user.address?.city);  // "서울"

// 메서드 호출에도 사용
user.sayHello?.();  // user에 sayHello가 있으면 호출, 없으면 undefined

// 배열 접근에도 사용
const arr = null;
arr?.[0];           // undefined (에러 없음)

// 체이닝 조합
const city = user?.address?.city ?? "알 수 없음";  // "서울"
```

> 💡 `?.`과 `??`는 세트로 쓰는 경우가 매우 많다:
> `const value = obj?.deeply?.nested?.value ?? "기본값";`

## 5. 타입 변환 🟡

### 암묵적 변환 (Implicit Coercion) — JS가 몰래 바꿈

```js
// 문자열 + 숫자 = 문자열 (가장 흔한 함정!)
"3" + 5        // "35" 😱 (문자열 연결)
5 + "3"        // "53" 😱

// 문자열 - 숫자 = 숫자 (뺄셈/곱셈/나눗셈은 숫자로 변환)
"10" - 5       // 5
"10" * 2       // 20
"abc" - 1      // NaN

// boolean → 숫자
true + 1       // 2
false + 1      // 1
```

### 명시적 변환 (Explicit Coercion) — 개발자가 직접 변환

```js
// 문자열로 변환
String(123)        // "123"
(123).toString()   // "123"
`${123}`           // "123" (템플릿 리터럴)

// 숫자로 변환
Number("123")      // 123
Number("abc")      // NaN
Number(true)       // 1
Number(null)       // 0
Number(undefined)  // NaN ⚠️
parseInt("123px")  // 123 (앞의 숫자만 파싱)
parseFloat("3.14") // 3.14
+"123"             // 123 (단항 + 연산자)

// boolean으로 변환
Boolean(0)         // false
Boolean("")        // false
Boolean(null)      // false
Boolean("hello")   // true
Boolean([])        // true ⚠️
!!value            // 이중 부정으로 boolean 변환 (실무에서 가장 많이 사용)
```

> 💡 **실무 팁**: `!!value`는 Truthy/Falsy를 명시적 boolean으로 바꾸는 관용구.
> `!!0` → `false`, `!!"hello"` → `true`

## 6. 실습: 연산자 퀴즈 챌린지

아래 각 줄의 결과를 **종이에 예측** 후 콘솔에서 확인:

```js
// 1. Truthy/Falsy
console.log(!!"");
console.log(!!0);
console.log(!![]);
console.log(!!null);
console.log(!!"false");

// 2. 단축 평가
console.log(null || "hello");
console.log("hi" || "bye");
console.log(0 && "hello");
console.log("hi" && "bye");

// 3. ?? vs ||
console.log(0 || 42);
console.log(0 ?? 42);
console.log("" || "기본");
console.log("" ?? "기본");
console.log(null ?? "기본");

// 4. 타입 변환 함정
console.log("5" + 3);
console.log("5" - 3);
console.log(true + true);
console.log([] + []);
console.log([] + {});
console.log({} + []);
```

<summary>정답 보기</summary>

```js
// 1. Truthy/Falsy
!!"" → false        // 빈 문자열은 Falsy
!!0 → false          // 0은 Falsy
!![] → true          // 빈 배열은 Truthy! ⚠️
!!null → false
!!"false" → true     // 비어있지 않은 문자열은 Truthy! ⚠️

// 2. 단축 평가
null || "hello" → "hello"   // null은 Falsy → 오른쪽
"hi" || "bye" → "hi"        // "hi"는 Truthy → 왼쪽
0 && "hello" → 0             // 0은 Falsy → 왼쪽에서 멈춤
"hi" && "bye" → "bye"        // "hi" Truthy → 오른쪽

// 3. ?? vs ||
0 || 42 → 42        // 0은 Falsy → 대체됨
0 ?? 42 → 0         // 0은 null/undefined가 아님 → 유지!
"" || "기본" → "기본"  // ""은 Falsy → 대체됨
"" ?? "기본" → ""      // ""은 null/undefined가 아님 → 유지!
null ?? "기본" → "기본"

// 4. 타입 변환 함정
"5" + 3 → "53"      // + 는 문자열 연결 우선
"5" - 3 → 2          // - 는 숫자 연산
true + true → 2      // true → 1
[] + [] → ""          // 둘 다 빈 문자열로 변환 후 연결
[] + {} → "[object Object]"
{} + [] → 0 또는 "[object Object]" (환경에 따라 다름)
```

## 면접 대비

### 🔴 필수 질문

**Q1. Truthy와 Falsy 값이란?**

> JavaScript에서 boolean이 아닌 값도 조건문에서 참/거짓으로 평가된다. Falsy 값은 `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, `NaN` 총 8개이며, 이 외의 모든 값은 Truthy다. 주의할 점은 빈 배열(`[]`)과 빈 객체(`{}`)는 Truthy라는 것이다.

**Q2. `||`와 `??`의 차이?**

> `||`는 왼쪽이 Falsy면 오른쪽을 반환한다. `??`(Nullish Coalescing)는 왼쪽이 `null` 또는 `undefined`일 때만 오른쪽을 반환한다. `0`이나 빈 문자열이 유효한 값인 경우 `||`를 쓰면 의도치 않게 대체되므로 `??`를 사용해야 한다.

**Q3. 옵셔널 체이닝(`?.`)이란?**

> 객체의 중첩 속성에 안전하게 접근하는 연산자다. 접근하려는 속성이 `null` 또는 `undefined`이면 TypeError를 발생시키지 않고 `undefined`를 반환한다. API 응답처럼 데이터 구조가 불확실한 경우에 `user?.address?.city`처럼 사용하여 런타임 에러를 방지한다.
