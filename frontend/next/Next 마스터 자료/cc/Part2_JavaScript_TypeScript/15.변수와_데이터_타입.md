## 1. 변수 선언: var vs let vs const 🔴

> 💡 **비유: 상자에 이름 붙이기**
> 변수는 데이터를 담는 **이름 붙은 상자**다.
> `var`는 낡은 상자 (구멍이 뚫려있어 위험), `let`은 튼튼한 상자, `const`는 잠긴 상자.

```js
var name = "홍길동";     // 🚫 옛날 방식. 쓰지 마라
let age = 25;            // ✅ 값이 바뀔 수 있는 변수
const PI = 3.14159;      // ✅ 값이 바뀌면 안 되는 상수
```

### 3가지 차이점 비교표

| 구분 | `var` | `let` | `const` |
|------|-------|-------|---------|
| **스코프** | 함수 스코프 | 블록 스코프 `{}` | 블록 스코프 `{}` |
| **재선언** | ✅ 가능 (위험!) | ❌ 불가 | ❌ 불가 |
| **재할당** | ✅ 가능 | ✅ 가능 | ❌ 불가 |
| **호이스팅** | ✅ (undefined로 초기화) | ✅ (TDZ에 갇힘) | ✅ (TDZ에 갇힘) |
| **사용 권장** | ❌ 절대 쓰지 마라 | 🟡 값이 바뀔 때만 | ✅ 기본값으로 사용 |

### var가 위험한 이유

```js
// ❌ var: 같은 이름으로 재선언 가능 → 실수로 덮어쓰기
var user = "철수";
var user = "영희";  // 에러 없이 덮어씌워짐! 😱

// ✅ let: 재선언 시도 → 즉시 에러로 알려줌
let user = "철수";
let user = "영희";  // SyntaxError: 이미 선언되었습니다!
```

```js
// ❌ var: 블록({})을 무시하고 밖으로 새어나감
if (true) {
  var leaked = "나는 밖에서도 보여요!";
}
console.log(leaked);  // "나는 밖에서도 보여요!" 😱

// ✅ let: 블록 안에서만 존재
if (true) {
  let safe = "나는 여기서만 살아요";
}
console.log(safe);    // ReferenceError: safe is not defined ✅
```

> 💡 **실무 원칙**: `const`를 기본으로 쓰고, 값이 재할당 되어야 할 때만 `let`을 써라. `var`는 절대 쓰지 마라.

## 2. 호이스팅(Hoisting) 🔴

> 💡 **비유: 수업 전 출석부**
> 선생님이 수업 시작 전에 학생 이름을 출석부에 먼저 적어둔다.
> 학생이 교실에 도착하기 전이어도, 이름은 이미 올라가 있다.
> JavaScript도 마찬가지. 코드 실행 전에 변수와 함수 **선언**을 먼저 메모리에 등록한다.

```js
// ❌ var 호이스팅: 에러 없이 undefined (무서움)
console.log(a);  // undefined (에러가 아니라 undefined!)
var a = 10;
console.log(a);  // 10

// 위 코드는 엔진이 이렇게 해석한다:
// var a;           ← 선언이 맨 위로 끌어올려짐 (값은 아직 없음)
// console.log(a);  ← undefined
// a = 10;          ← 여기서 비로소 값이 할당됨
```

```js
// ✅ let/const 호이스팅: TDZ로 에러 발생 (안전!)
console.log(b);  // ReferenceError! ← TDZ에 걸림
let b = 20;
```

### TDZ(Temporal Dead Zone) 🟡

```
코드 실행 흐름:

┌─── TDZ (일시적 사각지대) ─────┐
│ let b; 선언은 되었지만         │ ← 여기서 접근하면 ReferenceError!
│ 아직 초기화되지 않은 구간      │
├───────────────────────────────┤
│ let b = 20;  ← 여기서 초기화   │ ← 여기부터 접근 가능
└───────────────────────────────┘
```

> 💡 **면접 포인트**: "let/const도 호이스팅 됩니다. 하지만 var와 달리 TDZ(Temporal Dead Zone)에 갇혀서 초기화 전에 접근하면 ReferenceError가 발생합니다."

### 함수 호이스팅

```js
// 함수 선언문: 통째로 호이스팅 (선언 전에 호출 가능!)
sayHello();  // "안녕!" ← 에러 없음!
function sayHello() { console.log("안녕!"); }

// 함수 표현식: 변수 호이스팅 규칙을 따름
sayBye();  // TypeError: sayBye is not a function
const sayBye = function() { console.log("잘 가!"); };
```

## 3. 데이터 타입 🔴

### 원시 타입(Primitive) — 7가지

> 💡 **비유: 포스트잇에 직접 적은 값**
> 원시 타입은 값 자체가 변수에 저장된다. 복사하면 독립적인 새 값이 생긴다.

```js
// 1. string (문자열)
const name = "홍길동";
const greeting = `안녕, ${name}!`;  // 템플릿 리터럴 (백틱)

// 2. number (숫자 — 정수/소수 구분 없음)
const age = 25;
const pi = 3.14;
const infinity = Infinity;
const notANumber = NaN;  // "숫자가 아님"인데 타입은 number 😱

// 3. boolean (참/거짓)
const isLoggedIn = true;
const hasPermission = false;

// 4. null (의도적인 "비어있음")
const selectedItem = null;  // "아직 아무것도 선택 안 했어"

// 5. undefined (아직 값이 할당되지 않음)
let result;
console.log(result);  // undefined

// 6. symbol (유일한 식별자 — ES6)
const id = Symbol("id");

// 7. bigint (아주 큰 정수 — ES2020)
const huge = 9007199254740991n;
```

### null vs undefined 🔴

```js
// undefined: JavaScript 엔진이 "아직 값이 없어"라고 자동 할당
let a;
console.log(a);  // undefined

// null: 개발자가 의도적으로 "비어있음"을 명시
let b = null;
console.log(b);  // null

// 비교
console.log(null == undefined);   // true (느슨한 비교: 둘 다 "없음")
console.log(null === undefined);  // false (엄격한 비교: 타입이 다름)

// typeof 함정!
console.log(typeof undefined);  // "undefined"
console.log(typeof null);       // "object" 😱 (JavaScript의 유명한 버그!)
```

> 💡 **실무 원칙**: 빈 값을 표현할 때는 `null`을 명시적으로 써라. `undefined`는 "아직 할당 안 됨"의 의미로 엔진에게 맡겨라.

### 참조 타입(Reference) — 객체, 배열, 함수

> 💡 **비유: 사물함 열쇠**
> 참조 타입은 값이 직접 저장되는 게 아니라, 값이 보관된 **사물함의 주소(참조)**가 저장된다.
> 복사하면 같은 사물함을 가리키는 열쇠가 하나 더 생긴다!

```js
// 원시 타입: 값 복사 (독립적)
let a = 10;
let b = a;       // b에 10이 복사됨
b = 20;
console.log(a);  // 10 (a는 영향 없음 ✅)

// 참조 타입: 주소 복사 (같은 객체를 가리킴!)
let obj1 = { name: "철수" };
let obj2 = obj1;           // obj2는 obj1과 같은 객체를 가리킴
obj2.name = "영희";
console.log(obj1.name);    // "영희" 😱 (obj1도 바뀜! 같은 사물함이니까!)
```

```
원시 타입 복사:          참조 타입 복사:
┌───┐  ┌───┐            ┌───┐  ┌───┐
│ a │  │ b │            │obj1│ │obj2│
│10 │  │10 │            │ ●──┤ │ ●──┤  ← 둘 다 같은 곳을 가리킴!
└───┘  └───┘            └───┘  └───┘
  ↓      ↓                  ↘  ↙
독립된 두 값              ┌─────────┐
                          │name:"영희"│ ← 하나를 바꾸면 둘 다 바뀜
                          └─────────┘
```

> ⚠️ 이 차이를 모르면 React에서 상태 업데이트 버그가 끝없이 발생한다.
> Part 2 - 22번(불변성)에서 이 문제를 깊이 다룬다.

## 4. typeof 연산자와 함정 🟡

```js
typeof "hello"      // "string"
typeof 42           // "number"
typeof true         // "boolean"
typeof undefined    // "undefined"
typeof Symbol()     // "symbol"
typeof 42n          // "bigint"

// 함정들!
typeof null         // "object" 😱 (JS 초창기 버그. 고칠 수 없음)
typeof []           // "object" (배열도 객체!)
typeof {}           // "object"
typeof function(){} // "function" (객체이지만 특별 취급)
typeof NaN          // "number" 😱 ("숫자가 아님"인데 number...)

// 배열 확인은 Array.isArray()를 써라
Array.isArray([1, 2, 3]);  // true
Array.isArray("hello");    // false
```

## 5. `===` vs `==` 🔴

```js
// == (느슨한 비교): 타입을 자동 변환 후 비교 → 예측 불가능!
0 == ""         // true 😱
0 == false      // true 😱
"" == false     // true 😱
null == undefined // true
"1" == 1        // true

// === (엄격한 비교): 타입까지 정확히 비교 → 안전!
0 === ""        // false ✅
0 === false     // false ✅
"1" === 1       // false ✅
null === undefined // false ✅
```

> 💡 **절대 규칙**: 항상 `===`를 써라. `==`는 쓸 이유가 없다.
> 유일한 예외: `value == null`은 `null`과 `undefined`를 동시에 체크하는 관용적 표현으로 허용.

## 면접 대비

### 🔴 필수 질문

**Q1. var, let, const의 차이를 설명해주세요.**

> `var`는 함수 스코프이고 재선언이 가능하며, 호이스팅 시 undefined로 초기화되어 선언 전 접근이 가능하다. `let`은 블록 스코프이고 재선언이 불가하며, 호이스팅 시 TDZ에 갇혀 선언 전 접근하면 ReferenceError가 발생한다. `const`는 `let`과 동일하지만 재할당이 불가하다. 단, 객체/배열의 경우 내부 속성 변경은 가능하다. 실무에서는 `const`를 기본으로 사용하고, 재할당이 필요할 때만 `let`을 쓴다.

**Q2. 호이스팅이란?**

> JavaScript 엔진이 코드를 실행하기 전에 변수와 함수의 선언을 메모리에 먼저 등록하는 동작이다. `var`는 선언과 동시에 undefined로 초기화되어 선언 전에도 접근이 가능하고, `let`/`const`는 선언만 등록되고 초기화는 실행 시점에 이루어져 그 사이 구간(TDZ)에서 접근하면 에러가 발생한다. 함수 선언문은 통째로 호이스팅되어 선언 전에 호출이 가능하다.

**Q3. 원시 타입과 참조 타입의 차이?**

> 원시 타입(string, number, boolean, null, undefined, symbol, bigint)은 값 자체가 변수에 저장되어 복사 시 독립적인 값이 생긴다. 참조 타입(객체, 배열, 함수)은 메모리의 주소(참조)가 저장되어 복사 시 같은 객체를 가리키게 된다. 따라서 참조 타입은 한쪽을 변경하면 다른 쪽도 영향을 받는다.

**Q4. `null`과 `undefined`의 차이?**

> `undefined`는 변수가 선언되었지만 아직 값이 할당되지 않은 상태로, JavaScript 엔진이 자동으로 부여한다. `null`은 개발자가 의도적으로 "값이 비어있음"을 명시한 것이다. `typeof null`은 "object"를 반환하는데 이는 JavaScript의 유명한 설계 버그다.

```
│                                                │

```
