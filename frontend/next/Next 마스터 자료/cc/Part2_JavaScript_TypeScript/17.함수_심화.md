## 1. 함수를 만드는 3가지 방법 🔴

```js
// 1. 함수 선언문 (Function Declaration)
function greet(name) {
  return `안녕, ${name}!`;
}

// 2. 함수 표현식 (Function Expression)
const greet = function(name) {
  return `안녕, ${name}!`;
};

// 3. 화살표 함수 (Arrow Function) — ES6+, 실무에서 가장 많이 사용
const greet = (name) => {
  return `안녕, ${name}!`;
};

// 화살표 함수 축약형
const greet = (name) => `안녕, ${name}!`;    // 한 줄이면 return 생략
const double = x => x * 2;                    // 매개변수 1개면 괄호 생략
const getUser = () => ({ name: "철수" });      // 객체 반환 시 소괄호 필수!
```

### 핵심 차이

| 구분 | 함수 선언문 | 함수 표현식 | 화살표 함수 |
|------|-----------|-----------|-----------|
| **호이스팅** | ✅ 통째로 끌어올림 | ❌ 변수만 호이스팅 | ❌ 변수만 호이스팅 |
| **this 바인딩** | ✅ 자체 this | ✅ 자체 this | ❌ 상위 스코프 this 상속 |
| **arguments** | ✅ 있음 | ✅ 있음 | ❌ 없음 (rest 사용) |
| **생성자(new)** | ✅ 가능 | ✅ 가능 | ❌ 불가 |

```js
// 호이스팅 차이 실전
sayHi();  // ✅ "하이!" — 함수 선언문은 호이스팅됨
function sayHi() { console.log("하이!"); }

sayBye();  // ❌ TypeError — 변수는 호이스팅 되지만 함수는 아직 할당 안 됨
const sayBye = () => console.log("잘 가!");
```

> 💡 **실무 원칙**: 컴포넌트 내부 함수는 `const fn = () => {}` (화살표 함수)
> 독립 유틸 함수는 `function fn() {}`도 OK (호이스팅이 유리할 때)

## 2. 매개변수 심화 🟡

### 기본값 (Default Parameters)

```js
// ES6+ 기본값
function createUser(name, role = "viewer") {
  return { name, role };
}
createUser("철수");           // { name: "철수", role: "viewer" }
createUser("영희", "admin");  // { name: "영희", role: "admin" }
```

### Rest 파라미터 (...rest)

```js
// 나머지 인수를 배열로 모아줌
function sum(...numbers) {
  return numbers.reduce((acc, num) => acc + num, 0);
}
sum(1, 2, 3);       // 6
sum(1, 2, 3, 4, 5); // 15

// 첫 번째는 따로, 나머지는 배열로
function log(first, ...rest) {
  console.log("첫 번째:", first);
  console.log("나머지:", rest);
}
log("a", "b", "c");  // 첫 번째: "a", 나머지: ["b", "c"]
```

> ⚠️ rest는 반드시 마지막 매개변수여야 한다: `(a, b, ...rest)` ✅ / `(...rest, a)` ❌

## 3. 일급 객체(First-class Object) 🔴

> 💡 **비유: VIP 시민권**
> JavaScript에서 함수는 "일급 시민"이다. 숫자나 문자열처럼 어디서든 사용할 수 있다.
> 이것을 이해하면 React의 Props로 함수를 넘기는 패턴이 자연스러워진다.

```js
// 1. 변수에 할당 가능
const add = (a, b) => a + b;

// 2. 다른 함수의 인수(매개변수)로 전달 가능
function execute(fn, x, y) {
  return fn(x, y);
}
execute(add, 3, 5);  // 8

// 3. 다른 함수의 반환값으로 사용 가능
function createMultiplier(factor) {
  return (number) => number * factor;
}
const double = createMultiplier(2);
const triple = createMultiplier(3);
double(5);  // 10
triple(5);  // 15

// 4. 배열/객체에 저장 가능
const operations = [add, (a, b) => a - b, (a, b) => a * b];
operations[0](3, 5);  // 8
```

> 💡 **React 연결**: `<Button onClick={handleClick}>` — 함수를 Props로 넘기는 것
> `[1,2,3].map(x => x * 2)` — 함수를 인수로 넘기는 것
> 이 모든 것이 "함수가 일급 객체"이기 때문에 가능하다.

## 4. 고차 함수(Higher-Order Function) 🔴

> 함수를 **인수로 받거나**, 함수를 **반환**하는 함수

```js
// 1. 함수를 인수로 받는 고차 함수
const numbers = [1, 2, 3, 4, 5];

// map: 각 요소를 변환하여 새 배열 반환
numbers.map(n => n * 2);         // [2, 4, 6, 8, 10]

// filter: 조건에 맞는 요소만 추출
numbers.filter(n => n > 3);      // [4, 5]

// reduce: 하나의 값으로 축약
numbers.reduce((acc, n) => acc + n, 0);  // 15

// 2. 함수를 반환하는 고차 함수
function withLogging(fn) {
  return function(...args) {
    console.log(`[LOG] 함수 호출: ${fn.name}`, args);
    const result = fn(...args);
    console.log(`[LOG] 결과:`, result);
    return result;
  };
}

const loggedAdd = withLogging(add);
loggedAdd(3, 5);
// [LOG] 함수 호출: add [3, 5]
// [LOG] 결과: 8
```

> 💡 **React에서 고차 함수**: `useEffect(콜백함수, [의존성])` — 콜백 패턴
> `arr.map(item => <Card key={item.id} {...item} />)` — 리스트 렌더링

## 5. 콜백 함수(Callback) 🔴

> 💡 **비유: 음식점 진동벨**
> "음식이 나오면(조건 충족되면) 이 벨을 울려주세요(콜백 실행)"
> 음식이 나올 때까지 자리에서 기다리는 것이 아니라, 벨이 울리면 받으러 간다.

```js
// 동기 콜백 — 즉시 실행 (map, filter 등)
[1, 2, 3].forEach(num => console.log(num));

// 비동기 콜백 — 나중에 실행
setTimeout(() => {
  console.log("2초 뒤에 실행!");
}, 2000);

// 이벤트 콜백
document.addEventListener("click", (event) => {
  console.log("클릭됨!", event.target);
});
```

> 💡 콜백은 **비동기 프로그래밍의 시작**이다.
> 하지만 콜백이 겹겹이 쌓이면 "콜백 지옥(Callback Hell)"이 발생한다.
> → 이 문제를 해결하기 위해 Promise와 async/await가 등장한다 (24~26번에서 다룸)

function myMap(array, callback) {
  // 여기에 구현
  // hint: 빈 배열을 만들고, for문으로 돌면서 callback(요소)를 push

// 과제 1
function myMap(array, callback) {
  const result = [];
  for (let i = 0; i < array.length; i++) {
    result.push(callback(array[i], i, array));
  }
  return result;
}

// 과제 2
function myFilter(array, callback) {
  const result = [];
  for (let i = 0; i < array.length; i++) {
    if (callback(array[i], i, array)) {
      result.push(array[i]);
    }
  }
  return result;
}

## 면접 대비

### 🔴 필수 질문

**Q1. 함수 선언문과 함수 표현식의 차이?**

> 함수 선언문(`function fn() {}`)은 호이스팅되어 선언 전에 호출이 가능하다. 함수 표현식(`const fn = function() {}`)은 변수 호이스팅 규칙을 따라 선언 전에 호출하면 에러가 발생한다. 화살표 함수도 표현식이므로 동일하다.

**Q2. 화살표 함수와 일반 함수의 차이?**

> 화살표 함수는 자체 `this`를 바인딩하지 않고 상위 스코프의 `this`를 상속한다(Lexical this). `arguments` 객체가 없으며 `new` 키워드로 생성자 호출이 불가하다. 짧은 콜백이나 React 이벤트 핸들러에 적합하다.

**Q3. 일급 객체(First-class Object)란?**

> 변수에 할당할 수 있고, 함수의 인수로 전달할 수 있으며, 함수의 반환값으로 사용할 수 있는 객체를 말한다. JavaScript에서 함수는 일급 객체이므로 `onClick={handleClick}`처럼 Props로 전달하거나, `arr.map(fn)`처럼 고차 함수에 인수로 넘길 수 있다.

**Q4. 고차 함수란?**

> 함수를 인수로 받거나 함수를 반환하는 함수다. `map`, `filter`, `reduce`가 대표적이다. React의 `useEffect(callback, deps)`, `addEventListener("click", handler)` 등도 콜백 패턴의 고차 함수 활용이다.
