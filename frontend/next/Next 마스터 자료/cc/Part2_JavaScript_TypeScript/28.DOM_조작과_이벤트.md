## 1. DOM이란? 🔴

> 💡 **비유: 가계도(Family Tree)**
> HTML 문서를 브라우저가 읽으면 **트리 구조의 객체 모델**로 변환한다.
> 이것이 DOM(Document Object Model). JavaScript로 이 트리를 읽고, 수정하고, 삭제할 수 있다.

```
HTML:                          DOM 트리:
<html>                         Document
  <body>                         └─ html
    <div id="app">                   └─ body
      <h1>제목</h1>                      └─ div#app
      <p>본문</p>                            ├─ h1 → "제목"
    </div>                                   └─ p → "본문"
  </body>
</html>
```

## 2. DOM 선택 🔴

```js
// 단일 선택 (가장 많이 사용)
document.querySelector("#app");          // ID로 선택
document.querySelector(".card");         // 첫 번째 .card
document.querySelector("h1");           // 첫 번째 h1
document.querySelector("[data-id='1']"); // 속성 선택자

// 다중 선택 (NodeList 반환)
document.querySelectorAll(".card");      // 모든 .card (forEach 가능)
document.querySelectorAll("li");         // 모든 li

// 오래된 방식 (아직 쓰이지만 querySelector가 더 유연)
document.getElementById("app");
document.getElementsByClassName("card");  // HTMLCollection (유사 배열)
```

## 3. 이벤트 버블링과 캡처링 🔴

> 💡 **비유: 물에 돌 던지기**
> 물에 돌을 던지면 파문이 **안에서 바깥으로** 퍼진다. 이것이 **버블링**.
> 반대로 바깥에서 안으로 좁혀가는 것이 **캡처링**.

```html
<div id="grandparent">
  <div id="parent">
    <button id="child">클릭!</button>
  </div>
</div>
```

```
클릭 이벤트 전파 순서:

① 캡처링 단계 (위 → 아래): 잘 안 씀
   Document → html → body → grandparent → parent → child

② 타겟 단계: 클릭된 요소 자체
   child (이벤트 발생!)

③ 버블링 단계 (아래 → 위): 기본 동작! ⭐
   child → parent → grandparent → body → html → Document
```

```js
// 버블링 확인
document.getElementById("grandparent").addEventListener("click", () => {
  console.log("grandparent 클릭!");
});
document.getElementById("parent").addEventListener("click", () => {
  console.log("parent 클릭!");
});
document.getElementById("child").addEventListener("click", () => {
  console.log("child 클릭!");
});

// child 버튼 클릭 시 출력:
// "child 클릭!"      ← 타겟
// "parent 클릭!"     ← 버블링
// "grandparent 클릭!" ← 버블링

// 버블링 중단 (필요할 때만!)
element.addEventListener("click", (e) => {
  e.stopPropagation();  // 이 요소에서 버블링 중단
});
```

## 4. event.target vs event.currentTarget 🔴

```js
// target: 이벤트가 실제로 발생한 요소 (클릭된 그 놈!)
// currentTarget: 이벤트 리스너가 붙어있는 요소 (addEventListener한 그 놈!)

document.getElementById("parent").addEventListener("click", (e) => {
  console.log("target:", e.target);         // 실제 클릭된 요소 (child일 수도!)
  console.log("currentTarget:", e.currentTarget); // 항상 parent
});
```

> 💡 **React에서**: `e.target`은 실제 클릭된 DOM, `e.currentTarget`은 핸들러가 붙은 DOM.
> React의 합성 이벤트(SyntheticEvent)도 동일한 속성을 가진다.

## 5. 이벤트 위임(Event Delegation) 🔴

> 💡 **비유: 반 대표가 공지 전달**
> 학생 30명 각각에게 말하는 대신, 반 대표 1명에게 말하면 전달이 된다.
> 자식 100개에 각각 이벤트를 거는 대신, 부모 1개에 걸고 `event.target`으로 구분한다.

```js
// ❌ 나쁜 패턴: 각 li에 개별 이벤트 등록 (100개면 100개 리스너!)
document.querySelectorAll("li").forEach(li => {
  li.addEventListener("click", () => console.log(li.textContent));
});

// ✅ 이벤트 위임: 부모 ul에 1개만 등록!
document.querySelector("ul").addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    console.log(e.target.textContent);  // 클릭된 li의 텍스트
  }
});
```

> 💡 이벤트 위임은 **버블링**을 이용한 것!
> 자식 클릭 → 이벤트가 부모로 버블링 → 부모의 리스너가 받아서 target으로 구분

### 이벤트 위임이 중요한 이유

```
1. 성능: 리스너 1개 vs N개 (메모리 절약)
2. 동적 요소: 나중에 추가된 자식에도 자동으로 작동!
3. React 내부: React가 내부적으로 이벤트 위임을 사용한다
   (모든 이벤트를 root에 위임하여 처리)
```

## 6. DOM 조작 (React에서는 직접 하지 않지만 원리는 알아야 함) 🟡

```js
// 생성
const div = document.createElement("div");
div.textContent = "새 요소";
div.className = "card";
div.setAttribute("data-id", "123");

// 추가
document.body.appendChild(div);          // 맨 뒤에 추가
parent.insertBefore(div, referenceNode); // 특정 위치에 삽입

// 삭제
element.remove();                        // 자기 자신 삭제
parent.removeChild(child);               // 자식 삭제

// 수정
element.textContent = "새 텍스트";        // 텍스트만
element.innerHTML = "<b>HTML도 가능</b>"; // ⚠️ XSS 주의!
element.style.color = "red";              // 인라인 스타일
element.classList.add("active");          // 클래스 추가
element.classList.remove("active");       // 클래스 제거
element.classList.toggle("active");       // 토글
```

> ⚠️ **React에서는 DOM을 직접 조작하지 않는다!** Virtual DOM을 통해 간접적으로 처리.
> 하지만 `useRef`로 DOM에 접근하거나, 외부 라이브러리 통합 시 이 지식이 필요하다.

## 면접 대비

### 🔴 필수 질문

**Q1. 이벤트 버블링이란?**

> DOM에서 이벤트가 발생하면 해당 요소에서 시작하여 부모 → 조부모 → document까지 상위로 전파되는 현상이다. 이를 이용한 이벤트 위임 패턴에서는 부모 요소에 하나의 리스너만 등록하고 `event.target`으로 실제 클릭된 자식을 구분하여 처리한다.

**Q2. `event.target`과 `event.currentTarget`의 차이?**

> `event.target`은 이벤트가 실제로 발생한(클릭된) 요소이고, `event.currentTarget`은 이벤트 리스너가 등록된 요소이다. 이벤트 위임에서 부모에 리스너를 달면, `currentTarget`은 항상 부모이고 `target`은 실제 클릭된 자식이 된다.

**Q3. 이벤트 위임이란? 왜 사용하나요?**

> 자식 요소들에 개별적으로 이벤트를 등록하는 대신, 공통 부모에 하나의 리스너를 등록하고 버블링을 이용해 처리하는 패턴이다. 메모리 효율적이고, 동적으로 추가되는 요소에도 자동으로 작동한다. React 내부에서도 이벤트 위임을 사용하여 루트에서 모든 이벤트를 통합 관리한다.
