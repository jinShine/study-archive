# 30. 브라우저 저장소

> **키워드**: `localStorage` `sessionStorage` `Cookie` `IndexedDB` `HttpOnly` `Secure` `SameSite`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | localStorage vs sessionStorage vs Cookie 비교, Cookie 보안 속성 | 인증/상태 저장의 기본 |
| 🟡 이해 | 각 저장소의 실무 사용처, XSS/CSRF 위험 | 보안 면접 |
| 🟢 참고 | IndexedDB, Cache API | 오프라인/PWA |

---

## 1. 3대 저장소 비교표 🔴

| 구분 | `localStorage` | `sessionStorage` | `Cookie` |
|------|---------------|-----------------|----------|
| **용량** | ~5MB | ~5MB | ~4KB |
| **수명** | 영구 (직접 삭제까지) | 탭 닫으면 삭제 | `Max-Age`/`Expires` 설정 |
| **서버 전송** | ❌ 안 보냄 | ❌ 안 보냄 | ✅ 매 요청마다 자동 전송! |
| **접근** | JS로 자유롭게 | JS로 자유롭게 | `HttpOnly`면 JS 접근 차단! |
| **스코프** | 도메인 전체 | 탭 단위 | 도메인 + Path |

```js
// localStorage — 영구 저장 (다크 모드, 언어 설정 등)
localStorage.setItem("theme", "dark");
localStorage.getItem("theme");    // "dark"
localStorage.removeItem("theme");
localStorage.clear();              // 전체 삭제

// sessionStorage — 탭 한정 (임시 폼 데이터 등)
sessionStorage.setItem("step", "2");

// Cookie — 서버와 자동 통신 (인증 토큰!)
document.cookie = "token=abc123; Max-Age=3600; Secure; SameSite=Strict";
```

---

## 2. 실무에서 뭘 어디에 저장하나? 🔴

```
localStorage:  다크 모드, 언어 설정, 최근 검색어 (민감하지 않은 UI 설정)
sessionStorage: 폼 임시 저장, 한 탭에서만 필요한 상태
Cookie:        인증 토큰(JWT), 세션 ID (서버가 읽어야 하는 데이터)

⚠️ 절대 localStorage에 넣으면 안 되는 것:
  - JWT 토큰 (XSS로 털림!)
  - 비밀번호
  - 개인정보

✅ JWT는 HttpOnly Cookie에 저장하는 것이 보안 정석
```

---

## 3. Cookie 보안 속성 🔴

```
Set-Cookie: token=eyJhbG...; HttpOnly; Secure; SameSite=Strict; Max-Age=3600

HttpOnly   → JS(document.cookie)로 접근 불가! XSS 방어
Secure     → HTTPS에서만 전송! 도청 방어
SameSite   → 다른 사이트에서 요청 시 쿠키 안 보냄! CSRF 방어
  - Strict: 완전 차단 (같은 사이트에서만)
  - Lax: 링크 이동은 허용, POST는 차단 (기본값)
  - None: 제한 없음 (반드시 Secure와 함께)
```

> 💡 **면접 핵심**: "JWT를 localStorage가 아닌 HttpOnly Cookie에 저장해야 하는 이유?"
> → XSS 공격으로 `localStorage`는 JS 한 줄이면 털리지만, `HttpOnly` 쿠키는 JS 접근 자체가 차단된다.

## 면접 대비

### 🔴 필수 질문

**Q1. localStorage, sessionStorage, Cookie의 차이?**

> localStorage는 영구 저장(5MB)되고 서버에 전송되지 않는다. sessionStorage는 탭이 닫히면 삭제되고 서버 전송되지 않는다. Cookie는 용량이 작지만(4KB) 매 HTTP 요청마다 서버로 자동 전송되며, HttpOnly/Secure/SameSite 보안 속성을 설정할 수 있다.

**Q2. JWT를 localStorage에 저장하면 안 되는 이유?**

> XSS(Cross-Site Scripting) 공격에 취약하다. 악성 스크립트가 `window.localStorage.getItem("token")`으로 토큰을 탈취할 수 있다. HttpOnly Cookie에 저장하면 JavaScript에서 접근 자체가 차단되어 XSS로부터 안전하다.
