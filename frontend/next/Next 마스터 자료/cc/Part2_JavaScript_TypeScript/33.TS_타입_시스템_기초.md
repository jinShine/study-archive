## 1. TypeScript를 쓰는 이유 🔴

> 💡 **비유: 안전벨트**
> 안전벨트 없이 운전할 수는 있다. 하지만 사고가 나면 치명적이다.
> TS는 코드의 안전벨트. 실행하기 **전에** 버그를 잡아준다.

```js
// ❌ JavaScript — 실행해봐야 에러를 안다 (런타임 에러)
function add(a, b) {
  return a + b;
}
add("hello", 5);  // "hello5" 😱 (의도하지 않은 문자열 연결)

// ✅ TypeScript — 코드를 쓰는 순간 에러를 알려준다 (컴파일 타임 에러)
function add(a: number, b: number): number {
  return a + b;
}
add("hello", 5);  // ❌ 빨간 밑줄! "string은 number에 할당할 수 없습니다"
```

### TS의 3대 이점

```
1. 버그 조기 발견: 실행 전에 타입 에러 감지
2. 자동 완성(IntelliSense): 에디터가 속성/메서드를 제안
3. 문서화: 코드 자체가 타입 문서 역할 (주석 필요 ↓)
```

## 2. 타입 추론 (Type Inference) 🔴

> TS는 대부분의 경우 타입을 **자동으로 추론**한다. 모든 곳에 타입을 적을 필요 없다!

```ts
// 변수 — 초기값으로 추론
let name = "철수";        // string으로 추론됨
let age = 25;             // number로 추론됨
let isActive = true;      // boolean으로 추론됨

name = 42;  // ❌ 에러! string 변수에 number 할당 불가

// 함수 반환값 — 자동 추론
function add(a: number, b: number) {
  return a + b;  // 반환 타입이 number로 자동 추론됨
}

// const는 리터럴 타입으로 추론!
const greeting = "hello";  // 타입: "hello" (string이 아님!)
let greeting2 = "hello";   // 타입: string
```

> 💡 **실무 원칙**: 추론이 되는 곳에 타입을 적지 마라. 중복이다.
> 매개변수, 복잡한 반환값, export하는 것에만 명시적으로 적어라.

## 3. 기본 타입 🔴

```ts
// 원시 타입
let str: string = "hello";
let num: number = 42;
let bool: boolean = true;
let n: null = null;
let u: undefined = undefined;

// 배열
let numbers: number[] = [1, 2, 3];
let names: string[] = ["철수", "영희"];
let mixed: (string | number)[] = ["hello", 42];

// 제네릭 배열 문법 (같은 의미)
let numbers2: Array<number> = [1, 2, 3];

// 튜플 — 고정 길이 + 각 위치의 타입이 정해진 배열
let point: [number, number] = [10, 20];        // [x, y]
let entry: [string, number] = ["철수", 25];     // [이름, 나이]
// React의 useState 반환값이 사실 튜플!
// const [count, setCount] = useState(0);  → [number, (n: number) => void]

// 객체
let user: { name: string; age: number; email?: string } = {
  name: "철수",
  age: 25,
  // email은 선택적(?)이므로 없어도 됨
};

// 함수
function greet(name: string): string {
  return `안녕, ${name}!`;
}

// 화살표 함수 타입
const add = (a: number, b: number): number => a + b;

// void — 반환값이 없는 함수
function log(message: string): void {
  console.log(message);
}
```

## 4. 유니온(Union)과 인터섹션(Intersection) 🔴

### 유니온 (`|`) — "이것 또는 저것"

```ts
// string 또는 number
let id: string | number;
id = "abc";   // ✅
id = 123;     // ✅
id = true;    // ❌ boolean은 안 됨!

// 함수 매개변수에서
function formatId(id: string | number): string {
  // 타입 좁히기(Narrowing) 필요!
  if (typeof id === "string") {
    return id.toUpperCase();  // string 메서드 사용 가능
  }
  return String(id);          // number를 string으로 변환
}

// null 허용 패턴 (가장 많이 쓰는 유니온!)
let selectedUser: User | null = null;
```

### 인터섹션 (`&`) — "이것 그리고 저것"

```ts
type HasName = { name: string };
type HasAge = { age: number };

// 두 타입을 합침 (모든 속성을 가져야 함)
type Person = HasName & HasAge;

const person: Person = {
  name: "철수",
  age: 25,
  // 둘 다 필수!
};
```

## 5. 타입 별칭 (Type Alias) 🔴

```ts
// type 키워드로 타입에 이름 붙이기
type UserId = string | number;
type Point = { x: number; y: number };

// 복잡한 타입도 깔끔하게
type ApiResponse<T> = {
  data: T;
  status: number;
  message: string;
};

// 사용
const user: ApiResponse<{ name: string }> = {
  data: { name: "철수" },
  status: 200,
  message: "성공",
};
```

## 6. 리터럴 타입 🟡

```ts
// 특정 값만 허용
type Direction = "up" | "down" | "left" | "right";
let dir: Direction = "up";    // ✅
dir = "diagonal";             // ❌ 에러!

// 숫자 리터럴
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;

// 실전: API 상태
type Status = "idle" | "loading" | "success" | "error";

// React 컴포넌트 Props에서 매우 자주 사용!
type ButtonVariant = "primary" | "secondary" | "danger";
function Button({ variant }: { variant: ButtonVariant }) { }
```

## 7. 실습: 타입 붙이기 연습

```ts
// 아래 JavaScript 코드에 TypeScript 타입을 추가하라

// 과제 1
const users = [
  { id: 1, name: "철수", age: 25, isAdmin: true },
  { id: 2, name: "영희", age: 30, isAdmin: false },
];

// 과제 2
function getUser(id) {
  return users.find(u => u.id === id) || null;
}

// 과제 3
function formatPrice(price, currency) {
  if (currency === "KRW") return `${price.toLocaleString()}원`;
  if (currency === "USD") return `$${price.toFixed(2)}`;
  return `${price}`;
}
```

// 과제 1
type User = {
  id: number;

  age: number;
  isAdmin: boolean;

const users: User[] = [

function getUser(id: number): User | null {

function formatPrice(price: number, currency: Currency): string {

}

## 면접 대비

### 🔴 필수 질문

**Q1. TypeScript를 사용하는 이유?**

> 정적 타입 시스템을 통해 코드 실행 전(컴파일 타임)에 타입 에러를 감지하여 버그를 조기에 방지한다. 에디터의 자동 완성과 리팩토링 지원이 강화되고, 코드 자체가 타입 문서 역할을 하여 팀 협업 시 의사소통 비용이 줄어든다.

**Q2. 유니온 타입에서 타입 좁히기(Narrowing)란?**

> 유니온 타입(`string | number`)의 변수를 사용할 때, `typeof`, `in`, `instanceof` 등의 조건문으로 현재 정확한 타입을 판별하는 것이다. TypeScript 컴파일러는 조건문 내부에서 타입을 자동으로 좁혀주므로 해당 타입의 메서드를 안전하게 사용할 수 있다.
