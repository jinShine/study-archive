## 1. async/await = Promise의 문법적 설탕(Syntactic Sugar) 🔴

> 💡 **비유: 같은 요리, 다른 식기**
> Promise 체이닝 = 젓가락으로 먹기 (가능하지만 체이닝이 길어지면 불편)
> async/await = 포크와 나이프로 먹기 (동기 코드처럼 읽기 쉬움)
> 요리(동작)는 완전히 동일하고, 먹는 도구(문법)만 다르다.

```js
// Promise 체이닝 버전
function getUser() {
  return fetch("/api/user")
    .then(res => res.json())
    .then(user => {
      console.log(user.name);
      return user;
    })
    .catch(err => console.error(err));
}

// ✅ async/await 버전 (동기 코드처럼 읽힌다!)
async function getUser() {
  try {
    const res = await fetch("/api/user");
    const user = await res.json();
    console.log(user.name);
    return user;
  } catch (err) {
    console.error(err);
  }
}
```

### 핵심 규칙

```
1. async 함수는 항상 Promise를 반환한다
2. await는 async 함수 안에서만 사용 가능
3. await는 Promise가 settled될 때까지 실행을 "일시 중단"
4. await 뒤의 코드는 Microtask Queue에 등록됨 (24번 참고)
```

```js
async function foo() {
  return "hello";
}
foo();  // Promise { "hello" } ← async 함수는 항상 Promise 반환!
foo().then(val => console.log(val));  // "hello"
```

## 2. try-catch 에러 핸들링 🔴

```js
async function fetchData() {
  try {
    const res = await fetch("/api/data");

    if (!res.ok) {
      throw new Error(`HTTP 에러! 상태: ${res.status}`);
    }

    const data = await res.json();
    return data;

  } catch (error) {
    // fetch 실패(네트워크 에러) 또는 throw된 에러 모두 여기서 처리
    console.error("데이터 가져오기 실패:", error.message);
    return null;  // 또는 에러를 다시 던지기: throw error;

  } finally {
    // 성공/실패 무관하게 항상 실행 (로딩 상태 해제 등)
    console.log("요청 완료");
  }
}
```

> 💡 **실무 원칙**: `fetch`는 **네트워크 에러에서만** reject된다.
> 404나 500 같은 HTTP 에러는 reject가 아니므로 `res.ok`로 직접 체크해야 한다!

```js
// ⚠️ fetch의 함정
try {
  const res = await fetch("/api/not-exist");  // 404
  console.log(res.ok);    // false (reject 아님!)
  console.log(res.status); // 404
  // fetch는 에러를 안 던짐! 직접 체크해야 함!
} catch (err) {
  // 여기는 네트워크 자체가 끊겼을 때만 실행됨
}
```

## 3. 순차 실행 vs 병렬 실행 🔴

### ❌ 실수: 불필요한 순차 실행 (직렬)

```js
// 3개 API가 각각 1초씩 걸린다면...
async function loadDashboard() {
  const users = await fetch("/api/users").then(r => r.json());     // 1초
  const products = await fetch("/api/products").then(r => r.json()); // +1초
  const orders = await fetch("/api/orders").then(r => r.json());     // +1초
  // 총 3초! 😱 (서로 의존성이 없는데 기다림)
}
```

### ✅ 정석: 독립적인 요청은 병렬로!

```js
async function loadDashboard() {
  const [users, products, orders] = await Promise.all([
    fetch("/api/users").then(r => r.json()),
    fetch("/api/products").then(r => r.json()),
    fetch("/api/orders").then(r => r.json()),
  ]);
  // 총 1초! ⚡ (가장 오래 걸리는 것 기준)
}
```

### 판단 기준

```
"두 번째 요청이 첫 번째 결과에 의존하는가?"

YES → 순차(await → await)
  const user = await getUser(id);
  const orders = await getOrders(user.id);  // user.id가 필요!

NO → 병렬(Promise.all)
  const [user, products] = await Promise.all([
    getUser(id),
    getProducts(),  // user와 무관!
  ]);
```

## 4. 화살표 함수와 async 🟡

```js
// 화살표 함수에도 async 사용 가능
const fetchUser = async (id) => {
  const res = await fetch(`/api/users/${id}`);
  return res.json();
};

// React 이벤트 핸들러에서
const handleSubmit = async (e) => {
  e.preventDefault();
  try {
    await saveData(formData);
    alert("저장 완료!");
  } catch {
    alert("저장 실패!");
  }
};

// useEffect에서 async 사용 시 주의!
useEffect(() => {
  // ❌ useEffect 콜백 자체를 async로 하면 안 됨!
  // useEffect(async () => { ... })  ← 에러!

  // ✅ 내부에 async 함수를 만들어서 호출
  const loadData = async () => {
    const data = await fetchUser(1);
    setUser(data);
  };
  loadData();
}, []);
```

## 5. 실습: 비동기 데이터 로딩

```js
// 과제 1: 순차 → 병렬 리팩토링
// 아래 코드를 Promise.all로 최적화하라
async function loadPage() {
  const header = await fetch("/api/header").then(r => r.json());
  const sidebar = await fetch("/api/sidebar").then(r => r.json());
  const content = await fetch("/api/content").then(r => r.json());
  const footer = await fetch("/api/footer").then(r => r.json());
  return { header, sidebar, content, footer };
}

// 과제 2: 에러 핸들링이 포함된 데이터 로딩 함수
// - /api/users 호출
// - res.ok 체크 (아니면 에러 throw)
// - 성공 시 JSON 파싱 후 반환

  console.log("1");
  const result = await Promise.resolve("2");
  console.log(result);
  console.log("3");
}
console.log("A");
quiz();
console.log("B");
```

## 면접 대비

### 🔴 필수 질문

**Q1. async/await와 Promise의 관계?**

> async/await는 Promise의 문법적 설탕이다. `async` 함수는 항상 Promise를 반환하고, `await`는 Promise가 settled될 때까지 해당 async 함수의 실행을 일시 중단한다. 내부적으로는 `.then()` 체이닝과 동일하게 동작하지만, 동기 코드처럼 위에서 아래로 읽을 수 있어 가독성이 훨씬 뛰어나다.

**Q2. 여러 비동기 요청을 최적화하는 방법?**

> 서로 의존성이 없는 요청은 `Promise.all`로 병렬 실행한다. `await A(); await B();`는 A가 끝날 때까지 B가 시작되지 않아 불필요한 대기가 발생한다. `await Promise.all([A(), B()])`는 동시에 시작하여 가장 느린 요청 시간만큼만 대기한다. 의존성이 있는 경우에만 순차 `await`를 사용한다.

```
│            async/await 핵심 요약                │
├────────────────────────────────────────────────┤
│                                                │

│                                                │

│  순차 vs 병렬:                                   │
│    의존 있으면 → await A; await B (순차)          │

│  useEffect에서: 내부에 async 함수 선언 후 호출    │
```
