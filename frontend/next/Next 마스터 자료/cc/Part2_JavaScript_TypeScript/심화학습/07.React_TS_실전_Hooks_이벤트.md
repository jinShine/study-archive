## 1. useState 타이핑 🔴

```tsx
// 타입 추론이 되는 경우 → 제네릭 생략 OK
const [count, setCount] = useState(0);          // number로 추론
const [name, setName] = useState("철수");        // string으로 추론
const [isOpen, setIsOpen] = useState(false);     // boolean으로 추론

// ⚠️ 추론이 안 되는 경우 → 제네릭 명시 필수!

// 1. 초기값이 null인 경우
const [user, setUser] = useState<User | null>(null);
// 제네릭 없으면 → useState(null) → 타입이 null로 고정되어 User 할당 불가!

// 2. 빈 배열인 경우
const [items, setItems] = useState<Product[]>([]);
// 제네릭 없으면 → useState([]) → never[]로 추론되어 아무것도 push 불가!

// 3. 유니온 리터럴 타입인 경우
const [status, setStatus] = useState<"idle" | "loading" | "error">("idle");
// 제네릭 없으면 → useState("idle") → "idle"로만 추론

// 4. 복잡한 객체
const [formData, setFormData] = useState<{
  name: string;
  email: string;
  age: number | null;
}>({
  name: "",
  email: "",
  age: null,
});
```

### setState 콜백 패턴

```tsx
// ✅ 이전 상태를 기반으로 업데이트할 때 콜백 사용
setCount(prev => prev + 1);
setItems(prev => [...prev, newItem]);
setFormData(prev => ({ ...prev, name: "새이름" }));
// prev의 타입은 자동 추론됨!
```

## 2. useRef 타이핑 🔴

```tsx
// 1. DOM 접근용 ref (가장 많이 사용!)
const inputRef = useRef<HTMLInputElement>(null);
// 초기값 null! → 마운트 후 DOM에 연결됨

// 사용
<input ref={inputRef} />
inputRef.current?.focus();  // ?. 필수! (null일 수 있으므로)
inputRef.current?.value;    // HTMLInputElement의 모든 속성 자동완성!

// 2. 다른 HTML 요소
const divRef = useRef<HTMLDivElement>(null);
const buttonRef = useRef<HTMLButtonElement>(null);
const formRef = useRef<HTMLFormElement>(null);
const canvasRef = useRef<HTMLCanvasElement>(null);

// 3. 값 저장용 ref (리렌더링 없이 값 유지)
const renderCount = useRef<number>(0);
const timerIdRef = useRef<NodeJS.Timeout | null>(null);
const prevValueRef = useRef<string>("");

// ⚠️ DOM ref는 null로 초기화, 값 ref는 실제 값으로 초기화
// DOM: useRef<HTMLInputElement>(null)  → .current는 읽기 전용
// 값:  useRef<number>(0)             → .current는 수정 가능
```

## 3. 이벤트 타입 총정리 🔴

### 이벤트 핸들러 인라인 vs 분리

```tsx
// 인라인: 타입 추론이 자동으로 됨!
<input onChange={(e) => {
  console.log(e.target.value);  // e가 자동으로 ChangeEvent<HTMLInputElement>
}} />

// 분리: 타입을 직접 명시해야 함
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
  console.log(e.target.value);
};
<input onChange={handleChange} />
```

### 이벤트 타입 치트시트


### 이벤트 핸들러 타입 (Props로 넘길 때)

```tsx

type Props = {
  onClick: React.MouseEventHandler<HTMLButtonElement>;
  onChange: React.ChangeEventHandler<HTMLInputElement>;
  onSubmit: React.FormEventHandler<HTMLFormElement>;
};

// 위는 아래와 동일:
type Props = {
  onClick: (e: React.MouseEvent<HTMLButtonElement>) => void;
  onChange: (e: React.ChangeEvent<HTMLInputElement>) => void;
  onSubmit: (e: React.FormEvent<HTMLFormElement>) => void;
};
```

## 4. useReducer 타이핑 — 판별 유니온의 꽃 🟡

```tsx
// 1. State 타입
type CounterState = {
  count: number;
  step: number;
  history: number[];
};

// 2. Action 타입 (판별 유니온!)
type CounterAction =
  | { type: "INCREMENT" }
  | { type: "DECREMENT" }
  | { type: "SET_STEP"; payload: number }
  | { type: "RESET" }
  | { type: "UNDO" };

// 3. Reducer 함수
function counterReducer(state: CounterState, action: CounterAction): CounterState {
  switch (action.type) {
    case "INCREMENT":
      return {
        ...state,
        count: state.count + state.step,
        history: [...state.history, state.count],
      };
    case "DECREMENT":
      return {
        ...state,
        count: state.count - state.step,
        history: [...state.history, state.count],
      };
    case "SET_STEP":
      return { ...state, step: action.payload };
      // ✅ payload 타입이 자동으로 number! (판별 유니온 덕분)
    case "RESET":
      return { count: 0, step: 1, history: [] };
    case "UNDO":
      const prev = state.history[state.history.length - 1];
      return {
        ...state,
        count: prev ?? 0,
        history: state.history.slice(0, -1),
      };
  }
}

// 4. 컴포넌트에서 사용
function Counter() {
  const [state, dispatch] = useReducer(counterReducer, {
    count: 0,
    step: 1,
    history: [],
  });

  return (
    <div>
      <p>{state.count}</p>
      <button onClick={() => dispatch({ type: "INCREMENT" })}>+</button>
      <button onClick={() => dispatch({ type: "SET_STEP", payload: 5 })}>Step 5</button>
      {/* ❌ 에러! INCREMENT에는 payload가 없음! */}
      {/* dispatch({ type: "INCREMENT", payload: 5 }) */}
    </div>
  );
}
```

> 💡 `useReducer` + 판별 유니온 = 복잡한 상태 로직의 완벽한 타입 안전성!
> dispatch에 잘못된 action을 넘기면 즉시 컴파일 에러!

## 5. useCallback / useMemo 타이핑 🟡

```tsx
// useCallback — 보통 타입 추론이 됨
const handleClick = useCallback((id: number) => {
  console.log(id);
}, []);
// (id: number) => void 로 자동 추론

// useMemo — 보통 타입 추론이 됨
const expensiveValue = useMemo(() => {
  return items.filter(item => item.price > 1000);
}, [items]);
// Product[]로 자동 추론

// 명시가 필요한 경우 (드뭄)
const fn = useCallback<(id: number) => Promise<User>>(async (id) => {
  const res = await fetch(`/api/users/${id}`);
  return res.json();
}, []);
```

## 6. 실습: 타입 안전한 폼 컴포넌트

```tsx
// 과제: 회원가입 폼을 완전한 타입 안전성으로 구현

// 요구사항:
// 1. FormState 타입 (name, email, password, agreed: boolean)
// 2. handleChange — input의 name을 키로 사용하여 상태 업데이트
//    힌트: e.target.name을 keyof FormState로 타이핑
// 3. handleSubmit — e: FormEvent, preventDefault
// 4. 에러 상태: Partial<Record<keyof FormState, string>>
// 5. useRef로 이름 input에 자동 포커스
```

<summary>정답 보기</summary>

```tsx
type FormState = {
  name: string;
  email: string;
  password: string;
  agreed: boolean;
};

type FormErrors = Partial<Record<keyof FormState, string>>;

function SignupForm() {
  const [form, setForm] = useState<FormState>({
    name: "", email: "", password: "", agreed: false,
  });
  const [errors, setErrors] = useState<FormErrors>({});
  const nameRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    nameRef.current?.focus();
  }, []);

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const { name, value, type, checked } = e.target;
    setForm(prev => ({
      ...prev,
      [name]: type === "checkbox" ? checked : value,
    }));
  };

  const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
    e.preventDefault();
    const newErrors: FormErrors = {};
    if (!form.name) newErrors.name = "이름을 입력하세요";
    if (!form.email.includes("@")) newErrors.email = "올바른 이메일을 입력하세요";
    if (form.password.length < 8) newErrors.password = "8자 이상 입력하세요";
    if (!form.agreed) newErrors.agreed = "약관에 동의해주세요";

    if (Object.keys(newErrors).length > 0) {
      setErrors(newErrors);
      return;
    }
    console.log("제출:", form);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={nameRef} name="name" value={form.name} onChange={handleChange} />
      {errors.name && <p>{errors.name}</p>}

      <input name="email" type="email" value={form.email} onChange={handleChange} />
      {errors.email && <p>{errors.email}</p>}

      <input name="password" type="password" value={form.password} onChange={handleChange} />
      {errors.password && <p>{errors.password}</p>}

      <label>
        <input name="agreed" type="checkbox" checked={form.agreed} onChange={handleChange} />
        약관 동의
      </label>
      {errors.agreed && <p>{errors.agreed}</p>}

      <button type="submit">가입</button>
    </form>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. `useState`에 제네릭을 명시해야 하는 경우는?**

> 초기값으로 타입을 추론할 수 없을 때 제네릭이 필요하다. 대표적으로 `useState<User | null>(null)` (초기값 null), `useState<Product[]>([])` (빈 배열), `useState<"idle" | "loading">("idle")` (유니온 리터럴)이 있다. 초기값이 명확한 숫자나 문자열은 추론되므로 생략한다.

**Q2. React 이벤트 타입에서 `ChangeEvent`와 `FormEvent`의 차이?**

> `ChangeEvent<HTMLInputElement>`는 input, select, textarea의 값이 변경될 때 발생하며 `e.target.value`로 값에 접근한다. `FormEvent<HTMLFormElement>`는 form이 제출될 때 발생하며 `e.preventDefault()`로 기본 동작을 막고 `new FormData(e.currentTarget)`으로 전체 폼 데이터를 가져온다.
