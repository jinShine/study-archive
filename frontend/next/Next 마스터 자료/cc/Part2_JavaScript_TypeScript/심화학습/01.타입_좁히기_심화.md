# TS 심화 01. 타입 좁히기(Narrowing) 심화

> **키워드**: `typeof` `instanceof` `in` `판별 유니온` `사용자 정의 타입 가드` `Assertion Function` `제어 흐름 분석`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 모든 Narrowing 기법 총정리, 제어 흐름 분석 | 실무에서 매분 사용 |
| 🟡 이해 | Assertion Function, 커스텀 가드 설계 | 안전한 코드 |
| 🟢 참고 | 타입 predicate 고급, Branded Type | 라이브러리 설계 |

---

## 1. Narrowing이란? — TS의 진짜 힘 🔴

> 💡 **비유: 범인 좁히기**
> 처음에 "용의자는 아무나(string | number | null)"다.
> 단서(조건문)를 하나씩 추가하면 범인이 특정된다.
> TS 컴파일러가 조건문 안에서 타입을 자동으로 좁혀주는 것이 Narrowing이다.

```ts
function process(value: string | number | null) {
  // 이 시점: value는 string | number | null

  if (value === null) {
    return;  // 이후: value는 string | number (null 제거!)
  }

  // 이 시점: value는 string | number

  if (typeof value === "string") {
    value.toUpperCase();  // TS가 string으로 좁혀줌!
  } else {
    value.toFixed(2);     // TS가 number로 좁혀줌!
  }
}
```

## 2. 모든 Narrowing 기법 총정리 🔴

### 기법 1: typeof 가드

```ts
function format(value: string | number | boolean) {
  if (typeof value === "string") {
    return value.trim().toUpperCase();   // string
  }
  if (typeof value === "number") {
    return value.toFixed(2);             // number
  }
  return value ? "Yes" : "No";          // boolean
}

// typeof가 구분할 수 있는 것:
// "string", "number", "boolean", "undefined", "object", "function", "symbol", "bigint"
// ⚠️ typeof null === "object" (구분 불가! null은 === null로 체크)
// ⚠️ typeof [] === "object" (배열 구분 불가! Array.isArray() 사용)
```

### 기법 2: Truthiness 체크 (null/undefined 제거)

```ts
function greet(name: string | null | undefined) {
  if (name) {
    // name은 string (null, undefined, "" 모두 제거됨!)
    return `안녕, ${name}!`;
  }
  return "안녕, 손님!";
}

// ⚠️ 주의: 0과 ""도 falsy이므로 의도치 않게 제거될 수 있음!
function process(count: number | null) {
  if (count) {
    // count는 number이지만 0이 빠짐! 😱
  }
  if (count != null) {
    // ✅ null과 undefined만 제거 (0은 유지!)
  }
}
```

### 기법 3: === / !== 동등성 체크

```ts
function example(x: string | number, y: string | boolean) {
  if (x === y) {
    // x와 y 모두 string! (유일한 공통 타입)
    x.toUpperCase();
    y.toUpperCase();
  }
}

// null 체크 패턴
function process(value: string | null) {
  if (value !== null) {
    value.toUpperCase();  // string으로 좁혀짐
  }
}
```

### 기법 4: in 연산자 (속성 존재 여부)

```ts
type Fish = { swim: () => void; name: string };
type Bird = { fly: () => void; name: string };
type Pet = Fish | Bird;

function move(pet: Pet) {
  if ("swim" in pet) {
    pet.swim();  // Fish
  } else {
    pet.fly();   // Bird
  }
}
```

### 기법 5: instanceof (클래스/생성자 함수)

```ts
function handleError(error: unknown) {
  if (error instanceof TypeError) {
    console.error("타입 에러:", error.message);
  } else if (error instanceof RangeError) {
    console.error("범위 에러:", error.message);
  } else if (error instanceof Error) {
    console.error("일반 에러:", error.message);
  } else {
    console.error("알 수 없는 에러:", error);
  }
}

// Date 체크
function formatDate(input: string | Date): string {
  if (input instanceof Date) {
    return input.toISOString();  // Date
  }
  return new Date(input).toISOString();  // string
}
```

### 기법 6: 판별 유니온 (Discriminated Union) — 가장 강력! 🔴

```ts
// 공통 "판별자(discriminant)" 속성이 있는 유니온
type LoadingState = { status: "loading" };
type SuccessState = { status: "success"; data: string[] };
type ErrorState = { status: "error"; message: string; code: number };

type State = LoadingState | SuccessState | ErrorState;

function render(state: State) {
  switch (state.status) {
    case "loading":
      return <Spinner />;
    case "success":
      return <List data={state.data} />;      // ✅ data 접근 가능
    case "error":
      return <Error message={state.message} />; // ✅ message 접근 가능
  }
}
```

### 실전: API 응답 패턴

```ts
type ApiResult<T> =
  | { ok: true; data: T; status: number }
  | { ok: false; error: { message: string; code: string }; status: number };

async function fetchUser(id: number): Promise<ApiResult<User>> {
  const res = await fetch(`/api/users/${id}`);
  if (res.ok) {
    return { ok: true, data: await res.json(), status: res.status };
  }
  return { ok: false, error: await res.json(), status: res.status };
}

const result = await fetchUser(1);
if (result.ok) {
  console.log(result.data.name);      // ✅ User 타입
} else {
  console.error(result.error.message); // ✅ Error 타입
}
```

### 기법 7: Array.isArray()

```ts
function flatten(value: string | string[]) {
  if (Array.isArray(value)) {
    return value.join(", ");  // string[]
  }
  return value;               // string
}
```

## 3. 사용자 정의 타입 가드 (Type Predicate) 🟡

> `is` 키워드로 "이 함수가 true를 반환하면 인자는 특정 타입이다"라고 TS에게 알려준다.

```ts
// 기본 패턴
function isString(value: unknown): value is string {
  return typeof value === "string";
}

function isUser(obj: unknown): obj is User {
  return (
    typeof obj === "object" &&
    obj !== null &&
    "id" in obj &&
    "name" in obj &&
    typeof (obj as User).id === "number" &&
    typeof (obj as User).name === "string"
  );
}

// 사용
function process(data: unknown) {
  if (isUser(data)) {
    console.log(data.name);  // ✅ User 타입으로 좁혀짐!
  }
}

// 실전: null 필터링 (배열에서 null 제거)
function isNotNull<T>(value: T | null | undefined): value is T {
  return value != null;
}

const items: (string | null)[] = ["a", null, "b", null, "c"];
const filtered = items.filter(isNotNull);
// filtered: string[] ✅ (null이 타입에서도 제거됨!)

// filter만 쓰면:
const filtered2 = items.filter(item => item !== null);
// filtered2: (string | null)[] 😱 (타입이 좁혀지지 않음!)
```

> 💡 **핵심**: `.filter()`에 커스텀 타입 가드를 쓰면 배열의 타입이 자동으로 좁혀진다!

## 4. Assertion Function 🟡

> "이 조건이 거짓이면 에러를 던져라" → 이후 코드에서 타입이 좁혀짐

```ts
function assertIsString(value: unknown): asserts value is string {
  if (typeof value !== "string") {
    throw new Error(`Expected string, got ${typeof value}`);
  }
}

function assertIsDefined<T>(value: T | null | undefined): asserts value is T {
  if (value == null) {
    throw new Error("Value is null or undefined");
  }
}

// 사용
function process(input: unknown) {
  assertIsString(input);
  // 이 줄 이후: input은 string! (에러 안 나면 통과했으니까)
  console.log(input.toUpperCase());
}

// 실전: API 응답 검증
function assertApiSuccess<T>(result: ApiResult<T>): asserts result is { ok: true; data: T } {
  if (!result.ok) {
    throw new Error(result.error.message);
  }
}

const result = await fetchUser(1);
assertApiSuccess(result);
// 이 줄 이후: result.data가 확정! (에러면 위에서 throw)
console.log(result.data.name);
```

## 5. 제어 흐름 분석의 한계와 해결 🟡

```ts
// TS가 좁히지 못하는 경우
function example(arr: string[]) {
  let found: string | undefined;

  arr.forEach(item => {
    if (item === "target") {
      found = item;
    }
  });

  // ❌ TS는 forEach 콜백 안에서의 할당을 추적 못함!
  // found는 여전히 string | undefined
  if (found) {
    console.log(found.toUpperCase());  // 실제로는 되지만 TS가 불안해함
  }
}

// ✅ 해결: find 같은 반환값이 있는 메서드 사용
function example2(arr: string[]) {
  const found = arr.find(item => item === "target");
  if (found) {
    console.log(found.toUpperCase());  // ✅ string으로 좁혀짐
  }
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. TypeScript에서 타입을 좁히는 방법들을 설명해주세요.**

> `typeof`(원시 타입 판별), `instanceof`(클래스 인스턴스), `in`(속성 존재 여부), `Array.isArray()`(배열 판별), `===`/`!==`(동등성), Truthiness 체크(null/undefined 제거), 판별 유니온(`switch` + 리터럴 판별자)이 있다. 사용자 정의 타입 가드(`is` 키워드)를 사용하면 커스텀 조건으로 타입을 좁힐 수 있으며, Assertion 함수(`asserts`)로 조건 실패 시 에러를 던지고 이후 코드에서 타입을 확정할 수 있다.

**Q2. 판별 유니온(Discriminated Union)이란?**

> 유니온의 각 멤버가 공통된 리터럴 속성(판별자)을 가지는 패턴이다. 예를 들어 `{ type: "success"; data: T } | { type: "error"; message: string }`에서 `type`이 판별자다. `switch(result.type)`으로 분기하면 TS가 각 케이스에서 정확한 타입을 자동으로 좁혀준다. React의 `useReducer` action, API 응답 타입, 상태 머신 등에 광범위하게 활용된다.
