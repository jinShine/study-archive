## 1. 제네릭 컴포넌트 — 타입 안전한 재사용의 끝판왕 🔴

### 제네릭 리스트 컴포넌트

```tsx
// 어떤 타입의 배열이든 받아서 렌더링하는 범용 리스트
type ListProps<T> = {
  items: T[];
  renderItem: (item: T, index: number) => React.ReactNode;
  keyExtractor: (item: T) => string | number;
  emptyMessage?: string;
};

function List<T>({ items, renderItem, keyExtractor, emptyMessage = "데이터가 없습니다" }: ListProps<T>) {
  if (items.length === 0) {
    return <p className="text-gray-500">{emptyMessage}</p>;
  }

  return (
    <ul>
      {items.map((item, index) => (
        <li key={keyExtractor(item)}>{renderItem(item, index)}</li>
      ))}
    </ul>
  );
}

// 사용: T가 자동으로 추론됨!
type User = { id: number; name: string; email: string };
const users: User[] = [
  { id: 1, name: "철수", email: "chul@test.com" },
  { id: 2, name: "영희", email: "young@test.com" },
];

<List
  items={users}
  keyExtractor={(user) => user.id}           // user: User로 자동 추론!
  renderItem={(user) => (                     // user: User로 자동 추론!
    <div>
      <strong>{user.name}</strong>
      <span>{user.email}</span>              // 자동완성 완벽!
    </div>
  )}
/>
```

### 제네릭 Select 컴포넌트

```tsx
type SelectProps<T> = {
  options: T[];
  value: T | null;
  onChange: (selected: T) => void;
  getLabel: (option: T) => string;
  getValue: (option: T) => string | number;
  placeholder?: string;
};

function Select<T>({ options, value, onChange, getLabel, getValue, placeholder }: SelectProps<T>) {
  return (
    <select
      value={value ? String(getValue(value)) : ""}
      onChange={(e) => {
        const selected = options.find(opt => String(getValue(opt)) === e.target.value);
        if (selected) onChange(selected);
      }}
    >
      {placeholder && <option value="">{placeholder}</option>}
      {options.map(option => (
        <option key={getValue(option)} value={getValue(option)}>
          {getLabel(option)}
        </option>
      ))}
    </select>
  );
}

// 사용
type Country = { code: string; name: string };
const countries: Country[] = [
  { code: "KR", name: "대한민국" },
  { code: "US", name: "미국" },
];

<Select
  options={countries}
  value={selectedCountry}
  onChange={(country) => setSelected(country)}  // country: Country ✅
  getLabel={(c) => c.name}                      // c: Country ✅
  getValue={(c) => c.code}                      // c: Country ✅
  placeholder="국가를 선택하세요"
/>
```

## 2. Context 타이핑 🔴

### 기본 패턴

```tsx
// 1. Context 타입 정의
type ThemeContextType = {
  theme: "light" | "dark";
  toggleTheme: () => void;
};

// 2. Context 생성 (초기값 문제!)
// ❌ null로 시작하면 매번 null 체크를 해야 함
const ThemeContext = createContext<ThemeContextType | null>(null);

// ✅ 해결: 커스텀 훅에서 null 체크를 한 번만!
function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error("useTheme은 ThemeProvider 안에서만 사용할 수 있습니다");
  }
  return context;  // ThemeContextType (null 아님 보장!)
}

// 3. Provider 컴포넌트
function ThemeProvider({ children }: { children: React.ReactNode }) {
  const [theme, setTheme] = useState<"light" | "dark">("light");

  const toggleTheme = useCallback(() => {
    setTheme(prev => prev === "light" ? "dark" : "light");
  }, []);

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// 4. 소비자 컴포넌트
function Header() {
  const { theme, toggleTheme } = useTheme();  // ✅ null 체크 불필요!
  return (
    <header className={theme === "dark" ? "bg-gray-900" : "bg-white"}>
      <button onClick={toggleTheme}>테마 변경</button>
    </header>
  );
}
```

### Context 초기값 패턴 비교

```tsx
// 패턴 A: null + 커스텀 훅 (가장 추천! ✅)
const Ctx = createContext<CtxType | null>(null);
function useCtx() {
  const ctx = useContext(Ctx);
  if (!ctx) throw new Error("Provider 필요!");
  return ctx;
}

// 패턴 B: 기본값 제공 (Provider 없이도 동작하게)
const Ctx = createContext<CtxType>({
  theme: "light",
  toggleTheme: () => {},  // noop (아무것도 안 하는 함수)
});
// 단점: Provider를 안 감싸도 에러가 안 나서 버그 발견이 늦어짐

// 패턴 C: as 단언 (빠르지만 안전하지 않음)
const Ctx = createContext<CtxType>({} as CtxType);
// 단점: 런타임에 빈 객체의 메서드를 호출하면 폭발
```

> 💡 **실무 정석**: 패턴 A (null + 커스텀 훅). Provider 없이 사용하면 즉시 에러!

## 3. forwardRef — 부모가 자식의 DOM에 접근 🔴

```tsx
// ❌ 일반 컴포넌트에 ref를 전달하면 에러
function Input(props: InputProps) { return <input {...props} /> }
<Input ref={inputRef} />  // 에러! ref는 Props로 전달 안 됨

// ✅ forwardRef로 ref 전달 허용
import { forwardRef } from "react";

type InputProps = {
  label: string;
  error?: string;
} & React.InputHTMLAttributes<HTMLInputElement>;

const Input = forwardRef<HTMLInputElement, InputProps>(
  ({ label, error, ...rest }, ref) => {
    return (
      <div>
        <label>{label}</label>
        <input ref={ref} {...rest} />
        {error && <p className="text-red-500">{error}</p>}
      </div>
    );
  }
);

Input.displayName = "Input";  // DevTools에서 이름 표시

// 사용
function Form() {
  const emailRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    emailRef.current?.focus();  // 부모가 자식 input에 직접 접근!
  }, []);

  return <Input ref={emailRef} label="이메일" type="email" />;
}
```

### forwardRef 제네릭 문법

```
forwardRef<RefType, PropsType>((props, ref) => JSX)

  RefType: ref가 가리킬 DOM 요소 타입 (HTMLInputElement 등)
  PropsType: 컴포넌트의 Props 타입
```

## 4. useImperativeHandle — 부모에게 메서드 노출 🟡

```tsx
// 자식이 부모에게 커스텀 메서드를 제공하는 패턴
type FormHandle = {
  reset: () => void;
  validate: () => boolean;
  getData: () => FormData;
};

const MyForm = forwardRef<FormHandle, { onSubmit: (data: FormData) => void }>(
  ({ onSubmit }, ref) => {
    const [name, setName] = useState("");
    const [email, setEmail] = useState("");

    useImperativeHandle(ref, () => ({
      reset() {
        setName("");
        setEmail("");
      },
      validate() {
        return name.length > 0 && email.includes("@");
      },
      getData() {
        return { name, email } as unknown as FormData;
      },
    }));

    return (
      <form>
        <input value={name} onChange={e => setName(e.target.value)} />
        <input value={email} onChange={e => setEmail(e.target.value)} />
      </form>
    );
  }
);

// 부모에서 사용
function Parent() {
  const formRef = useRef<FormHandle>(null);

  const handleSave = () => {
    if (formRef.current?.validate()) {
      const data = formRef.current.getData();
      console.log(data);
    }
  };

  return (
    <>
      <MyForm ref={formRef} onSubmit={() => {}} />
      <button onClick={() => formRef.current?.reset()}>초기화</button>
      <button onClick={handleSave}>저장</button>
    </>
  );
}
```

## 5. 커스텀 훅 타이핑 🟡

```tsx
// 반환 타입이 자동 추론되면 명시 불필요!
function useToggle(initial = false) {
  const [value, setValue] = useState(initial);
  const toggle = useCallback(() => setValue(v => !v), []);
  const setTrue = useCallback(() => setValue(true), []);
  const setFalse = useCallback(() => setValue(false), []);
  return { value, toggle, setTrue, setFalse } as const;
}

// 제네릭 커스텀 훅
function useLocalStorage<T>(key: string, initialValue: T) {
  const [stored, setStored] = useState<T>(() => {
    if (typeof window === "undefined") return initialValue;
    try {
      const item = window.localStorage.getItem(key);
      return item ? (JSON.parse(item) as T) : initialValue;
    } catch {
      return initialValue;
    }
  });

  const setValue = useCallback((value: T | ((prev: T) => T)) => {
    setStored(prev => {
      const newValue = value instanceof Function ? value(prev) : value;
      window.localStorage.setItem(key, JSON.stringify(newValue));
      return newValue;
    });
  }, [key]);

  return [stored, setValue] as const;
  // as const → 튜플 타입 [T, (value: T | ((prev: T) => T)) => void]
}

// 사용
const [theme, setTheme] = useLocalStorage<"light" | "dark">("theme", "light");
setTheme("dark");          // ✅
setTheme(prev => prev === "light" ? "dark" : "light"); // ✅
```

> 💡 **`as const` 반환**: 배열을 반환하는 훅은 `as const`를 붙여야 튜플로 추론된다.
> 안 붙이면 `(string | Function)[]`으로 추론되어 구조 분해 시 타입이 섞인다.

## 6. 실습: 타입 안전한 모달 시스템

```tsx
// 과제: Context + forwardRef + 제네릭을 결합한 모달 시스템

// 1. ModalContext — isOpen, open(), close() 메서드 제공
// 2. ModalProvider — children을 감싸는 Provider
// 3. useModal() — 커스텀 훅 (null 체크 포함)
// 4. Modal 컴포넌트 — forwardRef로 외부에서 close 가능
// 5. ConfirmModal<T> — 제네릭으로 확인 버튼 클릭 시 T 타입 데이터 반환
```

<details>
<summary>핵심 타입 정답</summary>

```tsx
// Context 타입
type ModalContextType = {
  isOpen: boolean;
  open: () => void;
  close: () => void;
};

// 커스텀 훅
const ModalContext = createContext<ModalContextType | null>(null);
function useModal() {
  const ctx = useContext(ModalContext);
  if (!ctx) throw new Error("ModalProvider 필요!");
  return ctx;
}

// forwardRef Modal
type ModalHandle = { close: () => void };
type ModalProps = { title: string; children: React.ReactNode };

const Modal = forwardRef<ModalHandle, ModalProps>(({ title, children }, ref) => {
  const { isOpen, close } = useModal();

  useImperativeHandle(ref, () => ({ close }));

  if (!isOpen) return null;
  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center">
      <div className="bg-white p-6 rounded-xl">
        <h2>{title}</h2>
        {children}
        <button onClick={close}>닫기</button>
      </div>
    </div>
  );
});

// 제네릭 ConfirmModal
type ConfirmModalProps<T> = {
  data: T;
  onConfirm: (data: T) => void;
  renderContent: (data: T) => React.ReactNode;
};

function ConfirmModal<T>({ data, onConfirm, renderContent }: ConfirmModalProps<T>) {
  const { close } = useModal();
  return (
    <div>
      {renderContent(data)}
      <button onClick={() => { onConfirm(data); close(); }}>확인</button>
    </div>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. React에서 Context를 TypeScript로 타이핑하는 방법은?**

> `createContext<ContextType | null>(null)`로 생성하고, `useContext`를 감싼 커스텀 훅에서 null 체크를 수행하여 Provider 밖에서 사용하면 즉시 에러를 던진다. 이 패턴으로 소비자 컴포넌트에서는 null 체크 없이 안전하게 Context 값에 접근할 수 있다.

**Q2. `forwardRef`는 왜 필요하고 어떻게 타이핑하나요?**

> 부모 컴포넌트가 자식의 DOM 요소에 직접 접근(focus, scroll 등)하기 위해 `ref`를 전달하는 메커니즘이다. `forwardRef<HTMLInputElement, PropsType>((props, ref) => JSX)`로 타이핑하며, 첫 번째 제네릭이 ref가 가리킬 DOM 타입, 두 번째가 Props 타입이다.
