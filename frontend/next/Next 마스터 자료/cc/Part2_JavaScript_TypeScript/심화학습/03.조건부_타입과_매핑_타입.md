## 1. 조건부 타입 (Conditional Types) 🔴

> 💡 **비유: 삼항 연산자의 타입 버전**
> 값에 `조건 ? A : B`가 있듯이, 타입에도 `T extends 조건 ? A : B`가 있다.

```ts
// 기본 문법: T extends U ? TrueType : FalseType
type IsString<T> = T extends string ? "예" : "아니오";

type A = IsString<string>;  // "예"
type B = IsString<number>;  // "아니오"
type C = IsString<"hello">; // "예" (리터럴도 string을 extends)
```

### 실전: null 제거

```ts
// null/undefined를 제거하는 타입 (NonNullable의 원리!)
type NoNull<T> = T extends null | undefined ? never : T;

type Result = NoNull<string | null | undefined>;
// string (null과 undefined가 never로 바뀌어 사라짐!)

// 이것이 바로 내장 NonNullable<T>의 구현이다!
```

### 실전: Promise 내부 타입 추출

```ts
// Promise<T>에서 T를 꺼내기
type Unwrap<T> = T extends Promise<infer U> ? U : T;

type A = Unwrap<Promise<string>>;  // string
type B = Unwrap<Promise<number>>; // number
type C = Unwrap<string>;          // string (Promise가 아니면 그대로)

// 내장 Awaited<T>와 동일한 원리!
type D = Awaited<Promise<Promise<string>>>; // string (중첩도 벗겨냄)
```

## 2. infer — 타입 안에서 변수 추출 🟡

> 💡 **비유: 정규표현식의 캡처 그룹**
> `infer`는 타입 패턴 매칭에서 "여기에 뭐가 오든 그걸 변수로 잡아줘"라는 의미다.

```ts
// 함수 반환 타입 추출 (ReturnType의 원리!)
type MyReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type A = MyReturnType<() => string>;     // string
type B = MyReturnType<(x: number) => boolean>; // boolean

// 함수 매개변수 타입 추출 (Parameters의 원리!)
type MyParameters<T> = T extends (...args: infer P) => any ? P : never;

type C = MyParameters<(name: string, age: number) => void>;
// [name: string, age: number]

// 배열 요소 타입 추출
type ElementOf<T> = T extends (infer E)[] ? E : never;

type D = ElementOf<string[]>;  // string
type E = ElementOf<number[]>;  // number

// 첫 번째 요소 타입
type First<T> = T extends [infer F, ...any[]] ? F : never;
type F = First<[string, number, boolean]>; // string
```

## 3. keyof — 객체의 키를 유니온으로 🔴

```ts
type User = {
  id: number;
  name: string;
  email: string;
};

type UserKeys = keyof User;  // "id" | "name" | "email"

// 실전: 안전한 객체 접근 함수
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user: User = { id: 1, name: "철수", email: "test@test.com" };
getProperty(user, "name");   // string ✅
getProperty(user, "age");    // ❌ 에러! "age"는 User의 키가 아님

// typeof와 조합
const CONFIG = {
  apiUrl: "https://api.example.com",
  timeout: 3000,
  retries: 3,
} as const;

type ConfigKeys = keyof typeof CONFIG; // "apiUrl" | "timeout" | "retries"
```

## 4. 매핑 타입 (Mapped Types) — Partial/Pick의 원리 🔴

> 💡 **비유: 배열의 map()처럼, 객체의 모든 속성을 변환하는 타입 버전**

### Partial의 내부 구현

```ts
// Partial<T>가 실제로 하는 일:
type MyPartial<T> = {
  [K in keyof T]?: T[K];
  // keyof T의 모든 키(K)를 순회하면서
  // 각 속성을 선택적(?)으로 만든다
};

type User = { id: number; name: string; email: string };
type PartialUser = MyPartial<User>;
// { id?: number; name?: string; email?: string }
```

### Required의 내부 구현

```ts
type MyRequired<T> = {
  [K in keyof T]-?: T[K];
  // -? 는 ?를 제거한다는 의미!
};
```

### Pick의 내부 구현

```ts
type MyPick<T, K extends keyof T> = {
  [P in K]: T[P];
  // K에 포함된 키만 순회
};

type UserPreview = MyPick<User, "id" | "name">;
// { id: number; name: string }
```

### Omit의 내부 구현

```ts
type MyOmit<T, K extends keyof T> = {
  [P in Exclude<keyof T, K>]: T[P];
  // keyof T에서 K를 제외한 키만 순회
};

// Exclude: 유니온에서 특정 타입 제거
type Exclude<T, U> = T extends U ? never : T;
type A = Exclude<"a" | "b" | "c", "a">;  // "b" | "c"
```

### Record의 내부 구현

```ts
type MyRecord<K extends keyof any, V> = {
  [P in K]: V;
};

type StatusMap = MyRecord<"success" | "error", string>;
// { success: string; error: string }
```

### Readonly 매핑

```ts
type MyReadonly<T> = {
  readonly [K in keyof T]: T[K];
};

type ReadonlyUser = MyReadonly<User>;
// { readonly id: number; readonly name: string; readonly email: string }
```

## 5. Template Literal Types 🟡

> 문자열 리터럴 타입을 조합하여 새로운 문자열 타입을 생성

```ts
// 기본
type Greeting = `Hello, ${string}!`;
const a: Greeting = "Hello, World!";  // ✅
const b: Greeting = "Goodbye!";       // ❌

// 유니온 조합 (매우 강력!)
type Color = "red" | "blue" | "green";
type Size = "sm" | "md" | "lg";

type ClassName = `${Color}-${Size}`;
// "red-sm" | "red-md" | "red-lg" | "blue-sm" | ... (9가지!)

// 실전: CSS 클래스 타입
type Spacing = 0 | 1 | 2 | 4 | 8;
type SpacingClass = `p-${Spacing}` | `m-${Spacing}`;
// "p-0" | "p-1" | "p-2" | ... | "m-0" | "m-1" | ...

// 실전: 이벤트 핸들러 이름
type EventName = "click" | "change" | "submit";
type HandlerName = `on${Capitalize<EventName>}`;
// "onClick" | "onChange" | "onSubmit"
```

### 내장 문자열 조작 타입

```ts
type A = Uppercase<"hello">;     // "HELLO"
type B = Lowercase<"HELLO">;     // "hello"
type C = Capitalize<"hello">;    // "Hello"
type D = Uncapitalize<"Hello">;  // "hello"
```

## 면접 대비

### 🔴 필수 질문

**Q1. `keyof`는 무엇이고 어떻게 활용하나요?**

> `keyof T`는 타입 T의 모든 키를 문자열 리터럴 유니온으로 추출한다. `keyof { name: string; age: number }`는 `"name" | "age"`가 된다. 제네릭과 결합하여 `<K extends keyof T>`로 함수의 매개변수를 객체의 실제 키로 제한할 수 있어, 타이핑 실수를 컴파일 타임에 잡아낸다.

### 🟡 개념 질문

**Q2. Partial, Pick, Omit 같은 유틸리티 타입의 내부 구현 원리?**

> 매핑 타입(Mapped Types)으로 구현된다. `[K in keyof T]` 문법으로 T의 모든 키를 순회하며 새로운 타입을 생성한다. Partial은 `?`를 추가하고, Readonly는 `readonly`를 추가하고, Pick은 특정 키만 순회하며, Omit은 Exclude로 키를 제거한 후 나머지를 순회한다.
