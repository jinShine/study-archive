## 1. Props 기본 타이핑 🔴

```tsx
// 방법 1: type (React 커뮤니티 주류)
type ButtonProps = {
  label: string;
  onClick: () => void;
  disabled?: boolean;      // 선택적
};

function Button({ label, onClick, disabled = false }: ButtonProps) {
  return <button onClick={onClick} disabled={disabled}>{label}</button>;
}

// 방법 2: interface (extends 필요할 때)
interface CardProps {
  title: string;
  description?: string;
}
```

### Props 기본값 패턴

```tsx
type ChipProps = {
  label: string;
  color?: "blue" | "red" | "green";
  size?: "sm" | "md" | "lg";
};

// ✅ 구조 분해 할당에서 기본값 설정 (실무 정석!)
function Chip({ label, color = "blue", size = "md" }: ChipProps) {
  return <span className={`chip-${color} chip-${size}`}>{label}</span>;
}

// 사용: <Chip label="태그" /> → color="blue", size="md" 자동 적용
```

## 2. children 타이핑 🔴

```tsx
// React.ReactNode — 가장 범용적 (거의 모든 것)
type LayoutProps = {
  children: React.ReactNode;
  // ReactNode = string | number | boolean | null | undefined
  //           | ReactElement | ReactFragment | ReactPortal
};

function Layout({ children }: LayoutProps) {
  return <div className="layout">{children}</div>;
}

// React.PropsWithChildren — children을 자동 포함하는 유틸리티
type CardProps = React.PropsWithChildren<{
  title: string;
  className?: string;
}>;
// = { title: string; className?: string; children?: ReactNode }

// children 필수로 만들기
type WrapperProps = {
  children: React.ReactNode;  // 선택적이 아님! 필수!
};

// 특정 타입의 children만 허용 (드문 케이스)
type StringOnlyProps = {
  children: string;  // JSX가 아닌 문자열만 허용
};
```

### 어떤 children 타입을 써야 하나?

```
React.ReactNode    → 99% 이것! (모든 렌더링 가능한 것)
React.ReactElement → JSX만 허용 (문자열/숫자 제외)
string             → 텍스트만 허용
never              → children 금지
```

## 3. HTML 속성 확장 — 진짜 재사용 컴포넌트의 핵심 🔴

> 💡 커스텀 Button이 HTML button의 모든 속성(type, disabled, aria-label 등)을
> 자동으로 받을 수 있어야 한다!

```tsx
// ❌ 나쁜 패턴: 필요한 속성을 하나하나 정의
type ButtonProps = {
  label: string;
  onClick?: () => void;
  disabled?: boolean;
  type?: "button" | "submit";
  className?: string;
  // ... aria-label은? tabIndex는? 끝이 없다!
};

// ✅ 좋은 패턴: HTML 속성 전체를 확장
type ButtonProps = {
  variant?: "primary" | "secondary" | "danger";
  size?: "sm" | "md" | "lg";
  isLoading?: boolean;
} & React.ButtonHTMLAttributes<HTMLButtonElement>;
//   ↑ button의 모든 기본 속성(type, disabled, onClick, aria-*, ...)이 자동 포함!

function Button({ variant = "primary", size = "md", isLoading, children, ...rest }: ButtonProps) {
  return (
    <button
      className={`btn-${variant} btn-${size}`}
      disabled={isLoading || rest.disabled}
      {...rest}  // onClick, type, aria-label 등 HTML 속성 전부 전달!
    >
      {isLoading ? "로딩..." : children}
    </button>
  );
}

// 사용: 모든 HTML button 속성을 그대로 쓸 수 있다!
<Button variant="primary" onClick={handleClick} aria-label="제출" type="submit">
  저장
</Button>
```

### 다른 HTML 요소 확장

```tsx
// input 확장
type InputProps = {
  label: string;
  error?: string;
} & React.InputHTMLAttributes<HTMLInputElement>;

// div 확장
type CardProps = {
  title: string;
} & React.HTMLAttributes<HTMLDivElement>;

// a 확장 (링크)
type LinkProps = {
  variant?: "default" | "underline";
} & React.AnchorHTMLAttributes<HTMLAnchorElement>;
```

### ComponentPropsWithRef / ComponentPropsWithoutRef

```tsx
// 특정 컴포넌트의 Props를 가져오기
type NativeButtonProps = React.ComponentPropsWithoutRef<"button">;
type NativeInputProps = React.ComponentPropsWithoutRef<"input">;
type NativeDivProps = React.ComponentPropsWithRef<"div">; // ref 포함

// 다른 커스텀 컴포넌트의 Props 가져오기
type MyButtonProps = React.ComponentProps<typeof Button>;
```

## 4. 조건부 Props — "A를 주면 B도 필수" 🟡

> 특정 props 조합만 허용하고 싶을 때

### 판별 유니온 Props (가장 추천!)

```tsx
// variant가 "link"면 href 필수, "button"이면 onClick 필수
type ButtonProps =
  | {
      variant: "button";
      onClick: () => void;
      href?: never;           // href 금지!
    }
  | {
      variant: "link";
      href: string;
      onClick?: never;        // onClick 금지!
    };

type BaseProps = {
  children: React.ReactNode;
  className?: string;
};

function ActionButton(props: BaseProps & ButtonProps) {
  if (props.variant === "link") {
    return <a href={props.href} className={props.className}>{props.children}</a>;
  }
  return <button onClick={props.onClick} className={props.className}>{props.children}</button>;
}

// ✅ 올바른 사용
<ActionButton variant="link" href="/about">소개</ActionButton>
<ActionButton variant="button" onClick={handleClick}>클릭</ActionButton>

// ❌ 에러! variant="link"인데 href가 없음
<ActionButton variant="link" onClick={handleClick}>에러!</ActionButton>
```

### as prop — 다형성(Polymorphic) 컴포넌트 🟡

> "이 컴포넌트를 button으로도, a로도, div로도 렌더링하고 싶다"

```tsx
type BoxProps<C extends React.ElementType = "div"> = {
  as?: C;
  children?: React.ReactNode;
} & Omit<React.ComponentPropsWithoutRef<C>, "as" | "children">;

function Box<C extends React.ElementType = "div">({
  as,
  children,
  ...rest
}: BoxProps<C>) {
  const Component = as || "div";
  return <Component {...rest}>{children}</Component>;
}

// div로 렌더링 (기본)
<Box className="container">내용</Box>

// section으로 렌더링
<Box as="section" id="hero">히어로</Box>

// a로 렌더링 (href가 자동완성!)
<Box as="a" href="/about">링크</Box>

// button으로 렌더링 (onClick이 자동완성!)
<Box as="button" onClick={handleClick}>버튼</Box>
```

> 💡 **shadcn/ui**의 `Slot` 컴포넌트가 이 패턴의 실전 활용이다.

```tsx
// 과제 1: Input 컴포넌트
// - label, error 커스텀 props
// - HTML input의 모든 속성(type, placeholder, onChange 등) 자동 지원
// - error가 있으면 빨간 테두리

// 과제 2: Card 컴포넌트
// - title (필수), description (선택), footer (선택, ReactNode)
// - HTML div 속성 확장 (className, onClick 등)

// 과제 3: 판별 유니온 Alert 컴포넌트
// - variant: "success" → message만 필수
// - variant: "error" → message + errorCode 필수
// - variant: "warning" → message + dismissable(boolean) 필수
```

```tsx
// 과제 1
type InputProps = {
  label: string;
  error?: string;
} & React.InputHTMLAttributes<HTMLInputElement>;

function Input({ label, error, id, ...rest }: InputProps) {
  const inputId = id || label.toLowerCase().replace(/\s/g, "-");
  return (
    <div>
      <label htmlFor={inputId}>{label}</label>
      <input
        id={inputId}
        className={error ? "border-red-500" : "border-gray-300"}
        aria-invalid={!!error}
        aria-describedby={error ? `${inputId}-error` : undefined}
        {...rest}
      />
      {error && <p id={`${inputId}-error`} className="text-red-500 text-sm">{error}</p>}
    </div>
  );
}

// 과제 2
type CardProps = {
  title: string;
  description?: string;
  footer?: React.ReactNode;
} & React.HTMLAttributes<HTMLDivElement>;

function Card({ title, description, footer, children, ...rest }: CardProps) {
  return (
    <div {...rest}>
      <h3>{title}</h3>
      {description && <p>{description}</p>}
      {children}
      {footer && <div>{footer}</div>}
    </div>
  );
}

// 과제 3
type AlertProps =
  | { variant: "success"; message: string }
  | { variant: "error"; message: string; errorCode: string }
  | { variant: "warning"; message: string; dismissable: boolean };

function Alert(props: AlertProps) {
  switch (props.variant) {
    case "success":
      return <div className="bg-green-50">{props.message}</div>;
    case "error":
      return <div className="bg-red-50">{props.message} (코드: {props.errorCode})</div>;
    case "warning":
      return (
        <div className="bg-yellow-50">
          {props.message}
          {props.dismissable && <button>닫기</button>}
        </div>
      );
  }
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. React 컴포넌트의 Props를 TypeScript로 어떻게 정의하나요?**

> `type` 또는 `interface`로 Props 타입을 정의하고, 함수 매개변수에서 구조 분해 할당으로 받는다. 선택적 Props에는 `?`를 붙이고, HTML 요소의 기본 속성도 지원하려면 `React.ButtonHTMLAttributes<HTMLButtonElement>` 같은 내장 타입과 인터섹션(`&`)으로 합친다. `...rest`로 나머지 속성을 전달하면 완전한 재사용 컴포넌트가 된다.

**Q2. `React.ReactNode`와 `React.ReactElement`의 차이?**

> `ReactNode`는 React가 렌더링할 수 있는 모든 타입(string, number, boolean, null, undefined, JSX, 배열 등)이고, `ReactElement`는 JSX 요소(`<div>`, `<Component />`)만 포함한다. `children` Props에는 거의 항상 `ReactNode`를 사용한다.
