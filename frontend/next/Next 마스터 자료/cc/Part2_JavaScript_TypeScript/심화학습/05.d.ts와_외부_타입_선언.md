# TS 심화 05. d.ts와 외부 타입 선언

> **키워드**: `d.ts` `declare` `@types` `모듈 선언` `전역 타입 확장` `ambient declaration`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | d.ts가 뭔지, @types 패키지, 모듈 선언 | 외부 라이브러리 사용 필수 |
| 🟡 이해 | declare, 전역 타입 확장, 환경 변수 타입 | 실무 세팅 |
| 🟢 참고 | Triple-slash directive, 자체 라이브러리 타입 배포 | 라이브러리 개발 |

## 1. d.ts 파일이란? 🔴

> 💡 **비유: 메뉴판**
> 식당(라이브러리)의 실제 주방(JS 코드)을 볼 필요 없이,
> 메뉴판(d.ts)만 보면 "무슨 요리를 주문할 수 있는지" 알 수 있다.
> d.ts = 타입 정보만 담긴 선언 파일 (실제 코드 없음)

```ts
// 일반 .ts 파일 → 타입 + 실행 코드
export function add(a: number, b: number): number {
  return a + b;  // 실행 코드가 있음
}

// .d.ts 파일 → 타입 정보만! (실행 코드 없음)
export declare function add(a: number, b: number): number;
// 구현체가 없다! "이런 형태의 함수가 있다"는 선언만
```

### 어디서 만나나?

```
1. node_modules 안에 → 라이브러리가 제공하는 타입 정의
   └─ node_modules/react/index.d.ts
   └─ node_modules/next/types/index.d.ts

2. @types 패키지 → TS 타입이 없는 JS 라이브러리의 타입
   └─ @types/react        ← npm install @types/react
   └─ @types/node          ← Node.js API 타입

3. 프로젝트 내 직접 생성 → 커스텀 모듈/전역 타입
   └─ src/types/global.d.ts
   └─ src/types/modules.d.ts
```

## 2. @types 패키지 시스템 🔴

```bash
# 라이브러리에 타입이 내장되어 있으면 → 추가 설치 불필요
npm install zod          # zod는 TS로 작성되어 타입 내장 ✅
npm install zustand      # zustand도 타입 내장 ✅

# 라이브러리에 타입이 없으면 → @types 패키지 별도 설치
npm install lodash       # JS로 작성됨 (타입 없음)
npm install @types/lodash # 커뮤니티가 만든 타입 선언

# 확인 방법: node_modules/라이브러리/package.json에서
# "types" 또는 "typings" 필드가 있으면 타입 내장
```

### @types가 없는 라이브러리를 쓸 때

```ts
// ❌ 에러: "모듈 'some-old-lib'에 대한 선언 파일을 찾을 수 없습니다"
import something from "some-old-lib";

// ✅ 해결: 직접 선언 파일 생성
// src/types/some-old-lib.d.ts
declare module "some-old-lib" {
  export function doSomething(arg: string): void;
  export const version: string;
  // 타입을 모르면 최소한 이렇게라도:
  const lib: any;
  export default lib;
}
```

## 3. 자주 쓰는 declare 패턴 🟡

### CSS/이미지 모듈 선언

```ts
// src/types/modules.d.ts

// CSS Modules
declare module "*.module.css" {
  const classes: { readonly [key: string]: string };
  export default classes;
}

declare module "*.module.scss" {
  const classes: { readonly [key: string]: string };
  export default classes;
}

// 이미지 파일
declare module "*.png" {
  const src: string;
  export default src;
}
declare module "*.jpg" {
  const src: string;
  export default src;
}
declare module "*.svg" {
  import React from "react";
  const SVG: React.FC<React.SVGProps<SVGSVGElement>>;
  export default SVG;
}
```

### 환경 변수 타입 선언 (Next.js) 🔴

```ts
// src/types/env.d.ts

namespace NodeJS {
  interface ProcessEnv {
    // 서버 전용 환경 변수
    DATABASE_URL: string;
    JWT_SECRET: string;
    NEXTAUTH_SECRET: string;

    // 클라이언트 노출 환경 변수 (NEXT_PUBLIC_ 접두사)
    NEXT_PUBLIC_API_URL: string;
    NEXT_PUBLIC_GOOGLE_ANALYTICS_ID: string;

    // 선택적 환경 변수
    SENTRY_DSN?: string;
  }
}
```

```ts
// 이제 process.env에 자동완성이 된다!
const apiUrl = process.env.NEXT_PUBLIC_API_URL;  // string ✅ (자동완성!)
const secret = process.env.JWT_SECRET;            // string ✅
const missing = process.env.RANDOM;               // ❌ 에러! 선언 안 한 변수
```

### 전역 타입 확장 (Next-Auth 예시) 🟡

```ts
// src/types/next-auth.d.ts
import { DefaultSession } from "next-auth";

// NextAuth의 Session 타입에 커스텀 필드 추가
declare module "next-auth" {
  interface Session {
    user: {
      id: string;          // 추가
      role: "admin" | "user"; // 추가
    } & DefaultSession["user"];
  }

  interface User {
    role: "admin" | "user";
  }
}

// 이제 session.user.id, session.user.role에 타입이 붙음!
```

### 전역 변수/함수 선언

```ts
// src/types/global.d.ts

// window에 커스텀 속성 추가
declare global {
  interface Window {
    gtag: (...args: any[]) => void;
    kakao: any;
  }
}

// 전역 타입 선언 (import 없이 어디서든 사용)
declare global {
  type ID = string | number;
  type Nullable<T> = T | null;
}

export {}; // 이 파일을 모듈로 만들기 위한 빈 export (필수!)
```

> ⚠️ `declare global` 블록이 있는 파일에는 반드시 `export {}` 가 있어야 한다.
> 없으면 "스크립트 모드"로 인식되어 다른 파일의 타입과 충돌한다.

## 4. tsconfig의 include와 d.ts 🟡

```json
// tsconfig.json
{
  "include": [
    "next-env.d.ts",       // Next.js 기본 타입
    "src/**/*.ts",
    "src/**/*.tsx",
    "src/types/**/*.d.ts"  // 커스텀 타입 선언 파일들
  ]
}
```

> 💡 `src/types/` 폴더에 d.ts 파일을 모아두는 것이 실무 표준 구조.

## 면접 대비

### 🔴 필수 질문

**Q1. `.d.ts` 파일이란?**

> TypeScript의 타입 선언 파일로, 실행 코드 없이 타입 정보만 포함한다. JavaScript로 작성된 외부 라이브러리에 타입을 부여하거나, 전역 타입을 확장하거나, CSS/이미지 모듈의 import 타입을 정의할 때 사용한다. `@types/react` 같은 패키지가 이 `.d.ts` 파일들의 모음이다.

### 🟡 개념 질문

**Q2. `@types` 패키지는 왜 필요한가?**

> JavaScript로 작성된 라이브러리(lodash 등)에는 TypeScript 타입 정보가 없다. DefinitelyTyped 커뮤니티가 이런 라이브러리의 타입을 `@types/라이브러리명` 패키지로 제공한다. TypeScript로 작성된 라이브러리(zod, zustand 등)는 타입이 내장되어 있으므로 `@types` 패키지가 필요 없다.
