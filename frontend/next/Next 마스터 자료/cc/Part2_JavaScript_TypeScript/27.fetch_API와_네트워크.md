## 1. fetch 기본 — GET 요청 🔴

```js
// 가장 기본적인 데이터 가져오기
const response = await fetch("https://jsonplaceholder.typicode.com/users/1");
const user = await response.json();
console.log(user.name);  // "Leanne Graham"
```

### fetch 응답 처리 패턴 (실무 정석)

```js
async function fetchUser(id) {
  const res = await fetch(`/api/users/${id}`);

  // 1. HTTP 에러 체크 (fetch는 404/500에서 reject 안 함!)
  if (!res.ok) {
    throw new Error(`HTTP ${res.status}: ${res.statusText}`);
  }

  // 2. 응답 본문 파싱
  const data = await res.json();
  return data;
}
```

### 응답 본문 파싱 메서드

```js
const res = await fetch(url);

res.json()   // JSON 파싱 → 가장 많이 사용
res.text()   // 텍스트 문자열
res.blob()   // 파일(이미지 등) 바이너리
res.formData() // FormData 객체
res.arrayBuffer() // 바이너리 버퍼
```

## 2. POST / PATCH / DELETE 요청 🔴

```js
// POST — 데이터 생성
const res = await fetch("/api/users", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",  // 필수! 서버에 JSON임을 알려줌
  },
  body: JSON.stringify({
    name: "홍길동",
    email: "hong@test.com",
  }),
});
const newUser = await res.json();

// PATCH — 데이터 부분 수정
await fetch(`/api/users/${id}`, {
  method: "PATCH",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "새이름" }),
});

// DELETE — 데이터 삭제
await fetch(`/api/users/${id}`, {
  method: "DELETE",
});

// PUT — 데이터 전체 교체
await fetch(`/api/users/${id}`, {
  method: "PUT",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({ name: "새이름", email: "new@test.com", age: 30 }),
});
```

## 3. AbortController — 요청 취소 🟡

> 💡 검색창에 타이핑할 때마다 API를 호출하면, 이전 요청이 아직 진행 중일 수 있다.
> 새로운 요청을 보낼 때 이전 요청을 취소해야 Race Condition을 방지한다.

```js
// 기본 사용법
const controller = new AbortController();
const { signal } = controller;

fetch("/api/search?q=react", { signal })
  .then(res => res.json())
  .then(data => console.log(data))
  .catch(err => {
    if (err.name === "AbortError") {
      console.log("요청이 취소되었습니다");
    } else {
      throw err;
    }
  });

// 요청 취소!
controller.abort();

// React에서의 실전 패턴 (useEffect 클린업)
useEffect(() => {
  const controller = new AbortController();

  fetch(`/api/search?q=${query}`, { signal: controller.signal })
    .then(res => res.json())
    .then(setResults)
    .catch(err => {
      if (err.name !== "AbortError") console.error(err);
    });

  // 클린업: 컴포넌트 언마운트 또는 query 변경 시 이전 요청 취소
  return () => controller.abort();
}, [query]);
```

## 4. CORS — 왜 내 API 요청이 막히는가? 🔴

> 💡 **비유: 학교 출입증**
> 브라우저는 보안을 위해 "다른 출처(Origin)"의 서버에 요청을 기본적으로 차단한다.
> 서버가 "이 출처는 허용이야"라고 허가증(CORS 헤더)을 발급해야 통과된다.

```
같은 출처 (Same Origin):
  https://mysite.com/api/data     → mysite.com 페이지에서 요청 ✅

다른 출처 (Cross Origin):
  https://api.other.com/data      → mysite.com 페이지에서 요청 ❌ CORS!

출처(Origin) = 프로토콜 + 도메인 + 포트
  https://mysite.com:443  vs  http://mysite.com:80  → 다른 출처!
  https://mysite.com      vs  https://api.mysite.com → 다른 출처!
```

### CORS 에러 해결 방법

```
1. 서버에서 허용 (백엔드가 CORS 헤더 추가)
   Access-Control-Allow-Origin: https://mysite.com

2. Next.js에서는 같은 서버이므로 CORS 문제 없음!
   /api/... 라우트 핸들러는 같은 출처

3. 개발 시: next.config.ts의 rewrites로 프록시
   /api/external → https://external-api.com 으로 우회

4. 개발 도구: 브라우저 확장(CORS Unblock)은 개발용으로만!
```

> ⚠️ **CORS는 브라우저의 보안 정책이다.** 서버 → 서버(SSR, Server Actions)에서는 발생하지 않는다.
> Next.js의 서버 컴포넌트에서 외부 API를 호출하면 CORS 문제가 없는 이유!

## 면접 대비

### 🔴 필수 질문

**Q1. fetch의 에러 처리에서 주의할 점?**

> `fetch`는 네트워크 장애(인터넷 끊김 등)에서만 Promise를 reject한다. 404나 500 같은 HTTP 에러 상태에서는 reject되지 않고 정상적으로 Response 객체를 반환한다. 따라서 `response.ok` 속성(status 200~299)을 반드시 확인하고, false일 경우 명시적으로 에러를 throw해야 한다.

**Q2. CORS란 무엇이고, 왜 발생하며, 어떻게 해결하나요?**

> CORS(Cross-Origin Resource Sharing)는 브라우저의 보안 정책으로, 다른 출처(프로토콜+도메인+포트)의 서버에 요청하는 것을 기본적으로 차단한다. 서버가 응답 헤더에 `Access-Control-Allow-Origin`을 설정하여 특정 출처의 요청을 허용하면 해결된다. Next.js에서는 서버 컴포넌트나 Route Handler에서 외부 API를 호출하면 서버-서버 통신이므로 CORS가 발생하지 않는다.
