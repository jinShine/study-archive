## 1. 콜백 지옥 — Promise가 나온 이유 🔴

> 💡 **비유: 줄줄이 전화 돌리기**
> "A한테 전화해서 결과 받으면 → B한테 전화해서 결과 받으면 → C한테 전화..."
> 콜백이 겹겹이 중첩되면 코드가 오른쪽으로 밀려가며 읽기 불가능해진다.

```js
// ❌ 콜백 지옥 (Callback Hell) — 실제로 이렇게 썼었다!
getUser(userId, function(user) {
  getOrders(user.id, function(orders) {
    getOrderDetail(orders[0].id, function(detail) {
      getShipping(detail.shippingId, function(shipping) {
        console.log(shipping.status);
        // 더 깊어지면... 💀
      });
    });
  });
});

// ✅ Promise 체이닝으로 해결
getUser(userId)
  .then(user => getOrders(user.id))
  .then(orders => getOrderDetail(orders[0].id))
  .then(detail => getShipping(detail.shippingId))
  .then(shipping => console.log(shipping.status))
  .catch(error => console.error("에러:", error));
```

## 2. Promise의 3가지 상태 🔴

```
Promise = "미래에 완료될 작업의 약속"

┌─────────┐     성공     ┌───────────┐
│ Pending │ ──────────▶ │ Fulfilled │ → .then(결과)
│ (대기중) │             │  (이행됨)  │
└────┬────┘             └───────────┘
     │
     │       실패     ┌───────────┐
     └──────────────▶ │ Rejected  │ → .catch(에러)
                      │  (거부됨)  │
                      └───────────┘

한 번 Fulfilled/Rejected가 되면 다시는 바뀌지 않는다! (Settled)
```

```js
// Promise 생성 (라이브러리 내부에서 주로 사용)
const promise = new Promise((resolve, reject) => {
  // 비동기 작업 수행
  const success = true;

  if (success) {
    resolve("성공 데이터!");  // → Fulfilled 상태로 전환
  } else {
    reject(new Error("실패!")); // → Rejected 상태로 전환
  }
});

// Promise 소비
promise
  .then(data => console.log(data))     // "성공 데이터!"
  .catch(error => console.error(error)) // 에러 발생 시
  .finally(() => console.log("끝!"));   // 성공/실패 무관하게 항상 실행
```

## 3. .then / .catch / .finally 체이닝 🔴

```js
fetch("https://api.example.com/users/1")
  .then(response => {
    if (!response.ok) throw new Error(`HTTP ${response.status}`);
    return response.json();  // 다음 .then으로 전달
  })
  .then(user => {
    console.log(user.name);
    return fetch(`/api/orders/${user.id}`);  // 다음 Promise 반환
  })
  .then(response => response.json())
  .then(orders => console.log("주문:", orders))
  .catch(error => {
    // 위 체인 어디서든 에러가 나면 여기로!
    console.error("에러 발생:", error.message);
  })
  .finally(() => {
    // 로딩 스피너 끄기 등
    console.log("요청 완료 (성공이든 실패든)");
  });
```

> 💡 `.catch()`는 체인의 **어디서든** 발생한 에러를 잡는다.
> 마지막에 하나만 두면 모든 에러를 한 곳에서 처리 가능.

## 4. Promise 정적 메서드 🔴

### Promise.all — 모두 성공해야 성공 (AND)

```js
// 3개 요청을 동시에 보내고, 전부 완료되면 결과를 받는다
const [users, products, orders] = await Promise.all([
  fetch("/api/users").then(r => r.json()),
  fetch("/api/products").then(r => r.json()),
  fetch("/api/orders").then(r => r.json()),
]);

// ⚠️ 하나라도 실패하면 전체가 reject! (All or Nothing)
```

> 💡 **실무 활용**: 대시보드에서 여러 API를 동시에 호출할 때.
> 순차(직렬) 호출보다 훨씬 빠르다! (3초 + 2초 + 1초 = 직렬 6초 vs 병렬 3초)

### Promise.allSettled — 성공/실패 상관없이 전부 기다림 🟡

```js
const results = await Promise.allSettled([
  fetch("/api/users"),      // 성공
  fetch("/api/broken"),     // 실패
  fetch("/api/products"),   // 성공
]);
// [{status:"fulfilled", value:...}, {status:"rejected", reason:...}, {status:"fulfilled", value:...}]

// 실패한 것만 필터링
const failed = results.filter(r => r.status === "rejected");
```

### Promise.race — 가장 빠른 하나만 🟡

```js
// 타임아웃 패턴: 3초 안에 응답 없으면 에러!
const result = await Promise.race([
  fetch("/api/data"),
  new Promise((_, reject) =>
    setTimeout(() => reject(new Error("타임아웃!")), 3000)
  ),
]);
```

### Promise.any — 하나라도 성공하면 성공 (OR) 🟢

```js
// 여러 CDN 중 가장 빠른 것 사용
const fastest = await Promise.any([
  fetch("https://cdn1.example.com/data"),
  fetch("https://cdn2.example.com/data"),
  fetch("https://cdn3.example.com/data"),
]);
```

## 5. 실습: Promise 체이닝

```js
// 과제: 아래 콜백 지옥을 Promise 체이닝으로 리팩토링

// 콜백 버전 (변환 대상)
function getDataCallback() {
  setTimeout(() => {
    const userId = 1;
    console.log("1단계: userId =", userId);

    setTimeout(() => {
      const userName = "철수";
      console.log("2단계: userName =", userName);

      setTimeout(() => {
        const greeting = `안녕, ${userName}!`;
        console.log("3단계:", greeting);
      }, 500);
    }, 500);
  }, 500);
}

// Promise 체이닝 버전으로 변환해보자!
```

## 면접 대비

### 🔴 필수 질문

**Q1. Promise의 3가지 상태?**

> Pending(대기) — 아직 완료되지 않은 초기 상태. Fulfilled(이행) — 비동기 작업이 성공하여 `resolve`가 호출된 상태로 `.then()`으로 결과를 받는다. Rejected(거부) — 비동기 작업이 실패하여 `reject`가 호출된 상태로 `.catch()`로 에러를 받는다. 한 번 settled(이행/거부)되면 다시 변경되지 않는다.

**Q2. Promise.all과 Promise.allSettled의 차이?**

> `Promise.all`은 모든 Promise가 성공해야 결과를 반환하며, 하나라도 실패하면 즉시 reject된다. `Promise.allSettled`은 성공/실패 여부와 관계없이 모든 Promise가 완료될 때까지 기다리고, 각각의 상태(`fulfilled`/`rejected`)와 결과를 배열로 반환한다.
