## 1. 제네릭이란? — "타입의 변수" 🔴

> 💡 **비유: 택배 상자의 라벨**
> 같은 크기의 상자라도 "과일용", "전자제품용"이라고 라벨을 붙이면
> 상자 안에 뭐가 있는지 열기 전에 알 수 있다.
> 제네릭 = 함수나 타입에 붙이는 "타입 라벨"

```ts
// ❌ 제네릭 없이: any를 쓰면 타입 안전성이 사라짐
function getFirst(arr: any[]): any {
  return arr[0];
}
const first = getFirst([1, 2, 3]);
// first의 타입이 any → 자동완성 안 됨 😱

// ✅ 제네릭: 호출할 때 타입이 결정됨!
function getFirst<T>(arr: T[]): T {
  return arr[0];
}
const first = getFirst([1, 2, 3]);       // first: number ✅
const first2 = getFirst(["a", "b"]);     // first2: string ✅
// 타입 추론이 되므로 <number>를 명시할 필요 없음!
```

### T는 무엇인가?

```
<T> = 타입 변수 (Type Parameter)

함수의 매개변수가 "값의 placeholder"이듯,
제네릭의 T는 "타입의 placeholder"이다.

T는 관례적인 이름일 뿐, 아무 이름이나 가능:
  <T>     → Type (가장 흔함)
  <K, V>  → Key, Value (Map 등)
  <E>     → Element
  <P>     → Props (React)
```

## 2. 제네릭 함수 🔴

```ts
// 항등 함수 (받은 그대로 반환)
function identity<T>(value: T): T {
  return value;
}
identity(42);        // T = number → 반환: number
identity("hello");   // T = string → 반환: string

// 두 개의 타입 변수
function pair<A, B>(first: A, second: B): [A, B] {
  return [first, second];
}
pair("이름", 25);     // [string, number]
pair(true, [1,2,3]);  // [boolean, number[]]

// 배열 유틸 함수
function last<T>(arr: T[]): T | undefined {
  return arr[arr.length - 1];
}
last([1, 2, 3]);      // number | undefined
last(["a", "b"]);     // string | undefined
```

## 3. 제약 조건 (extends) 🟡

> "아무 타입이나 받으면 안 되고, 최소한 이 형태는 갖춰야 해!"

```ts
// ❌ T가 아무 타입이나 올 수 있어서 .length를 쓸 수 없음
function getLength<T>(value: T): number {
  return value.length;  // ❌ 에러! T에 length가 있다는 보장이 없음
}

// ✅ extends로 제약: "최소한 length 속성을 가진 타입만 받겠다"
function getLength<T extends { length: number }>(value: T): number {
  return value.length;  // ✅ length가 보장됨
}

getLength("hello");     // 5 ✅ (string에 length 있음)
getLength([1, 2, 3]);   // 3 ✅ (배열에 length 있음)
getLength(42);          // ❌ 에러! number에 length 없음

// 실전: 객체의 키만 허용
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { name: "철수", age: 25 };
getProperty(user, "name");   // string ✅
getProperty(user, "age");    // number ✅
getProperty(user, "email");  // ❌ 에러! "email"은 User의 키가 아님
```

## 4. 제네릭 타입/인터페이스 🟡

### API 응답 래퍼 (실무에서 가장 많이 쓰는 패턴!)

```ts
// 모든 API 응답의 공통 구조
type ApiResponse<T> = {
  data: T;
  status: number;
  message: string;
  timestamp: string;
};

// 사용
type UserResponse = ApiResponse<{ id: number; name: string }>;
type ProductListResponse = ApiResponse<Product[]>;

// fetch 래퍼
async function fetchApi<T>(url: string): Promise<ApiResponse<T>> {
  const res = await fetch(url);
  return res.json();
}

const users = await fetchApi<User[]>("/api/users");
// users.data는 User[] 타입! 자동완성 완벽! ✅
```

### React 컴포넌트에서

```tsx
// 제네릭 컴포넌트 (리스트)
type ListProps<T> = {
  items: T[];
  renderItem: (item: T) => React.ReactNode;
};

function List<T>({ items, renderItem }: ListProps<T>) {
  return <ul>{items.map(renderItem)}</ul>;
}

// 사용 — T가 자동 추론!
<List
  items={[{ id: 1, name: "철수" }]}
  renderItem={(user) => <li>{user.name}</li>}  // user가 자동으로 { id, name } 타입!
/>
```

function randomPick<T>(arr: T[]): T {

## 면접 대비

### 🟡 이해 질문

**Q1. 제네릭이란? 왜 필요한가?**

> 함수나 타입을 정의할 때 타입을 매개변수화하여, 호출 시점에 구체적인 타입이 결정되도록 하는 기능이다. `any`를 쓰면 타입 안전성이 사라지지만, 제네릭을 쓰면 다양한 타입에 재사용하면서도 타입 안전성과 자동완성을 유지할 수 있다. `Array<T>`, `Promise<T>`, `useState<T>` 등 기본 내장 타입부터 모두 제네릭이다.

**Q2. 제네릭의 `extends`는 무슨 역할?**

> 타입 변수가 받을 수 있는 타입의 범위를 제한(제약)한다. `<T extends { length: number }>`는 "T는 최소한 length 속성을 가진 타입이어야 한다"는 의미다. `keyof T`와 결합하면 객체의 실제 키만 허용하는 등 정밀한 타입 안전성을 구현할 수 있다.
