## 1. Set — 중복 없는 값의 집합 🔴

> 💡 **비유: 출석부**
> 같은 이름을 두 번 적어도 출석부에는 한 번만 기록된다.

```js
const set = new Set();

// 추가
set.add(1);
set.add(2);
set.add(2);  // 무시됨! 이미 있으니까
set.add(3);
console.log(set);       // Set(3) {1, 2, 3}
console.log(set.size);  // 3

// 확인/삭제
set.has(2);     // true
set.delete(2);  // true
set.has(2);     // false

// 실전: 배열 중복 제거 (가장 많이 쓰는 패턴!)
const arr = [1, 2, 2, 3, 3, 3, 4];
const unique = [...new Set(arr)];  // [1, 2, 3, 4] ✅

// 문자열 중복 제거
const tags = ["react", "next", "react", "vue", "next"];
const uniqueTags = [...new Set(tags)];  // ["react", "next", "vue"]

// 교집합 / 합집합 / 차집합 (ES2025+)
const a = new Set([1, 2, 3]);
const b = new Set([2, 3, 4]);
a.intersection(b);  // Set {2, 3}
a.union(b);          // Set {1, 2, 3, 4}
a.difference(b);     // Set {1}
```

## 2. Map — 키-값 쌍의 컬렉션 🟡

> 💡 **비유: 사전(Dictionary)**
> 일반 객체와 비슷하지만, **어떤 타입이든 키로 사용** 가능하다.

```js
const map = new Map();

// 추가 (어떤 타입이든 키로 사용 가능!)
map.set("name", "철수");
map.set(42, "숫자 키");
map.set(true, "불린 키");

const objKey = { id: 1 };
map.set(objKey, "객체도 키로 가능!");

// 조회
map.get("name");     // "철수"
map.get(42);         // "숫자 키"
map.has("name");     // true
map.size;            // 4

// 순회 (삽입 순서 보장!)
map.forEach((value, key) => console.log(`${key}: ${value}`));

// for...of로 순회
for (const [key, value] of map) {
  console.log(`${key}: ${value}`);
}
```

### Map vs 일반 객체

| 구분 | `Object` | `Map` |
|------|----------|-------|
| **키 타입** | 문자열/Symbol만 | 모든 타입 (객체, 함수, 숫자 등) |
| **순서** | 보장 안 될 수 있음 | 삽입 순서 보장 |
| **크기** | `Object.keys().length` | `.size` (O(1)) |
| **순회** | `Object.entries()` 필요 | 직접 이터러블 (`for...of`) |
| **성능** | 잦은 추가/삭제에 불리 | 잦은 추가/삭제에 최적화 |
| **JSON** | 직접 변환 가능 | 변환 불가 (수동 처리 필요) |

> 💡 **선택 기준**: JSON 데이터를 다루거나 API 응답이면 일반 객체.
> 키가 문자열이 아니거나, 잦은 삽입/삭제/크기 조회가 필요하면 Map.

## 3. WeakMap / WeakSet 🟡

```js
// WeakMap: 키가 가비지 컬렉션(GC)의 대상이 될 수 있음
// → 메모리 누수 방지! 캐시, DOM 요소 관련 데이터 저장에 유용

let user = { name: "철수" };
const cache = new WeakMap();
cache.set(user, "캐싱된 데이터");

user = null;  // user 참조가 사라지면, WeakMap의 항목도 자동으로 GC됨!
// 일반 Map이었다면 메모리에 계속 남아있었을 것!
```

> 💡 WeakMap은 순회(`forEach`)가 불가하고, `size`도 없다. 오직 `get/set/has/delete`만 가능.

## 4. Symbol 🟡

```js
// Symbol: 절대 겹치지 않는 유일한 식별자
const id1 = Symbol("id");
const id2 = Symbol("id");
console.log(id1 === id2);  // false! (설명이 같아도 다른 Symbol)

// 객체의 "숨겨진" 키로 사용 (for...in에 안 잡힘)
const user = {
  name: "철수",
  [id1]: "비밀 ID"
};
console.log(user[id1]);        // "비밀 ID"
console.log(Object.keys(user)); // ["name"] — Symbol 키는 안 보임!

// 글로벌 심볼 레지스트리
const s1 = Symbol.for("app.id");  // 레지스트리에 등록
const s2 = Symbol.for("app.id");  // 같은 키로 조회
console.log(s1 === s2);           // true! (같은 심볼)
```

> 💡 실무에서 직접 Symbol을 쓸 일은 적지만, 라이브러리 내부(React의 `$$typeof` 등)에서 사용된다.

## 5. for...of vs for...in 🟡

```js
const arr = ["a", "b", "c"];

// for...of → 값(value)을 순회 (배열, Set, Map, 문자열)
for (const value of arr) {
  console.log(value);  // "a", "b", "c"
}

// for...in → 키(key)를 순회 (객체의 프로퍼티)
const obj = { name: "철수", age: 25 };
for (const key in obj) {
  console.log(key);       // "name", "age"
  console.log(obj[key]);  // "철수", 25
}

// ⚠️ 배열에 for...in 쓰지 마라! (인덱스를 문자열로 반환 + 프로토타입 키도 포함)
for (const i in arr) {
  console.log(typeof i);  // "string" 😱 (숫자가 아니라 문자열 "0", "1", "2")
}
```

> 💡 배열 → `for...of` 또는 `forEach`/`map`
> 객체 → `Object.entries()` + `for...of` 또는 `for...in`

const nums = [5, 3, 1, 3, 2, 5, 4, 1];

## 면접 대비

### 🔴 필수 질문

**Q1. 배열에서 중복을 제거하는 가장 간단한 방법은?**

> `[...new Set(array)]`로 Set을 거쳐 중복을 제거한 후 다시 배열로 변환한다. Set은 같은 값을 중복 저장하지 않는 자료구조이므로 O(n) 시간에 중복이 제거된다.

### 🟡 개념 질문

**Q2. Map과 일반 객체의 차이?**

> Map은 모든 타입을 키로 사용할 수 있고, 삽입 순서가 보장되며, `.size`로 O(1)에 크기를 확인할 수 있고, 잦은 삽입/삭제에 최적화되어 있다. 일반 객체는 키가 문자열/Symbol로 제한되지만, JSON 직렬화가 가능하고 리터럴 문법이 간결하다.
