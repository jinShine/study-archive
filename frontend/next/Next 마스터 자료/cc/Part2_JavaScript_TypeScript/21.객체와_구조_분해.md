## 1. 객체 리터럴 심화 🔴

```js
// 단축 속성명 (Shorthand Property)
const name = "철수";
const age = 25;

// ❌ 옛날 방식
const user = { name: name, age: age };

// ✅ 변수명과 키가 같으면 생략
const user = { name, age };  // { name: "철수", age: 25 }

// 단축 메서드
const calculator = {
  // ❌ 옛날
  add: function(a, b) { return a + b; },
  // ✅ 단축
  subtract(a, b) { return a - b; },
};

// 계산된 속성명 (Computed Property)
const key = "score";
const obj = {
  [key]: 100,            // { score: 100 }
  [`${key}Double`]: 200, // { scoreDouble: 200 }
};
```

## 2. 구조 분해 할당 (Destructuring) 🔴

> 💡 **비유: 택배 상자 풀기**
> 큰 상자(객체/배열)에서 필요한 물건만 쏙쏙 꺼내서 개별 변수에 담는 것

### 객체 구조 분해

```js
const user = { name: "철수", age: 25, email: "chul@test.com" };

// ❌ 일일이 꺼내기
const name = user.name;
const age = user.age;

// ✅ 구조 분해로 한 번에!
const { name, age, email } = user;
console.log(name);  // "철수"
console.log(age);   // 25

// 다른 변수명으로 받기 (별칭)
const { name: userName, age: userAge } = user;
console.log(userName);  // "철수"

// 기본값 설정
const { name, role = "viewer" } = user;
console.log(role);  // "viewer" (user에 role이 없으므로 기본값)

// 중첩 객체 분해
const response = {
  data: {
    user: { name: "철수", age: 25 }
  },
  status: 200
};
const { data: { user: { name } }, status } = response;
console.log(name);    // "철수"
console.log(status);  // 200
```

### 배열 구조 분해

```js
const colors = ["빨강", "파랑", "초록"];

// ❌ 인덱스로 접근
const first = colors[0];

// ✅ 구조 분해
const [first, second, third] = colors;
console.log(first);   // "빨강"
console.log(second);  // "파랑"

// 건너뛰기 (쉼표로)
const [, , last] = colors;
console.log(last);  // "초록"

// 변수 교환 (스왑)
let a = 1, b = 2;
[a, b] = [b, a];
console.log(a, b);  // 2, 1

// React의 useState가 바로 이 문법!
const [count, setCount] = useState(0);
//      값       세터      = 배열 구조 분해!
```

### 함수 매개변수에서 구조 분해 🔴

```js
// ❌ 객체 통째로 받기
function greet(user) {
  console.log(`안녕, ${user.name}!`);
}

// ✅ 매개변수에서 바로 분해 (React Props 패턴!)
function greet({ name, age = 0 }) {
  console.log(`안녕, ${name}! (${age}세)`);
}

greet({ name: "철수", age: 25 });  // "안녕, 철수! (25세)"

// React 컴포넌트에서:
function UserCard({ name, email, avatar }) {
  return (
    <div>
      <img src={avatar} alt={name} />
      <h2>{name}</h2>
      <p>{email}</p>
    </div>
  );
}
```

## 3. 스프레드 연산자 (...) 🔴

### 객체 스프레드

```js
const user = { name: "철수", age: 25 };

// 복사 (얕은 복사)
const copy = { ...user };

// 속성 추가/덮어쓰기
const updated = { ...user, age: 26, role: "admin" };
// { name: "철수", age: 26, role: "admin" }

// 순서가 중요! 뒤에 오는 것이 앞을 덮어씀
const a = { x: 1, y: 2 };
const b = { y: 3, z: 4 };
const merged = { ...a, ...b };  // { x: 1, y: 3, z: 4 }
```

### 배열 스프레드

```js
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];

// 합치기
const combined = [...arr1, ...arr2];  // [1, 2, 3, 4, 5, 6]

// 복사
const copy = [...arr1];

// 요소 추가
const withNew = [...arr1, 99];        // [1, 2, 3, 99]
const withFront = [0, ...arr1];       // [0, 1, 2, 3]
```

### Rest 패턴 (나머지 모으기)

```js
// 객체 rest
const { name, ...rest } = { name: "철수", age: 25, email: "test@test.com" };
console.log(name);  // "철수"
console.log(rest);  // { age: 25, email: "test@test.com" }

// 배열 rest
const [first, ...others] = [1, 2, 3, 4, 5];
console.log(first);   // 1
console.log(others);  // [2, 3, 4, 5]

// React에서 Props 나머지 전달 패턴!
function Button({ children, variant, ...rest }) {
  return <button className={variant} {...rest}>{children}</button>;
}
// <Button variant="primary" onClick={fn} disabled>클릭</Button>
// → rest = { onClick: fn, disabled: true }
```

## 4. 얕은 복사 vs 깊은 복사 🔴

```js
const user = {
  name: "철수",
  address: {
    city: "서울",
    zip: "06000"
  },
  hobbies: ["코딩", "독서"]
};

// 얕은 복사 (Shallow Copy) — 1단계만 복사, 중첩 객체는 참조 공유!
const shallow = { ...user };
shallow.name = "영희";
console.log(user.name);  // "철수" ✅ (1단계는 독립적)

shallow.address.city = "부산";
console.log(user.address.city);  // "부산" 😱 (중첩 객체는 공유됨!)

// 깊은 복사 (Deep Copy) — 모든 단계를 완전히 독립적으로 복사
const deep = structuredClone(user);  // ✅ 2022+ 표준!
deep.address.city = "제주";
console.log(user.address.city);  // "서울" ✅ (완전 독립!)
```

```
얕은 복사 시각화:

user              shallow
┌──────────┐      ┌──────────┐
│name:"철수"│      │name:"영희"│  ← 1단계: 독립적 ✅
│address: ●─┤──┐   │address: ●─┤──┐
└──────────┘  │   └──────────┘  │
              ▼                 ▼
          ┌─────────────┐
          │city: "부산"  │  ← 중첩: 같은 객체를 공유! 😱
          └─────────────┘
```

### 깊은 복사 방법 비교

```js
// ✅ 1. structuredClone (2022+ 표준. 가장 추천!)
const deep1 = structuredClone(original);

// 🟡 2. JSON 트릭 (함수, undefined, Date 등은 소실됨!)
const deep2 = JSON.parse(JSON.stringify(original));

// 🟡 3. 수동 스프레드 (2단계만 필요할 때)
const deep3 = {
  ...user,
  address: { ...user.address },
  hobbies: [...user.hobbies],
};
```

## 5. Object 주요 메서드 🟡

```js
const user = { name: "철수", age: 25, email: "test@test.com" };

// 키 배열
Object.keys(user);     // ["name", "age", "email"]

// 값 배열
Object.values(user);   // ["철수", 25, "test@test.com"]

// [키, 값] 쌍 배열
Object.entries(user);  // [["name","철수"], ["age",25], ["email","test@test.com"]]

// entries → 객체로 복원
Object.fromEntries([["a", 1], ["b", 2]]);  // { a: 1, b: 2 }

// 객체 병합
Object.assign({}, user, { role: "admin" });  // 스프레드와 동일 효과

// 객체 동결 (수정 불가)
const frozen = Object.freeze({ name: "철수" });
frozen.name = "영희";  // 무시됨 (strict mode에서는 에러)
```

```js
// 실전: 객체 순회
Object.entries(user).forEach(([key, value]) => {
  console.log(`${key}: ${value}`);
});
// name: 철수
// age: 25
// email: test@test.com

// 실전: 객체 변환
const prices = { apple: 1000, banana: 2000, cherry: 3000 };
const discounted = Object.fromEntries(
  Object.entries(prices).map(([fruit, price]) => [fruit, price * 0.9])
);
// { apple: 900, banana: 1800, cherry: 2700 }
```

## 면접 대비

### 🔴 필수 질문

**Q1. 구조 분해 할당이란?**

> 배열이나 객체에서 값을 추출하여 개별 변수에 할당하는 문법이다. 객체는 `const { name, age } = user`, 배열은 `const [first, second] = arr` 형태다. 함수 매개변수에서 `function fn({ name, age })` 형태로 사용하면 Props를 깔끔하게 받을 수 있어 React에서 필수적으로 사용된다.

**Q2. 얕은 복사와 깊은 복사의 차이?**

> 얕은 복사(Shallow Copy)는 1단계 속성만 새로 복사하고, 중첩된 객체/배열은 원본과 참조를 공유한다. 스프레드(`{...obj}`)가 대표적이다. 깊은 복사(Deep Copy)는 모든 중첩 단계를 완전히 독립적으로 복사한다. `structuredClone()`이 2022년부터 표준이며, JSON 트릭(`JSON.parse(JSON.stringify())`)은 함수와 특수 값이 소실되는 한계가 있다.

**Q3. 스프레드 연산자의 활용?**

> 객체/배열의 얕은 복사(`{...obj}`), 병합(`{...a, ...b}`), 속성 추가/덮어쓰기, 배열 합치기(`[...arr1, ...arr2]`)에 사용한다. React에서는 불변성을 지키며 상태를 업데이트할 때 `setState({...state, name: "새값"})` 패턴으로 필수적이다.
