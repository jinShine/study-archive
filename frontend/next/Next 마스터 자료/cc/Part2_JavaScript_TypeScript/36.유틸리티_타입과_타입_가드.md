# 36. 유틸리티 타입과 타입 가드

> **키워드**: `Partial` `Pick` `Omit` `Record` `ReturnType` `typeof` `in` `is` `판별 유니온`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | Partial, Pick, Omit, Record, 타입 가드(typeof/in) | 실무 매일 사용 |
| 🟡 이해 | ReturnType, Parameters, 사용자 정의 타입 가드(is), 판별 유니온 | 안전한 코드 |
| 🟢 참고 | Exclude, Extract, NonNullable, Awaited | 고급 타입 조작 |

## 1. 유틸리티 타입 — 기존 타입을 변환하는 도구 🔴

> 💡 **비유: 사진 필터**
> 원본 사진(타입)은 그대로 두고, 필터(유틸리티)를 씌워서 새 버전을 만든다.

```ts
type User = {
  id: number;
  name: string;
  email: string;
  age: number;
  role: "admin" | "user";
};
```

### Partial<T> — 모든 속성을 선택적으로 🔴

```ts
type UpdateUserInput = Partial<User>;
// { id?: number; name?: string; email?: string; age?: number; role?: ... }

// 실전: API 수정 요청 (일부만 보내도 됨)
async function updateUser(id: number, data: Partial<User>) {
  await fetch(`/api/users/${id}`, {
    method: "PATCH",
    body: JSON.stringify(data),
  });
}
updateUser(1, { name: "새이름" });  // name만 보내도 OK!
```

### Required<T> — 모든 속성을 필수로

```ts
type RequiredUser = Required<User>;
// 모든 ? 가 제거됨. 전부 필수!
```

### Pick<T, Keys> — 특정 속성만 추출 🔴

```ts
type UserPreview = Pick<User, "id" | "name">;
// { id: number; name: string }

// 실전: 목록에서 보여줄 최소 데이터
type UserListItem = Pick<User, "id" | "name" | "email">;
```

### Omit<T, Keys> — 특정 속성만 제외 🔴

```ts
type CreateUserInput = Omit<User, "id">;
// { name: string; email: string; age: number; role: ... }
// id는 서버가 자동 생성하므로 제외!

// 실전: 비밀번호 제외한 사용자 공개 정보
type PublicUser = Omit<User & { password: string }, "password">;
```

### Record<Keys, Value> — 키-값 매핑 🔴

```ts
// 특정 키들의 값 타입을 한 번에 지정
type StatusColors = Record<"success" | "error" | "warning", string>;
// { success: string; error: string; warning: string }

const colors: StatusColors = {
  success: "#10B981",
  error: "#EF4444",
  warning: "#F59E0B",
};

// 동적 키의 객체
type UserMap = Record<string, User>;  // 어떤 문자열 키든, 값은 User
```

### ReturnType<T> — 함수 반환 타입 추출 🟡

```ts
function getUser() {
  return { id: 1, name: "철수", age: 25 };
}

type UserType = ReturnType<typeof getUser>;
// { id: number; name: string; age: number }

// 실전: Server Action의 반환 타입 추출
type ActionResult = ReturnType<typeof createProductAction>;
```

### Parameters<T> — 함수 매개변수 타입 추출 🟡

```ts
function greet(name: string, age: number): void { }

type GreetParams = Parameters<typeof greet>;
// [string, number] (튜플)
```

## 2. 타입 가드 (Type Guard) — 안전한 타입 좁히기 🔴

### typeof 가드

```ts
function format(value: string | number): string {
  if (typeof value === "string") {
    return value.toUpperCase();  // TS가 string으로 좁혀줌
  }
  return value.toFixed(2);       // TS가 number로 좁혀줌
}
```

### in 가드 (속성 존재 여부)

```ts
type Dog = { bark: () => void; breed: string };
type Cat = { meow: () => void; color: string };
type Pet = Dog | Cat;

function interact(pet: Pet) {
  if ("bark" in pet) {
    pet.bark();    // TS: Dog로 좁혀짐
    pet.breed;     // ✅ 접근 가능
  } else {
    pet.meow();    // TS: Cat으로 좁혀짐
  }
}
```

### instanceof 가드

```ts
function handleError(error: unknown) {
  if (error instanceof Error) {
    console.error(error.message);  // Error 타입으로 좁혀짐
  } else {
    console.error("알 수 없는 에러:", error);
  }
}
```

### 판별 유니온 (Discriminated Union) 🟡

> 💡 공통 속성(판별자)으로 타입을 구분하는 강력한 패턴

```ts
// 모든 타입에 공통 'type' 속성 (판별자)
type SuccessResponse = {
  type: "success";    // 리터럴 타입!
  data: User[];
};
type ErrorResponse = {
  type: "error";      // 리터럴 타입!
  message: string;
  code: number;
};
type ApiResult = SuccessResponse | ErrorResponse;

function handleResponse(result: ApiResult) {
  switch (result.type) {
    case "success":
      console.log(result.data);     // TS: SuccessResponse ✅
      break;
    case "error":
      console.log(result.message);  // TS: ErrorResponse ✅
      break;
  }
}
```

> 💡 **React에서**: `useReducer`의 action 타입이 판별 유니온의 대표적 활용!
> `{ type: "INCREMENT" } | { type: "SET_VALUE"; value: number }`

### 사용자 정의 타입 가드 (is) 🟡

```ts
// 반환 타입에 "is"를 써서 TS에게 타입 정보를 알려줌
function isString(value: unknown): value is string {
  return typeof value === "string";
}

function process(value: unknown) {
  if (isString(value)) {
    value.toUpperCase();  // TS가 string으로 좁혀줌!
  }
}

// 실전: null 제거 필터
function isNotNull<T>(value: T | null): value is T {
  return value !== null;
}

const items: (string | null)[] = ["a", null, "b", null, "c"];
const filtered = items.filter(isNotNull);
// filtered: string[] ✅ (null이 제거된 타입!)
```

## 면접 대비

### 🔴 필수 질문

**Q1. `Partial`, `Pick`, `Omit`을 설명해주세요.**

> `Partial<T>`는 T의 모든 속성을 선택적(`?`)으로 만든다. API 수정(PATCH) 요청에 유용하다. `Pick<T, K>`는 T에서 지정한 키만 추출한 새 타입이다. `Omit<T, K>`는 T에서 지정한 키를 제외한 새 타입이다. 예를 들어 `Omit<User, "password">`는 비밀번호를 제외한 공개 사용자 타입이 된다.

### 🟡 개념 질문

**Q2. 타입 가드란?**

> 유니온 타입의 변수를 조건문으로 좁혀서 특정 타입의 메서드를 안전하게 사용할 수 있게 하는 것이다. `typeof`, `in`, `instanceof`가 기본 타입 가드이고, 반환 타입에 `is`를 사용하는 사용자 정의 타입 가드도 있다. 판별 유니온은 공통 리터럴 속성(판별자)으로 `switch`문을 통해 타입을 좁히는 강력한 패턴이다.
