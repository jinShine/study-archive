# 78. 낙관적 업데이트 (useOptimistic)

> **키워드**: `useOptimistic` `0초 렌더링` `자동 롤백` `서버 지연 숨기기`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | useOptimistic 기본, 즉각 반응 UI, 롤백 메커니즘 | UX 극대화 |
| 🟡 이해 | 서버 에러 시 롤백, revalidatePath 연동 | 견고한 구현 |
| 🟢 참고 | TanStack Query의 Optimistic Update와 비교 | 아키텍처 |

---

## 1. 낙관적 업데이트란? 🔴

> 💡 **비유: 자판기**
> 버튼을 누르면 음료가 나올 거라고 **낙관적으로 믿고** 바로 손을 뻗는다.
> 서버 응답을 기다리지 않고 UI를 먼저 변경! 실패하면 원래대로 돌려놓는다!

```
❌ 비관적(Pessimistic): 버튼 클릭 → 스피너 🔄 → 1초 대기 → 서버 응답 → 화면 변경
✅ 낙관적(Optimistic): 버튼 클릭 → 즉시 화면 변경! → 백그라운드에서 서버 통신
                      (실패하면 자동 롤백!)
```

## 2. useOptimistic 기본 🔴

```tsx
"use client";
import { useOptimistic } from "react";
import { toggleLikeAction } from "@/actions";

export default function LikeButton({
  productId,
  initialLiked,
}: {
  productId: string;
  initialLiked: boolean;
}) {
  // useOptimistic(진짜 상태, 업데이트 함수)
  const [optimisticLiked, setOptimisticLiked] = useOptimistic(
    initialLiked,
    (currentState, newValue: boolean) => newValue
  );

  const handleLike = async () => {
    const newStatus = !optimisticLiked;

    // 1. UI를 먼저 변경! (서버 기다리지 않음!)
    setOptimisticLiked(newStatus);

    try {
      // 2. 백그라운드에서 서버 통신 (1초 걸려도 OK!)
      await toggleLikeAction(productId, newStatus);
    } catch {
      // 3. 실패하면 자동 롤백! (initialLiked로 복구!)
      alert("서버 통신 실패! 원래 상태로 복구됩니다.");
    }
  };

  return (
    <form action={handleLike}>
      <button type="submit" className="text-2xl">
        {optimisticLiked ? "❤️" : "🤍"}
      </button>
    </form>
  );
}
```

```tsx
// actions.ts
"use server";
import { revalidatePath } from "next/cache";

export async function toggleLikeAction(productId: string, newStatus: boolean) {
  // 네트워크 지연 시뮬레이션 (1초)
  await new Promise(resolve => setTimeout(resolve, 1000));

  // 10% 확률로 에러 (롤백 테스트!)
  if (Math.random() < 0.1) {
    throw new Error("서버 에러!");
  }

  await db.product.update({
    where: { id: productId },
    data: { liked: newStatus },
  });

  revalidatePath(`/products/${productId}`);
}
```

## 3. 핵심 동작 흐름 🔴

```
시나리오: 좋아요 버튼 클릭 (서버 응답 1초)

낙관적 업데이트 없이:
  T=0초: 클릭! → 스피너 시작 🔄
  T=1초: 서버 응답 → 하트 빨갛게! ❤️
  → 사용자가 1초 기다려야 함!

낙관적 업데이트:
  T=0초: 클릭! → 즉시 하트 빨갛게! ❤️ (가짜 상태)
  T=0초~1초: 백그라운드에서 서버 통신 (사용자는 모름!)
  T=1초: 서버 응답 성공 → 캐시 갱신 → 진짜 상태로 승격!
  → 사용자는 0초만에 반응! ⚡

서버 실패 시:
  T=0초: 클릭! → 즉시 하트 빨갛게! ❤️ (가짜 상태)
  T=1초: 서버 에러! → 자동으로 원래 상태(🤍)로 롤백!
  → 사용자에게 "실패!" 알림
```

## 4. 롤백 메커니즘 🔴

```
useOptimistic의 마법:

  setOptimisticLiked(true)로 가짜 상태를 부여
  → optimisticLiked = true (화면 즉시 변경!)

  서버 액션 성공:
  → revalidatePath가 서버 데이터를 갱신
  → initialLiked가 true로 업데이트
  → 가짜 상태가 진짜 상태로 자연스럽게 승격!

  서버 액션 실패 (catch):
  → 가짜 상태가 메모리에서 자동 증발!
  → initialLiked(원래 값)로 자동 복구!
  → 개발자가 별도로 setState를 안 해도 됨!
```

## 5. 실습

```
과제: 낙관적 UI가 적용된 좋아요 시스템

1. 상품 상세 페이지 (서버 컴포넌트)
   - DB에서 상품 정보 + liked 상태 조회
   - LikeButton에 initialLiked 전달

2. LikeButton (클라이언트 컴포넌트)
   - useOptimistic으로 즉시 UI 변경
   - 서버 액션에 1초 지연 + 10% 에러 시뮬레이션

3. 테스트:
   - 클릭 즉시 하트 변경 → 0초! (스피너 없음!)
   - 서버 터미널에서 1초 후 DB 업데이트 로그 확인
   - 10% 확률 에러 → 하트가 원래대로 돌아가는지 확인!
```

## 면접 대비

### 🔴 필수 질문

**Q1. 낙관적 업데이트(Optimistic Update)란?**

> 서버 응답을 기다리지 않고 UI를 먼저 변경하여 사용자에게 즉각적인 피드백을 제공하는 패턴이다. React 19의 `useOptimistic` 훅을 사용하면 가짜 상태를 즉시 적용하고, 서버 성공 시 진짜 상태로 승격, 실패 시 자동 롤백한다. 좋아요, 장바구니 담기 등 즉각 반응이 중요한 인터랙션에 사용한다.
