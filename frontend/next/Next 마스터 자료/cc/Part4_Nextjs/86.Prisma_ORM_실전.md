## 1. Prisma란? 🔴

> 💡 **비유: 통역사**
> SQL을 직접 쓰는 대신, TypeScript 코드로 DB를 조작!
> Prisma가 TypeScript → SQL로 자동 번역해준다!

```bash
npm install prisma @prisma/client
npx prisma init  # schema.prisma + .env 생성!
```

## 2. Schema 정의 🔴

```prisma
// prisma/schema.prisma

generator client {
  provider = "prisma-client-js"
}

datasource db {
  provider = "postgresql"  // postgresql, mysql, sqlite
  url      = env("DATABASE_URL")
}

// 사용자 모델
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String
  role      Role     @default(USER)
  posts     Post[]   // 1:N 관계! (한 유저가 여러 게시글)
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}

// 게시글 모델
model Post {
  id        Int      @id @default(autoincrement())
  title     String
  content   String?  // ? = nullable
  published Boolean  @default(false)
  author    User     @relation(fields: [authorId], references: [id])
  authorId  Int
  tags      Tag[]    // N:M 관계! (암묵적 중간 테이블)
  createdAt DateTime @default(now())
}

// 태그 모델
model Tag {
  id    Int    @id @default(autoincrement())
  name  String @unique
  posts Post[] // N:M 관계
}

// Enum
enum Role {
  USER
  ADMIN
}
```

### Migration 실행

```bash
# Schema 변경 → SQL 마이그레이션 자동 생성 + 적용!
npx prisma migrate dev --name init

# Prisma Client 재생성 (타입 업데이트!)
npx prisma generate

# DB 내용 시각적으로 확인!
npx prisma studio  # → http://localhost:5555
```

## 3. Prisma Client — CRUD 🔴

```tsx
// lib/db.ts — Prisma Client 싱글턴 (Next.js 필수 패턴!)
import { PrismaClient } from "@prisma/client";

const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };

export const db = globalForPrisma.prisma || new PrismaClient();

if (process.env.NODE_ENV !== "production") globalForPrisma.prisma = db;
// 개발 모드에서 Hot Reload 시 연결이 누적되는 것을 방지!
```

### Create (생성)

```tsx
const user = await db.user.create({
  data: {
    email: "chul@test.com",
    name: "철수",
    role: "ADMIN",
  },
});

// 관계 데이터와 함께 생성!
const post = await db.post.create({
  data: {
    title: "첫 게시글",
    content: "안녕하세요!",
    author: { connect: { id: userId } },  // 기존 유저 연결!
    tags: {
      connectOrCreate: [
        { where: { name: "react" }, create: { name: "react" } },
        { where: { name: "nextjs" }, create: { name: "nextjs" } },
      ],
    },
  },
});
```

### Read (조회)

```tsx
// 전체 조회
const users = await db.user.findMany();

// 조건 조회
const admins = await db.user.findMany({
  where: { role: "ADMIN" },
  orderBy: { createdAt: "desc" },
  take: 10,  // LIMIT 10
  skip: 0,   // OFFSET 0 (페이징!)
});

// 단건 조회
const user = await db.user.findUnique({
  where: { id: 1 },
  include: { posts: true },  // 관계 데이터도 함께! (JOIN)
});

// 검색
const results = await db.post.findMany({
  where: {
    OR: [
      { title: { contains: "react", mode: "insensitive" } },
      { content: { contains: "react", mode: "insensitive" } },
    ],
    published: true,
  },
});
```

### Update (수정)

```tsx
const updated = await db.user.update({
  where: { id: 1 },
  data: { name: "새이름" },
});
```

### Delete (삭제)

```tsx
await db.post.delete({ where: { id: 1 } });
```

## 4. Server Actions + Prisma 실전 패턴 🔴

```tsx
// actions.ts
"use server";
import { db } from "@/lib/db";
import { revalidatePath } from "next/cache";
import { redirect } from "next/navigation";

export async function createPost(prevState: any, formData: FormData) {
  const title = formData.get("title") as string;
  const content = formData.get("content") as string;

  if (!title || title.length < 2) {
    return { success: false, message: "제목 2자 이상!" };
  }

  await db.post.create({
    data: { title, content, authorId: 1 },  // TODO: 인증된 유저 ID
  });

  revalidatePath("/posts");
  redirect("/posts");
}

export async function deletePost(id: number) {
  await db.post.delete({ where: { id } });
  revalidatePath("/posts");
}
```

```tsx
// app/posts/page.tsx — 서버 컴포넌트에서 직접 DB!
import { db } from "@/lib/db";

export default async function PostsPage() {
  const posts = await db.post.findMany({
    include: { author: true },
    orderBy: { createdAt: "desc" },
  });

  return (
    <ul>
      {posts.map(post => (
        <li key={post.id}>
          <h2>{post.title}</h2>
          <p>by {post.author.name}</p>
        </li>
      ))}
    </ul>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. Prisma ORM이란?**

> TypeScript/JavaScript를 위한 타입 안전한 ORM(Object-Relational Mapping)이다. `schema.prisma` 파일에 데이터 모델을 선언하면, Prisma가 데이터베이스 마이그레이션과 타입이 완전한 Client 코드를 자동 생성한다. `db.user.findMany()` 같은 직관적인 API로 CRUD를 수행하며, 관계 조회(`include`), 검색(`where`), 페이징(`take`/`skip`)을 타입 안전하게 지원한다.

│    db.user.delete({ where })                    │
│                                                │
│  관계: include: { posts: true } (JOIN!)          │
│  검색: where: { title: { contains: "검색어" } }  │
│                                                │
│  Next.js 패턴:                                   │
│    서버 컴포넌트 → db.post.findMany() (직접!)    │
│    Server Action → db.post.create() + revalidate│
│                                                │
│  싱글턴: globalForPrisma 패턴 (Hot Reload 방지!) │
