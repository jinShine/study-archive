# 84. 병렬 라우트 (Parallel Routes)

> **키워드**: `@folder` `슬롯` `default.tsx` `독립 로딩/에러` `장애 격리`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🟡 이해 | @folder 슬롯, layout 조립, default.tsx 방어 | 대시보드 설계 |
| 🟢 참고 | 병렬 + 인터셉팅 융합 (모달), Hard Navigation 대응 | 고급 라우팅 |

---

## 1. 병렬 라우트란? 🟡

> 💡 **비유: TV 화면 분할**
> 하나의 화면(URL)에 여러 독립적인 콘텐츠를 동시에 렌더링!
> 한 영역이 에러나도 다른 영역은 정상 작동! (장애 격리!)

```
app/dashboard/
├── @analytics/        ← 📊 통계 슬롯
│   ├── page.tsx
│   └── loading.tsx    ← 독립적 로딩!
├── @feed/             ← 📰 피드 슬롯
│   ├── page.tsx
│   └── error.tsx      ← 독립적 에러 처리!
├── @notice/           ← 📢 공지 슬롯
│   └── page.tsx
├── layout.tsx         ← 조립 공장!
└── page.tsx           ← 기본 콘텐츠 (children)
```

```tsx
// layout.tsx — 슬롯들을 조립!
export default function DashboardLayout({
  children,    // 기본 page.tsx
  analytics,   // @analytics 폴더
  feed,        // @feed 폴더
  notice,      // @notice 폴더
}: {
  children: React.ReactNode;
  analytics: React.ReactNode;
  feed: React.ReactNode;
  notice: React.ReactNode;
}) {
  return (
    <div className="grid grid-cols-12 gap-4">
      <main className="col-span-8">{children}</main>
      <aside className="col-span-4 space-y-4">
        {analytics}  {/* 독립 로딩! 독립 에러! */}
        {feed}
        {notice}
      </aside>
    </div>
  );
}
```

### 장점

```
1. 장애 격리: 피드에서 에러 → 통계/공지는 정상!
2. 독립 로딩: 통계가 느려도 → 피드/공지는 먼저 표시!
3. 팀 분업: 각 슬롯을 다른 개발자가 독립적으로 개발!
4. 독립 캐싱: 각 슬롯 별도 캐시 전략 가능!
```

## 2. default.tsx — 필수 방어막! 🟡

```
⚠️ 하드 네비게이션(새로고침, URL 직접 접속) 시:
  프레임워크가 각 슬롯의 page.tsx를 찾으려 함
  → 현재 URL에 맞는 파일이 없으면 → 404!

해결: default.tsx = "매칭 실패 시 보여줄 기본 UI"
```

```tsx
// @analytics/default.tsx
export default function AnalyticsDefault() {
  return <p>통계 데이터를 선택하세요.</p>;
}

// @feed/default.tsx
export default function FeedDefault() {
  return null;  // 아무것도 안 보여줌 (투명!)
}
```

> 💡 **황금 규칙**: 병렬 라우트를 사용하면 **모든 @folder에 default.tsx를 넣어라!**
> 안 넣으면 새로고침 시 404 에러!

## 3. 인터셉팅 라우트 & 모달 (간략 소개) 🟢

```
병렬 라우트(@modal) + 인터셉팅((..)photo) = 인스타그램 스타일 모달!

내부 클릭 (Soft Nav): 피드 위에 모달 팝업! (URL은 /photo/1로 변경!)
새로고침 (Hard Nav): 모달 대신 전체 화면 사진 페이지!

→ URL과 화면 상태의 완벽한 동기화!
→ Part 2 강의 자료 (part2/04.md)에서 상세히 다룸!
```
