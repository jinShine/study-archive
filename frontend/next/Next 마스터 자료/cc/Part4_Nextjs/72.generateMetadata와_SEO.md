## 1. 메타데이터 — 검색엔진과 소셜의 관문 🔴

```
메타데이터 = 페이지의 명함!

구글이 보는 것: <title>, <meta name="description">
카카오톡이 보는 것: <meta property="og:title">, og:description, og:image
트위터가 보는 것: <meta name="twitter:card">, twitter:title

메타데이터가 없으면:
  구글 검색: "localhost" 이라고 표시 😱
  카톡 공유: 밋밋한 텍스트 링크만
  → 클릭률(CTR) 급감! 유입 손실!
```

## 2. 정적 메타데이터 🔴

```tsx
// app/layout.tsx — 전역 기본 메타데이터
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    default: "내 쇼핑몰",               // 기본 제목
    template: "%s | 내 쇼핑몰",          // 하위 페이지: "소개 | 내 쇼핑몰"
  },
  description: "최고의 상품을 최저가로!",
  openGraph: {
    title: "내 쇼핑몰",
    description: "최고의 상품을 최저가로!",
    url: "https://myshop.com",
    siteName: "내 쇼핑몰",
    images: [
      { url: "/og-image.png", width: 1200, height: 630 },
    ],
    locale: "ko_KR",
    type: "website",
  },
  twitter: {
    card: "summary_large_image",
    title: "내 쇼핑몰",
    description: "최고의 상품을 최저가로!",
    images: ["/og-image.png"],
  },
  robots: {
    index: true,
    follow: true,
  },
};

// app/about/page.tsx — 페이지별 메타데이터
export const metadata: Metadata = {
  title: "회사 소개",           // → "회사 소개 | 내 쇼핑몰" (template 적용!)
  description: "우리 회사를 소개합니다",
};
```

## 3. 동적 메타데이터 (generateMetadata) 🔴

> 상품 상세처럼 URL의 [id]에 따라 제목이 달라져야 할 때!

```tsx
// app/products/[id]/page.tsx
import type { Metadata } from "next";

type Props = {
  params: Promise<{ id: string }>;
};

// 1. 동적 메타데이터 함수
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { id } = await params;

  const res = await fetch(`https://api.example.com/products/${id}`);
  const product = await res.json();

  return {
    title: product.name,                    // "초경량 노트북 | 내 쇼핑몰"
    description: product.description.slice(0, 100),
    openGraph: {
      title: `${product.name} - 특가 판매중!`,
      description: "지금 바로 확인하세요!",
      images: [product.imageUrl],
    },
  };
}

// 2. 페이지 컴포넌트 (같은 데이터를 다시 fetch해도 OK!)
export default async function ProductPage({ params }: Props) {
  const { id } = await params;
  const res = await fetch(`https://api.example.com/products/${id}`);
  const product = await res.json();

  return <h1>{product.name}</h1>;
}
```

### "fetch를 2번이나?" — Request Memoization! 🔴

```
generateMetadata에서 fetch 1번
ProductPage에서 fetch 1번
→ 총 2번 호출?

❌ 아니다! 실제 네트워크 요청은 1번만!
✅ Next.js가 동일한 URL의 fetch를 자동으로 중복 제거(Memoization)!

1번째 호출: 실제 서버 통신 → 결과를 메모리에 캐시
2번째 호출: 메모리에서 즉시 반환 (네트워크 0!)

→ Props Drilling 없이 각 함수에서 독립적으로 fetch해도 성능 문제 없음!
```

## 4. SEO 체크리스트 🟡

```
✅ 필수:
  - title: 고유하고 설명적인 제목 (50~60자)
  - description: 핵심 내용 요약 (120~160자)
  - og:title, og:description, og:image (소셜 공유)
  - 시맨틱 HTML (h1, main, article, nav)
  - alt 텍스트 (모든 이미지)
  - 모바일 반응형 (viewport 메타)
  - HTTPS

🟡 권장:
  - sitemap.xml (검색엔진에 페이지 목록 알려주기)
  - robots.txt (크롤링 규칙)
  - canonical URL (중복 페이지 방지)
  - JSON-LD (구조화 데이터)
  - og:image 크기: 1200x630px

📊 확인 도구:
  - Google Search Console (검색 결과 확인)
  - Lighthouse → SEO 점수
  - https://www.opengraph.xyz/ (OG 미리보기)
  - 카카오톡에 링크 보내서 카드 미리보기 확인!
```

## 5. Next.js의 SEO 자동 기능 🟡

```tsx
// sitemap.xml 자동 생성
// app/sitemap.ts
export default async function sitemap() {
  const products = await db.product.findMany();

  return [
    { url: "https://myshop.com", lastModified: new Date() },
    { url: "https://myshop.com/about", lastModified: new Date() },
    ...products.map(p => ({
      url: `https://myshop.com/products/${p.id}`,
      lastModified: p.updatedAt,
    })),
  ];
}

// robots.txt 자동 생성
// app/robots.ts
export default function robots() {
  return {
    rules: { userAgent: "*", allow: "/", disallow: "/admin/" },
    sitemap: "https://myshop.com/sitemap.xml",
  };
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. generateMetadata란?**

> 동적 라우팅 페이지에서 URL 파라미터에 따라 다른 메타데이터를 생성하는 async 함수다. 상품 상세 페이지에서 각 상품의 이름과 설명을 title, description, Open Graph에 반영하여 SEO와 소셜 공유를 최적화한다. 이 함수 내에서 페이지 컴포넌트와 같은 데이터를 fetch해도 Request Memoization에 의해 실제 요청은 1번만 발생한다.

**Q2. Next.js에서 SEO를 최적화하는 방법?**

> 서버 컴포넌트(RSC)로 완성된 HTML을 제공하여 검색엔진이 콘텐츠를 즉시 읽을 수 있게 한다. `metadata` 객체나 `generateMetadata`로 title, description, OG 태그를 설정하고, `sitemap.ts`와 `robots.ts`로 크롤링을 안내한다. `next/image`로 이미지를 최적화하고, 시맨틱 HTML을 사용하며, 모든 이미지에 alt를 부여한다.
