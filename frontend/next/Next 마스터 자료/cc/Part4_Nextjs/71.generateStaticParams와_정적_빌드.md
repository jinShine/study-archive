## 1. generateStaticParams란? 🔴

> 💡 **비유: VIP 행사 초대장**
> 행사 전에 VIP 명단(ID 목록)을 인쇄소에 넘기면,
> 인쇄소가 100장의 초대장을 미리 인쇄해둔다.
> 행사 날 VIP가 오면 즉시 전달! (인쇄 기다릴 필요 없음!)

```
generateStaticParams = "빌드 시 미리 만들어둘 페이지의 ID 목록을 Next.js에게 알려주는 함수"

빌드 전: /products/[id] → 어떤 id가 올지 모름
빌드 후: /products/1, /products/2, ... /products/100
         → 100개의 HTML 파일이 미리 생성됨!
         → 0.01초 만에 로드! CDN에서 바로 서빙!
```

## 2. 기본 구현 🔴

```tsx
// app/products/[id]/page.tsx

// 1. 빌드 시 실행: "어떤 id들을 미리 만들어야 하는지" 반환
export async function generateStaticParams() {
  const products = await fetch("https://api.example.com/products")
    .then(res => res.json());

  // ⚠️ 반드시 문자열(string) 형태로 반환!
  return products.map((product: { id: number }) => ({
    id: String(product.id),  // { id: "1" }, { id: "2" }, ...
  }));
}

// 2. 각 id에 대해 페이지를 생성
export default async function ProductPage({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;

  const res = await fetch(`https://api.example.com/products/${id}`);
  if (!res.ok) notFound();
  const product = await res.json();

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.price}원</p>
    </div>
  );
}
```

## 3. dynamicParams — 명단에 없는 ID 처리 🔴

```tsx
// 기본(true): 명단에 없는 ID도 허용! (서버에서 즉석 생성)
// /products/999 → 빌드 시 안 만들었지만, 접속 시 서버가 생성!
export const dynamicParams = true;  // 기본값

// 엄격(false): 명단에 없는 ID → 무조건 404!
// /products/999 → 빌드 때 안 만들었으면 404!
export const dynamicParams = false;
```

```
dynamicParams: true (기본):
  빌드 시 1~100번 생성
  101번 접속 → 서버에서 즉석 생성 → 성공 후 캐시에 저장!
  102번 접속 → 또 즉석 생성 → 캐시에 저장!
  → 점진적으로 정적 페이지가 늘어남! (Incremental!)

dynamicParams: false:
  빌드 시 1~100번 생성
  101번 접속 → 404!
  → 보안상 허용된 ID만 접근 가능하게 할 때 유용!
```

## 4. 반환 형식 주의사항 🔴

```tsx
// ⚠️ 폴더 이름과 반환 객체의 키가 정확히 일치해야!

// [id] 폴더:
return [{ id: "1" }, { id: "2" }];

// [slug] 폴더:
return [{ slug: "react-basics" }, { slug: "nextjs-routing" }];

// [category]/[itemId] 폴더 (중첩 동적 라우팅):
return [
  { category: "electronics", itemId: "laptop-1" },
  { category: "clothing", itemId: "shirt-2" },
];

// ⚠️ 값은 반드시 문자열(string)!
// ❌ { id: 1 }    → 숫자 → 에러!
// ✅ { id: "1" }  → 문자열 → OK!
```

## 5. ISR과 결합 🟡

```tsx
// generateStaticParams + revalidate = 빌드 시 생성 + 주기적 갱신!

export async function generateStaticParams() {
  const products = await fetchProducts();
  return products.map(p => ({ id: String(p.id) }));
}

export default async function ProductPage({ params }: Props) {
  const { id } = await params;

  // revalidate로 60초마다 갱신!
  const res = await fetch(`/api/products/${id}`, {
    next: { revalidate: 60 },
  });
  // → 빌드 시 생성 + 60초마다 갱신 = 속도 + 신선도!
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. generateStaticParams란?**

> 동적 라우팅(`[id]`) 페이지에서 빌드 시 미리 생성할 파라미터 목록을 반환하는 함수다. 반환된 각 파라미터에 대해 Next.js가 빌드 타임에 HTML을 생성하여 CDN에서 즉시 서빙한다. `dynamicParams: true`(기본)이면 명단에 없는 ID도 서버에서 즉석 생성하고, `false`이면 404를 반환한다. 반환값의 키는 폴더명과 일치해야 하며 값은 반드시 문자열이어야 한다.
