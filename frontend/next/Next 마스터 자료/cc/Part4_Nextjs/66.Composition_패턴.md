# 66. Composition 패턴

> **키워드**: `나뭇잎 패턴` `children 주입` `'use client' 전염성` `Provider 래퍼` `번들 최적화`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 나뭇잎 패턴, children 주입, 전염성 방지 | Next.js 아키텍처의 핵심 |
| 🟡 이해 | Provider 래퍼, 서버 컴포넌트 + 클라이언트 합성 | 실무 설계 |
| 🟢 참고 | 직렬화 가능한 Props만 전달 가능 | 제약 사항 |

---

> ⚠️ 이 내용은 57번(렌더링 패러다임)과 65번(서버/클라이언트)에서 다룬 합성 패턴의 **실전 종합편**이다.

## 1. 핵심 규칙 3가지 🔴

```
규칙 1: 기본은 서버! 인터랙션만 클라이언트로!
  → "페이지 전체를 'use client'로 만들지 마라!"

규칙 2: 클라이언트에서 서버를 직접 import하면 오염!
  → "children으로 주입하라!"

규칙 3: 서버 → 클라이언트로 전달 가능한 Props는 직렬화 가능한 것만!
  → string, number, boolean, 배열, 객체 ✅
  → 함수, 클래스 인스턴스, Date 객체 ❌
```

---

## 2. 실전 패턴 5가지 🔴

### 패턴 1: 데이터 표시(서버) + 인터랙션(클라이언트) 분리

```tsx
// page.tsx (서버 — 데이터!)
export default async function Page() {
  const products = await db.product.findMany();
  return (
    <div>
      <h1>상품 목록</h1>
      {products.map(p => (
        <div key={p.id}>
          <h2>{p.name}</h2>
          <p>{p.price}원</p>
          <AddToCartButton product={{ id: p.id, name: p.name, price: p.price }} />
        </div>
      ))}
    </div>
  );
}

// AddToCartButton.tsx (클라이언트 — 인터랙션!)
"use client";
export default function AddToCartButton({ product }: { product: { id: number; name: string; price: number } }) {
  const addItem = useCartStore(s => s.addItem);
  return <button onClick={() => addItem(product)}>장바구니 담기</button>;
}
```

### 패턴 2: 클라이언트 래퍼 + 서버 children

```tsx
// Accordion.tsx (클라이언트 — 토글 로직)
"use client";
export default function Accordion({ title, children }: { title: string; children: React.ReactNode }) {
  const [isOpen, setIsOpen] = useState(false);
  return (
    <div>
      <button onClick={() => setIsOpen(!isOpen)}>{title} {isOpen ? "▲" : "▼"}</button>
      {isOpen && <div>{children}</div>}
    </div>
  );
}

// page.tsx (서버 — 무거운 콘텐츠를 children으로!)
export default async function Page() {
  const details = await fetchHeavyContent();  // 서버에서 실행!
  return (
    <Accordion title="상세 정보">
      <p>{details.content}</p>  {/* 서버에서 렌더링! 번들에 안 포함! */}
    </Accordion>
  );
}
```

### 패턴 3: Provider 래퍼 (65번 복습)

```tsx
// Providers.tsx ('use client')
"use client";
export default function Providers({ children }: { children: React.ReactNode }) {
  return (
    <QueryClientProvider client={queryClient}>
      <ThemeProvider>
        {children}  {/* 서버 컴포넌트 children은 서버 성질 유지! */}
      </ThemeProvider>
    </QueryClientProvider>
  );
}

// layout.tsx (서버)
export default function Layout({ children }: { children: React.ReactNode }) {
  return <html><body><Providers>{children}</Providers></body></html>;
}
```

### 패턴 4: 서버에서 데이터 조회 → 클라이언트에 Props로 전달

```tsx
// page.tsx (서버)
export default async function Page() {
  const user = await getUser();  // 서버에서 인증 정보 조회

  return (
    <div>

      <CartView initialItems={initialCart} userId={user.id} />
      {/* 서버 데이터를 클라이언트의 초기값으로 전달! */}
    </div>
  );
}

// CartView.tsx ('use client')
"use client";
export default function CartView({ initialItems, userId }: Props) {
  const [items, setItems] = useState(initialItems);  // 서버 데이터로 초기화!
  // 이후 클라이언트에서 인터랙티브하게 관리
}
```

### 패턴 5: 서버 액션을 클라이언트에서 호출

```tsx
// actions.ts ('use server')
"use server";
export async function deleteProduct(id: number) {
  await db.product.delete({ where: { id } });
  revalidatePath("/products");
}

// DeleteButton.tsx ('use client')
"use client";
import { deleteProduct } from "@/actions";

export default function DeleteButton({ productId }: { productId: number }) {
  return (
    <button onClick={() => deleteProduct(productId)}>
      삭제
    </button>
  );
}

// page.tsx (서버)
export default async function Page() {
  const products = await db.product.findMany();
  return products.map(p => (
    <div key={p.id}>
      <span>{p.name}</span>
      <DeleteButton productId={p.id} />
    </div>
  ));
}
```

## 3. Props 직렬화 제약 🟡

> 서버 → 클라이언트로 전달하는 Props는 **직렬화 가능**해야 한다!

```tsx
// ✅ 직렬화 가능: 전달 OK!
<ClientComp
  name="철수"           // string
  count={42}            // number
  isActive={true}       // boolean
  tags={["a", "b"]}     // 배열
  user={{ id: 1 }}      // 객체
/>

// ❌ 직렬화 불가: 전달 불가!
<ClientComp
  onClick={() => {}}    // 함수 ❌
  date={new Date()}     // Date 객체 ❌
  ref={myRef}           // Ref ❌
  instance={new MyClass()} // 클래스 인스턴스 ❌
/>

// 해결: 함수는 Server Action으로, Date는 ISO 문자열로
<ClientComp
  onDelete={deleteServerAction}   // 'use server' 함수는 OK!
  dateStr={date.toISOString()}    // 문자열로 변환!
/>
```

## 4. 실습

```
과제: Composition 패턴으로 쇼핑몰 상품 페이지

1. 서버 컴포넌트:
   - 상품 데이터 DB 조회 (하드코딩으로 시뮬레이션)
   - 상품 이름, 가격, 설명 렌더링

2. 클라이언트 컴포넌트 (나뭇잎!):
   - LikeButton (좋아요 토글)
   - AddToCartButton (Zustand 장바구니에 추가)
   - ImageGallery (이미지 슬라이더)

3. Accordion (클라이언트 래퍼):
   - "상세 설명" 토글
   - children으로 서버에서 렌더링된 무거운 콘텐츠 주입

4. 확인사항:
   - 페이지의 대부분 = 서버 (터미널 로그)
   - 버튼만 = 클라이언트 (브라우저 로그)
   - Network 탭에서 JS 번들 크기 최소화 확인
```

## 면접 대비

### 🔴 필수 질문

**Q1. Next.js에서 서버 컴포넌트와 클라이언트 컴포넌트를 올바르게 조합하는 방법?**

> 나뭇잎 패턴으로 페이지 전체를 서버 컴포넌트로 유지하고, useState/onClick이 필요한 최소 단위만 별도 파일로 분리하여 'use client'를 선언한다. 클라이언트 안에 서버를 넣어야 할 때는 직접 import 대신 children Props로 주입하여(Composition) 서버 컴포넌트의 Zero Bundle Size 이점을 보존한다. Provider는 별도 클라이언트 래퍼로 분리한다.
