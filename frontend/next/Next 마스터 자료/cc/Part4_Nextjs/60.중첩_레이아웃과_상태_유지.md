# 60. 중첩 레이아웃과 상태 유지

> **키워드**: `Nested Layout` `children` `레이아웃 불변성` `Partial Rendering` `metadata`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 중첩 레이아웃, 상태 유지 원리, metadata 설정 | 매일 사용 |
| 🟡 이해 | Partial Rendering, 레이아웃 간 데이터 전달 | 성능/설계 |
| 🟢 참고 | template.tsx vs layout.tsx, Route Groups 고급 | 특수 상황 |

---

## 1. 중첩 레이아웃이 동작하는 원리 🔴

> 💡 **비유: TV 프레임**
> layout.tsx = TV 테두리 (바뀌지 않음!)
> page.tsx = TV 화면 속 영상 (채널 돌리면 바뀜!)
> 채널(페이지)이 바뀌어도 TV 테두리(레이아웃)는 그대로!

```
/dashboard/settings 접속 시:

┌── Root Layout ────────────────────────────────┐
│  [네비게이션 바]                                │
│  ┌── Dashboard Layout ──────────────────────┐ │
│  │ [사이드바]  ┌── Settings Page ──────────┐ │ │
│  │            │                            │ │ │
│  │  📊 홈     │  ⚙️ 환경 설정               │ │ │
│  │  ⚙️ 설정   │  여기만 바뀜! ← page.tsx    │ │ │
│  │  👤 프로필  │                            │ │ │
│  │            └────────────────────────────┘ │ │
│  └──────────────────────────────────────────┘ │
│  [푸터]                                        │
└───────────────────────────────────────────────┘

/dashboard/profile로 이동하면:
  ❌ Root Layout 다시 렌더링 안 함!
  ❌ Dashboard Layout 다시 렌더링 안 함!
  ✅ page.tsx만 Profile Page로 교체! (Partial Rendering!)
  ✅ 사이드바의 검색어, 스크롤 위치 유지!
```

## 2. 실전: 대시보드 레이아웃 구축 🔴

```tsx
// app/dashboard/layout.tsx
import Link from "next/link";

export default function DashboardLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="flex h-screen">
      {/* 사이드바 (페이지 전환해도 유지!) */}
      <aside className="w-64 bg-gray-900 text-white p-6 flex flex-col">
        <h1 className="text-xl font-bold mb-6 text-blue-400">Dashboard</h1>

        {/* ⭐ 핵심 테스트: 이 input에 타이핑 후 메뉴 이동해도 유지! */}
        <input
          type="text"
          placeholder="검색..."
          className="w-full p-2 rounded bg-gray-800 text-sm mb-4"
        />

        <nav className="space-y-2">
          <Link href="/dashboard" className="block p-2 rounded hover:bg-gray-800">
            📊 홈
          </Link>
          <Link href="/dashboard/settings" className="block p-2 rounded hover:bg-gray-800">
            ⚙️ 설정
          </Link>
          <Link href="/dashboard/profile" className="block p-2 rounded hover:bg-gray-800">
            👤 프로필
          </Link>
        </nav>
      </aside>

      {/* 메인 콘텐츠 (여기만 바뀜!) */}
      <main className="flex-1 overflow-y-auto p-8 bg-gray-50">
        {children}
      </main>
    </div>
  );
}
```

### 테스트 방법

```
1. /dashboard 접속
2. 사이드바 검색창에 "테스트" 입력
3. "⚙️ 설정" 클릭 → /dashboard/settings로 이동
4. 검색창에 "테스트"가 그대로 있으면 ✅ 성공!
   (레이아웃이 리렌더링되지 않았다는 증거!)
5. 브라우저 F5 새로고침 → 검색어 사라짐 (당연! State는 메모리)
```

## 3. metadata — SEO와 소셜 공유 🔴

### 정적 메타데이터

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    default: "내 사이트",           // 기본 제목
    template: "%s | 내 사이트",     // 하위 페이지 제목 템플릿
  },
  description: "Next.js로 만든 멋진 사이트",
  openGraph: {
    title: "내 사이트",
    description: "Next.js로 만든 사이트",
    images: ["/og-image.png"],
  },
};

// app/about/page.tsx
export const metadata: Metadata = {
  title: "소개",  // → "소개 | 내 사이트" (template 적용!)
  description: "우리 팀을 소개합니다",
};
```

### 동적 메타데이터 (generateMetadata) 🟡

```tsx
// app/products/[id]/page.tsx
import type { Metadata } from "next";

type Props = { params: Promise<{ id: string }> };

// ⭐ 동적 메타데이터: URL의 id에 따라 제목이 달라짐!
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { id } = await params;
  const product = await fetchProduct(id);

  return {
    title: product.name,
    description: product.description.slice(0, 100),
    openGraph: {
      title: `${product.name} - 특가!`,
      description: "지금 바로 확인하세요!",
      images: [product.imageUrl],
    },
  };
}

export default async function ProductPage({ params }: Props) {
  const { id } = await params;
  const product = await fetchProduct(id);
  return <h1>{product.name}</h1>;
}
```

> 💡 **Request Memoization**: generateMetadata와 페이지 컴포넌트에서
> 같은 fetch를 두 번 호출해도 실제 네트워크 요청은 1번만! (자동 중복 제거)

## 4. Partial Rendering — 바뀌는 것만 교체! 🟡

```
Soft Navigation 시:

Before: /dashboard (Home)
  Root Layout → Dashboard Layout → [Home Page] ← 이것만 교체!

After: /dashboard/settings
  Root Layout → Dashboard Layout → [Settings Page]

서버에서 전송되는 것:
  ❌ 전체 HTML (X)
  ✅ Settings Page의 RSC Payload만! (아주 작은 데이터)

레이아웃의 JS는 다시 다운로드하지 않음!
레이아웃의 State도 유지!
→ 이것이 Next.js가 빠른 핵심 이유!
```

## 5. 실습: 다중 레이아웃 프로젝트

```
과제:
1. Root Layout: 상단 네비게이션 + 푸터
2. (marketing) Route Group: 전체 너비 레이아웃
   - / → 랜딩 페이지 (히어로 + 기능 소개)
   - /pricing → 가격표
3. (app) Route Group: 사이드바 레이아웃
   - /dashboard → 통계 카드 3개
   - /dashboard/settings → 설정 폼
   - /dashboard/profile → 프로필
4. 각 페이지에 metadata 설정
5. 레이아웃 상태 유지 테스트 (검색어 + 스크롤)
6. loading.tsx 추가 (대시보드만)
```

## 면접 대비

### 🔴 필수 질문

**Q1. Next.js에서 레이아웃이 리렌더링되지 않는 이유?**

> Next.js의 Soft Navigation은 페이지 전환 시 변경된 page.tsx의 RSC Payload만 서버에서 받아와 children 자리에 교체한다(Partial Rendering). 상위 layout.tsx는 이미 렌더링되어 있고 교체 대상이 아니므로 리렌더링이 발생하지 않는다. 이 덕분에 사이드바의 검색어, 스크롤 위치, 토글 상태 등 레이아웃의 State가 페이지 전환 후에도 유지된다.

**Q2. metadata를 설정하는 방법?**

> 정적 메타데이터는 `export const metadata: Metadata = { title, description, openGraph }`로 선언한다. 동적 메타데이터(상품 상세 등)는 `generateMetadata` async 함수를 export하여 params에 따라 다른 메타데이터를 반환한다. Root Layout에서 `title.template: "%s | 사이트명"`을 설정하면 하위 페이지 제목이 자동으로 "페이지명 | 사이트명" 형태가 된다.
