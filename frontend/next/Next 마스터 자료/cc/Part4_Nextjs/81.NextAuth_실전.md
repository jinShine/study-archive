# 81. NextAuth.js (Auth.js) 실전

> **키워드**: `NextAuth` `Auth.js` `OAuth` `Google/GitHub Provider` `Session` `JWT` `Protected Routes`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | NextAuth 세팅, OAuth 소셜 로그인, Session 접근, Protected Routes | 실무 인증 표준 |
| 🟡 이해 | JWT vs Session 전략, 커스텀 사용자 필드, RBAC | 권한 관리 |
| 🟢 참고 | Credentials Provider, 커스텀 로그인 UI, 토큰 갱신 | 고급 인증 |

---

## 1. NextAuth.js란? 🔴

> 💡 **비유: 호텔 프론트 데스크**
> 복잡한 인증 로직(OAuth, JWT, Session, 쿠키...)을 NextAuth가 대신 처리!
> 개발자는 "Google 로그인 켜줘!"만 설정하면 끝!

```
NextAuth.js(= Auth.js v5)가 자동으로 하는 것:
  ✅ OAuth 흐름 전체 (리다이렉트, 토큰 교환, 프로필 가져오기)
  ✅ 세션 관리 (JWT 또는 DB 세션)
  ✅ CSRF 방어
  ✅ 쿠키 보안 (HttpOnly, Secure, SameSite)
  ✅ 콜백 URL 검증
  ✅ 토큰 자동 갱신
```

## 2. 설치 및 세팅 🔴

```bash
npm install next-auth@beta  # Auth.js v5 (Next.js App Router 지원!)
```

### 환경 변수

```env
# .env.local
NEXTAUTH_SECRET=your-super-secret-key-here  # openssl rand -base64 32 로 생성!
NEXTAUTH_URL=http://localhost:3000

# Google OAuth
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

# GitHub OAuth
GITHUB_CLIENT_ID=your-github-client-id
GITHUB_CLIENT_SECRET=your-github-client-secret
```

### Auth 설정 파일

```tsx
// auth.ts (프로젝트 루트 또는 src/)
import NextAuth from "next-auth";
import Google from "next-auth/providers/google";
import GitHub from "next-auth/providers/github";

export const { handlers, signIn, signOut, auth } = NextAuth({
  providers: [
    Google({
      clientId: process.env.GOOGLE_CLIENT_ID!,
      clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
    }),
    GitHub({
      clientId: process.env.GITHUB_CLIENT_ID!,
      clientSecret: process.env.GITHUB_CLIENT_SECRET!,
    }),
  ],
  callbacks: {
    // JWT에 커스텀 필드 추가
    async jwt({ token, user }) {
      if (user) {
        token.role = user.role ?? "user";
      }
      return token;
    },
    // Session에 커스텀 필드 추가
    async session({ session, token }) {
      session.user.id = token.sub!;
      session.user.role = token.role as string;
      return session;
    },
  },
  pages: {
    signIn: "/login",  // 커스텀 로그인 페이지
  },
});
```

### API Route 연결

```tsx
// app/api/auth/[...nextauth]/route.ts
import { handlers } from "@/auth";
export const { GET, POST } = handlers;
// → /api/auth/signin, /api/auth/callback 등 자동 생성!
```

## 3. 서버 컴포넌트에서 세션 접근 🔴

```tsx
// 서버 컴포넌트에서 인증 확인!
import { auth } from "@/auth";

export default async function DashboardPage() {
  const session = await auth();  // 서버에서 세션 조회!

  if (!session) {
    redirect("/login");  // 미인증 → 로그인으로!
  }

  return (
    <div>
      <h1>안녕하세요, {session.user.name}님!</h1>
      <img src={session.user.image} alt="프로필" />
      <p>이메일: {session.user.email}</p>
      <p>권한: {session.user.role}</p>
    </div>
  );
}
```

### 클라이언트 컴포넌트에서 세션 접근

```tsx
"use client";
import { useSession } from "next-auth/react";

export default function UserMenu() {
  const { data: session, status } = useSession();

  if (status === "loading") return <p>로딩...</p>;
  if (!session) return <a href="/login">로그인</a>;

  return (
    <div>
      <span>{session.user.name}</span>
      <SignOutButton />
    </div>
  );
}
```

## 4. 로그인/로그아웃 버튼 🔴

```tsx
// 서버 컴포넌트에서 (Server Action!)
import { signIn, signOut } from "@/auth";

export function SignInButton() {
  return (
    <form action={async () => {
      "use server";
      await signIn("google");  // Google 로그인!
    }}>
      <button type="submit">Google로 로그인</button>
    </form>
  );
}

export function SignOutButton() {
  return (
    <form action={async () => {
      "use server";
      await signOut();
    }}>
      <button type="submit">로그아웃</button>
    </form>
  );
}
```

## 5. Middleware + NextAuth = 완벽한 보호! 🔴

```tsx
// src/middleware.ts
import { auth } from "@/auth";

export default auth((req) => {
  const isLoggedIn = !!req.auth;
  const isOnDashboard = req.nextUrl.pathname.startsWith("/dashboard");
  const isOnLogin = req.nextUrl.pathname.startsWith("/login");

  if (isOnDashboard && !isLoggedIn) {
    return Response.redirect(new URL("/login", req.nextUrl));
  }

  if (isOnLogin && isLoggedIn) {
    return Response.redirect(new URL("/dashboard", req.nextUrl));
  }
});

export const config = {
  matcher: ["/dashboard/:path*", "/login"],
};
```

## 7. 실습

```
과제: NextAuth 소셜 로그인 구현

1. NextAuth 설치 + auth.ts 설정 (GitHub Provider)
2. API Route 연결 ([...nextauth]/route.ts)
3. /login 페이지: "GitHub로 로그인" 버튼
4. /dashboard: 서버 컴포넌트에서 auth() → 세션 확인 → 사용자 정보 표시
5. Middleware: /dashboard 미인증 → /login 리다이렉트
6. 로그아웃 버튼 → signOut()
7. 커스텀: session에 role 필드 추가 (callbacks)
```

## 면접 대비

### 🔴 필수 질문

**Q1. NextAuth.js(Auth.js)란?**

> Next.js에서 인증을 쉽게 구현할 수 있는 라이브러리다. OAuth 소셜 로그인(Google, GitHub, Kakao 등)의 복잡한 흐름(리다이렉트, 토큰 교환, 프로필 조회)을 자동으로 처리하고, 세션 관리(JWT 또는 DB 세션), 쿠키 보안, CSRF 방어를 기본 제공한다. 서버 컴포넌트에서는 `auth()`, 클라이언트에서는 `useSession()`으로 세션에 접근한다.
