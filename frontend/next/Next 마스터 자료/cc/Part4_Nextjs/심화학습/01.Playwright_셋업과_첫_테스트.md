## 1. E2E 테스트란? — 테스트 피라미드 🟡

> 💡 **비유: 자동차 검사**
> 유닛 테스트 = 엔진, 타이어, 브레이크를 각각 따로 점검
> 통합 테스트 = 엔진 + 변속기가 잘 맞물리는지 점검
> E2E 테스트 = **실제로 도로에서 출발→주행→도착까지** 전체 흐름 점검!

```
테스트 피라미드:

        /  E2E  \          ← 느리지만 신뢰도 최고! (Playwright)
       / 통합 테스트 \       ← API + DB 연동 확인
      / 유닛 테스트    \     ← 함수 하나하나 (Vitest/Jest)
     ──────────────────

E2E (End-to-End):
  실제 브라우저를 열어서 사용자처럼 클릭, 입력, 이동!
  "회원가입 → 로그인 → 상품 검색 → 장바구니 → 결제" 전체 흐름!

왜 Playwright인가?
  ✅ 크로스 브라우저 (Chromium, Firefox, WebKit 동시!)
  ✅ Auto-waiting (요소가 나타날 때까지 자동 대기!)
  ✅ Codegen (마우스 클릭만으로 테스트 코드 자동 생성!)
  ✅ Trace Viewer (실패 시 단계별 스크린샷 + 네트워크 로그!)
  ✅ Next.js 공식 지원 (webServer 연동!)
```

## 3. playwright.config.ts — 핵심 설정 🔴

> 이 파일이 Playwright의 **두뇌**다! Next.js와 연동하는 핵심!

```ts
// playwright.config.ts
import { defineConfig, devices } from "@playwright/test";

export default defineConfig({
  // 📁 테스트 파일 위치
  testDir: "./tests",

  // ⏱️ 전체 테스트 타임아웃 (기본 30초)
  timeout: 30 * 1000,

  // 🔄 실패 시 재시도 횟수 (CI에서만 2회 재시도)
  retries: process.env.CI ? 2 : 0,

  // 👷 병렬 실행 워커 수 (CI에서는 1개로 안정적으로)
  workers: process.env.CI ? 1 : undefined,

  // 📊 리포트 형식
  reporter: "html",

  // 🌐 모든 테스트에 적용되는 공통 설정
  use: {
    // 기본 URL (page.goto("/about")처럼 상대 경로 사용 가능!)
    baseURL: "http://localhost:3000",

    // 실패 시 자동 스크린샷
    screenshot: "only-on-failure",

    // 실패 시 액션 추적 (Trace Viewer)
    trace: "on-first-retry",
  },

  // 🖥️ 테스트할 브라우저 목록
  projects: [
    {
      name: "chromium",
      use: { ...devices["Desktop Chrome"] },
    },
    {
      name: "firefox",
      use: { ...devices["Desktop Firefox"] },
    },
    {
      name: "webkit",
      use: { ...devices["Desktop Safari"] },
    },
    // 📱 모바일 테스트도 가능!
    {
      name: "mobile-chrome",
      use: { ...devices["Pixel 5"] },
    },
  ],

  // ⭐ Next.js 연동 핵심! 테스트 전에 dev 서버 자동 실행!
  webServer: {
    command: "npm run dev",          // 실행할 명령어
    url: "http://localhost:3000",    // 서버가 뜰 주소
    reuseExistingServer: !process.env.CI, // 로컬: 기존 서버 재사용 / CI: 새로 실행
    timeout: 120 * 1000,             // 서버 시작 대기 시간 (2분)
  },
});
```

### 설정 상세 해설

```
webServer (가장 중요!):
  Playwright가 테스트를 시작하기 전에 자동으로 Next.js 서버를 켜줌!
  로컬에서 이미 npm run dev가 돌고 있으면 → 재사용! (reuseExistingServer)
  CI 환경에서는 → 새로 실행!

baseURL:
  "http://localhost:3000"을 설정하면
  page.goto("/about") ← 이렇게 상대 경로로 쓸 수 있음!
  page.goto("http://localhost:3000/about") ← 이거 안 써도 됨!

projects:
  하나의 테스트를 여러 브라우저에서 동시에 실행!
  → 크로스 브라우저 호환성 자동 검증!

reporter: "html":
  테스트 끝나면 예쁜 HTML 리포트 자동 생성!
  npx playwright show-report 로 열어볼 수 있음!
```

## 4. 첫 테스트 작성 🔴

```ts
// tests/home.spec.ts
import { test, expect } from "@playwright/test";

// test(): 하나의 테스트 케이스를 정의!
// 첫 번째 인자: 테스트 이름 (한글 OK!)
// 두 번째 인자: 실제 테스트 로직 (page 객체가 자동 주입!)
test("홈페이지가 정상적으로 렌더링된다", async ({ page }) => {
  // 1. 홈페이지로 이동 (baseURL 덕분에 상대 경로!)
  await page.goto("/");

  // 2. 페이지 제목 확인
  await expect(page).toHaveTitle(/Next/);
  // toHaveTitle: <title> 태그의 텍스트를 검증!
  // 정규식(/Next/)으로 "Next"가 포함되어 있는지!

  // 3. 특정 텍스트가 화면에 보이는지 확인
  await expect(page.getByRole("heading", { name: /welcome/i })).toBeVisible();
  // getByRole: 접근성 역할(heading = h1~h6)로 요소 찾기!
  // /welcome/i: 대소문자 무시하고 "welcome" 포함 확인!
});

test("About 페이지로 이동할 수 있다", async ({ page }) => {
  // 1. 홈에서 시작
  await page.goto("/");

  // 2. "About" 링크 클릭
  await page.getByRole("link", { name: "About" }).click();

  // 3. URL이 /about으로 변경되었는지 확인
  await expect(page).toHaveURL("/about");

  // 4. About 페이지의 콘텐츠 확인
  await expect(page.getByText("소개 페이지")).toBeVisible();
});
```

### 코드 상세 해설

```
test("이름", async ({ page }) => { ... })
  → page = Playwright가 열어주는 실제 브라우저 탭!
  → 이 안에서 실제 사용자처럼 클릭, 입력, 이동!

page.goto("/")
  → 브라우저 주소창에 URL을 입력하는 것과 동일!
  → baseURL + "/" = "http://localhost:3000/"

expect(page).toHaveTitle(...)
  → 브라우저 탭의 제목을 검증!
  → 정규식, 문자열 모두 사용 가능!

page.getByRole("link", { name: "About" })
  → <a> 태그 중 텍스트가 "About"인 것을 찾기!
  → 접근성(a11y) 기반 선택 → 가장 권장되는 방식!

.click()
  → 해당 요소를 실제로 클릭!
  → Auto-waiting: 요소가 보이고 클릭 가능할 때까지 자동 대기!

expect(page).toHaveURL("/about")
  → 현재 URL이 /about인지 확인!
```

## 5. 테스트 실행 명령어 🔴

```bash
# 모든 테스트 실행 (헤드리스 = 브라우저 안 보임)
npx playwright test

# 브라우저 화면을 보면서 실행! (디버깅용)
npx playwright test --headed

# 특정 파일만 실행
npx playwright test tests/home.spec.ts

# 특정 브라우저만
npx playwright test --project=chromium

# UI 모드 (인터랙티브 디버깅!)
npx playwright test --ui

# 코드젠 (마우스 클릭만으로 테스트 코드 자동 생성!)
npx playwright codegen http://localhost:3000

# HTML 리포트 열기
npx playwright show-report
```

### 가장 많이 쓰는 명령어 3가지

```
개발 중:      npx playwright test --ui
              (UI 모드에서 테스트 선택/실행/디버깅!)

디버깅:       npx playwright codegen http://localhost:3000
              (마우스로 클릭하면 테스트 코드가 자동 생성!)

CI/커밋 전:   npx playwright test
              (전체 테스트 한 방에!)
```

## 6. 테스트 파일 네이밍 규칙

```
tests/
├── home.spec.ts          ← 홈페이지 테스트
├── auth.spec.ts          ← 로그인/회원가입 테스트
├── products.spec.ts      ← 상품 CRUD 테스트
├── navigation.spec.ts    ← 라우팅/네비게이션 테스트
└── cart.spec.ts          ← 장바구니 테스트

규칙:
  파일명: [기능].spec.ts
  .spec.ts 확장자 → Playwright가 자동 인식!
  describe()로 관련 테스트 그룹화 가능!
```

```ts
// tests/products.spec.ts
import { test, expect } from "@playwright/test";

test.describe("상품 페이지", () => {
  test("상품 목록이 표시된다", async ({ page }) => {
    await page.goto("/products");
    await expect(page.getByRole("heading", { name: "상품 목록" })).toBeVisible();
  });

  test("상품 상세로 이동할 수 있다", async ({ page }) => {
    await page.goto("/products");
    await page.getByText("상세보기").first().click();
    await expect(page).toHaveURL(/\/products\/\d+/);
  });
});
```

## 면접 대비

### 🔴 필수 질문

**Q1. E2E 테스트란? 유닛 테스트와의 차이?**

> E2E(End-to-End) 테스트는 실제 브라우저를 자동화하여 사용자의 전체 사용 흐름(회원가입→로그인→구매)을 검증하는 테스트다. 유닛 테스트는 함수 하나를 격리하여 검증하지만, E2E는 프론트엔드→백엔드→DB까지 전체 시스템이 연동되는 것을 확인한다. Playwright는 크로스 브라우저 지원, Auto-waiting, Trace Viewer 등으로 안정적인 E2E 테스트를 지원한다.

**Q2. Playwright의 webServer 설정이 하는 역할?**

> 테스트 실행 전에 Next.js 개발 서버를 자동으로 시작하는 설정이다. 로컬에서는 이미 실행 중인 서버를 재사용(reuseExistingServer)하고, CI 환경에서는 새로 실행한다. 이 설정 덕분에 별도로 서버를 수동으로 켜지 않아도 `npx playwright test` 한 줄로 서버 시작부터 테스트 실행까지 자동화된다.
