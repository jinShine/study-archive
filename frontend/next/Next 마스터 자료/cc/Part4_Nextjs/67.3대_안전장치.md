## 1. 안전장치 1: not-found.tsx — 길 잃은 사용자의 나침반 🔴

### 자동 404 (경로 자체가 없을 때)

```
사용자가 /nothing-here 접속
→ app 폴더에 해당 경로 없음
→ 자동으로 가장 가까운 not-found.tsx 표시!
```

### 수동 404 (데이터가 없을 때)

```tsx
// app/products/[id]/page.tsx
import { notFound } from "next/navigation";

export default async function ProductPage({ params }: Props) {
  const { id } = await params;
  const product = await db.product.findUnique({ where: { id: Number(id) } });

  // URL은 유효하지만 데이터가 없을 때!
  if (!product) {
    notFound();  // → not-found.tsx 화면 + HTTP 404 상태 코드!
  }

  return <h1>{product.name}</h1>;
}
```

### not-found.tsx 구현

```tsx
// app/not-found.tsx (글로벌 404)
import Link from "next/link";

export default function NotFound() {
  return (
    <div className="min-h-screen flex flex-col items-center justify-center">
      <h1 className="text-9xl font-extrabold text-gray-200">404</h1>
      <h2 className="text-2xl font-bold mt-4">페이지를 찾을 수 없습니다</h2>
      <p className="text-gray-500 mt-2">요청하신 페이지가 삭제되었거나 이동되었을 수 있습니다.</p>
      <Link href="/" className="mt-6 px-6 py-3 bg-blue-600 text-white rounded-lg hover:bg-blue-700">
        홈으로 돌아가기
      </Link>
    </div>
  );
}

// app/products/not-found.tsx (상품 전용 404)
// → /products/... 경로에서 발생한 404는 이 파일이 우선!
export default function ProductNotFound() {
  return <p>해당 상품을 찾을 수 없습니다. 상품 목록으로 돌아가세요.</p>;
}
```

> 💡 **계층 구조**: 에러 발생 지점에서 가장 가까운 not-found.tsx가 우선!
> `app/products/not-found.tsx` > `app/not-found.tsx`

## 2. 안전장치 2: loading.tsx — 기다림조차 디자인하기 🔴

### 마법의 원리: Streaming SSR + Suspense

```
loading.tsx가 있으면 Next.js가 내부적으로 이렇게 감싸줌:

<Suspense fallback={<Loading />}>
  <Page />
</Suspense>

1. 서버가 Loading UI를 먼저 브라우저에 보냄 (0.001초!)
2. Page 데이터가 준비되면 Loading을 Page로 교체 (스트리밍!)
3. 사용자는 빈 화면 대신 스켈레톤을 봄 → 체감 속도 ↑↑
```

### 글로벌 로딩 (스피너)

```tsx
// app/loading.tsx — 모든 페이지에 적용
export default function Loading() {
  return (
    <div className="flex items-center justify-center min-h-[400px]">
      <div className="w-12 h-12 border-4 border-blue-600 border-t-transparent rounded-full animate-spin" />
    </div>
  );
}
```

### 세그먼트 전용 스켈레톤 (실무 추천!) ⭐

```tsx
// app/products/loading.tsx — 상품 목록 전용!
export default function ProductsLoading() {
  return (
    <div className="grid grid-cols-3 gap-4">
      {/* 실제 상품 카드와 동일한 구조의 뼈대! */}
      {[...Array(6)].map((_, i) => (
        <div key={i} className="bg-white rounded-xl shadow-sm p-6">
          <div className="h-48 bg-gray-200 rounded-lg animate-pulse mb-4" />
          <div className="h-5 bg-gray-200 rounded w-3/4 animate-pulse mb-2" />
          <div className="h-4 bg-gray-200 rounded w-1/2 animate-pulse" />
        </div>
      ))}
    </div>
  );
}
```

> 💡 **Layout Shift 방지**: 스켈레톤의 크기/구조를 실제 콘텐츠와 동일하게!
> 데이터가 로드되면 스켈레톤 → 실제 콘텐츠로 매끄럽게 교체!

## 3. 안전장치 3: error.tsx — 최후의 방어선 🔴

### 규칙: error.tsx는 반드시 'use client'!

```
에러가 발생하면 → error.tsx가 해당 page.tsx를 대체!
레이아웃(layout.tsx)은 살아있음! → 네비게이션 유지!

구조:
  Layout (살아있음!)
    └── Error Boundary (error.tsx가 담당)
         └── Page (에러 발생 → 교체됨!)
```

```tsx
// app/products/error.tsx — ⚠️ 'use client' 필수!
"use client";
import { useEffect } from "react";

export default function ProductError({
  error,
  reset,
}: {
  error: Error & { digest?: string };
  reset: () => void;
}) {
  useEffect(() => {
    // Sentry 등 에러 로깅 서비스에 보고!
    console.error("에러 발생:", error);
    // Sentry.captureException(error);
  }, [error]);

  return (
    <div className="p-8 bg-red-50 border border-red-200 rounded-xl text-center">
      <div className="text-4xl mb-4">💥</div>
      <h2 className="text-xl font-bold text-red-600 mb-2">문제가 발생했습니다</h2>
      <p className="text-red-500 mb-4">
        일시적인 오류입니다. 잠시 후 다시 시도해주세요.
      </p>

      {/* digest: 서버 에러의 추적 코드 (보안상 실제 메시지는 숨겨짐) */}
      {error.digest && (
        <p className="text-xs text-gray-400 mb-4">Error ID: {error.digest}</p>
      )}

      {/* reset: 에러난 Page를 다시 렌더링 시도! (새로고침 없이!) */}
      <button
        onClick={reset}
        className="px-6 py-2 bg-red-600 text-white rounded-lg hover:bg-red-700"
      >
        다시 시도
      </button>
    </div>
  );
}
```

### Props 설명

```
error: 발생한 에러 객체
  - error.message: 에러 메시지 (개발 모드에서만 상세)
  - error.digest: 서버 에러 추적 코드 (프로덕션에서 실제 메시지 대신)
  → 보안: 프로덕션에서는 민감한 서버 에러 메시지가 브라우저에 노출 안 됨!

reset: 에러 복구 함수
  - 호출하면 해당 Page를 새로고침 없이 다시 렌더링 시도
  - 일시적 에러(네트워크 불안정)면 성공할 수 있음!
```

### global-error.tsx — 지구 멸망급 에러 대응 🟡

```tsx
// app/global-error.tsx — Root Layout 자체가 에러날 때 최후의 보루!
"use client";

export default function GlobalError({
  error,
  reset,
}: {
  error: Error & { digest?: string };
  reset: () => void;
}) {
  return (
    // ⚠️ Layout이 깨졌으므로 html, body를 직접 작성!
    <html lang="ko">
      <body className="flex items-center justify-center min-h-screen bg-red-50">
        <div className="text-center">
          <h1 className="text-3xl font-bold text-red-600">치명적 오류</h1>
          <p className="mt-2">서비스에 심각한 문제가 발생했습니다.</p>
          <button onClick={reset} className="mt-4 px-6 py-2 bg-red-600 text-white rounded">
            다시 시도
          </button>
        </div>
      </body>
    </html>
  );
}
```

## 4. 3대 안전장치 계층 구조 🔴

```
┌── Layout (항상 살아있음!) ─────────────────────┐
│                                                │
│  ┌── Error Boundary (error.tsx) ─────────────┐ │
│  │                                           │ │
│  │  ┌── Suspense (loading.tsx) ─────────────┐│ │
│  │  │                                       ││ │
│  │  │  page.tsx (실제 콘텐츠)                ││ │
│  │  │  → 로딩 중: loading.tsx 표시           ││ │
│  │  │  → 에러 발생: error.tsx 표시           ││ │
│  │  │  → 404: not-found.tsx 표시            ││ │
│  │  │                                       ││ │
│  │  └───────────────────────────────────────┘│ │
│  └───────────────────────────────────────────┘ │
│                                                │
│  네비게이션은 항상 살아있다! (레이아웃 불변!)      │
└────────────────────────────────────────────────┘
```

## 5. 실습

```
과제: 3대 안전장치가 모두 갖춰진 대시보드

1. not-found.tsx: 글로벌 커스텀 404 (홈으로 돌아가기 버튼)
2. loading.tsx: 대시보드 전용 스켈레톤 UI (카드 3개 + 테이블)
3. error.tsx: 에러 카드 + "다시 시도" 버튼 + error.digest 표시
4. 인위적 지연: page.tsx에 await sleep(3000) → loading.tsx 확인!
5. 인위적 에러: if (Math.random() < 0.5) throw new Error("폭발!")
   → error.tsx 확인! → reset 버튼으로 복구 시도!
6. notFound(): /products/999 접속 → 커스텀 404 확인!
```

## 면접 대비

### 🔴 필수 질문

**Q1. Next.js의 3대 Special File의 역할?**

> `not-found.tsx`는 404 에러 시 커스텀 UI를 제공하며, `notFound()` 함수로 프로그래밍 방식으로도 트리거할 수 있다. `loading.tsx`는 페이지 데이터가 준비되는 동안 자동으로 Suspense fallback으로 사용되어 스켈레톤 UI를 보여준다(Streaming SSR). `error.tsx`는 'use client' 필수이며, 하위 컴포넌트의 런타임 에러를 잡아 에러 UI와 복구 버튼(`reset`)을 제공한다. 이 3가지가 레이아웃 아래에 계층적으로 감싸져서 서비스의 안정성을 보장한다.

**Q2. error.tsx에서 Layout은 왜 살아있나요?**

> error.tsx는 Layout 아래에 위치하는 Error Boundary이므로, page.tsx에서 에러가 발생해도 Layout은 영향을 받지 않는다. 따라서 네비게이션 바, 사이드바 등 레이아웃 UI는 유지되어 사용자가 다른 페이지로 이동하거나 복구를 시도할 수 있다. Layout 자체의 에러는 `global-error.tsx`가 잡으며, 이 경우 html/body 태그를 직접 포함해야 한다.
