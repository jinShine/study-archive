# 87. Server Actions + Zod + Prisma 통합

> **키워드**: `Zod 서버 검증` `safeParse` `FormData → Zod → Prisma` `3중 방어선` `에러 메시지 한글화`

| 등급 | 섹션 | 의미 |

| 🔴 필수 | Zod 서버 사이드 검증, Server Action 통합 패턴, 3중 방어선 | 실무 데이터 무결성! |

## 1. 3중 방어선 — 데이터 무결성의 정석! 🔴

```
1층: HTML 내장 검증 (required, minlength, type="email")
  → 브라우저에서 기본 차단 (가장 가벼움, 개발자 도구로 뚫림!)

2층: 클라이언트 Zod (React Hook Form + zodResolver)
  → 실시간 피드백! (blur/change 시 에러 메시지)
  → 하지만 JS를 끄거나 API를 직접 호출하면 우회 가능!

3층: 서버 Zod (Server Action 내부)          ← 최종 방어선! 절대 우회 불가!
  → FormData → Zod safeParse → 통과해야만 DB 저장!
```

## 2. 통합 패턴 — FormData → Zod → Prisma 🔴

```tsx
// lib/validations.ts — Zod 스키마 (클라이언트 + 서버 공유!)
import { z } from "zod";

export const createPostSchema = z.object({
  title: z.string()
    .min(2, "제목은 2자 이상이어야 합니다")
    .max(100, "제목은 100자 이하여야 합니다"),
  content: z.string()
    .min(10, "내용은 10자 이상이어야 합니다")
    .max(5000, "내용은 5000자 이하여야 합니다"),
  published: z.coerce.boolean().default(false),
  // z.coerce → FormData의 문자열을 자동 타입 변환!
});

export type CreatePostInput = z.infer<typeof createPostSchema>;
// { title: string; content: string; published: boolean }
```

```tsx
// actions.ts — Server Action
"use server";
import { db } from "@/lib/db";
import { createPostSchema } from "@/lib/validations";
import { revalidatePath } from "next/cache";
import { redirect } from "next/navigation";

export type ActionState = {
  success: boolean;
  message: string;
  errors?: Record<string, string[]>;  // 필드별 에러!
};

export async function createPost(
  prevState: ActionState,
  formData: FormData
): Promise<ActionState> {

  // 1. FormData → 객체로 변환
  const raw = {
    title: formData.get("title"),
    content: formData.get("content"),
    published: formData.get("published") === "on",
  };

  // 2. Zod로 서버 검증! (3층 방어선!)
  const result = createPostSchema.safeParse(raw);

  if (!result.success) {
    // 3. 검증 실패 → 필드별 에러 메시지 반환!
    return {
      success: false,
      message: "입력값을 확인해주세요",
      errors: result.error.flatten().fieldErrors,
      // { title: ["2자 이상!"], content: ["10자 이상!"] }
    };
  }

  // 4. 검증 통과! 타입 안전한 데이터로 DB 저장!
  try {
    await db.post.create({
      data: {
        title: result.data.title,     // ← z.infer 타입! 안전!
        content: result.data.content,
        published: result.data.published,
        authorId: 1,  // TODO: 인증된 유저
      },
    });
  } catch (error) {
    return { success: false, message: "서버 오류가 발생했습니다" };
  }

  // 5. 캐시 갱신 + 리다이렉트
  revalidatePath("/posts");
  redirect("/posts");
}
```

```tsx
// app/posts/new/page.tsx — 폼 UI
"use client";
import { useActionState } from "react";
import { createPost, type ActionState } from "@/actions";
import SubmitButton from "@/components/SubmitButton";

export default function NewPostPage() {
  const initialState: ActionState = { success: false, message: "" };
  const [state, formAction] = useActionState(createPost, initialState);

  return (
    <form action={formAction} className="space-y-4 max-w-lg">
      <div>
        <label htmlFor="title">제목</label>
        <input id="title" name="title" className="w-full border rounded p-2" />
        {state.errors?.title && (
          <p className="text-red-500 text-sm mt-1">{state.errors.title[0]}</p>
        )}
      </div>

      <div>
        <label htmlFor="content">내용</label>
        <textarea id="content" name="content" rows={5} className="w-full border rounded p-2" />
        {state.errors?.content && (
          <p className="text-red-500 text-sm mt-1">{state.errors.content[0]}</p>
        )}
      </div>

      <label className="flex items-center gap-2">
        <input type="checkbox" name="published" />
        공개
      </label>

      <SubmitButton />

      {state.message && !state.success && (
        <p className="text-red-500 bg-red-50 p-3 rounded">{state.message}</p>
      )}
    </form>
  );
}
```

## 3. 클라이언트 + 서버 동일 스키마 공유! 🔴

```tsx
// 같은 Zod 스키마를 클라이언트(RHF)와 서버(Server Action)에서 공유!

// 클라이언트: React Hook Form + Zod (실시간 검증!)
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { createPostSchema, type CreatePostInput } from "@/lib/validations";

const { register, handleSubmit, formState: { errors } } = useForm<CreatePostInput>({
  resolver: zodResolver(createPostSchema),  // 같은 스키마!
});

// 서버: Server Action에서 같은 스키마로 한 번 더 검증!
const result = createPostSchema.safeParse(raw);  // 같은 스키마!

// → 검증 규칙이 하나의 파일에서 관리됨! (Single Source of Truth!)
// → 스키마 수정하면 클라이언트/서버 모두 자동 반영!
```

## 4. 실습

```
과제: 풀스택 게시판 (Zod + Server Actions + Prisma)

1. lib/validations.ts: createPostSchema + updatePostSchema
2. actions.ts: createPost + updatePost + deletePost
   - safeParse로 서버 검증
   - fieldErrors 반환
   - Prisma로 DB CRUD
   - revalidatePath + redirect
3. 게시글 작성 폼: useActionState + 필드별 에러 표시
4. 게시글 수정 폼: 기존 데이터 defaultValue + 수정 액션
5. 게시글 삭제: bind 패턴으로 ID 전달
6. 동일 스키마를 RHF(클라이언트)와 Server Action(서버)에서 공유!
```

### 🔴 필수 질문

**Q1. 서버에서도 유효성 검사를 해야 하는 이유?**

> 클라이언트 검증은 JavaScript를 끄거나 개발자 도구로 우회할 수 있다. API를 직접 호출하면 클라이언트 검증을 완전히 건너뛸 수 있다. 따라서 서버에서 Zod `safeParse`로 최종 검증하는 것이 3중 방어선의 핵심이다. 클라이언트 검증은 UX를 위한 것이고, 서버 검증은 보안과 데이터 무결성을 위한 것이다.
