## 1. Next.js의 SEO 자동 기능들 🔴

### sitemap.xml — 검색엔진에 페이지 목록 알려주기

```tsx
// app/sitemap.ts — Next.js가 자동으로 /sitemap.xml 생성!
import { db } from "@/lib/db";

export default async function sitemap() {
  const products = await db.product.findMany({
    select: { id: true, updatedAt: true },
  });

  return [
    { url: "https://myshop.com", lastModified: new Date(), changeFrequency: "daily" as const, priority: 1 },
    { url: "https://myshop.com/about", lastModified: new Date(), changeFrequency: "monthly" as const, priority: 0.5 },
    ...products.map(p => ({
      url: `https://myshop.com/products/${p.id}`,
      lastModified: p.updatedAt,
      changeFrequency: "weekly" as const,
      priority: 0.8,
    })),
  ];
}
// → https://myshop.com/sitemap.xml 으로 자동 서빙!
```

### robots.txt — 크롤러 규칙

```tsx
// app/robots.ts
export default function robots() {
  return {
    rules: [
      {
        userAgent: "*",
        allow: "/",
        disallow: ["/admin/", "/api/", "/dashboard/"],
        // 관리자 페이지와 API는 크롤링 제외!
      },
    ],
    sitemap: "https://myshop.com/sitemap.xml",
  };
}
// → https://myshop.com/robots.txt 으로 자동 서빙!
```

## 2. SEO 메타데이터 총정리 🔴

```tsx
// app/layout.tsx — 전역 메타데이터
export const metadata: Metadata = {
  metadataBase: new URL("https://myshop.com"),

  title: {
    default: "내 쇼핑몰 - 최고의 상품",
    template: "%s | 내 쇼핑몰",
  },
  description: "최고의 상품을 최저가로 만나보세요",

  // Open Graph (카카오톡, 슬랙 등)
  openGraph: {
    type: "website",
    locale: "ko_KR",
    url: "https://myshop.com",
    siteName: "내 쇼핑몰",
    title: "내 쇼핑몰",
    description: "최고의 상품을 최저가로!",
    images: [{ url: "/og-image.png", width: 1200, height: 630, alt: "내 쇼핑몰" }],
  },

  // Twitter Card
  twitter: {
    card: "summary_large_image",
    title: "내 쇼핑몰",
    description: "최고의 상품을 최저가로!",
    images: ["/og-image.png"],
  },

  // 크롤링 설정
  robots: { index: true, follow: true },

  // 파비콘
  icons: {
    icon: "/favicon.ico",
    apple: "/apple-touch-icon.png",
  },

  // 언어 대체 (다국어)
  alternates: {
    canonical: "https://myshop.com",
    languages: {
      "ko-KR": "https://myshop.com/ko",
      "en-US": "https://myshop.com/en",
    },
  },
};
```

## 3. JSON-LD — 구조화 데이터 (구글 리치 결과!) 🟡

```tsx
// app/products/[id]/page.tsx
export default async function ProductPage({ params }: Props) {
  const { id } = await params;
  const product = await db.product.findUnique({ where: { id: Number(id) } });

  // JSON-LD: 구글이 상품 정보를 이해하여 가격/평점을 검색 결과에 표시!
  const jsonLd = {
    "@context": "https://schema.org",
    "@type": "Product",
    name: product.name,
    description: product.description,
    image: product.imageUrl,
    offers: {
      "@type": "Offer",
      price: product.price,
      priceCurrency: "KRW",
      availability: product.inStock
        ? "https://schema.org/InStock"
        : "https://schema.org/OutOfStock",
    },
  };

  return (
    <>
      {/* Head에 JSON-LD 삽입 */}
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />

      <h1>{product.name}</h1>
      <p>{product.price}원</p>
    </>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. Next.js에서 SEO를 최적화하는 방법?**

> 첫째, 서버 컴포넌트로 완성된 HTML을 제공하여 크롤러가 콘텐츠를 즉시 읽게 한다. 둘째, `metadata` 객체/`generateMetadata` 함수로 title, description, OG 태그를 설정한다. 셋째, `sitemap.ts`로 검색엔진에 전체 페이지 목록을 알리고, `robots.ts`로 크롤링 규칙을 설정한다. 넷째, `next/image`로 이미지를 최적화하고 width/height를 명시하여 CLS를 방지한다. 다섯째, JSON-LD로 구조화 데이터를 제공하면 구글 리치 결과(가격, 평점 표시)가 가능하다.
