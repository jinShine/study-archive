## 91. 프로젝트 설계

### 프로젝트 아이디어 (실무에 가까운 것!)

```
추천 프로젝트 (택 1):
  1. 쇼핑몰 — 상품 CRUD + 장바구니 + 결제 페이지
  2. 블로그 플랫폼 — 게시글 CRUD + 카테고리 + 검색 + 댓글
  3. 대시보드 — 통계 카드 + 차트 + 데이터 테이블 + 필터
  4. 소셜 피드 — 게시글 + 좋아요 + 팔로우 + 무한 스크롤

⚠️ 너무 단순하면(Todo 앱) 취업에 도움 안 됨!
⚠️ 너무 복잡하면(카카오톡 클론) 완성 못 함!
→ "CRUD + 검색 + 인증 + 배포"가 핵심!
```

### 기술 스택 (2026년 실무 표준!)

```
프레임워크:   Next.js 15+ (App Router)
언어:        TypeScript
스타일링:     Tailwind CSS + shadcn/ui
상태 관리:    Zustand (클라이언트) + TanStack Query (서버)
폼 처리:     React Hook Form + Zod
인증:        NextAuth.js (Auth.js)
DB/ORM:      Prisma + PostgreSQL (또는 SQLite)
배포:        Vercel
테스트:      Vitest + React Testing Library
에러 추적:    Sentry (선택)
```

### 설계 산출물

```
□ 요구사항 정리 (기능 목록 + 우선순위)
□ 와이어프레임 (Figma 또는 손 그림 → 사진!)
□ DB 스키마 (ERD — Prisma schema로!)
□ 폴더 구조 설계
□ 기술 선택 이유 정리 (면접에서 물어봄!)
```

## 92. 풀스택 CRUD 구현

### 핵심 기능 체크리스트

```
□ Prisma 모델 설계 + Migration
□ 게시글/상품 목록 (서버 컴포넌트 + ISR!)
□ 게시글/상품 상세 (동적 라우팅 + generateMetadata!)
□ 생성 (Server Actions + Zod + useActionState!)
□ 수정 (기존 데이터 로드 + 수정 Server Action!)
□ 삭제 (bind 패턴 + revalidatePath!)
□ 검색 (URL searchParams + 서버 사이드 필터링!)
□ 카테고리 필터 (URL 기반!)
□ 페이지네이션 (커서 기반 추천!)
□ 이미지 업로드 (next/image + 외부 스토리지!)
```

### 실무에서 차별화되는 포인트

```
✅ loading.tsx 스켈레톤 UI (모든 페이지!)

✅ not-found.tsx 커스텀 404
✅ Optimistic Update (좋아요 등!)
✅ 빈 상태(Empty State) UI
✅ 반응형 (모바일/태블릿/데스크톱)
✅ 다크 모드
```

```markdown
## 🚀 배포 URL
https://my-project.vercel.app

## 📝 프로젝트 소개
한 줄 설명 + 주요 기능 나열

## 🛠 기술 스택
Next.js 15 / TypeScript / Tailwind CSS / Prisma / NextAuth / Zustand / TanStack Query

## ⭐ 주요 기능
- [ ] 소셜 로그인 (Google OAuth)
- [ ] 게시글 CRUD + Zod 유효성 검사
- [ ] 서버 사이드 검색 + 필터링 + 페이지네이션
- [ ] ISR로 목록 캐싱 + Optimistic Update
- [ ] 반응형 디자인 + 다크 모드
- [ ] Lighthouse 90+ 성능

## 📐 아키텍처
서버 컴포넌트(데이터 조회) + 클라이언트 컴포넌트(인터랙션)
서버 상태: TanStack Query / 클라이언트 상태: Zustand

## 🎯 기술 선택 이유
- Next.js: SSR/ISR로 SEO + 성능 최적화
- Zustand: Redux 대비 보일러플레이트 90% 감소
- TanStack Query: 서버 상태 캐싱/동기화 자동화
- Prisma: 타입 안전한 DB 접근
- shadcn/ui: 접근성 보장 + 커스터마이징 자유

## 🏃 실행 방법
git clone, npm install, .env 설정, prisma migrate, npm run dev
```
