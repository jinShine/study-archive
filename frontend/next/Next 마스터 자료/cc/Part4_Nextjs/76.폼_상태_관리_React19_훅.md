# 76. 폼 상태 관리 (useFormStatus & useActionState)

> **키워드**: `useFormStatus` `useActionState` `pending` `prevState` `formAction` `다중 클릭 방어`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | useFormStatus(pending), useActionState(결과 동기화) | Server Actions UI 필수 |
| 🟡 이해 | prevState 누적, isPending, TypeScript 타이핑 | 복잡한 폼 |
| 🟢 참고 | useFormStatus의 4가지 프로퍼티, 에러/성공 피드백 패턴 | 고급 UX |

---

## 1. useFormStatus — 전송 중 상태 추적 🔴

> 💡 "폼이 전송되는 동안 버튼을 비활성화하고 스피너를 보여줘!"

### 절대 규칙: `<form>` 내부의 **자식 컴포넌트**에서만 사용!

```tsx
// components/SubmitButton.tsx
"use client";
import { useFormStatus } from "react-dom";

export default function SubmitButton() {
  const { pending } = useFormStatus();
  // pending = true: 서버 액션 실행 중!
  // pending = false: 유휴 상태

  return (
    <button
      type="submit"
      disabled={pending}
      className={pending ? "opacity-50 cursor-not-allowed" : "bg-blue-600 text-white"}
    >
      {pending ? (
        <span className="flex items-center gap-2">
          <span className="animate-spin">🔄</span> 저장 중...
        </span>
      ) : (
        "저장"
      )}
    </button>
  );
}

// 사용: form 안에 자식으로!
<form action={serverAction}>
  <input name="title" />
  <SubmitButton />  {/* form의 자식! useFormStatus가 부모 form을 감지! */}
</form>
```

> ⚠️ **form과 같은 레벨(같은 파일)에서 useFormStatus를 호출하면 안 됨!**
> 반드시 **자식 컴포넌트로 분리**해야 부모 form의 상태를 감지 가능!

## 2. useActionState — 서버 응답을 화면에 반영! 🔴

> 💡 "서버에서 '이름이 너무 짧아!'라는 에러를 보냈으면, 화면에 빨간 글씨로 보여줘!"

```tsx
// actions.ts
"use server";

export type FormState = {
  success: boolean;
  message: string;
  errors?: Record<string, string[]>;
};

export async function createProduct(
  prevState: FormState,  // ⚠️ 첫 번째 인자: 이전 상태 (프레임워크가 자동 주입!)
  formData: FormData
): Promise<FormState> {
  const name = formData.get("name") as string;
  const price = Number(formData.get("price"));

  // 유효성 검사
  if (!name || name.length < 2) {
    return { success: false, message: "상품명은 2자 이상!", errors: { name: ["2자 이상 입력하세요"] } };
  }
  if (price <= 0) {
    return { success: false, message: "가격은 양수!", errors: { price: ["0보다 커야 합니다"] } };
  }

  // DB 저장
  await db.product.create({ data: { name, price } });
  return { success: true, message: "상품이 등록되었습니다!" };
}
```

```tsx
// page.tsx
"use client";
import { useActionState } from "react";
import { createProduct, type FormState } from "@/actions";
import SubmitButton from "@/components/SubmitButton";

export default function NewProductPage() {
  const initialState: FormState = { success: false, message: "" };

  // useActionState: 서버 액션 + 초기 상태 → [최신 상태, 새 액션, 펜딩]
  const [state, formAction, isPending] = useActionState(createProduct, initialState);

  return (
    <form action={formAction}>  {/* ⚠️ 원본(createProduct) 대신 formAction 사용! */}
      <div>
        <input name="name" placeholder="상품명" />
        {state.errors?.name && (
          <p className="text-red-500 text-sm">{state.errors.name[0]}</p>
        )}
      </div>

      <div>
        <input name="price" type="number" placeholder="가격" />
        {state.errors?.price && (
          <p className="text-red-500 text-sm">{state.errors.price[0]}</p>
        )}
      </div>

      <SubmitButton />

      {/* 서버 응답 피드백! */}
      {state.message && (
        <div className={state.success ? "text-green-600 bg-green-50 p-3 rounded" : "text-red-600 bg-red-50 p-3 rounded"}>
          {state.message}
        </div>
      )}
    </form>
  );
}
```

### 핵심 포인트

```
useActionState의 3가지 반환값:
  state      → 서버 액션이 반환한 최신 상태 (에러/성공 메시지)
  formAction → 프레임워크가 래핑한 새 액션 (form에 이것을 연결!)
  isPending  → 전송 중 여부 (useFormStatus 대신 사용 가능!)

prevState:
  서버 액션의 첫 번째 인자로 자동 주입!
  이전 제출의 결과를 기억 → 시도 횟수 누적 등에 활용!
```

## 3. 두 훅의 조합 패턴 🟡

```
useFormStatus: 전송 중 UI (버튼 비활성화, 스피너)
  → 자식 컴포넌트에서!

useActionState: 전송 결과 (성공/에러 메시지)
  → 폼이 있는 컴포넌트에서!

조합:
  <form action={formAction}>           ← useActionState
    <input ... />
    <SubmitButton />                   ← useFormStatus (자식!)
    {state.message && <Feedback />}    ← useActionState의 state
  </form>
```

## 4. 실습

```
과제: 회원가입 폼 (완전한 Server Actions 패턴)

1. actions.ts: createUser(prevState, formData)
   - name: 2~20자
   - email: 이메일 형식
   - password: 8자+
   - 검증 실패 → { success: false, errors: { 필드: ["메시지"] } }
   - 성공 → { success: true, message: "가입 완료!" }

2. page.tsx: useActionState + formAction
   - 각 필드 아래 에러 메시지
   - 성공/실패 피드백 표시

3. SubmitButton.tsx: useFormStatus
   - pending 시 스피너 + 비활성화

4. 테스트:
   - 빈 필드 제출 → 에러 메시지!
   - 올바른 데이터 → 성공 메시지!
   - 전송 중 버튼 연타 → 1번만 전송!
```

## 면접 대비

### 🔴 필수 질문

**Q1. useFormStatus와 useActionState의 차이?**

> `useFormStatus`는 부모 `<form>`의 전송 상태(pending)를 자식 컴포넌트에서 감지하여 버튼 비활성화/스피너 표시에 사용한다. 반드시 form 내부의 자식 컴포넌트에서만 호출 가능하다. `useActionState`는 Server Action의 반환값을 컴포넌트 State에 자동 동기화하여 에러/성공 메시지를 화면에 표시한다. formAction(래핑된 액션)을 form에 연결하면 TypeScript 타입 에러 없이 prevState가 자동 주입된다.

```
│                                                │

```
