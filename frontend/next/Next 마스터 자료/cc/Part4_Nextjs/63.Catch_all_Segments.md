# 63. Catch-all Segments

> **키워드**: `[...slug]` `[[...slug]]` `배열 params` `Breadcrumb` `무한 깊이 경로`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | [...slug] vs [[...slug]], 배열로 경로 처리 | 카테고리/위키/파일 탐색기 |
| 🟡 이해 | Breadcrumb 구현, notFound 연동, 깊이별 조건 처리 | 실전 UI |
| 🟢 참고 | 라우트 그룹과 결합, SEO 시 generateStaticParams | 최적화 |

---

## 1. 왜 Catch-all이 필요한가? 🔴

```
쇼핑몰 카테고리:
  /shop/electronics
  /shop/electronics/laptops
  /shop/electronics/laptops/gaming
  /shop/electronics/components/gpu/nvidia/rtx-4090

깊이가 제각각! 폴더를 미리 만들 수 없다!
→ Catch-all Segments로 어떤 깊이든 하나의 파일에서 처리!
```

---

## 2. 기본형 `[...slug]` vs 선택형 `[[...slug]]` 🔴

### `[...slug]` — 최소 1개 이상의 경로 필수

```
app/shop/[...slug]/page.tsx

/shop/a         → slug = ["a"]           ✅
/shop/a/b       → slug = ["a", "b"]      ✅
/shop/a/b/c     → slug = ["a", "b", "c"] ✅
/shop           → 404! ❌ (slug가 비어있으면 안 됨!)
```

### `[[...slug]]` — 경로 없어도 OK (Optional) ⭐ 추천!

```
app/shop/[[...slug]]/page.tsx

/shop           → slug = undefined (또는 빈 배열) ✅ ← 이게 차이!
/shop/a         → slug = ["a"]                    ✅
/shop/a/b       → slug = ["a", "b"]               ✅
/shop/a/b/c     → slug = ["a", "b", "c"]          ✅
```

> 💡 `[[...slug]]` (이중 대괄호)를 쓰면 /shop 메인도 같은 파일에서 처리 가능!

---

## 3. 실전 구현: 카테고리 페이지 🔴

```tsx
// app/shop/[[...slug]]/page.tsx

import { notFound } from "next/navigation";
import Link from "next/link";

type Props = {
  params: Promise<{ slug?: string[] }>;  // Optional Catch-all이므로 ?
};

export default async function ShopPage({ params }: Props) {
  const { slug = [] } = await params;  // slug 없으면 빈 배열!

  const depth = slug.length;
  const currentCategory = depth > 0 ? slug[depth - 1] : "전체";

  return (
    <div className="p-8">
      {/* Breadcrumb 네비게이션 */}
      <nav className="flex items-center gap-2 text-sm text-gray-500 mb-6">
        <Link href="/shop" className="text-blue-600 hover:underline">Shop</Link>
        {slug.map((segment, index) => {
          // 현재 위치까지의 경로를 조립!
          const href = `/shop/${slug.slice(0, index + 1).join("/")}`;
          const isLast = index === slug.length - 1;

          return (
            <span key={index} className="flex items-center gap-2">
              <span>/</span>
              {isLast ? (
                <span className="font-bold text-gray-900">{segment}</span>
              ) : (
                <Link href={href} className="text-blue-600 hover:underline">
                  {segment}
                </Link>
              )}
            </span>
          );
        })}
      </nav>

      {/* 메인 콘텐츠 */}
      {depth === 0 ? (
        <div>
          <h1 className="text-3xl font-bold">전체 카테고리</h1>
          <div className="grid grid-cols-3 gap-4 mt-6">
            <Link href="/shop/electronics" className="p-4 border rounded hover:bg-gray-50">
              전자제품
            </Link>
            <Link href="/shop/clothing" className="p-4 border rounded hover:bg-gray-50">
              의류
            </Link>
            <Link href="/shop/food" className="p-4 border rounded hover:bg-gray-50">
              식품
            </Link>
          </div>
        </div>
      ) : (
        <div>
          <h1 className="text-3xl font-bold capitalize">{currentCategory}</h1>
          <p className="text-gray-500 mt-2">깊이: {depth}단계</p>
          <p className="mt-4">
            현재 경로: <code className="bg-gray-100 px-2 py-1 rounded">
              /shop/{slug.join("/")}
            </code>
          </p>
        </div>
      )}
    </div>
  );
}
```

### 핵심 배열 조작 패턴

```ts
const slug = ["electronics", "laptops", "gaming"];

// 현재 카테고리 (마지막)
slug[slug.length - 1]  // "gaming"
slug.at(-1)            // "gaming" (ES2022)

// 깊이
slug.length  // 3

// 부모 경로
slug.slice(0, -1).join("/")  // "electronics/laptops"

// Breadcrumb용: 각 단계까지의 경로
slug.slice(0, 1).join("/")  // "electronics"
slug.slice(0, 2).join("/")  // "electronics/laptops"
slug.slice(0, 3).join("/")  // "electronics/laptops/gaming"

// 특정 키워드 포함 여부
slug.includes("sale")  // 경로에 "sale"이 포함되었는지
```

## 4. 실습

```
과제: 문서 위키 시스템

1. app/docs/[[...slug]]/page.tsx
2. /docs → "문서 홈" (추천 문서 카드 3개)
3. /docs/react → "React 문서"
4. /docs/react/hooks → "React Hooks 문서"
5. /docs/react/hooks/useState → "useState 문서"
6. 모든 경로에 Breadcrumb (중간 단계 클릭 가능!)
7. 없는 경로(/docs/없는주제) → notFound()
8. 경로 깊이에 따라 다른 레이아웃 (1단계: 목록, 2단계+: 상세)
```

## 면접 대비

### 🔴 필수 질문

**Q1. `[...slug]`와 `[[...slug]]`의 차이?**

> `[...slug]`는 최소 1개 이상의 경로 세그먼트가 필수이며, 루트 경로에서는 404가 발생한다. `[[...slug]]`(이중 대괄호)는 Optional로, 경로가 없어도 매칭되어 루트 경로와 하위 경로를 같은 파일에서 처리할 수 있다. slug는 문자열 배열로 전달되며, Optional일 경우 undefined가 될 수 있어 기본값 처리(`slug = []`)가 필요하다.
