## 1. 동적 라우팅이란? 🔴

> 💡 **비유: 호텔의 빈 명찰**
> `[id]` 폴더 = 어떤 이름이든 받아들이는 빈 명찰
> /products/1 이든, /products/macbook 이든, 하나의 page.tsx가 모두 처리!

```
app/
└── products/
    └── [id]/            ← 대괄호가 핵심!
        └── page.tsx     ← /products/무엇이든 여기로!

/products/1       → id = "1"
/products/123     → id = "123"
/products/macbook → id = "macbook"
```

### 기본 구현 (Next.js 15 — params는 Promise!) 🔴

```tsx
// app/products/[id]/page.tsx

// ⚠️ Next.js 15부터 params는 Promise! 반드시 await!
type Props = {
  params: Promise<{ id: string }>;
};

export default async function ProductDetailPage({ params }: Props) {
  const { id } = await params;  // ← await 필수!

  // id는 항상 string! 숫자가 필요하면 parseInt!
  const numericId = parseInt(id);

  // 데이터 조회 (서버 컴포넌트니까 직접 DB/API 호출!)
  const product = await fetchProduct(numericId);

  // 존재하지 않는 상품 처리
  if (!product) {
    notFound();  // → not-found.tsx 화면 표시 + 404 상태코드
  }

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.price.toLocaleString()}원</p>
      <p>{product.description}</p>
    </div>
  );
}
```

> ⚠️ **면접 필수**: "Next.js 15부터 params가 비동기(Promise)로 바뀌었습니다.
> 반드시 `await params`를 해야 하며, 이전 버전처럼 동기적으로 접근하면 에러가 발생합니다."

## 2. 목록 → 상세 패턴 (가장 흔한 실무 패턴!) 🔴

```tsx
// app/products/page.tsx — 목록
import Link from "next/link";

const products = [
  { id: 1, name: "노트북", price: 1200000 },
  { id: 2, name: "키보드", price: 85000 },
  { id: 3, name: "마우스", price: 45000 },
];

export default function ProductListPage() {
  return (
    <div>
      <h1>상품 목록</h1>
      <div className="grid grid-cols-3 gap-4">
        {products.map(product => (
          <div key={product.id} className="border p-4 rounded-lg">
            <h2>{product.name}</h2>
            <p>{product.price.toLocaleString()}원</p>
            {/* 동적 경로로 이동! */}
            <Link href={`/products/${product.id}`} className="text-blue-600">
              상세보기 →
            </Link>
          </div>
        ))}
      </div>
    </div>
  );
}
```

```tsx
// app/products/[id]/page.tsx — 상세
import { notFound } from "next/navigation";

const products = [
  { id: 1, name: "노트북", price: 1200000, description: "초경량 노트북" },
  { id: 2, name: "키보드", price: 85000, description: "무소음 기계식" },
  { id: 3, name: "마우스", price: 45000, description: "인체공학 버티컬" },
];

export default async function ProductDetailPage({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const product = products.find(p => p.id === parseInt(id));

  if (!product) {
    notFound();  // 존재하지 않는 상품 → 404!
  }

  return (
    <div>
      <h1>{product.name}</h1>
      <p className="text-2xl font-bold">{product.price.toLocaleString()}원</p>
      <p>{product.description}</p>
    </div>
  );
}
```

## 3. 라우팅 우선순위 🔴

> 정적 경로가 동적 경로보다 항상 우선!

```
app/
└── products/
    ├── [id]/
    │   └── page.tsx    → /products/123 (동적)
    └── new/
        └── page.tsx    → /products/new (정적)

/products/new → new/ 폴더가 먼저 매칭! (정적 우선!)
/products/123 → [id]/ 폴더가 매칭 (id = "123")
/products/abc → [id]/ 폴더가 매칭 (id = "abc")

우선순위: 정적(new) > 동적([id]) > Catch-all([...slug])
```

## 4. 중첩 동적 라우팅 🟡

```
app/
└── shop/
    └── [category]/
        └── [itemId]/
            └── page.tsx   → /shop/electronics/iphone-15

params = { category: "electronics", itemId: "iphone-15" }
```

```tsx
type Props = {
  params: Promise<{ category: string; itemId: string }>;
};

export default async function ShopItemPage({ params }: Props) {
  const { category, itemId } = await params;

  return (
    <div>
      <p>카테고리: {category}</p>
      <p>상품: {itemId}</p>
    </div>
  );
}
```

## 5. params 타입 주의사항 🔴

```tsx
// ⚠️ params의 값은 항상 string!
const { id } = await params;
console.log(typeof id);  // "string" (항상!)

// 숫자 비교가 필요하면 변환!
const product = products.find(p => p.id === parseInt(id));
// 또는
const product = products.find(p => String(p.id) === id);

// ⚠️ NaN 주의!
parseInt("abc");  // NaN
Number("abc");    // NaN
// → isNaN() 또는 예외 처리 필요!
```

```
과제: 블로그 시스템의 목록→상세 구현

1. app/posts/page.tsx — 게시글 목록 (5개 하드코딩)
   - 제목 + 작성일 + "자세히 보기" Link
2. app/posts/[slug]/page.tsx — 게시글 상세

   - 없는 slug → notFound()
   - 제목 + 작성일 + 본문 + "목록으로" Link

   - 각 게시글 제목이 브라우저 탭에 표시
```

```tsx
// 데이터
const posts = [
  { slug: "react-basics", title: "React 기초", date: "2026-03-20", body: "React는..." },
  { slug: "nextjs-routing", title: "Next.js 라우팅", date: "2026-03-21", body: "라우팅은..." },
  { slug: "typescript-tips", title: "TS 실무 팁", date: "2026-03-22", body: "TS에서는..." },
];

// app/posts/[slug]/page.tsx
import { notFound } from "next/navigation";
import type { Metadata } from "next";
import Link from "next/link";

type Props = { params: Promise<{ slug: string }> };

export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { slug } = await params;
  const post = posts.find(p => p.slug === slug);
  if (!post) return { title: "Not Found" };
  return { title: post.title, description: post.body.slice(0, 100) };
}

export default async function PostPage({ params }: Props) {
  const { slug } = await params;
  const post = posts.find(p => p.slug === slug);
  if (!post) notFound();

  return (
    <article>
      <h1>{post.title}</h1>
      <time>{post.date}</time>
      <p>{post.body}</p>
      <Link href="/posts">← 목록으로</Link>
    </article>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. Next.js 15에서 동적 라우팅의 params가 바뀐 점?**

> Next.js 15부터 `params`가 비동기 객체(Promise)로 변경되었다. `const { id } = await params;`로 반드시 await해야 값에 접근할 수 있다. 이는 서버 컴포넌트의 스트리밍 최적화를 위한 변경이며, await 없이 접근하면 런타임 에러가 발생한다.

**Q2. 동적 경로에서 존재하지 않는 데이터를 어떻게 처리하나요?**

> `next/navigation`의 `notFound()` 함수를 호출하면 해당 컴포넌트의 렌더링이 즉시 중단되고, 가장 가까운 `not-found.tsx` 파일이 렌더링되며 HTTP 404 상태 코드가 반환된다. 데이터베이스 조회 후 결과가 없을 때 `if (!data) notFound();` 패턴으로 사용한다.
