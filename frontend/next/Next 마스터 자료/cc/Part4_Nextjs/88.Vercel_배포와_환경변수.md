## 1. Vercel — Next.js의 공식 배포 플랫폼 🔴

```bash
# 방법 1: Vercel CLI
npm i -g vercel
vercel  # 프로젝트 폴더에서 실행!

# 방법 2: Git 연동 자동 배포 (가장 추천!)
# 1. GitHub에 코드 push
# 2. vercel.com에서 GitHub 리포지토리 연결
# 3. 끝! push할 때마다 자동 빌드+배포!
```

### 자동 배포 파이프라인

```
main 브랜치 push → Production 자동 배포! (실서비스!)
feature 브랜치 PR → Preview 자동 배포! (테스트용 URL 생성!)

Preview: https://my-app-abc123.vercel.app (PR마다 고유!)
Production: https://my-app.vercel.app (커스텀 도메인 가능!)
```

## 2. 환경 변수 — 서버 vs 클라이언트 구분! 🔴

```env
# .env.local (로컬 개발용, Git에 커밋 안 됨!)

# 서버 전용 변수 (브라우저에 노출 안 됨! ✅)
DATABASE_URL="postgresql://user:pass@localhost:5432/mydb"
JWT_SECRET="super-secret-key-1234"
STRIPE_SECRET_KEY="sk_test_..."
NEXTAUTH_SECRET="auth-secret..."

# 클라이언트 노출 변수 (NEXT_PUBLIC_ 접두사 필수!)
NEXT_PUBLIC_API_URL="https://api.example.com"
NEXT_PUBLIC_GA_ID="G-XXXXXXXXXX"
NEXT_PUBLIC_SITE_URL="https://mysite.com"
```

### 접두사 규칙 🔴

```
❌ NEXT_PUBLIC_ 없음 → 서버에서만 접근 가능!
  process.env.DATABASE_URL  ← 서버 컴포넌트/Server Action에서만!
  process.env.JWT_SECRET    ← 클라이언트에서 접근하면 undefined!

✅ NEXT_PUBLIC_ 있음 → 브라우저에도 노출!
  process.env.NEXT_PUBLIC_API_URL  ← 어디서든 접근 가능!
  ⚠️ 민감한 정보는 절대 NEXT_PUBLIC_에 넣지 마!
```

### Vercel에서 환경 변수 설정

```
Vercel 대시보드 → Settings → Environment Variables

Production: 실서비스용 값 (실제 DB URL, 실제 API 키)
Preview: 테스트용 값 (테스트 DB, 테스트 API 키)
Development: 로컬 개발용 값

각 환경별로 다른 값 설정 가능!
→ 실수로 테스트 DB를 실서비스에서 쓰는 사고 방지!
```

## 3. 배포 전 체크리스트 🔴

```
✅ 빌드 성공: npm run build 에러 없이 완료?
✅ 타입 에러 없음: TypeScript 에러 0개?
✅ 환경 변수 설정: Vercel에 모든 필수 변수 등록?
✅ 데이터베이스 연결: Production DB URL 설정?
✅ 이미지 도메인: next.config.ts에 외부 이미지 도메인 등록?
✅ HTTPS: Vercel은 자동 HTTPS! (커스텀 도메인도!)
✅ 에러 페이지: not-found.tsx, error.tsx 커스터마이징?
✅ SEO: metadata, sitemap.xml, robots.txt 설정?
✅ 성능: Lighthouse 90+ 목표?
```

## 4. 실습

```
과제: Vercel에 배포하기!

1. GitHub 리포지토리 생성 + 코드 push
2. vercel.com에서 리포지토리 연동
3. 환경 변수 설정 (DATABASE_URL, NEXTAUTH_SECRET 등)
4. 자동 배포 확인 (빌드 로그 확인!)
5. Production URL로 접속하여 동작 확인
6. 새 브랜치 → PR → Preview URL 생성 확인!
7. 커스텀 도메인 연결 (선택)
```
