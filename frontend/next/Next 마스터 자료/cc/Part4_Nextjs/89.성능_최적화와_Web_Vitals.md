## 1. Core Web Vitals — 구글이 측정하는 성능 지표 🔴

```
LCP (Largest Contentful Paint) — "화면이 뜨기까지"
  가장 큰 콘텐츠(이미지, 텍스트 블록)가 화면에 나타나는 시간
  ✅ 좋음: ≤ 2.5초
  ❌ 나쁨: > 4초
  개선: 이미지 최적화, SSR, 코드 스플리팅

INP (Interaction to Next Paint) — "반응 속도"
  사용자 클릭/입력 후 화면이 변하기까지 시간
  ✅ 좋음: ≤ 200ms
  ❌ 나쁨: > 500ms
  개선: 무거운 계산 분리, useMemo, useTransition

CLS (Cumulative Layout Shift) — "레이아웃 안정성"
  페이지 로드 중 요소가 갑자기 밀리는 정도
  ✅ 좋음: ≤ 0.1
  ❌ 나쁨: > 0.25
  개선: 이미지 width/height 명시, 폰트 font-display: swap
```

## 2. next/image — 이미지 자동 최적화 🔴

```tsx
import Image from "next/image";

// ✅ next/image: 자동으로 WebP 변환, lazy loading, 크기 최적화!
<Image
  src="/hero.jpg"
  alt="히어로 이미지"
  width={1200}
  height={600}
  priority            // LCP 이미지에만! (즉시 로드)
  className="rounded-xl"
/>

// 외부 이미지
<Image
  src="https://images.unsplash.com/photo-abc"
  alt="외부 이미지"
  width={400}
  height={300}
  // ⚠️ next.config.ts에 도메인 등록 필수!
/>

// fill 모드 (부모 크기에 맞춤)
<div className="relative w-full h-64">
  <Image
    src="/product.jpg"
    alt="상품"
    fill                    // 부모를 꽉 채움!
    className="object-cover" // 비율 유지하며 채우기
    sizes="(max-width: 768px) 100vw, 50vw"  // 반응형 힌트!
  />
</div>
```

### next/image가 자동으로 하는 것

```
✅ WebP/AVIF 자동 변환 (브라우저 지원 시)
✅ Lazy loading (뷰포트에 보일 때 로드)
✅ 반응형 srcset 자동 생성
✅ Blur placeholder (로딩 중 흐릿한 미리보기)
✅ 크기 최적화 (디바이스에 맞는 크기)
✅ width/height로 CLS 방지!
```

## 3. next/font — 폰트 최적화 🟡

```tsx
// app/layout.tsx
import { Noto_Sans_KR } from "next/font/google";

const notoSansKR = Noto_Sans_KR({
  subsets: ["latin"],
  weight: ["400", "700"],
  display: "swap",      // 폰트 로딩 전 시스템 폰트 먼저!
  variable: "--font-noto", // CSS 변수로 사용
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ko" className={notoSansKR.variable}>
      <body className="font-sans">{children}</body>
    </html>
  );
}

// tailwind.config.ts에서 연결
// theme: { fontFamily: { sans: ["var(--font-noto)", "sans-serif"] } }
```

### next/font가 자동으로 하는 것

```
✅ 빌드 시 폰트 파일을 로컬에 번들링 (CDN 요청 0!)
✅ FOIT(글자 안 보이는 현상) 방지 (font-display: swap)
✅ CLS 방지 (폰트 크기 자동 조정)
✅ Google Fonts 프라이버시 보호 (구글 서버에 요청 안 감!)
```

## 5. 번들 분석 🟡

```bash
# 번들 크기 분석 도구
npm install @next/bundle-analyzer

# next.config.ts
import withBundleAnalyzer from "@next/bundle-analyzer";

const config = withBundleAnalyzer({
  enabled: process.env.ANALYZE === "true",
})({
  // ... 기존 설정
});

# 분석 실행
ANALYZE=true npm run build
# → 브라우저에 번들 트리맵 표시! 어떤 라이브러리가 크기를 차지하는지!
```

### 번들 줄이기

```
1. dynamic import (next/dynamic): 큰 컴포넌트를 필요할 때만 로드!
2. 트리 셰이킹: named import 사용! (import { debounce } from "lodash-es")
3. 서버 컴포넌트 활용: 무거운 라이브러리를 서버에서만! (Zero Bundle!)
4. next/image, next/font: 자동 최적화!
5. 사용하지 않는 패키지 제거!
```

## 면접 대비

### 🔴 필수 질문

**Q1. Core Web Vitals 3가지?**

> LCP(Largest Contentful Paint)는 가장 큰 콘텐츠의 렌더링 시간으로 2.5초 이하가 목표. INP(Interaction to Next Paint)는 사용자 상호작용 후 화면 반응 속도로 200ms 이하가 목표. CLS(Cumulative Layout Shift)는 로드 중 레이아웃이 밀리는 정도로 0.1 이하가 목표. next/image로 이미지를 최적화하고, 서버 컴포넌트로 JS 번들을 줄이며, loading.tsx로 레이아웃 이동을 방지하여 개선한다.

**Q2. `next/image`를 사용해야 하는 이유?**

> 자동으로 WebP 변환, lazy loading, 반응형 srcset 생성, blur placeholder를 제공한다. width/height를 명시하여 CLS를 방지하고, 브라우저와 디바이스에 최적화된 크기로 전송하여 LCP를 개선한다. 일반 `<img>` 태그보다 대역폭 사용이 크게 감소한다.
