## 1. Middleware란? 🔴

> 💡 **비유: 건물 입구의 경비원**
> 모든 방문자(요청)가 건물에 들어오기 전에 경비원(미들웨어)을 거친다.
> "사원증 있나요?" → 있으면 통과! 없으면 로비(로그인)로 돌려보냄!

```
사용자 요청 → middleware.ts → 페이지 렌더링

미들웨어에서 할 수 있는 것:
  ✅ 인증 확인 → 미인증 시 /login으로 리다이렉트
  ✅ 요청 헤더 추가/수정
  ✅ 응답 헤더 추가/수정
  ✅ 조건부 리다이렉트/리라이트
  ✅ 쿠키 읽기/쓰기
```

## 2. middleware.ts 위치와 기본 구조 🔴

```
⚠️ 파일 위치: 프로젝트 루트(src/ 사용 시 src/ 안)에 딱 1개!

src/
├── middleware.ts    ← 여기! (app 폴더 밖!)
├── app/
│   ├── layout.tsx
│   └── ...
```

```tsx
// src/middleware.ts
import { NextRequest, NextResponse } from "next/server";

export function middleware(request: NextRequest) {
  // 모든 매칭된 요청에 대해 실행!
  console.log("미들웨어 실행:", request.nextUrl.pathname);

  // 그냥 통과
  return NextResponse.next();
}

// 어떤 경로에 미들웨어를 적용할지 (Matcher)
export const config = {
  matcher: [
    // 정적 파일과 API를 제외한 모든 경로에 적용
    "/((?!api|_next/static|_next/image|favicon.ico).*)",
  ],
};
```

## 3. 인증 가드 — 가장 흔한 사용 패턴! 🔴

```tsx
// src/middleware.ts
import { NextRequest, NextResponse } from "next/server";

// 보호할 경로 목록
const protectedPaths = ["/dashboard", "/settings", "/profile", "/admin"];
// 인증된 사용자가 접근하면 안 되는 경로 (로그인/회원가입)
const authPaths = ["/login", "/register"];

export function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  // 쿠키에서 토큰 확인
  const token = request.cookies.get("token")?.value;
  const isAuthenticated = !!token;  // 토큰 있으면 인증됨!

  // 1. 보호된 경로에 미인증 접근 → 로그인으로!
  if (protectedPaths.some(path => pathname.startsWith(path))) {
    if (!isAuthenticated) {
      const loginUrl = new URL("/login", request.url);
      loginUrl.searchParams.set("callbackUrl", pathname);  // 로그인 후 원래 페이지로!
      return NextResponse.redirect(loginUrl);
    }
  }

  // 2. 이미 로그인한 사용자가 로그인 페이지 접근 → 대시보드로!
  if (authPaths.some(path => pathname.startsWith(path))) {
    if (isAuthenticated) {
      return NextResponse.redirect(new URL("/dashboard", request.url));
    }
  }

  // 3. 관리자 전용 경로
  if (pathname.startsWith("/admin")) {
    // JWT 디코딩하여 role 확인 (간단한 예시)
    try {
      const payload = JSON.parse(atob(token!.split(".")[1]));
      if (payload.role !== "admin") {
        return NextResponse.redirect(new URL("/unauthorized", request.url));
      }
    } catch {
      return NextResponse.redirect(new URL("/login", request.url));
    }
  }

  // 4. 통과!
  return NextResponse.next();
}

export const config = {
  matcher: ["/dashboard/:path*", "/settings/:path*", "/profile/:path*", "/admin/:path*", "/login", "/register"],
};
```

## 4. Matcher — 어떤 경로에 적용할지 🔴

```tsx
export const config = {
  matcher: [
    // 특정 경로만
    "/dashboard/:path*",          // /dashboard 와 하위 모든 경로
    "/admin/:path*",

    // 정규식으로 제외 패턴
    "/((?!api|_next/static|_next/image|favicon.ico).*)",
    // api, 정적 파일, 이미지, 파비콘 제외!

    // 여러 경로 배열
    ["/dashboard/:path*", "/settings/:path*", "/profile"],
  ],
};
```

> 💡 **성능 주의**: 미들웨어는 **매 요청마다** 실행되므로,
> 불필요한 경로에 적용하면 성능이 저하된다!
> Matcher로 필요한 경로만 정확히 지정하라!

## 5. 요청/응답 조작 🟡

```tsx
export function middleware(request: NextRequest) {
  // 요청 헤더 추가
  const requestHeaders = new Headers(request.headers);
  requestHeaders.set("x-pathname", request.nextUrl.pathname);
  requestHeaders.set("x-request-id", crypto.randomUUID());

  // 응답 헤더 추가
  const response = NextResponse.next({
    request: { headers: requestHeaders },
  });
  response.headers.set("x-middleware-executed", "true");

  return response;
}
```

## 6. 실습

```
과제: 인증 미들웨어 구현

1. middleware.ts 생성 (src/ 루트)
2. /dashboard, /settings → 토큰 없으면 /login으로 리다이렉트
3. /login → 토큰 있으면 /dashboard로 리다이렉트
4. callbackUrl: 로그인 후 원래 가려던 페이지로 이동!
5. 쿠키에 가짜 토큰 설정/삭제하여 동작 확인
6. Matcher로 정적 파일 제외
```

## 면접 대비

### 🔴 필수 질문

**Q1. Next.js Middleware란?**

> 페이지 렌더링 전에 모든 요청을 가로채어 처리하는 서버 측 함수다. `src/middleware.ts`에 위치하며, 인증 확인(쿠키의 토큰 검증), 리다이렉트, 헤더 조작 등에 사용한다. Matcher로 적용 경로를 제한하여 성능을 보장한다. 보호된 라우트 구현의 첫 번째 방어선이다.
