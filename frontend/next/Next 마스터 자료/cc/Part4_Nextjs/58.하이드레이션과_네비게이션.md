## 1. 하이드레이션(Hydration)이란? 🔴

> 💡 **비유: 컵라면의 건조 채소**
> 서버가 보내준 HTML = 바짝 말린 건조 채소 (모양은 있지만 맛이 없다)
> JavaScript = 뜨거운 물
> 하이드레이션 = 물을 부어서 생생하게 살리는 것!

```
Step 1: 서버가 완성된 HTML을 보냄 (눈으로 볼 수 있지만 버튼이 안 눌림!)
Step 2: 브라우저가 JS를 다운로드
Step 3: JS가 HTML에 이벤트 리스너를 붙임 (하이드레이션!)
Step 4: 버튼이 작동하기 시작! (인터랙티브!)

서버 HTML:
  <button>좋아요</button>  ← 보이지만 안 눌림!

하이드레이션 후:
  <button onclick="handleLike()">좋아요</button>  ← 이제 눌림! ✅
```

### Next.js의 부분 하이드레이션 (Partial Hydration)

```
서버 컴포넌트: 하이드레이션 대상에서 제외! (JS 0바이트)
  → 텍스트, 이미지 등 정적 콘텐츠 → 하이드레이션 불필요!

클라이언트 컴포넌트: 하이드레이션 대상!
  → 버튼, 입력 필드 등 인터랙티브 요소만 JS가 활성화!

결과: 하이드레이션해야 할 JS가 극도로 적어짐 → 빠른 TTI! ⚡
```

## 2. Hydration Mismatch — 서버와 브라우저의 불일치 🔴

> ⚠️ 서버가 그린 HTML과 브라우저가 그리려는 HTML이 다르면 에러!

### 흔한 원인과 해결법

```tsx
// ❌ 원인 1: 시간 (서버 시간 ≠ 브라우저 시간)
export default function TimeDisplay() {
  return <p>{new Date().toLocaleTimeString()}</p>;
  // 서버: 10:00:00 / 브라우저: 10:00:01 → 불일치!
}

// ✅ 해결: Two-Pass Rendering
"use client";
export default function TimeDisplay() {
  const [time, setTime] = useState<string | null>(null);

  useEffect(() => {
    setTime(new Date().toLocaleTimeString());
  }, []);

  return <p>{time ?? "로딩..."}</p>;
  // 서버: "로딩..." / 브라우저 첫 렌더: "로딩..." → 일치! ✅
  // useEffect 후: 실제 시간 표시
}

// ❌ 원인 2: 랜덤 값 (서버와 브라우저의 랜덤이 다름!)
export default function BadRandom() {
  const randomId = Math.random();  // 서버: 0.12 / 브라우저: 0.99 → 불일치!
  return <div id={randomId.toString()}>내용</div>;
}
// ✅ 해결: useId() 훅 사용 (서버/클라이언트에서 동일한 ID 생성!)
import { useId } from "react";
export default function GoodId() {
  const id = useId();  // 서버/브라우저 동일! ✅
  return <div id={id}>내용</div>;
}

// ❌ 원인 3: 잘못된 HTML 중첩 (브라우저가 몰래 고치다가 불일치!)
export default function BadNesting() {
  return (
    <p>
      {/* p 태그 안에 div 태그는 HTML 문법 위반! */}
      {/* 브라우저가 자동으로 구조를 "수정"하면서 서버 HTML과 달라짐! */}
      <div>이러면 안 돼요!</div>
    </p>
  );
}
// ✅ 해결: 올바른 HTML 구조 사용!
// p 안에는 인라인 요소(span 등)만! 블록 요소(div) 금지!

// ❌ 원인 4: 브라우저 확장 프로그램이 HTML을 수정
// 예: 광고 차단기, 번역기, 다크모드 확장이 DOM을 변경!
// ✅ 해결: html/body 태그에 suppressHydrationWarning 추가!
<html lang="ko" suppressHydrationWarning>
  <body suppressHydrationWarning>
    {children}
  </body>
</html>

// ❌ 원인 5: typeof window !== "undefined" 분기
export default function BadCheck() {
  // 서버: false → "서버입니다"
  // 브라우저: true → "브라우저입니다" → 불일치!
  if (typeof window !== "undefined") {
    return <p>브라우저입니다</p>;
  }
  return <p>서버입니다</p>;
}
// ✅ 해결: useEffect에서 클라이언트 전용 로직 처리!

// ✅ 패턴 2: suppressHydrationWarning (치트키)
// "텍스트가 다를 수 있다는 거 알아. 에러 띄우지 마!"
<p suppressHydrationWarning>
  {new Date().toLocaleTimeString()}
</p>
// ⚠️ 텍스트 불일치에만 사용! 태그 구조 불일치에 쓰면 앱이 깨짐!
```

> 💡 **면접 포인트**: "서버와 클라이언트의 초기 렌더링 결과가 반드시 일치해야 합니다.
> 시간이나 랜덤 값처럼 불일치가 발생하는 값은 useEffect에서 클라이언트 전용으로 처리합니다."

## 3. Hard Navigation vs Soft Navigation 🔴

### Hard Navigation — 새로고침 (전체 페이지 재로드)

```
<a href="/about">소개</a>  ← 브라우저 기본 동작

클릭 → 현재 페이지 파괴 → 서버에 새 HTML 요청 → 전체 다시 로드
  ❌ 화면 깜빡임 (White Flash)
  ❌ 모든 State 소멸 (입력값, 스크롤 위치, 사이드바 상태 등)
  ❌ 공통 UI(헤더, 푸터)까지 다시 렌더링
```

### Soft Navigation — SPA 전환 (부분 업데이트) ✅

```
<Link href="/about">소개</Link>  ← Next.js의 Link 컴포넌트

클릭 → 서버에서 변경된 부분(RSC Payload)만 받아옴 → 필요한 부분만 교체
  ✅ 화면 깜빡임 없음!
  ✅ 레이아웃 State 유지! (사이드바, 스크롤, 입력값)
  ✅ 공통 UI는 그대로, 바뀌는 부분만 교체!
  ✅ Prefetching으로 즉시 전환!
```

```tsx
// ❌ <a> 태그 — Hard Navigation (새로고침!)
<a href="/about">소개</a>

// ✅ <Link> 컴포넌트 — Soft Navigation (부드러운 전환!)
import Link from "next/link";
<Link href="/about">소개</Link>

// 동적 경로
<Link href={`/products/${product.id}`}>상세보기</Link>

// 새 탭에서 열기 (외부 링크는 <a> 사용!)
<a href="https://google.com" target="_blank" rel="noopener noreferrer">구글</a>
```

## 4. Prefetching — 클릭 전에 미리 로드! 🟡

> Next.js의 `<Link>`는 뷰포트에 보이면 자동으로 다음 페이지를 미리 가져온다!

```
사용자가 네비게이션 메뉴를 봄
  → <Link href="/about">이 뷰포트에 들어옴
  → Next.js가 백그라운드에서 /about의 RSC Payload를 미리 다운로드!
  → 사용자가 클릭! → 이미 다운로드 완료 → 0초 전환! ⚡

프로덕션(build → start)에서만 작동!
개발 모드(dev)에서는 hover 시 프리페치.
```

```tsx
// Prefetch 비활성화 (거의 안 씀)
<Link href="/heavy-page" prefetch={false}>무거운 페이지</Link>
```

## 5. 프로그래밍 방식 네비게이션 🟡

```tsx
"use client";
import { useRouter } from "next/navigation";  // ⚠️ next/navigation! (next/router 아님!)

function LoginButton() {
  const router = useRouter();

  const handleLogin = async () => {
    const success = await login(email, password);
    if (success) {
      router.push("/dashboard");     // 이동 (히스토리에 추가)
      router.replace("/dashboard");  // 이동 (히스토리 교체 — 뒤로 가기 안 됨)
      router.refresh();              // 현재 페이지 서버에서 다시 가져오기 (캐시 무효화)
      router.back();                 // 뒤로 가기
      router.forward();              // 앞으로 가기
    }
  };
}
```

### redirect vs router.push

```tsx
// 서버 컴포넌트 / Server Action에서:
import { redirect } from "next/navigation";

async function createPost(formData: FormData) {
  "use server";
  await db.post.create({ data: { title: formData.get("title") } });
  redirect("/posts");  // 서버에서 리다이렉트! (함수 마지막에!)
}

// 클라이언트 컴포넌트에서:
"use client";
const router = useRouter();
router.push("/posts");  // 클라이언트에서 이동!
```

## 면접 대비

### 🔴 필수 질문

**Q1. 하이드레이션(Hydration)이란?**

> 서버에서 생성된 정적 HTML에 JavaScript를 결합하여 인터랙티브한 앱으로 만드는 과정이다. 서버는 완성된 HTML을 먼저 보내 사용자가 즉시 화면을 볼 수 있게 하고, 이후 JS가 다운로드되어 이벤트 핸들러와 상태를 활성화한다. Next.js는 서버 컴포넌트에 대해서는 하이드레이션을 건너뛰어(Partial Hydration) JS 전송량을 최소화한다.

**Q2. Hydration Mismatch란? 어떻게 방지하나요?**

> 서버에서 렌더링한 HTML과 클라이언트에서 첫 렌더링한 결과가 다를 때 발생하는 에러다. 시간(`new Date()`), 랜덤 값(`Math.random()`), 잘못된 HTML 중첩(`<p>` 안의 `<div>`) 등이 원인이다. 해결법으로 Two-Pass Rendering(초기값을 null로 두고 useEffect에서 설정)이나 `suppressHydrationWarning`(텍스트 불일치만)을 사용한다.

**Q3. `<Link>`와 `<a>` 태그의 차이?**

> `<a>` 태그는 Hard Navigation으로 전체 페이지를 새로고침하여 모든 State가 초기화된다. Next.js의 `<Link>` 컴포넌트는 Soft Navigation으로 변경된 부분만 RSC Payload로 받아와 교체하므로 레이아웃 State가 유지되고 깜빡임이 없다. 또한 뷰포트에 보이는 Link는 자동으로 Prefetching하여 클릭 시 즉시 전환된다.

```

│      하이드레이션과 네비게이션 핵심 요약           │

│                                                │
│  하이드레이션 = 정적 HTML + JS → 인터랙티브 앱   │

│                                                │

│    시간/랜덤 → useEffect에서 설정 (Two-Pass)     │
│    HTML 중첩 → <p> 안에 <div> 금지!              │
│                                                │
│  <a> = Hard Nav (새로고침, State 소멸)           │
│  <Link> = Soft Nav (부분 교체, State 유지!) ✅   │
│  Prefetching: 뷰포트에 보이면 자동 로드!          │

│                                                │
└────────────────────────────────────────────────┘
```
