# 64. Route Groups & Private Folders

> **키워드**: `(소괄호)` `_언더바` `다중 루트 레이아웃` `URL 미반영` `Colocation 강화`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | Route Group으로 레이아웃 분리, Private Folder로 코드 격리 | 프로젝트 구조 |
| 🟡 이해 | 다중 루트 레이아웃, suppressHydrationWarning | 대규모 프로젝트 |
| 🟢 참고 | Route Group 중첩, 동적 Route Group | 고급 구조 |

## 1. Route Groups — URL에서 사라지는 투명 폴더 🔴

> 💡 **비유: 서류함의 칸막이**
> 서류함(프로젝트) 안의 칸막이(Route Group)는 정리 용도일 뿐,
> 서류(URL)의 이름에는 영향을 주지 않는다!

```
소괄호 ()로 감싼 폴더 = URL에 반영 안 됨!

app/
├── (marketing)/          ← URL에서 사라짐!
│   ├── layout.tsx        ← 마케팅 전용 레이아웃
│   ├── page.tsx          → / (marketing이 안 나옴!)
│   └── pricing/
│       └── page.tsx      → /pricing
│
├── (app)/                ← URL에서 사라짐!
│   ├── layout.tsx        ← 앱 전용 레이아웃 (사이드바)
│   ├── dashboard/
│   │   └── page.tsx      → /dashboard
│   └── settings/
│       └── page.tsx      → /settings
```

### 왜 필요한가?

```
마케팅 페이지: 풀 너비 + 상단 바 + 화려한 푸터
대시보드:     사이드바 + 좁은 메인 + 심플한 디자인

❌ Route Group 없이:
  하나의 layout.tsx에서 if문으로 분기? → 코드 복잡!

✅ Route Group으로:
  각 그룹이 독자적인 layout.tsx를 가짐!
  URL은 깔끔하게 유지!
```

### 다중 루트 레이아웃 🟡

```tsx
// ⚠️ 기존 app/layout.tsx를 삭제하고, 각 그룹에 루트 레이아웃 생성!

// app/(marketing)/layout.tsx — 마케팅 루트 레이아웃
import "@/app/globals.css";  // ← 각 그룹에서 직접 import!

export default function MarketingLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ko" suppressHydrationWarning>
      <body className="bg-white">
        <header className="sticky top-0 bg-white/80 backdrop-blur border-b">
          마케팅 네비게이션
        </header>
        <main>{children}</main>
        <footer className="bg-gray-900 text-white py-12">화려한 푸터</footer>
      </body>
    </html>
  );
}

// app/(app)/layout.tsx — 앱 루트 레이아웃
import "@/app/globals.css";

export default function AppLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ko" suppressHydrationWarning>
      <body className="bg-gray-50">
        <div className="flex h-screen">
          <aside className="w-64 bg-white border-r">사이드바</aside>
          <main className="flex-1 overflow-y-auto p-8">{children}</main>
        </div>
      </body>
    </html>
  );
}
```

> ⚠️ **주의**: 다중 루트 레이아웃을 쓸 때:
> 1. `app/layout.tsx`를 삭제 (충돌 방지!)
> 2. 각 그룹의 layout.tsx에 `<html>`, `<body>` 포함
> 3. `globals.css`를 각 그룹에서 직접 import
> 4. `suppressHydrationWarning`으로 브라우저 확장 경고 방지

## 2. Private Folders — 라우팅에서 제외 🔴

```
_언더바로 시작하는 폴더 = URL로 접근 불가!

app/
├── _components/          ← /_ components → 404!
│   ├── Header.tsx
│   └── Footer.tsx
├── _lib/                 ← /_lib → 404!
│   └── utils.ts
├── _utils/               ← /_utils → 404!
│   └── date-formatter.ts
└── dashboard/
    ├── page.tsx
    └── _components/      ← 대시보드 전용 부품 (URL 접근 불가)
        └── Chart.tsx
```

### Private Folder vs Colocation

```
Colocation (page.tsx가 아닌 파일):
  app/dashboard/Chart.tsx → 라우트 아님 (page.tsx가 아니니까)
  BUT: 혹시 나중에 page.tsx로 이름 변경하면 라우트가 될 수 있음!

Private Folder (_로 시작):
  app/dashboard/_components/Chart.tsx → 절대 라우트 아님!
  _로 시작하는 폴더는 라우팅 시스템이 완전히 무시!
  → 더 안전한 격리!
```

> 💡 **실무 원칙**: 컴포넌트/유틸은 `_components/`, `_lib/`에 두면 확실히 안전!
> 또는 `src/components/`, `src/lib/`처럼 app 폴더 밖에 두는 것도 좋은 방법.

## 3. 실습

```
과제: Route Group으로 프로젝트 구조 잡기

1. (marketing) 그룹:
   - / → 랜딩 페이지 (히어로 + CTA)
   - /pricing → 가격표
   - 레이아웃: 상단 바 + 푸터 (풀 너비)

2. (app) 그룹:
   - /dashboard → 통계 대시보드
   - /dashboard/settings → 설정
   - 레이아웃: 사이드바(240px) + 메인 (Flexbox)

3. _components/ 폴더에 공통 컴포넌트 (Header, Footer, Sidebar)
4. /_components URL로 접근 시 404 확인!
5. /(marketing) URL로 접근 시 404 확인!
```

## 면접 대비

### 🔴 필수 질문

**Q1. Route Group이란?**

> 소괄호 `()`로 감싼 폴더로, URL 경로에 반영되지 않으면서 레이아웃을 논리적으로 분리할 수 있게 한다. 예를 들어 `(marketing)`과 `(app)` 그룹에 각각 다른 `layout.tsx`를 두면, 마케팅 페이지는 풀 너비 레이아웃, 대시보드는 사이드바 레이아웃으로 완전히 분리된다. URL에는 그룹 이름이 나타나지 않으므로 깔끔한 주소를 유지할 수 있다.

**Q2. Private Folder(`_`)의 역할?**

> 폴더 이름 앞에 `_`를 붙이면 Next.js 라우팅 시스템이 해당 폴더를 완전히 무시한다. `/_components` URL로 접근하면 404가 발생한다. Colocation보다 더 강력한 격리를 제공하여, 내부 컴포넌트나 유틸리티가 실수로 라우트가 되는 것을 원천적으로 방지한다.
