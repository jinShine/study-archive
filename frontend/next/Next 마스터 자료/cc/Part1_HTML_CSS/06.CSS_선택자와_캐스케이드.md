# 6. CSS 선택자와 캐스케이드

> **키워드**: `선택자` `특이성(Specificity)` `캐스케이드` `상속` `!important`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 기본 선택자, 특이성 점수 계산, 캐스케이드 규칙 | 면접 단골 + CSS 디버깅의 근본 |
| 🟡 이해 | 결합자, 의사 클래스/요소, 상속 | 실무에서 자주 사용 |
| 🟢 참고 | :has(), :is(), :where() | 최신 선택자 |

## 1. CSS를 HTML에 연결하는 3가지 방법 🔴

```html
<!-- 1. 외부 스타일시트 (✅ 실무 표준) -->
<link rel="stylesheet" href="style.css">

<!-- 2. 내부 스타일 (간단한 테스트용) -->
<style>
  h1 { color: blue; }
</style>

<!-- 3. 인라인 스타일 (❌ 지양. 유지보수 최악) -->
<h1 style="color: blue;">제목</h1>
```

> 💡 **실무에서는** 1번(외부 스타일시트) 또는 Tailwind CSS의 유틸리티 클래스를 사용한다.
> 인라인 스타일은 React의 `style` prop이나 긴급 오버라이드 외에는 쓰지 않는다.

## 2. 선택자(Selector) 총정리 🔴

### 기본 선택자

```css
/* 전체 선택자 — 모든 요소 */
* { margin: 0; padding: 0; }

/* 태그 선택자 — 특정 HTML 태그 */
p { color: gray; }
h1 { font-size: 32px; }

/* 클래스 선택자 — .으로 시작 (가장 많이 사용!) */
.card { border: 1px solid #ddd; }
.card-title { font-weight: bold; }

/* ID 선택자 — #으로 시작 (특이성 높아서 지양) */
#header { background: navy; }

/* 속성 선택자 — 특정 속성을 가진 요소 */
input[type="email"] { border-color: blue; }
a[target="_blank"] { color: red; }
```

> 💡 **실무 원칙**: 클래스 선택자(`.class`)를 90% 이상 사용한다.
> ID 선택자(`#id`)는 특이성이 너무 높아서 오버라이드가 어렵다.

### 결합자(Combinator) 🟡

```css
/* 자손 선택자 (공백) — 모든 하위 요소 */
.card p { color: gray; }       /* .card 안의 모든 p (손자까지) */

/* 자식 선택자 (>) — 직계 자식만 */
.card > p { color: gray; }     /* .card의 직계 자식 p만 */

/* 인접 형제 (+) — 바로 다음 형제 1개 */
h2 + p { margin-top: 0; }     /* h2 바로 다음에 오는 p 1개만 */

/* 일반 형제 (~) — 이후 모든 형제 */
h2 ~ p { color: gray; }       /* h2 이후의 모든 형제 p */
```

```
<div class="card">
  <p>직계 자식 ✅ (> 선택)</p>         ← .card > p 선택됨
  <div>
    <p>손자 ✅ (공백 선택)</p>           ← .card p 선택됨, .card > p 선택 안 됨
  </div>
</div>
```

### 의사 클래스(Pseudo-class) 🟡

```css
/* 상태 기반 */
a:hover { color: red; }           /* 마우스 올릴 때 */
a:active { color: darkred; }      /* 클릭 중 */
input:focus { border-color: blue; } /* 포커스 받을 때 */
input:disabled { opacity: 0.5; }   /* 비활성화 */
input:checked { accent-color: green; } /* 체크됨 */

/* 구조 기반 */
li:first-child { font-weight: bold; }  /* 첫 번째 자식 */
li:last-child { border: none; }        /* 마지막 자식 */
li:nth-child(2) { color: blue; }       /* 2번째 자식 */
li:nth-child(odd) { background: #f5f5f5; } /* 홀수 번째 (줄무늬 테이블) */
li:nth-child(even) { background: white; }  /* 짝수 번째 */

/* 부정 선택자 */
p:not(.special) { color: gray; }   /* .special이 아닌 모든 p */
```

### 의사 요소(Pseudo-element) 🟡

```css
/* 요소의 특정 부분을 선택 (:: 두 개) */
p::first-line { font-weight: bold; }   /* 첫 줄만 */
p::first-letter { font-size: 2em; }    /* 첫 글자만 (드롭캡 효과) */
p::selection { background: yellow; }    /* 드래그 선택 영역 */

/* 가상 요소 생성 */
.required::after {
  content: " *";       /* 텍스트 뒤에 별표 추가 */
  color: red;
}
.icon::before {
  content: "🔥";       /* 텍스트 앞에 아이콘 추가 */
}
```

## 3. 특이성(Specificity) — CSS 충돌의 심판관 🔴

> 💡 **비유: 카드 게임의 패 등급**
> 두 CSS 규칙이 같은 요소를 선택하면 누가 이기는가?
> 가위바위보처럼 "등급이 높은 쪽"이 이긴다.

### 특이성 점수 계산법

```
특이성은 (A, B, C) 세 자리 숫자로 계산한다:

A = ID 선택자(#) 개수
B = 클래스(.) / 속성([]) / 의사 클래스(:) 개수
C = 태그 / 의사 요소(::) 개수

점수가 높은 쪽이 승리!
```

### 실전 예제

```css
p { }                        /* (0, 0, 1) */
.card { }                    /* (0, 1, 0) */
p.card { }                   /* (0, 1, 1) */
#header { }                  /* (1, 0, 0) */
#header .nav a { }           /* (1, 1, 1) */
#header .nav a:hover { }     /* (1, 2, 1) */
```

```
승부 예시:

.card { color: blue; }           (0, 1, 0)
p.card { color: red; }           (0, 1, 1) ← 🏆 승리! (C가 더 높음)

.card { color: blue; }           (0, 1, 0)
#main { color: red; }            (1, 0, 0) ← 🏆 승리! (A가 있으면 무조건)

결과: <p class="card" id="main"> → 빨간색
```

### 특이성 서열 (낮은 → 높은)

```
태그(p)  <  클래스(.card)  <  ID(#main)  <  인라인(style="")  <  !important

(0,0,1)    (0,1,0)          (1,0,0)        (∞)                (핵폭탄)
```

> ⚠️ **`!important`는 핵폭탄이다**. 한 번 쓰면 이걸 이기려면 또 `!important`를 써야 하고,
> 결국 코드 전체가 `!important` 지옥이 된다. **절대 쓰지 마라** (외부 라이브러리 오버라이드 제외).

## 4. 캐스케이드(Cascade) — CSS의 C 🔴

> CSS(Cascading Style Sheets)의 "Cascading"은 **폭포처럼 위에서 아래로 흐른다**는 뜻이다.

같은 특이성일 때 최종 승자를 결정하는 3단계 규칙:

```
Step 1: 중요도 (Importance)
  !important  >  일반 선언

Step 2: 특이성 (Specificity)
  인라인  >  ID  >  클래스  >  태그

Step 3: 소스 순서 (Source Order)
  나중에 선언된 것이 이긴다 (덮어쓰기)
```

```css
/* 소스 순서 예시 — 특이성이 같으면 나중 것이 승리 */
.title { color: blue; }    /* 먼저 선언 */
.title { color: red; }     /* 나중 선언 ← 🏆 승리! 빨간색 적용 */
```

## 5. 상속(Inheritance) 🟡

일부 CSS 속성은 부모 → 자식으로 자동 전달된다.

```css
body {
  font-family: 'Pretendard', sans-serif;   /* ✅ 상속됨 → 모든 자식에 적용 */
  color: #333;                              /* ✅ 상속됨 */
  border: 1px solid red;                    /* ❌ 상속 안 됨 */
  padding: 20px;                            /* ❌ 상속 안 됨 */
}
```

| 상속 ✅ (텍스트 관련) | 상속 ❌ (박스 관련) |
|---------------------|-------------------|
| `color`, `font-*`, `line-height` | `margin`, `padding`, `border` |
| `text-align`, `letter-spacing` | `width`, `height`, `display` |
| `visibility`, `cursor` | `background`, `position` |

```css
/* 강제로 상속 받기/끊기 */
.child {
  color: inherit;    /* 부모 값을 강제 상속 */
  border: initial;   /* 브라우저 기본값으로 리셋 */
  margin: unset;     /* 상속 가능하면 inherit, 아니면 initial */
}
```

## 6. 실습: 특이성 전투

아래 HTML에 어떤 색상이 적용될지 **종이에 먼저 예측**한 후, 브라우저에서 확인해보자.

```html
<style>
  p { color: black; }
  .text { color: blue; }
  #special { color: green; }
  .container .text { color: purple; }
  .container p.text { color: orange; }
</style>

<div class="container">
  <p class="text" id="special">나는 무슨 색일까?</p>
</div>
```

<details>
<summary>정답 보기</summary>

```
p               → (0, 0, 1) black
.text           → (0, 1, 0) blue
#special        → (1, 0, 0) green     ← 🏆 ID가 있으면 무조건 우위!
.container .text → (0, 2, 0) purple
.container p.text → (0, 2, 1) orange

최종 승자: #special → 초록색(green)

ID 선택자(1, 0, 0)는 클래스를 아무리 많이 조합해도(0, 99, 99) 이길 수 없다.
이것이 ID 선택자를 지양하는 이유다.
```

</details>

## 면접 대비

### 🔴 필수 질문

**Q1. CSS 특이성(Specificity)을 설명하고, 점수를 계산하는 방법은?**

> 같은 요소에 여러 CSS 규칙이 적용될 때 어떤 규칙이 우선하는지 결정하는 가중치 체계다. (A, B, C) 세 자리로 계산하며, A는 ID 선택자 수, B는 클래스/속성/의사 클래스 수, C는 태그/의사 요소 수다. 더 높은 자릿수가 무조건 우위이므로 ID(1,0,0)는 클래스(0,10,0)보다 항상 이긴다.

**Q2. `!important`를 쓰면 안 되는 이유?**

> `!important`는 모든 특이성을 무시하고 최우선으로 적용된다. 이를 오버라이드하려면 또 다른 `!important`가 필요해져서 코드가 통제 불능이 된다. 외부 라이브러리의 스타일을 부득이하게 덮어쓸 때를 제외하면 사용을 절대 지양해야 한다.

**Q3. CSS에서 상속되는 속성과 되지 않는 속성은?**

> 텍스트 관련 속성(color, font-family, line-height, text-align 등)은 부모에서 자식으로 상속된다. 박스 모델 관련 속성(margin, padding, border, width, height, background 등)은 상속되지 않는다.
