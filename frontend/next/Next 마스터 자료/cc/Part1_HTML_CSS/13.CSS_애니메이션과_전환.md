## 1. transition — 상태 변화를 부드럽게 🔴

> 💡 **비유: 조명 디머(Dimmer)**
> 스위치를 탁 누르면 불이 확 켜지는 것(transition 없음) vs
> 디머를 돌리면 서서히 밝아지는 것(transition 있음)

```css
/* ❌ transition 없음: 색상이 확 바뀜 */
.button { background: blue; }
.button:hover { background: red; }

/* ✅ transition 있음: 0.3초에 걸쳐 부드럽게 변함 */
.button {
  background: blue;
  transition: background 0.3s ease;
}
.button:hover { background: red; }
```

### transition 기본 문법

```css
.element {
  /* 개별 속성 */
  transition-property: background, transform;  /* 어떤 속성을 */
  transition-duration: 0.3s;                   /* 얼마 동안 */
  transition-timing-function: ease;            /* 어떤 속도 곡선으로 */
  transition-delay: 0s;                        /* 지연 시간 */

  /* 단축 속성 (실무에서 가장 많이 사용) */
  transition: background 0.3s ease;
  transition: all 0.3s ease;                   /* 모든 속성에 적용 */
  transition: transform 0.2s, opacity 0.3s;   /* 속성별 개별 시간 */
}
```

### 타이밍 함수 (Timing Function)

```
ease        ████▓▓░░  처음 빠르게, 끝에서 느리게 (기본값, 가장 자연스러움)
ease-in     ░░▓▓████  처음 느리게, 점점 빨라짐
ease-out    ████▓▓░░  처음 빠르게, 점점 느려짐
ease-in-out ░▓████▓░  양쪽 끝이 느리고 중간이 빠름
linear      ████████  일정한 속도 (로봇 같은 느낌)

cubic-bezier(0.4, 0, 0.2, 1)  → 커스텀 곡선 (구글 Material Design 표준)
```

> 💡 **실무 팁**: 대부분의 UI 전환에 `ease` 또는 `cubic-bezier(0.4, 0, 0.2, 1)` 사용.
> 시간은 `0.15s ~ 0.3s`가 가장 자연스럽다. 0.5s 이상은 느리게 느껴진다.

## 2. transform — 요소를 변형 🔴

> transform은 레이아웃을 건드리지 않고 시각적으로만 변형한다.
> → **Reflow가 발생하지 않아 성능이 좋다!**

```css
.element {
  /* 이동 */
  transform: translateX(50px);        /* 오른쪽으로 50px */
  transform: translateY(-20px);       /* 위로 20px */
  transform: translate(50px, -20px);  /* X, Y 동시 */

  /* 크기 */
  transform: scale(1.1);             /* 10% 확대 */
  transform: scale(0.9);             /* 10% 축소 */
  transform: scaleX(1.5);            /* 가로만 1.5배 */

  /* 회전 */
  transform: rotate(45deg);          /* 시계방향 45도 */
  transform: rotate(-90deg);         /* 반시계방향 90도 */

  /* 기울이기 */
  transform: skewX(10deg);

  /* 복합 (순서 중요! 왼쪽부터 적용) */
  transform: translate(-50%, -50%) scale(1.1) rotate(5deg);
}
```

### 실무 hover 효과 5가지

```css
/* 1. 카드 살짝 떠오르기 */
.card {
  transition: transform 0.2s ease, box-shadow 0.2s ease;
}
.card:hover {
  transform: translateY(-4px);
  box-shadow: 0 8px 25px rgba(0,0,0,0.15);
}

/* 2. 버튼 눌리는 효과 */
.button:active {
  transform: scale(0.95);
}

/* 3. 아이콘 회전 */
.arrow { transition: transform 0.3s ease; }
.arrow.open { transform: rotate(180deg); }

/* 4. 이미지 확대 (overflow: hidden으로 컨테이너 내에서) */
.image-wrapper { overflow: hidden; border-radius: 8px; }
.image-wrapper img {
  transition: transform 0.3s ease;
}
.image-wrapper:hover img {
  transform: scale(1.05);
}

/* 5. 링크 밑줄 슬라이드 */
.link {
  position: relative;
}
.link::after {
  content: '';
  position: absolute;
  bottom: 0; left: 0;
  width: 0; height: 2px;
  background: currentColor;
  transition: width 0.3s ease;
}
.link:hover::after {
  width: 100%;
}
```

## 3. @keyframes & animation — 반복/자동 애니메이션 🟡

> transition은 "A → B" 2개 상태 전환만 가능.
> animation은 "A → B → C → D..." 여러 단계, 반복, 자동 실행 가능.

```css
/* Step 1: 키프레임 정의 (애니메이션 시나리오) */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pulse {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.05); }
  100% { transform: scale(1); }
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

/* Step 2: 요소에 적용 */
.fade-in {
  animation: fadeIn 0.5s ease forwards;
  /* 이름 / 시간 / 속도곡선 / forwards(끝 상태 유지) */
}

.pulse {
  animation: pulse 2s ease-in-out infinite;
  /* infinite: 무한 반복 */
}

.spinner {
  animation: spin 1s linear infinite;
  /* linear: 일정 속도로 빙글빙글 */
}
```

### animation 속성 상세

```css
.element {
  animation-name: fadeIn;           /* 키프레임 이름 */
  animation-duration: 0.5s;         /* 재생 시간 */
  animation-timing-function: ease;  /* 속도 곡선 */
  animation-delay: 0.2s;            /* 시작 전 대기 */
  animation-iteration-count: 1;     /* 반복 횟수 (infinite = 무한) */
  animation-direction: normal;      /* normal / reverse / alternate */
  animation-fill-mode: forwards;    /* forwards = 끝 상태 유지 */

  /* 단축 */
  animation: fadeIn 0.5s ease 0.2s 1 normal forwards;
}
```

## 4. 성능을 해치지 않는 애니메이션 🔴

### GPU 가속이 되는 속성 vs Reflow를 유발하는 속성

```
✅ GPU 가속 (성능 좋음, 60fps 가능):
  transform    → translate, scale, rotate
  opacity      → 투명도

❌ Reflow 유발 (성능 나쁨, 버벅거림):
  width, height     → 크기 변경
  top, left         → 위치 변경
  margin, padding   → 간격 변경
  font-size         → 글자 크기
```

```css
/* ❌ 나쁜 예: left로 이동 → Reflow 발생 */
.box { position: absolute; left: 0; transition: left 0.3s; }
.box:hover { left: 100px; }

/* ✅ 좋은 예: transform으로 이동 → GPU 가속 */
.box { transition: transform 0.3s; }
.box:hover { transform: translateX(100px); }
```

> 💡 **핵심 규칙**: 애니메이션에는 `transform`과 `opacity`만 사용해라!
> 위치 이동 → `translate`, 크기 변경 → `scale`, 나타남/사라짐 → `opacity`

### will-change — GPU에게 미리 알려주기 🟢

```css
/* 애니메이션이 시작되기 전에 GPU에게 준비시킴 */
.animated-element {
  will-change: transform, opacity;
}
```

> ⚠️ **남용 금지!** 모든 요소에 `will-change`를 달면 오히려 메모리를 과도하게 사용한다.
> 실제로 애니메이션이 자주 일어나는 요소에만 선택적으로 적용.

## 면접 대비

### 🔴 필수 질문

**Q1. Reflow와 Repaint의 차이는? 성능에 영향을 주는 CSS 속성은?**

> Reflow는 요소의 크기나 위치가 변해 레이아웃을 다시 계산하는 과정이다(width, height, margin 변경 시). Repaint는 시각적 변화만 있어 다시 칠하는 과정이다(color, background 변경 시). Reflow가 Repaint보다 비용이 훨씬 크다. 애니메이션에는 `transform`과 `opacity`만 사용해야 GPU 가속이 되어 Reflow를 피하고 60fps를 유지할 수 있다.

**Q2. `transition`과 `animation`의 차이?**

> `transition`은 두 상태(A→B) 사이의 전환을 부드럽게 하며, hover 같은 트리거가 필요하다. `animation`은 `@keyframes`로 여러 단계를 정의할 수 있고, 자동 시작/무한 반복이 가능하며, 트리거 없이도 동작한다.

### 🟡 개념 질문

**Q3. CSS 애니메이션에서 `transform`만 써야 하는 이유?**

> `transform`과 `opacity`는 브라우저의 Compositor 단계에서 처리되어 GPU 가속이 적용된다. 반면 `width`, `left`, `margin` 등은 Layout 단계부터 다시 계산(Reflow)해야 하므로 프레임 드랍이 발생한다. 이동은 `translate`, 크기는 `scale`로 대체해야 60fps를 유지할 수 있다.
