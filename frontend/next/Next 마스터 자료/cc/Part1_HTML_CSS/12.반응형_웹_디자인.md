# 12. 반응형 웹 디자인

> **키워드**: `@media` `모바일 퍼스트` `브레이크포인트` `viewport` `srcset` `container query`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 미디어 쿼리, 모바일 퍼스트, 브레이크포인트 전략 | 모든 실무 프로젝트 필수 |
| 🟡 이해 | 반응형 이미지, 유동적 타이포그래피 | 성능 최적화 |
| 🟢 참고 | Container Query, 논리적 속성 | 최신 CSS |

---

## 1. 반응형 웹이란? 🔴

> 💡 **비유: 물**
> 물은 컵에 담으면 컵 모양, 병에 담으면 병 모양이 된다.
> 반응형 웹도 마찬가지. 스마트폰이든, 태블릿이든, 4K 모니터든
> **화면 크기에 따라 레이아웃이 자연스럽게 변하는** 웹을 만드는 것이다.

```
같은 웹사이트가 기기에 따라 다르게 보임:

📱 모바일 (375px)         📱 태블릿 (768px)         🖥️ 데스크톱 (1280px)
┌─────────────┐           ┌──────────────────┐       ┌────────────────────────┐
│ ☰ 로고      │           │ 로고    메뉴 메뉴 │       │ 로고  메뉴1 메뉴2 메뉴3│
├─────────────┤           ├────────┬─────────┤       ├──────┬────────┬────────┤
│   카드 1    │           │ 카드1  │ 카드2   │       │사이드│ 카드1  │ 카드3  │
├─────────────┤           ├────────┼─────────┤       │ 바   ├────────┼────────┤
│   카드 2    │           │ 카드3  │ 카드4   │       │      │ 카드2  │ 카드4  │
├─────────────┤           └────────┴─────────┘       └──────┴────────┴────────┘
│   카드 3    │
└─────────────┘
  1열 세로 쌓기              2열 그리드                  사이드바 + 3열 그리드
```

## 2. 뷰포트 메타 태그 — 반응형의 시작점 🔴

```html
<!-- 이 한 줄이 없으면 모바일에서 데스크톱 사이즈로 축소되어 보임! -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

| 속성 | 의미 |
|------|------|
| `width=device-width` | 페이지 너비를 기기 화면 너비에 맞춤 |
| `initial-scale=1.0` | 초기 확대/축소 비율 1:1 |

> ⚠️ 이 태그가 없으면 미디어 쿼리가 제대로 작동하지 않는다! 반응형 웹의 **전제 조건**.

## 3. 미디어 쿼리 (@media) 🔴

```css
/* 기본 문법 */
@media (조건) {
  /* 조건이 참일 때 적용할 스타일 */
}

/* 화면 너비가 768px 이상일 때 */
@media (min-width: 768px) {
  .container { max-width: 720px; }
}

/* 화면 너비가 480px 이하일 때 */
@media (max-width: 480px) {
  .sidebar { display: none; }  /* 모바일에서 사이드바 숨김 */
}
```

### 모바일 퍼스트 vs 데스크톱 퍼스트 🔴

```css
/* ❌ 데스크톱 퍼스트 (max-width: 큰 → 작은) */
.card { width: 33.33%; }  /* 기본: 3열 */
@media (max-width: 768px) { .card { width: 50%; } }   /* 태블릿: 2열 */
@media (max-width: 480px) { .card { width: 100%; } }  /* 모바일: 1열 */

/* ✅ 모바일 퍼스트 (min-width: 작은 → 큰) ← 실무 표준! */
.card { width: 100%; }    /* 기본: 모바일 1열 */
@media (min-width: 640px) { .card { width: 50%; } }    /* 태블릿: 2열 */
@media (min-width: 1024px) { .card { width: 33.33%; } } /* 데스크톱: 3열 */
```

> 💡 **왜 모바일 퍼스트인가?**
> 1. 전 세계 웹 트래픽의 60%+ 이상이 모바일
> 2. 작은 화면에서 큰 화면으로 확장하는 것이 반대보다 쉬움
> 3. 기본 CSS가 가벼워서 모바일 성능에 유리
> 4. Tailwind CSS도 모바일 퍼스트 (`sm:`, `md:`, `lg:`는 모두 min-width)

## 4. 브레이크포인트 전략 🔴

### 실무 표준 브레이크포인트 (Tailwind CSS 기준)

```css
/* Tailwind CSS의 기본 브레이크포인트 */
/* sm  */  @media (min-width: 640px)  { }  /* 큰 모바일 / 작은 태블릿 */
/* md  */  @media (min-width: 768px)  { }  /* 태블릿 */
/* lg  */  @media (min-width: 1024px) { }  /* 노트북 */
/* xl  */  @media (min-width: 1280px) { }  /* 데스크톱 */
/* 2xl */  @media (min-width: 1536px) { }  /* 큰 데스크톱 */
```

```
브레이크포인트 시각화:

0        640px     768px     1024px    1280px    1536px
|──모바일──|──sm────|───md───|───lg────|───xl────|──2xl──|
```

> 💡 **실무 팁**: 기기 크기가 아니라 **콘텐츠가 깨지는 지점**에서 브레이크포인트를 잡아라.
> "아이폰 크기에서"가 아니라 "카드가 2개 들어가기엔 좁아지는 시점"에서 전환.

## 5. 반응형 이미지 🟡

```html
<!-- 기본: 컨테이너 너비에 맞춤 -->
<img src="photo.jpg" alt="설명" style="max-width: 100%; height: auto;">

<!-- srcset: 화면 크기에 따라 다른 이미지 로드 (성능!) -->
<img
  src="photo-400.jpg"
  srcset="photo-400.jpg 400w,
          photo-800.jpg 800w,
          photo-1200.jpg 1200w"
  sizes="(max-width: 640px) 100vw,
         (max-width: 1024px) 50vw,
         33vw"
  alt="반응형 이미지"
>
```

```css
/* CSS로 반응형 이미지 */
img {
  max-width: 100%;   /* 부모보다 안 커짐 */
  height: auto;      /* 비율 유지 */
  display: block;    /* 하단 갭 제거 */
}

/* 비율 유지하며 꽉 채우기 */
.cover-image {
  width: 100%;
  height: 300px;
  object-fit: cover;    /* 비율 유지하며 영역 채움 (잘림 발생) */
  object-position: center; /* 잘릴 때 중앙 기준 */
}
```

> 💡 Next.js에서는 `next/image`가 이 모든 것(srcset, lazy loading, WebP 변환)을 자동 처리한다.

## 6. 반응형 레이아웃 실전 패턴 🔴

### 패턴 1: 미디어 쿼리 없는 반응형 (CSS Grid)

```css
/* 이 한 줄이면 자동 반응형 카드 그리드! */
.grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 16px;
}
```

### 패턴 2: 사이드바 레이아웃 (모바일에서 숨김)

```css
.layout {
  display: grid;
  grid-template-columns: 1fr;  /* 모바일: 1열 */
}
.sidebar { display: none; }     /* 모바일: 숨김 */

@media (min-width: 1024px) {
  .layout {
    grid-template-columns: 250px 1fr;  /* 데스크톱: 사이드바 + 메인 */
  }
  .sidebar { display: block; }
}
```

### 패턴 3: 햄버거 메뉴 (모바일 네비게이션)

```css
.nav-menu { display: none; }    /* 모바일: 메뉴 숨김 */
.hamburger { display: block; }  /* 모바일: 햄버거 아이콘 표시 */

@media (min-width: 768px) {
  .nav-menu { display: flex; }  /* 태블릿+: 메뉴 표시 */
  .hamburger { display: none; } /* 태블릿+: 햄버거 숨김 */
}
```

## 7. Container Query — 미래의 반응형 🟢

```css
/* 기존: 뷰포트(화면) 크기 기준 → 컴포넌트 재사용 시 문제 */
@media (min-width: 768px) { ... }

/* 미래: 부모 컨테이너 크기 기준 → 컴포넌트가 어디에 놓이든 적응 */
.card-wrapper {
  container-type: inline-size;  /* 이 요소를 컨테이너로 지정 */
}

@container (min-width: 400px) {
  .card { flex-direction: row; }  /* 부모가 400px 이상이면 가로 배치 */
}
```

> 💡 같은 카드 컴포넌트가 사이드바(좁음)에서는 세로, 메인(넓음)에서는 가로로 자동 전환!
> 아직 최신 기술이지만, 주요 브라우저 모두 지원 (2024~).

## 8. 실습: 반응형 포트폴리오 페이지

```
요구사항:
1. 모바일 퍼스트 접근
2. 헤더: 로고 + 햄버거 (모바일) → 로고 + 메뉴 (데스크톱)
3. 히어로 섹션: 전체 화면 높이 (100vh), 중앙 정렬 텍스트
4. 프로젝트 카드: 1열(모바일) → 2열(태블릿) → 3열(데스크톱)
   - auto-fit + minmax 사용 (미디어 쿼리 최소화)
5. 이미지: max-width: 100% + object-fit: cover
6. 푸터: 하단 고정
7. 브레이크포인트: 640px / 1024px
```

## 면접 대비

### 🔴 필수 질문

**Q1. 반응형 웹을 구현하는 방법은?**

> 미디어 쿼리(`@media`)로 화면 크기에 따라 CSS를 분기하고, 뷰포트 메타 태그를 설정하며, 상대 단위(rem, %, vw)와 유동적 레이아웃(Flexbox, Grid)을 사용한다. 또한 이미지에 `max-width: 100%`를 적용하고, `clamp()`로 유동적 타이포그래피를 만든다.

**Q2. 모바일 퍼스트란?**

> 기본 CSS를 모바일 화면에 맞춰 작성하고, `min-width` 미디어 쿼리로 큰 화면에 스타일을 추가하는 접근법이다. 전 세계 웹 트래픽의 과반이 모바일이고, 작은 화면에서 큰 화면으로 확장하는 것이 반대보다 쉬우며, 기본 CSS가 가벼워 모바일 성능에 유리하다.

### 🟡 개념 질문

**Q3. `srcset` 속성은 왜 필요한가?**

> 모바일에서 4K 이미지를 로드하면 불필요한 데이터 낭비가 발생한다. `srcset`을 사용하면 브라우저가 화면 크기와 해상도에 맞는 적절한 크기의 이미지를 자동으로 선택하여 로드한다.
