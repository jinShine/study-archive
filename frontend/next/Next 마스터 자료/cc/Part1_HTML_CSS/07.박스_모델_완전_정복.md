# 7. 박스 모델 완전 정복

> **키워드**: `content` `padding` `border` `margin` `box-sizing` `마진 겹침`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 박스 모델 4개 영역, box-sizing, 마진 겹침 | 면접 단골 + CSS 레이아웃의 근간 |

## 1. 모든 HTML 요소는 "상자(Box)"다 🔴

> 💡 **비유: 택배 상자**
> - **Content**: 상자 안의 실제 물건 (텍스트, 이미지)
> - **Padding**: 물건과 상자 벽 사이의 뽁뽁이 (내부 여백)
> - **Border**: 상자의 테두리 (눈에 보이는 경계선)
> - **Margin**: 이 상자와 옆 상자 사이의 거리 (외부 여백)

```
┌─────────────────────────── Margin (외부 여백) ──┐
│                                                 │
│  ┌──────────────── Border (테두리) ──────────┐  │
│  │                                           │  │
│  │  ┌──────── Padding (내부 여백) ────────┐  │  │
│  │  │                                     │  │  │
│  │  │      ┌── Content (실제 내용) ──┐    │  │  │
│  │  │      │   Hello World!          │    │  │  │
│  │  │      └─────────────────────────┘    │  │  │
│  │  │                                     │  │  │
│  │  └─────────────────────────────────────┘  │  │
│  │                                           │  │
│  └───────────────────────────────────────────┘  │
│                                                 │
└─────────────────────────────────────────────────┘
```

```css
.box {
  width: 200px;          /* 콘텐츠 너비 */
  height: 100px;         /* 콘텐츠 높이 */
  padding: 20px;         /* 내부 여백 (상하좌우 동일) */
  border: 2px solid #333; /* 테두리 */
  margin: 30px;          /* 외부 여백 */
}
```

## 2. box-sizing: 가장 중요한 CSS 한 줄 🔴

> 💡 **비유: 피자 주문**
> "12인치 피자"를 시켰는데, 상자 포함 12인치인가? 피자만 12인치인가?

### content-box (기본값 — 함정!)

```css
.box {
  box-sizing: content-box;  /* 기본값 */
  width: 200px;
  padding: 20px;
  border: 2px solid black;
}
/* 실제 차지하는 너비 = 200 + 20*2 + 2*2 = 244px 😱 */
```

```
❌ content-box: width가 content만 의미
┌─margin─┐
│ ┌─border(2)─┐
│ │ ┌─padding(20)─┐
│ │ │ ┌content(200)┐ │ │ │
│ │ │ └────────────┘ │ │ │
│ │ └────────────────┘ │ │
│ └────────────────────┘ │
└────────────────────────┘
  실제 너비: 200+20+20+2+2 = 244px 😱
```

### border-box (✅ 실무 표준)

```css
.box {
  box-sizing: border-box;  /* 실무 필수! */
  width: 200px;
  padding: 20px;
  border: 2px solid black;
}
/* 실제 차지하는 너비 = 정확히 200px ✅ */
/* content가 자동으로 200 - 20*2 - 2*2 = 156px 으로 줄어듦 */
```

```
✅ border-box: width가 border까지 포함
┌─margin─┐
│ ┌──────── 200px ────────┐ │
│ │border│padding│content│padding│border│ │
│ │  2   │  20   │ 156   │  20   │  2  │ │
│ └──────────────────────────────────────┘ │
└──────────────────────────────────────────┘
  실제 너비: 정확히 200px ✅
```

> ⚠️ **전역 리셋 필수**: 모든 프로젝트에서 이 한 줄을 제일 먼저 적어라!

```css
*, *::before, *::after {
  box-sizing: border-box;
}
```

## 3. margin과 padding 단축 속성 🔴

```css
/* 4방향 개별 지정 */
margin-top: 10px;
margin-right: 20px;
margin-bottom: 10px;
margin-left: 20px;

/* 단축 속성 (시계 방향: 상 → 우 → 하 → 좌) */
margin: 10px 20px 10px 20px;  /* 상 우 하 좌 */
margin: 10px 20px 10px;       /* 상 좌우 하 */
margin: 10px 20px;            /* 상하 좌우 */
margin: 10px;                 /* 상하좌우 동일 */

/* 가로 중앙 정렬 (블록 요소) */
margin: 0 auto;  /* 상하 0, 좌우 auto → 가로 중앙 */
```

> 💡 **암기법**: 시계 방향! **12시(top) → 3시(right) → 6시(bottom) → 9시(left)**

## 4. 마진 겹침(Margin Collapsing) 🔴

> 면접에서 "마진 겹침이 뭔지 아세요?" 라고 물어본다. 모르면 CSS 기초가 부실하다고 판단.

```
두 블록 요소의 수직(상하) 마진이 만나면, 합쳐지지 않고 큰 쪽만 적용된다!
```

```css
.box-a { margin-bottom: 30px; }
.box-b { margin-top: 20px; }

/* 예상: 30 + 20 = 50px 간격 */
/* 실제: max(30, 20) = 30px 간격 😱 (겹침 발생!) */
```

```
❌ 예상                    ✅ 실제 (마진 겹침)
┌───────┐                 ┌───────┐
│ Box A │                 │ Box A │
└───────┘                 └───────┘
  30px                      30px ← 큰 쪽만 적용!
  20px
┌───────┐                 ┌───────┐
│ Box B │                 │ Box B │
└───────┘                 └───────┘
```

### 마진 겹침이 발생하는 3가지 경우

```
1. 인접 형제 요소의 상하 마진
2. 부모와 첫 번째/마지막 자식의 상하 마진
3. 빈 블록 요소의 상하 마진 (자기 자신끼리 겹침)
```

### 마진 겹침 방지법

```css
/* 방법 1: padding이나 border로 경계를 만듦 */
.parent { padding-top: 1px; }  /* 또는 border-top: 1px solid transparent; */

/* 방법 2: display: flex 또는 grid 사용 (겹침 발생 안 함) */
.parent { display: flex; flex-direction: column; }

/* 방법 3: overflow: hidden */
.parent { overflow: hidden; }
```

> ⚠️ **좌우 마진은 겹치지 않는다!** 오직 수직(상하) 마진만 겹친다.
> Flexbox/Grid 컨테이너 안에서는 마진 겹침이 발생하지 않는다.

## 5. outline vs border 🟡

```css
/* border — 박스 모델에 포함 (공간 차지) */
.box { border: 2px solid blue; }  /* 박스 크기에 영향 */

/* outline — 박스 모델 바깥 (공간 차지 안 함) */
.box { outline: 2px solid red; }  /* 박스 크기에 영향 없음. 레이아웃 안 밀림 */
```

> 💡 `outline`은 주로 **포커스 링**에 사용된다.
> `:focus-visible { outline: 2px solid blue; }` — 레이아웃을 밀지 않으면서 포커스 표시

## 6. 개발자 도구로 박스 모델 확인하기 (실습) 🔴

```
1. 크롬 F12 → Elements 탭에서 아무 요소 클릭
2. 우측 하단에 박스 모델 다이어그램이 표시됨
   - 파란색: content
   - 초록색: padding
   - 노란색: border
   - 주황색: margin
3. 각 영역 위에 마우스를 올리면 페이지에서 해당 영역이 하이라이트됨
4. 숫자를 더블클릭하면 실시간으로 값을 수정할 수 있음!
```

### 실습: 박스 모델 계산 문제

```css
.card {
  box-sizing: content-box;
  width: 300px;
  padding: 15px;
  border: 3px solid gray;
  margin: 10px;
}
```

**Q: 이 요소가 화면에서 차지하는 총 너비는?**

<details>
<summary>정답 보기</summary>

```
box-sizing: content-box이므로:
총 너비 = margin-left + border-left + padding-left + width + padding-right + border-right + margin-right
       = 10 + 3 + 15 + 300 + 15 + 3 + 10
       = 356px

만약 box-sizing: border-box였다면:
총 너비 = margin-left + width + margin-right
       = 10 + 300 + 10
       = 320px (padding과 border가 300px 안에 포함됨)
```

</details>

## 면접 대비

### 🔴 필수 질문

**Q1. CSS 박스 모델을 설명해주세요.**

> 모든 HTML 요소는 Content(실제 내용), Padding(내부 여백), Border(테두리), Margin(외부 여백) 4개의 영역으로 구성된 직사각형 상자(Box)로 렌더링된다. width와 height는 기본적으로 Content 영역만을 의미하며, `box-sizing: border-box`를 적용하면 Border까지 포함한 크기를 의미하게 된다.

**Q2. `box-sizing: border-box`는 왜 쓰나요?**

> 기본값인 `content-box`는 width에 padding과 border가 더해져서 실제 크기가 예상보다 커지는 문제가 있다. `border-box`를 쓰면 padding과 border가 width 안에 포함되어 직관적인 크기 계산이 가능하다. 실무에서는 `*, *::before, *::after { box-sizing: border-box; }`를 전역 리셋으로 사용한다.

**Q3. 마진 겹침(Margin Collapsing)이란?**

> 두 블록 요소의 수직(상하) 마진이 인접할 때, 합산되지 않고 더 큰 쪽의 마진만 적용되는 현상이다. 인접 형제, 부모-자식, 빈 블록 3가지 경우에 발생하며, Flexbox/Grid 컨테이너 내부에서는 발생하지 않는다. 좌우 마진은 겹치지 않는다.
