# 8. display와 position

> **키워드**: `block` `inline` `inline-block` `none` `static` `relative` `absolute` `fixed` `sticky` `z-index`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | display 4종, position 5종, z-index | 면접 단골 + 레이아웃의 기본기 |
| 🟡 이해 | 쌓임 맥락(Stacking Context) | 디버깅에 필수 |
| 🟢 참고 | display: contents, isolation | 고급 패턴 |

---

## 1. display — 요소의 배치 성격을 결정 🔴

### 4가지 핵심 display 값

```
block          한 줄을 통째로 차지. 다음 요소는 아래로.
─────────────────────────────────
|          <div>                |
─────────────────────────────────
|          <p>                  |
─────────────────────────────────

inline         콘텐츠만큼만 차지. 옆으로 나란히.
이것은 <span>인라인</span>이고 <a>링크</a>도 인라인이다.

inline-block   인라인처럼 나란히 + 블록처럼 width/height 설정 가능
┌──────┐ ┌──────┐ ┌──────┐
│ 150px│ │ 150px│ │ 150px│  ← 나란히 배치되면서 크기 설정 가능
└──────┘ └──────┘ └──────┘

none           화면에서 완전히 사라짐. 공간도 차지하지 않음.
(vs visibility: hidden → 보이진 않지만 공간은 차지)
```

### 비교표

| 속성 | 줄바꿈 | width/height | 상하 margin | 대표 태그 |
|------|--------|-------------|------------|----------|
| `block` | 새 줄 시작 | ✅ 설정 가능 | ✅ 적용 | div, p, h1~h6 |
| `inline` | 같은 줄 | ❌ 불가 | ❌ 무시 | span, a, strong |
| `inline-block` | 같은 줄 | ✅ 설정 가능 | ✅ 적용 | img, button, input |
| `none` | 사라짐 | - | - | - |

### none vs visibility: hidden vs opacity: 0

```css
/* 완전히 사라짐 — 공간도 없음, 스크린 리더도 무시 */
.gone { display: none; }

/* 보이지 않지만 공간 차지 — 레이아웃 유지 */
.invisible { visibility: hidden; }

/* 투명하지만 공간 차지 + 클릭 가능 — 애니메이션에 활용 */
.transparent { opacity: 0; }
```

## 2. position — 요소의 위치 기준을 결정 🔴

### 5가지 position 값

```css
/* 1. static (기본값) — 문서 흐름대로 배치. top/left 무시 */
.box { position: static; }

/* 2. relative — 원래 위치 기준으로 이동. 원래 공간은 유지됨 */
.box {
  position: relative;
  top: 20px;    /* 원래 위치에서 아래로 20px */
  left: 10px;   /* 원래 위치에서 오른쪽으로 10px */
}

/* 3. absolute — 가장 가까운 position 조상 기준으로 배치. 문서 흐름에서 빠짐 */
.parent { position: relative; }  /* 기준점 역할 */
.child {
  position: absolute;
  top: 0;
  right: 0;     /* 부모의 오른쪽 상단에 배치 */
}

/* 4. fixed — 뷰포트(브라우저 창) 기준. 스크롤해도 안 움직임 */
.navbar {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;  /* 상단 고정 네비게이션 바 */
}

/* 5. sticky — 평소엔 relative, 스크롤 지점 도달 시 fixed처럼 고정 */
.sidebar {
  position: sticky;
  top: 20px;    /* 스크롤하다가 뷰포트 상단에서 20px 되면 고정 */
}
```

### 시각적 비교

```
┌── 문서 흐름 ─────────────────────────────────┐
│                                               │
│  [static]  문서 흐름 그대로. 가장 평범한 상태    │
│                                               │
│  [relative] 원래 자리를 기억하고 살짝 이동       │
│        ↗ (원래 자리는 빈 채로 남아있음)          │
│                                               │
│  [absolute] 문서 흐름에서 완전히 이탈!           │
│  (relative 부모를 기준으로 자유롭게 배치)         │
│                                               │
└───────────────────────────────────────────────┘

┌── 뷰포트(브라우저 창) ─────────────────────────┐
│ [fixed] 스크롤해도 이 자리에 꼼짝 안 함          │
│                                               │
│ [sticky] 여기까지 스크롤하면 그때부터 고정!       │
│  ↕ (평소엔 문서 흐름, 임계점 도달 시 fixed)      │
│                                               │
└───────────────────────────────────────────────┘
```

## 3. absolute의 핵심 — 기준점 찾기 🔴

> 💡 `absolute`는 부모를 타고 올라가며 `position`이 `static`이 아닌 첫 번째 조상을 기준으로 삼는다.
> 조상이 모두 `static`이면 **최상위 `<html>`** 이 기준이 된다.

```
실무 패턴: 부모에 relative, 자식에 absolute

┌─── .card (position: relative) ───────────┐
│                                          │
│  상품 이미지                               │
│                                          │
│  ┌─── .badge (position: absolute) ───┐   │
│  │  HOT 🔥                           │   │  ← 카드 우측 상단에 배지
│  └───────────────────────────────────┘   │
│  top: 10px; right: 10px;                 │
│                                          │
│  상품 이름                                │
│  가격: 29,000원                           │
└──────────────────────────────────────────┘
```

```css
.card {
  position: relative;  /* 기준점 역할 (실제로 이동하지 않음) */
}
.badge {
  position: absolute;
  top: 10px;
  right: 10px;
}
```

> ⚠️ 실수 포인트: 부모에 `position: relative`를 안 주면 badge가 페이지 최상단으로 날아간다!

## 4. z-index — 요소의 쌓임 순서 🔴

> 💡 **비유: 카드 덱**
> z-index가 높을수록 위에 쌓인다. 기본값은 `auto`(0과 동일).

```css
.back { z-index: 1; }    /* 뒤에 깔림 */
.middle { z-index: 10; }  /* 중간 */
.front { z-index: 100; }  /* 맨 앞에 보임 */
```

```
화면에서 보이는 순서 (위에서 아래 = 앞에서 뒤):

   ┌──────────┐ z-index: 100 (맨 앞)
   │  FRONT   │
   └──┬───────┘
      │  ┌──────────┐ z-index: 10
      │  │  MIDDLE  │
      │  └──┬───────┘
      │     │  ┌──────────┐ z-index: 1 (맨 뒤)
      │     │  │  BACK    │
      │     │  └──────────┘
```

> ⚠️ **z-index는 `position: static`이 아닌 요소에서만 작동한다!**
> `position: relative`, `absolute`, `fixed`, `sticky`에서만 의미가 있음.

### 쌓임 맥락(Stacking Context) 🟡

```
z-index가 안 먹힐 때 99%의 원인: 쌓임 맥락

부모가 새로운 쌓임 맥락을 만들면,
자식의 z-index는 그 부모 안에서만 비교된다.
부모의 z-index가 1이면, 자식이 z-index: 9999여도
옆에 있는 z-index: 2인 요소보다 뒤에 간다!
```

```
❌ 흔한 실수
.parent-a { z-index: 1; }   /* 쌓임 맥락 생성 */
  .child-a { z-index: 9999; }  /* 부모(1) 안에서만 9999 */

.parent-b { z-index: 2; }   /* 쌓임 맥락 생성 */
  .child-b { z-index: 1; }    /* 부모(2) 안에서 1 */

→ child-b가 child-a보다 위에 보인다! (부모의 z-index가 더 높으므로)
```

### 새로운 쌓임 맥락을 만드는 조건

```
- position: relative/absolute/fixed/sticky + z-index 값 설정
- opacity가 1 미만
- transform, filter, perspective 속성 사용
- isolation: isolate
- display: flex/grid 자식에 z-index 설정
```

## 5. 실전 패턴 모음 🟡

### 패턴 1: 상단 고정 헤더

```css
.header {
  position: fixed;
  top: 0;
  left: 0;
  width: 100%;
  z-index: 1000;
  background: white;
}
body {
  padding-top: 60px;  /* 헤더 높이만큼 밀어줘야 콘텐츠가 안 가려짐 */
}
```

### 패턴 2: 중앙 정렬 (absolute)

```css
.center {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);  /* 자기 크기의 절반만큼 역이동 */
}
```

### 패턴 3: 오버레이 (모달 배경)

```css
.overlay {
  position: fixed;
  inset: 0;           /* top:0; right:0; bottom:0; left:0; 의 단축 */
  background: rgba(0, 0, 0, 0.5);
  z-index: 999;
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. `position: relative`와 `position: absolute`의 차이?**

> `relative`는 원래 문서 흐름의 위치를 기준으로 이동하며, 원래 자리는 빈 채로 유지된다. `absolute`는 문서 흐름에서 완전히 빠지며, position이 static이 아닌 가장 가까운 조상을 기준으로 배치된다.

**Q2. `z-index`가 안 먹히는 이유?**

> 가장 흔한 원인은 쌓임 맥락(Stacking Context)이다. 부모가 새로운 쌓임 맥락을 생성하면, 자식의 z-index는 부모 내부에서만 비교된다. 부모의 z-index가 낮으면 자식이 아무리 높은 값을 가져도 외부 요소보다 아래에 깔린다. 또한 z-index는 position이 static이 아닌 요소에서만 작동한다.

**Q3. `display: none`과 `visibility: hidden`의 차이?**

> `display: none`은 요소를 DOM에서 공간까지 완전히 제거하여 레이아웃에 영향을 주지 않는다. `visibility: hidden`은 보이지 않지만 공간은 그대로 차지한다. 접근성 측면에서 `display: none`은 스크린 리더도 무시하지만, `visibility: hidden`은 상황에 따라 읽힐 수 있다.

```
┌────────────────────────────────────────────────┐
│          display와 position 핵심 요약           │

```
