# 2. HTML 기본 태그와 문서 구조

> **키워드**: `<!DOCTYPE>` `<html>` `<head>` `<body>` `메타 태그` `블록/인라인` `기본 태그`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 문서 구조, 블록 vs 인라인, 핵심 태그 | 면접 단골 + 매일 사용 |
| 🟡 이해 | 메타 태그, head 내부 요소 | 개념 이해하면 OK |
| 🟢 참고 | 문자 인코딩, DOCTYPE 역사 | 있다는 것만 알면 됨 |

---

## 1. HTML 문서의 뼈대 🔴

> 💡 **비유: 사람의 골격**
> `<html>`은 몸 전체, `<head>`는 뇌(보이진 않지만 중요한 정보), `<body>`는 눈에 보이는 몸통이다.

```html
<!DOCTYPE html>           <!-- "이 문서는 HTML5야!" 선언 -->
<html lang="ko">          <!-- 문서 전체를 감싸는 최상위 태그 -->

  <head>                   <!-- 🧠 보이지 않는 정보 (브라우저/검색엔진용) -->
    <meta charset="UTF-8">                          <!-- 한글 깨짐 방지 -->
    <meta name="viewport" content="width=device-width, initial-scale=1.0">  <!-- 모바일 대응 필수 -->
    <meta name="description" content="나의 첫 웹페이지">  <!-- 구글 검색 결과 설명 -->
    <title>나의 첫 웹페이지</title>                    <!-- 브라우저 탭 제목 -->
    <link rel="stylesheet" href="style.css">          <!-- CSS 연결 -->
  </head>

  <body>                   <!-- 👀 눈에 보이는 모든 콘텐츠 -->
    <h1>안녕하세요!</h1>
    <p>나의 첫 번째 웹페이지입니다.</p>
  </body>

</html>
```

### 각 줄이 하는 일

| 태그 | 역할 |
|------|------|
| `<!DOCTYPE html>` | 브라우저에게 "HTML5 표준으로 해석해"라고 선언. 없으면 쿼크 모드(옛날 방식)로 동작 |
| `<html lang="ko">` | 문서 전체의 루트. `lang`은 스크린 리더와 검색엔진에게 언어를 알려줌 |
| `<meta charset="UTF-8">` | 문자 인코딩. 없으면 한글이 깨질 수 있음 |
| `<meta name="viewport">` | 모바일 기기에서 화면 너비를 기기 너비에 맞춤. **반응형 웹의 시작** |
| `<title>` | 브라우저 탭에 표시되는 제목 + 구글 검색 결과의 파란 제목 |

## 2. 블록(Block) 요소 vs 인라인(Inline) 요소 🔴

> HTML 태그는 크게 두 종류로 나뉜다. 이걸 모르면 CSS 레이아웃에서 막힌다.

```
블록 요소 (Block)
┌─────────────────────────────────────┐
│ <div> 나는 한 줄을 통째로 차지한다    │
└─────────────────────────────────────┘
┌─────────────────────────────────────┐
│ <p> 나도 한 줄을 통째로 차지한다      │
└─────────────────────────────────────┘

인라인 요소 (Inline)
"나는 <span>흐르는 텍스트</span> 안에서 <a href="#">링크처럼</a> 끼어든다"
```

| 구분 | 블록(Block) | 인라인(Inline) |
|------|------------|---------------|
| **줄바꿈** | 항상 새 줄에서 시작 | 같은 줄에 나란히 배치 |
| **너비** | 부모의 100% 차지 | 콘텐츠만큼만 차지 |
| **width/height** | 설정 가능 | ❌ 설정 불가 |
| **margin/padding** | 상하좌우 모두 적용 | 좌우만 정상 적용 (상하 margin 무시) |
| **대표 태그** | `<div>`, `<p>`, `<h1>~<h6>`, `<ul>`, `<section>` | `<span>`, `<a>`, `<strong>`, `<em>`, `<img>` |

> ⚠️ `<img>`는 인라인이면서 width/height 설정이 가능한 특수한 **대체 요소(Replaced Element)**다.

## 3. 핵심 태그 총정리 🔴

### 제목 (Heading)

```html
<h1>가장 큰 제목 (페이지당 1개 권장)</h1>
<h2>두 번째 제목</h2>
<h3>세 번째 제목</h3>
<!-- h4, h5, h6까지 있음 -->
```

> ⚠️ `<h1>`은 SEO에서 매우 중요. 한 페이지에 **1개만** 사용하는 것이 권장된다.
> 글자 크기를 키우고 싶다고 `<h1>`을 남발하면 안 된다. 크기는 CSS로 조절!

### 텍스트

```html
<p>문단(paragraph). 블록 요소. 텍스트 덩어리를 감싼다.</p>
<span>인라인 텍스트 조각. 스타일링 용도로 자주 씀.</span>
<strong>중요한 텍스트 (굵게, 의미적으로 중요)</strong>
<em>강조 텍스트 (기울임, 의미적으로 강조)</em>
<br>  <!-- 줄바꿈. 남발 금지! 간격은 CSS margin으로 -->
<hr>  <!-- 수평선. 주제 전환을 나타냄 -->
```

> ❌ `<b>` `<i>` → 단순 시각적 효과만. 의미 없음
> ✅ `<strong>` `<em>` → 시각적 효과 + **의미(시맨틱)** 가 있음. 스크린 리더가 강조해서 읽어줌

### 링크와 이미지

```html
<!-- 링크 -->
<a href="https://google.com">구글로 이동</a>
<a href="https://google.com" target="_blank" rel="noopener noreferrer">새 탭에서 열기</a>
<a href="#section2">페이지 내 이동 (앵커)</a>
<a href="mailto:hello@example.com">이메일 보내기</a>

<!-- 이미지 -->
<img src="photo.jpg" alt="설명 텍스트" width="300" height="200">
```

> ⚠️ `target="_blank"` 사용 시 보안을 위해 반드시 `rel="noopener noreferrer"` 추가!
> ⚠️ `<img>`의 `alt` 속성은 **접근성 필수**. 이미지를 볼 수 없을 때 대체 텍스트 역할

### 리스트

```html
<!-- 순서 없는 리스트 (점) -->
<ul>
  <li>사과</li>
  <li>바나나</li>
</ul>

<!-- 순서 있는 리스트 (번호) -->
<ol>
  <li>첫 번째</li>
  <li>두 번째</li>
</ol>

<!-- 설명 리스트 (용어 + 정의) -->
<dl>
  <dt>HTML</dt>
  <dd>웹 문서의 구조를 정의하는 마크업 언어</dd>
</dl>
```

### 구역 나누기

```html
<div>  <!-- 범용 블록 컨테이너. 의미 없음. 스타일링/그룹핑 용도 -->
  <span>범용 인라인 컨테이너. 의미 없음.</span>
</div>
```

> 💡 `<div>`와 `<span>`은 **의미가 없는** 태그다.
> 의미가 있는 상황에서는 다음 시간에 배울 **시맨틱 태그**(`<header>`, `<main>`, `<section>` 등)를 써야 한다.

## 4. 실습: 자기소개 페이지 만들기

아래 요구사항에 맞는 HTML 파일을 **직접** 작성해보자 (CSS 없이, 순수 HTML만):

```
요구사항:
1. 페이지 제목(title): "홍길동의 자기소개"
2. 큰 제목(h1): 이름
3. 소제목(h2): "소개", "기술 스택", "연락처" 3개 섹션
4. 소개 섹션: 자기소개 문단(p) 2~3줄
5. 기술 스택: 순서 없는 리스트(ul)로 3개 이상
6. 연락처: 이메일 링크(a href="mailto:...")
7. 프로필 사진 자리: img 태그 (src는 아무 이미지 URL)
8. charset, viewport 메타 태그 포함
```

<summary>정답 예시 보기</summary>

```html

  <h2>소개</h2>

  <h2>기술 스택</h2>

    <li>HTML5</li>
    <li>CSS3</li>
    <li>JavaScript (ES6+)</li>
    <li>React</li>
  </ul>

  <h2>연락처</h2>
  <p>이메일: <a href="mailto:hong@example.com">hong@example.com</a></p>
  <p>깃허브: <a href="https://github.com/hong" target="_blank" rel="noopener noreferrer">github.com/hong</a></p>

</html>
```

## 면접 대비

### 🔴 필수 질문

**Q1. `<!DOCTYPE html>`은 왜 필요한가요?**

> 브라우저에게 이 문서가 HTML5 표준을 따른다는 것을 선언한다. 이 선언이 없으면 브라우저는 쿼크 모드(Quirks Mode)로 동작하여 CSS 해석이 예상과 다르게 될 수 있다.

**Q2. 블록 요소와 인라인 요소의 차이?**

> 블록 요소는 항상 새 줄에서 시작하며 부모 너비의 100%를 차지한다. width/height, margin/padding이 모두 적용된다. 인라인 요소는 같은 줄에 나란히 배치되며 콘텐츠 크기만큼만 차지한다. width/height 설정이 불가하고, 상하 margin이 무시된다.

**Q3. `<strong>`과 `<b>`의 차이는?**

> 둘 다 시각적으로 굵은 글씨를 만들지만, `<strong>`은 "이 텍스트가 중요하다"는 의미(시맨틱)를 가진다. 스크린 리더가 강조해서 읽어준다. `<b>`는 단순히 시각적 효과만 있고 의미가 없다. 시맨틱 HTML을 위해 `<strong>` 사용을 권장한다.

### 🟡 개념 질문

**Q4. `<meta name="viewport">`는 왜 필요한가요?**

> 모바일 브라우저에게 페이지의 너비를 기기 화면 너비에 맞추라고 지시한다. 이 태그가 없으면 모바일에서 데스크톱 사이즈로 축소되어 글자가 매우 작게 보인다. 반응형 웹의 필수 조건이다.

## 핵심 요약

```
┌─────────────────────────────────────────────────┐
│           HTML 문서 구조 핵심 요약                │
├─────────────────────────────────────────────────┤
│                                                 │
│  <!DOCTYPE html>  → HTML5 표준 선언             │
│  <html lang="ko"> → 문서 루트 + 언어 설정       │
│  <head>           → 보이지 않는 정보 (메타/타이틀)│
│  <body>           → 눈에 보이는 모든 콘텐츠      │
│                                                 │
│  블록(Block): 한 줄 통째로 차지 (div, p, h1)     │
│  인라인(Inline): 흐름 속에 끼어듦 (span, a, img)  │
│                                                 │
│  의미 있는 태그 ✅: <strong>, <em>               │
│  의미 없는 태그 ❌: <b>, <i>                     │
│                                                 │
└─────────────────────────────────────────────────┘
```
