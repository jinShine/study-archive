## 1. 색상 표기법 🔴

```css
/* 1. 키워드 — 간단하지만 제한적 */
color: red;
color: transparent;

/* 2. HEX — 실무에서 가장 많이 사용 */
color: #ff0000;          /* 빨강 */
color: #333;             /* #333333의 축약 (어두운 회색) */
color: #ff000080;        /* 뒤 2자리 = 투명도 (50%) */

/* 3. RGB/RGBA — 투명도 제어 시 */
color: rgb(255, 0, 0);
color: rgba(255, 0, 0, 0.5);   /* 50% 투명 */

/* 4. HSL — 디자이너 친화적 (색상/채도/밝기) */
color: hsl(0, 100%, 50%);       /* 빨강 */
color: hsla(0, 100%, 50%, 0.5); /* 50% 투명 */

/* 5. 현대 CSS — oklch (인간의 눈에 균일한 색 공간) */
color: oklch(0.7 0.15 30);
```

> 💡 **실무 팁**: Tailwind CSS를 쓰면 색상을 직접 작성할 일이 거의 없다.
> `text-red-500`, `bg-blue-100` 같은 유틸리티 클래스를 사용한다.
> 커스텀 색상은 `tailwind.config` 또는 CSS 변수(`--primary: #4F46E5`)로 관리.

## 2. CSS 단위 완전 정복 🔴

### 절대 단위 vs 상대 단위

```
절대 단위: 어디서든 같은 크기 (고정)
  px → 1px = 화면의 1점. 가장 직관적

상대 단위: 기준에 따라 크기가 변함 (반응형!)
  rem  → 루트(html)의 font-size 기준 (보통 16px)
  em   → 부모의 font-size 기준
  %    → 부모의 해당 속성 기준
  vw   → 뷰포트 너비의 1%
  vh   → 뷰포트 높이의 1%
```

### px vs rem vs em

```css
/* 브라우저 기본: html { font-size: 16px; } */

.title {
  font-size: 32px;     /* 항상 32px. 사용자가 브라우저 글꼴 키워도 안 변함 ❌ */
}

.title {
  font-size: 2rem;     /* 16px × 2 = 32px. 사용자가 글꼴 키우면 따라 커짐 ✅ */
}
```

> ⚠️ **rem을 써야 하는 핵심 이유: 접근성**
> 시력이 나쁜 사용자는 브라우저 설정에서 기본 글꼴 크기를 키운다.
> `px`로 고정하면 이 설정이 무시되어 접근성 위반이다.
> `rem`을 쓰면 사용자 설정에 비례하여 자연스럽게 커진다.

### rem vs em 차이

```css
html { font-size: 16px; }

/* rem = Root Em. 항상 html의 font-size 기준 → 예측 가능 */
.a { font-size: 1.5rem; }  /* 항상 16 × 1.5 = 24px */

/* em = 부모의 font-size 기준 → 중첩되면 예측 어려움 */
.parent { font-size: 20px; }
.child { font-size: 1.5em; }  /* 20 × 1.5 = 30px */
.grandchild { font-size: 1.5em; }  /* 30 × 1.5 = 45px 😱 중첩 폭발! */
```

> 💡 **실무 원칙**: font-size는 `rem`, padding/margin도 `rem`.
> em은 해당 요소의 font-size에 비례해야 하는 경우(버튼 내부 padding 등)에만 제한적으로 사용.

### 뷰포트 단위 (vw, vh)

```css
/* 화면 전체를 꽉 채우는 히어로 섹션 */
.hero {
  width: 100vw;     /* 뷰포트 너비의 100% */
  height: 100vh;    /* 뷰포트 높이의 100% */
}

/* 뷰포트 기반 글꼴 크기 (주의: 너무 작거나 커질 수 있음) */
.responsive-text {
  font-size: 5vw;   /* 화면 너비에 비례하여 커짐 */
}
```

> ⚠️ **모바일 주의**: `100vh`는 모바일 브라우저에서 주소창 높이 때문에 넘칠 수 있다.
> 해결: `height: 100dvh;` (Dynamic Viewport Height, 최신 CSS)

### clamp() — 반응형의 끝판왕 🟡

```css
/* clamp(최소값, 선호값, 최대값) */
.title {
  font-size: clamp(1.5rem, 4vw, 3rem);
  /* 최소 1.5rem, 화면에 따라 4vw, 최대 3rem */
  /* → 화면이 작으면 1.5rem, 크면 3rem, 중간은 자연스럽게 스케일 */
}

.container {
  width: clamp(320px, 90%, 1200px);
  /* 최소 320px, 기본 부모의 90%, 최대 1200px */
}
```

> 💡 `clamp()`는 미디어 쿼리 없이 반응형 타이포그래피를 만드는 현대적 방법이다.

## 3. 타이포그래피 🟡

```css
.text {
  /* 글꼴 패밀리 (왼쪽부터 시도, 없으면 다음 폰트) */
  font-family: 'Pretendard', 'Noto Sans KR', sans-serif;

  font-size: 1rem;           /* 글자 크기 */
  font-weight: 400;          /* 굵기 (100~900, 400=normal, 700=bold) */
  font-style: italic;        /* 기울임 */

  line-height: 1.6;          /* 줄 간격 (단위 없이! 1.6 = font-size의 1.6배) */
  letter-spacing: -0.02em;   /* 자간 (한글은 약간 좁히면 예쁨) */
  word-spacing: 0.05em;      /* 단어 간격 */

  text-align: center;        /* 정렬: left / center / right / justify */
  text-decoration: underline; /* 밑줄 / line-through / none */
  text-transform: uppercase;  /* 대문자 변환 */
  text-overflow: ellipsis;    /* 넘치는 텍스트 ... 처리 */
}

/* 한 줄 말줄임 (3가지 세트) */
.ellipsis {
  white-space: nowrap;       /* 줄바꿈 금지 */
  overflow: hidden;          /* 넘치는 부분 숨김 */
  text-overflow: ellipsis;   /* 넘치면 ... 표시 */
}

/* 여러 줄 말줄임 (2줄까지만 보여주고 ...) */
.line-clamp {
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

> 💡 **line-height 팁**: 단위 없는 숫자(1.5, 1.6)를 권장.
> `line-height: 24px`은 font-size가 바뀌어도 고정되지만,
> `line-height: 1.6`은 font-size에 비례하여 자연스럽게 조절된다.

### 웹 폰트 로딩 🟡

```css
/* 방법 1: Google Fonts CDN */
@import url('https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@400;700&display=swap');

/* 방법 2: @font-face (자체 호스팅) */
@font-face {
  font-family: 'Pretendard';
  src: url('/fonts/Pretendard-Regular.woff2') format('woff2');
  font-weight: 400;
  font-display: swap;  /* 폰트 로딩 중에도 텍스트를 시스템 폰트로 먼저 보여줌 */
}
```

> ⚠️ `font-display: swap`은 **성능 필수 옵션**이다.
> 없으면 폰트 로딩 완료까지 텍스트가 안 보이는 FOIT(Flash of Invisible Text) 발생.
> Next.js에서는 `next/font`가 이 모든 것을 자동 최적화해준다.

## 4. CSS 변수 (커스텀 속성) 🟡

```css
/* 선언: --이름: 값 */
:root {
  --color-primary: #4F46E5;
  --color-text: #1F2937;
  --font-size-base: 1rem;
  --radius: 8px;
}

/* 사용: var(--이름) */
.button {
  background: var(--color-primary);
  color: white;
  border-radius: var(--radius);
  font-size: var(--font-size-base);
}

/* 다크 모드에서 변수만 바꾸면 전체 테마가 변경! */
@media (prefers-color-scheme: dark) {
  :root {
    --color-primary: #818CF8;
    --color-text: #F9FAFB;
  }
}
```

> 💡 Tailwind CSS의 `tailwind.config`에서 커스텀 색상을 정의하면
> 내부적으로 CSS 변수로 변환된다.

```

1. CSS 변수로 색상 팔레트 정의 (primary, gray 3단계)

6. 한 줄 말줄임 + 2줄 말줄임 예시 포함
```

## 면접 대비

### 🔴 필수 질문

**Q1. `px`과 `rem`의 차이? 왜 `rem`을 권장하나요?**

> `px`은 고정 단위로 브라우저 설정에 영향받지 않는다. `rem`은 루트 요소(html)의 font-size를 기준으로 하는 상대 단위다. 사용자가 브라우저에서 글꼴 크기를 키웠을 때 `rem`은 비례하여 커지지만 `px`는 변하지 않아 접근성을 해친다. 반응형 설계와 접근성을 위해 `rem` 사용이 권장된다.

**Q2. `em`과 `rem`의 차이?**

> `rem`은 항상 루트(html)의 font-size 기준이므로 예측 가능하다. `em`은 부모 요소의 font-size 기준이라 중첩되면 값이 누적되어 계산이 어려워진다. font-size에는 `rem`을 쓰고, 해당 요소의 font-size에 비례해야 하는 padding 등에만 `em`을 제한적으로 사용한다.

### 🟡 개념 질문

**Q3. `clamp()` 함수의 역할은?**

> `clamp(최소값, 선호값, 최대값)` 형태로, 미디어 쿼리 없이 반응형 크기를 만든다. 선호값이 최소값과 최대값 사이에서 유동적으로 변한다. 예를 들어 `clamp(1rem, 3vw, 2rem)`은 화면 크기에 따라 1rem~2rem 사이에서 자연스럽게 조절된다.
