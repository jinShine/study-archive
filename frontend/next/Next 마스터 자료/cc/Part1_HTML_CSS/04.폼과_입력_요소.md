# 4. 폼과 입력 요소

> **키워드**: `<form>` `<input>` `<label>` `<select>` `<textarea>` `name` `action` `method`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | form/input/label 기본, name 속성, 제출 흐름 | React/Next.js 폼의 기초 |
| 🟡 이해 | input 타입 종류, 내장 유효성 검사 | 실무에서 자주 사용 |
| 🟢 참고 | FormData API, 접근성 고급 | 프레임워크에서 활용 |

## 1. 폼(Form)이란? 🔴

> 💡 **비유: 관공서 신청서**
> `<form>`은 빈 신청서 양식이고, `<input>`은 빈칸(이름, 전화번호 등)이다.
> `<button type="submit">`은 접수 창구에 제출하는 행위다.
> `name` 속성은 각 빈칸 옆에 적힌 "이름:", "전화:" 같은 라벨이다.
> 서버(공무원)는 `name`을 보고 어떤 데이터인지 판별한다.

```html
<form action="/api/register" method="POST">
  <!--  action: 신청서를 어디로 보낼지 (서버 주소)  -->
  <!--  method: 어떤 방법으로 보낼지 (GET/POST)    -->

  <label for="username">이름</label>
  <input type="text" id="username" name="username">
  <!--  name="username" → 서버가 { username: "홍길동" } 으로 받음  -->

  <button type="submit">제출</button>
</form>
```

### 제출 시 일어나는 일 (Step-by-step)

```
Step 1: 사용자가 입력란을 채우고 "제출" 버튼 클릭
Step 2: 브라우저가 모든 input의 name=value 쌍을 수집
        → { username: "홍길동", email: "hong@test.com" }
Step 3: action 주소(/api/register)로 method(POST) 방식으로 전송
Step 4: 서버가 데이터를 받아 처리 후 응답
Step 5: 브라우저가 응답에 따라 화면 갱신 (보통 새로고침 발생)
```

> ⚠️ **React/Next.js에서는** `e.preventDefault()`로 새로고침을 막거나,
> Next.js 15의 Server Actions에서는 `<form action={serverFunction}>` 패턴을 사용한다.

## 2. `<label>` — 클릭 영역을 넓히는 마법 🔴

```html
<!-- ❌ 나쁜 예: label 없음 → 작은 체크박스만 클릭 가능 -->
<input type="checkbox" id="agree"> 동의합니다

<!-- ✅ 좋은 예: label로 연결 → "동의합니다" 텍스트 클릭해도 체크됨 -->
<label for="agree">
  <input type="checkbox" id="agree"> 동의합니다
</label>

<!-- 또는 for 속성으로 연결 -->
<label for="email">이메일</label>
<input type="email" id="email" name="email">
```

> `<label>`의 `for` 값과 `<input>`의 `id` 값이 일치해야 연결된다.
> **접근성 필수**: 스크린 리더가 "이메일 입력란"이라고 안내해준다.

## 3. input 타입 총정리 🟡

```html
<!-- 텍스트 계열 -->
<input type="text" placeholder="일반 텍스트">
<input type="email" placeholder="이메일 (@ 필수 검증)">
<input type="password" placeholder="비밀번호 (●●●● 마스킹)">
<input type="tel" placeholder="전화번호 (모바일: 숫자 키패드)">
<input type="url" placeholder="URL (https:// 검증)">
<input type="search" placeholder="검색어 (X 버튼 자동 생성)">

<!-- 숫자/날짜 -->
<input type="number" min="0" max="100" step="1">
<input type="range" min="0" max="100">   <!-- 슬라이더 -->
<input type="date">                       <!-- 달력 UI -->
<input type="time">
<input type="datetime-local">

<!-- 선택 계열 -->
<input type="checkbox">   <!-- 다중 선택 (0개 이상) -->
<input type="radio" name="gender" value="male">     <!-- 단일 선택 -->
<input type="radio" name="gender" value="female">   <!-- 같은 name끼리 그룹 -->

<!-- 파일/색상 -->
<input type="file" accept="image/*">        <!-- 파일 업로드 -->
<input type="color">                        <!-- 색상 선택기 -->

<!-- 숨김 -->
<input type="hidden" name="userId" value="123">  <!-- 화면에 안 보이지만 전송됨 -->
```

> 💡 **실무 팁**: `type`을 정확히 지정하면:
> - 모바일에서 **적절한 키보드**가 올라옴 (`email` → @ 키보드, `tel` → 숫자 키패드)
> - 브라우저 **내장 유효성 검사**가 자동 적용됨

## 4. 기타 폼 요소 🟡

```html
<!-- 드롭다운 (선택 상자) -->
<label for="city">도시</label>
<select id="city" name="city">
  <option value="">선택하세요</option>
  <option value="seoul">서울</option>
  <option value="busan">부산</option>
  <option value="jeju">제주</option>
</select>

<!-- 여러 줄 텍스트 -->
<label for="bio">자기소개</label>
<textarea id="bio" name="bio" rows="5" cols="40" placeholder="자기소개를 입력하세요"></textarea>

<!-- 버튼 종류 -->
<button type="submit">제출 (폼 전송)</button>
<button type="reset">초기화 (입력값 리셋)</button>
<button type="button">일반 버튼 (JS 이벤트용)</button>
```

> ⚠️ `<button>`의 기본 `type`은 `submit`이다.
> 폼 안에서 `type="button"`을 명시하지 않으면 클릭 시 폼이 제출되는 버그가 발생한다!

## 5. HTML 내장 유효성 검사 🟡

JavaScript 없이 HTML 속성만으로 기본적인 검증이 가능하다:

```html
<form>
  <!-- 필수 입력 -->
  <input type="text" name="name" required>

  <!-- 최소/최대 길이 -->
  <input type="text" name="nick" minlength="2" maxlength="10">

  <!-- 숫자 범위 -->
  <input type="number" name="age" min="1" max="150">

  <!-- 정규식 패턴 -->
  <input type="text" name="phone" pattern="[0-9]{3}-[0-9]{4}-[0-9]{4}"
         title="000-0000-0000 형식으로 입력하세요">

  <button type="submit">제출</button>
</form>
```

> 💡 **실무에서는** HTML 검증 + JavaScript 검증 + **서버 검증** 3중 방어선을 친다.
> HTML 검증은 개발자 도구에서 `required` 속성을 지우면 뚫리기 때문에 절대 이것만 믿으면 안 된다.

## 6. `name` 속성이 핵심인 이유 🔴

```html
<form action="/api/login" method="POST">
  <input type="email" name="email" value="hong@test.com">
  <input type="password" name="password" value="1234">
  <button type="submit">로그인</button>
</form>
```

이 폼이 제출되면 서버는 이렇게 받는다:

```
// 서버가 받는 데이터
{
  "email": "hong@test.com",
  "password": "1234"
}
```

> ⚠️ `name` 속성이 없으면 **해당 input의 값은 전송되지 않는다!**
> React Hook Form의 `register("email")`도, Next.js의 `formData.get("email")`도
> 결국 이 `name` 속성을 기반으로 동작한다.

## 7. 실습: 회원가입 폼 만들기

```

1. 이름 (text, 필수, 2~20자)
2. 이메일 (email, 필수)
3. 비밀번호 (password, 필수, 8자 이상)

```

## 면접 대비

### 🔴 필수 질문

**Q1. `<form>` 태그의 `action`과 `method` 속성은?**

> `action`은 폼 데이터를 전송할 서버의 URL 경로이고, `method`는 HTTP 메서드(GET 또는 POST)를 지정한다. GET은 쿼리 스트링으로, POST는 요청 본문으로 데이터를 전달한다.

**Q2. `name` 속성이 왜 중요한가?**

> 폼이 제출될 때 서버가 데이터를 식별하는 키(Key) 역할을 한다. `name`이 없는 input의 값은 서버로 전송되지 않는다. React Hook Form이나 Next.js Server Actions도 내부적으로 이 `name` 속성을 기반으로 데이터를 추출한다.

### 🟡 개념 질문

**Q3. `<label>`은 왜 필요한가?**

> 첫째, 접근성 — 스크린 리더가 입력란의 용도를 읽어준다. 둘째, UX — label을 클릭해도 연결된 input에 포커스가 가므로 클릭 영역이 넓어진다. `for` 속성과 `id`를 매칭하거나, label 안에 input을 중첩하여 연결한다.
