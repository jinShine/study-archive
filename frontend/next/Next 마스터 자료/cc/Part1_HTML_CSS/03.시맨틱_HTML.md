# 3. 시맨틱 HTML

> **키워드**: `<header>` `<nav>` `<main>` `<section>` `<article>` `<aside>` `<footer>` `SEO` `스크린 리더`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 시맨틱 태그의 종류와 의미, div 남발이 나쁜 이유 | 면접 단골 + SEO 필수 |
| 🟡 이해 | 올바른 태그 선택 기준, 검색엔진과의 관계 | 설명할 수 있으면 OK |
| 🟢 참고 | WHATWG 명세, 콘텐츠 모델 | 있다는 것만 알면 됨 |

---

## 1. 시맨틱(Semantic)이 뭔데? 🔴

> 💡 **비유: 택배 상자**
> 모든 물건을 "상자"라고만 적어서 보내면 받는 사람이 뭔지 모른다.
> "냉동식품", "깨지기 쉬움", "서류" 라고 적혀 있으면 열기 전에도 안다.
> HTML도 마찬가지. `<div>`만 쓰면 기계(검색엔진/스크린 리더)가 내용을 모른다.

```
❌ div 지옥 (Non-semantic)
<div class="header">
  <div class="nav">
    <div class="nav-item">메뉴1</div>
  </div>
</div>
<div class="main">
  <div class="article">
    <div class="title">제목</div>
    <div class="content">본문</div>
  </div>
</div>
<div class="footer">바닥글</div>

✅ 시맨틱 HTML (Semantic)
<header>
  <nav>
    <a href="/">메뉴1</a>
  </nav>
</header>
<main>
  <article>
    <h2>제목</h2>
    <p>본문</p>
  </article>
</main>
<footer>바닥글</footer>
```

둘 다 **화면에 보이는 결과는 완전히 동일**하다. 하지만 시맨틱 HTML은:

1. **검색엔진(SEO)**: 구글 봇이 "아, 여기가 본문이고 여기가 네비게이션이구나" 파악 → 검색 랭킹 상승
2. **접근성**: 스크린 리더가 "메인 콘텐츠로 건너뛰기" 같은 내비게이션 기능 제공
3. **유지보수**: 개발자가 코드만 보고도 구조를 즉시 파악 가능

## 2. 시맨틱 태그 완전 정복 🔴

### 페이지 구조를 잡는 7대 태그

```
┌──────────────────────────────────────┐
│  <header>  로고, 네비게이션, 검색바    │
│  ┌──────────────────────────────┐    │
│  │  <nav>  주요 메뉴 링크 모음    │    │
│  └──────────────────────────────┘    │
├──────────────────────────────────────┤
│  <main>  페이지의 핵심 콘텐츠 (1개만!) │
│  ┌────────────────┐ ┌──────────┐    │
│  │  <article>      │ │ <aside>  │    │
│  │  독립적 콘텐츠   │ │ 사이드바  │    │
│  │  (블로그 글 등)  │ │ 광고/위젯 │    │
│  │  ┌────────────┐ │ │          │    │
│  │  │ <section>  │ │ │          │    │
│  │  │ 주제별 구역 │ │ │          │    │
│  │  └────────────┘ │ │          │    │
│  └────────────────┘ └──────────┘    │
├──────────────────────────────────────┤
│  <footer>  저작권, 연락처, 링크        │
└──────────────────────────────────────┘
```

| 태그 | 의미 | 사용 예시 | 주의사항 |
|------|------|----------|---------|
| `<header>` | 소개 영역, 내비게이션 | 로고 + 메뉴 바 | 페이지/섹션마다 여러 개 가능 |
| `<nav>` | 주요 내비게이션 링크 모음 | 메인 메뉴, 사이드 메뉴 | 모든 링크 묶음에 쓰지 말 것. **주요** 내비게이션만 |
| `<main>` | 페이지의 핵심 콘텐츠 | 본문 영역 | **페이지당 1개만!** 사이드바/헤더/푸터 제외 |
| `<article>` | 독립적으로 배포 가능한 콘텐츠 | 블로그 글, 뉴스 기사, 상품 카드 | RSS에 단독으로 실려도 의미가 통하는 콘텐츠 |
| `<section>` | 주제별 그룹핑 | 소개/기능/가격 섹션 | 보통 `<h2>`~`<h6>` 제목을 포함해야 함 |
| `<aside>` | 주요 콘텐츠와 간접적 관련 | 사이드바, 광고, 관련 글 | 없어도 본문 이해에 지장 없는 보조 콘텐츠 |
| `<footer>` | 바닥글 | 저작권, 연락처, 사이트맵 | 페이지/섹션마다 여러 개 가능 |

### article vs section 선택 기준

```
"이 콘텐츠를 따로 떼어내서 다른 곳에 붙여도 의미가 통하는가?"

  ✅ YES → <article> (블로그 글, 댓글, 상품 카드)
  ❌ NO  → <section> (소개 구역, 기능 설명 구역)
```

## 3. 기타 시맨틱 태그 🟡

```html
<!-- 시간 -->
<time datetime="2026-03-22">2026년 3월 22일</time>

<!-- 그림 + 설명 -->
<figure>
  <img src="chart.png" alt="매출 차트">
  <figcaption>2026년 1분기 매출 현황</figcaption>
</figure>

<!-- 접기/펼치기 (JS 없이 동작!) -->
<details>
  <summary>더 보기</summary>
  <p>숨겨진 상세 내용이 여기에...</p>
</details>

<!-- 주소 정보 -->
<address>
  <a href="mailto:contact@example.com">contact@example.com</a>
</address>

<!-- 텍스트 하이라이트 -->
<p>이 부분이 <mark>중요한 키워드</mark>입니다.</p>
```

## 4. 올바른 태그 선택 플로우차트 🔴

어떤 태그를 써야 할지 모를 때 이 순서를 따르자:

```
질문 1: 이 콘텐츠에 적합한 시맨틱 태그가 있는가?
  ├─ YES → 그 태그를 사용 (header, nav, main, article, section, aside, footer)
  └─ NO  → 질문 2로

질문 2: 스타일링이나 레이아웃 목적인가?
  ├─ 블록 레벨 그룹핑 → <div>
  └─ 인라인 텍스트 꾸미기 → <span>
```

> ⚠️ **황금 규칙**: `<div>`는 **마지막 수단**이다. 의미 있는 태그가 있으면 반드시 그걸 먼저 써라.

## 5. 실습: 뉴스 기사 페이지 구조 잡기

아래 요구사항에 맞게 **시맨틱 태그로만** HTML을 작성해보자:

```
요구사항:
1. 상단: 사이트 로고 + 네비게이션 메뉴 (홈, 정치, 경제, 스포츠)
2. 본문: 뉴스 기사 3개 (각각 제목 + 작성일 + 본문 + 이미지)
3. 사이드바: 인기 기사 목록 (순서 있는 리스트)
4. 하단: 저작권 정보 + 이메일 연락처
5. div는 최대 2개까지만 사용 가능!
```

<details>
<summary>정답 예시 보기</summary>

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>데일리 뉴스</title>
</head>
<body>
  <header>
    <h1>데일리 뉴스</h1>
    <nav>
      <ul>
        <li><a href="/">홈</a></li>
        <li><a href="/politics">정치</a></li>
        <li><a href="/economy">경제</a></li>
        <li><a href="/sports">스포츠</a></li>
      </ul>
    </nav>
  </header>

  <div> <!-- 레이아웃 목적의 div (main + aside 가로 배치용) -->
    <main>
      <article>
        <h2>첫 번째 뉴스 제목</h2>
        <time datetime="2026-03-22">2026년 3월 22일</time>
        <figure>
          <img src="news1.jpg" alt="뉴스 관련 이미지">
          <figcaption>사진 설명</figcaption>
        </figure>
        <p>뉴스 본문 내용...</p>
      </article>

      <article>
        <h2>두 번째 뉴스 제목</h2>
        <time datetime="2026-03-22">2026년 3월 22일</time>
        <p>뉴스 본문 내용...</p>
      </article>

      <article>
        <h2>세 번째 뉴스 제목</h2>
        <time datetime="2026-03-21">2026년 3월 21일</time>
        <p>뉴스 본문 내용...</p>
      </article>
    </main>

    <aside>
      <h2>인기 기사</h2>
      <ol>
        <li><a href="#">인기 기사 1</a></li>
        <li><a href="#">인기 기사 2</a></li>
        <li><a href="#">인기 기사 3</a></li>
      </ol>
    </aside>
  </div>

  <footer>
    <p>&copy; 2026 데일리 뉴스. All rights reserved.</p>
    <address>
      <a href="mailto:news@example.com">news@example.com</a>
    </address>
  </footer>
</body>
</html>
```

## 면접 대비

### 🔴 필수 질문

**Q1. 시맨틱 HTML을 사용해야 하는 이유는?**

> 3가지 이점이 있다. 첫째, SEO — 검색엔진이 페이지 구조를 정확히 파악하여 검색 랭킹을 높인다. 둘째, 접근성 — 스크린 리더 사용자가 페이지를 탐색할 때 "메인 콘텐츠로 이동"처럼 구조적 내비게이션이 가능해진다. 셋째, 유지보수 — 개발자가 코드를 읽을 때 `<div class="header">` 보다 `<header>`가 의도를 훨씬 빠르게 전달한다.

**Q2. `<section>`과 `<article>`의 차이는?**

> `<article>`은 독립적으로 의미가 통하는 완결된 콘텐츠다. 블로그 글이나 뉴스 기사처럼 다른 사이트에 옮겨도 이해되는 단위다. `<section>`은 주제별로 콘텐츠를 그룹핑하는 용도로, 단독으로는 의미가 불완전하며 보통 제목(h2~h6) 태그를 포함한다.

**Q3. `<div>`를 언제 사용하나요?**

> 적합한 시맨틱 태그가 존재하지 않을 때, 순수하게 스타일링이나 레이아웃 목적으로 요소를 그룹핑하기 위해 사용한다. `<div>`는 의미가 없는 범용 컨테이너이므로, 시맨틱 태그가 있는 상황에서는 사용을 지양해야 한다.

### 🟡 개념 질문

**Q4. `<main>` 태그의 규칙은?**

> 페이지의 핵심 콘텐츠를 감싸며, 한 페이지에 반드시 1개만 존재해야 한다. 사이드바, 헤더, 푸터, 네비게이션 등 반복되는 UI는 포함하지 않는다.
