# 11. Grid 완전 정복

> **키워드**: `display: grid` `grid-template` `fr` `repeat` `minmax` `auto-fill` `grid-area`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 기본 그리드 선언, fr, repeat, gap, 영역 배치 | 2차원 레이아웃의 핵심 |
| 🟡 이해 | minmax, auto-fill/fit, grid-area, 반응형 그리드 | 실무 패턴 |
| 🟢 참고 | subgrid, 이름 붙은 라인 | 최신 CSS |

---

## 1. Flexbox vs Grid — 언제 뭘 쓸까? 🔴

```
Flexbox = 1차원 (한 방향: 가로 또는 세로)
  → 네비게이션 바, 카드 한 줄, 입력 필드+버튼

Grid = 2차원 (가로 + 세로 동시)
  → 대시보드 레이아웃, 이미지 갤러리, 전체 페이지 구조

둘 다 알아야 한다! 상황에 맞는 도구를 골라 쓰자.
```

```
Flex (1차원):                   Grid (2차원):
┌──┐ ┌──┐ ┌──┐                ┌──────┬──────┬──────┐
│  │ │  │ │  │ → 한 줄          │      │      │      │
└──┘ └──┘ └──┘                │  1   │  2   │  3   │
                               ├──────┼──────┼──────┤
                               │  4   │  5   │  6   │
                               └──────┴──────┴──────┘
```

## 2. Grid 기본 — 행과 열 정의 🔴

```css
.grid {
  display: grid;

  /* 열(Column) 정의 */
  grid-template-columns: 200px 200px 200px;  /* 200px 3열 */
  grid-template-columns: 1fr 1fr 1fr;        /* 균등 3열 */
  grid-template-columns: 1fr 2fr 1fr;        /* 1:2:1 비율 */
  grid-template-columns: 200px 1fr;          /* 사이드바(고정) + 메인(가변) */

  /* 행(Row) 정의 */
  grid-template-rows: 60px 1fr 40px;         /* 헤더(60) + 메인(가변) + 푸터(40) */

  /* 간격 */
  gap: 16px;             /* 행+열 간격 동일 */
  gap: 20px 16px;        /* row-gap 20px, column-gap 16px */
}
```

### fr 단위 — 남는 공간 비율 분배

```
grid-template-columns: 1fr 2fr 1fr;

전체 너비: 800px, gap: 0

┌──200px──┬──400px──┬──200px──┐
│  1fr    │   2fr   │  1fr    │
└─────────┴─────────┴─────────┘
  (1/4)      (2/4)     (1/4)
```

> 💡 `fr`은 Flexbox의 `flex-grow`와 비슷하다. 남은 공간을 비율로 나눈다.
> `200px 1fr`은 "200px 고정 + 나머지 전부"

## 3. repeat() — 반복 선언을 간결하게 🔴

```css
/* 이거를... */
grid-template-columns: 1fr 1fr 1fr 1fr 1fr 1fr;

/* 이렇게! */
grid-template-columns: repeat(6, 1fr);    /* 1fr 6번 반복 */

/* 혼합도 가능 */
grid-template-columns: 200px repeat(3, 1fr) 200px;
/* = 200px 1fr 1fr 1fr 200px */
```

## 4. minmax()와 auto-fill/auto-fit — 반응형 그리드의 핵심 🟡

### minmax(최소, 최대)

```css
.grid {
  grid-template-columns: repeat(3, minmax(200px, 1fr));
  /* 각 열: 최소 200px, 최대 1fr(균등 분배) */
  /* 화면이 좁아져도 200px 이하로 줄어들지 않음 */
}
```

### auto-fill vs auto-fit

```css
/* auto-fill: 빈 공간에도 빈 열을 만듦 (공간을 꽉 안 채움) */
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

/* auto-fit: 아이템이 없는 열은 0으로 축소 (공간을 꽉 채움 ✅) */
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
```

```
auto-fill (아이템 3개, 넓은 화면):
┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ← 빈 4번째 열이 공간 차지
│  1   │ │  2   │ │  3   │ │(빈칸)│
└──────┘ └──────┘ └──────┘ └──────┘

auto-fit (아이템 3개, 넓은 화면):
┌──────────┐ ┌──────────┐ ┌──────────┐ ← 3개가 공간을 꽉 채움!
│     1    │ │     2    │ │     3    │
└──────────┘ └──────────┘ └──────────┘
```

> 💡 **실무 공식**: 반응형 카드 그리드는 이 한 줄이면 끝:
> `grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));`

## 5. 아이템 배치 — 그리드 라인 번호 🟡

```
라인 번호 시각화 (3열 2행 그리드):

     1      2      3      4    ← 열 라인
  1  ┌──────┬──────┬──────┐
     │  A   │  B   │  C   │
  2  ├──────┼──────┼──────┤    ← 행 라인
     │  D   │  E   │  F   │
  3  └──────┴──────┴──────┘
```

```css
/* 아이템을 특정 위치에 배치 */
.item-a {
  grid-column: 1 / 3;   /* 열 라인 1~3 (2칸 차지) */
  grid-row: 1 / 2;      /* 행 라인 1~2 (1칸 차지) */
}

/* 축약 */
.item-a {
  grid-column: 1 / span 2;  /* 1번 라인부터 2칸 */
}

/* 전체 너비 차지 */
.full-width {
  grid-column: 1 / -1;  /* 첫 번째 ~ 마지막 라인 */
}
```

## 6. grid-template-areas — 직관적 영역 배치 🟡

> 💡 이름으로 레이아웃을 "그림 그리듯" 배치하는 방법

```css
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: 60px 1fr 40px;
  grid-template-areas:
    "header  header  header"
    "sidebar main   ads"
    "footer  footer  footer";
  min-height: 100vh;
  gap: 8px;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.ads     { grid-area: ads; }
.footer  { grid-area: footer; }
```

```
결과 시각화:
┌──────────────────────────────────┐
│           header                 │  60px
├─────────┬──────────────┬────────┤
│         │              │        │
│ sidebar │    main      │  ads   │  1fr (남은 전체)
│ (200px) │   (가변)     │(200px) │
│         │              │        │
├─────────┴──────────────┴────────┤
│           footer                 │  40px
└──────────────────────────────────┘
```

> 💡 빈 영역은 `.`(점)으로 표시: `"header header ."`

## 7. 자주 쓰는 정렬 속성 🟡

```css
.grid {
  /* 그리드 전체의 정렬 (컨테이너에) */
  justify-items: center;    /* 각 셀 안에서 가로 중앙 */
  align-items: center;      /* 각 셀 안에서 세로 중앙 */
  place-items: center;      /* 위 두 줄의 단축 → 완벽 중앙 정렬! */
}

.item {
  /* 개별 아이템 정렬 */
  justify-self: end;        /* 이 아이템만 셀 내 오른쪽 */
  align-self: start;        /* 이 아이템만 셀 내 위쪽 */
}
```

> 💡 **Grid 완벽 중앙 정렬 (최단 코드)**:
> `display: grid; place-items: center;` — 단 2줄!

## 8. 실습: Grid로 대시보드 레이아웃

```
과제 1: 반응형 카드 갤러리
  - 카드 8개
  - auto-fit + minmax로 반응형 (최소 250px)
  - gap: 16px
  - 미디어 쿼리 없이 자동 줄바꿈

과제 2: 대시보드 레이아웃 (grid-template-areas)
  - 헤더: 상단 전체 (60px)
  - 사이드바: 왼쪽 (240px 고정)
  - 메인: 중앙 (가변)
  - 위젯: 오른쪽 (300px 고정)
  - 푸터: 하단 전체 (40px)
  - 최소 높이 100vh
```

## 면접 대비

### 🔴 필수 질문

**Q1. Flexbox와 Grid의 차이점은? 언제 각각 쓰나요?**

> Flexbox는 1차원(한 방향) 레이아웃에 적합하다. 네비게이션, 카드 한 줄, 입력 그룹 등 요소를 가로 또는 세로로 배치할 때 사용한다. Grid는 2차원(행+열) 레이아웃에 적합하다. 대시보드, 갤러리, 전체 페이지 구조처럼 행과 열을 동시에 제어해야 할 때 사용한다. 실무에서는 전체 구조를 Grid로 잡고, 내부 요소 정렬은 Flex로 하는 조합이 흔하다.

**Q2. `fr` 단위란?**

> fraction(분수)의 약자로, 그리드 컨테이너에서 사용 가능한 남은 공간을 비율로 분배하는 단위다. `1fr 2fr`이면 남은 공간을 1:2로 나눈다. 고정 크기(px)와 혼합하면 `200px 1fr`처럼 "200px 고정 + 나머지 전부" 패턴이 가능하다.

### 🟡 개념 질문

**Q3. `auto-fill`과 `auto-fit`의 차이?**

> `auto-fill`은 아이템 수가 적어도 빈 열 공간을 유지한다. `auto-fit`은 빈 열을 0으로 축소하여 아이템들이 공간을 꽉 채운다. 반응형 카드 그리드에서는 `auto-fit`이 더 직관적인 결과를 준다.
