## 1. 커스텀 훅이란? 🔴

> 💡 **비유: 레시피 카드**
> 같은 요리를 매번 처음부터 만들지 말고, 레시피(커스텀 훅)를 만들어두면
> 어떤 주방(컴포넌트)에서든 같은 요리를 빠르게 만들 수 있다.

```
커스텀 훅 = "use"로 시작하는 함수
           내부에서 다른 Hooks(useState, useEffect 등)를 사용
           로직을 재사용 가능한 단위로 추출

장점:
  1. 로직 재사용: 같은 패턴을 여러 컴포넌트에서
  2. 관심사 분리: UI 로직과 비즈니스 로직을 분리
  3. 테스트 용이: 훅만 따로 테스트 가능
  4. 가독성: 컴포넌트가 깔끔해짐
```

### "언제" 커스텀 훅을 만들어야 하나?

```
✅ 만들어야 할 때:
  - 같은 useState + useEffect 패턴이 2곳 이상 반복
  - 컴포넌트에 비즈니스 로직이 너무 많아서 UI가 안 보임
  - "이 로직을 다른 컴포넌트에서도 쓸 수 있겠다"

❌ 만들 필요 없을 때:
  - 한 곳에서만 사용하는 단순 로직 (과도한 추상화!)
  - 로직이 3줄 이하로 간단할 때
```

## 2. 훅의 규칙 (Rules of Hooks) 🔴

```
규칙 1: 최상위에서만 호출
  ✅ 컴포넌트/훅의 함수 본문 최상위
  ❌ 조건문, 반복문, 중첩 함수 안에서 호출 금지!

규칙 2: React 함수에서만 호출
  ✅ 함수형 컴포넌트, 커스텀 훅 안에서
  ❌ 일반 JS 함수, 클래스에서 호출 금지!

규칙 3: "use"로 시작하는 이름
  ✅ useToggle, useFetch, useAuth
  ❌ toggle, fetchData, getAuth
```

```tsx
// ❌ 규칙 위반: 조건문 안에서 훅 호출
function Bad({ isLoggedIn }: { isLoggedIn: boolean }) {
  if (isLoggedIn) {
    const [user, setUser] = useState(null);  // 💀 조건에 따라 훅 호출 순서가 달라짐!
  }
}

// ✅ 올바른 방법: 항상 호출하고 조건은 내부에서
function Good({ isLoggedIn }: { isLoggedIn: boolean }) {
  const [user, setUser] = useState(null);
  // 조건 처리는 훅 호출 이후에!
  if (!isLoggedIn) return <LoginForm />;
  return <Profile user={user} />;
}
```

> 💡 **왜 이 규칙이 필요한가?** React는 훅을 **호출 순서**로 식별한다.
> 조건문에서 훅을 건너뛰면 순서가 꼬여서 State가 뒤섞인다!

## 3. 실전 커스텀 훅 5가지 ⭐🔴

### 훅 1: useToggle — 가장 간단한 커스텀 훅

```tsx
function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);

  const toggle = useCallback(() => setValue(v => !v), []);
  const setTrue = useCallback(() => setValue(true), []);
  const setFalse = useCallback(() => setValue(false), []);

  return { value, toggle, setTrue, setFalse } as const;
}

// 사용
function Modal() {
  const { value: isOpen, toggle, setFalse: close } = useToggle();

  return (
    <>
      <button onClick={toggle}>모달 {isOpen ? "닫기" : "열기"}</button>
      {isOpen && (
        <div className="modal">
          <p>모달 내용</p>
          <button onClick={close}>닫기</button>
        </div>
      )}
    </>
  );
}
```

### 훅 2: useDebounce — 검색 최적화 ⭐ 매우 자주 사용!

```tsx
function useDebounce<T>(value: T, delay: number = 300): T {
  const [debouncedValue, setDebouncedValue] = useState(value);

  useEffect(() => {
    const timer = setTimeout(() => {
      setDebouncedValue(value);
    }, delay);

    return () => clearTimeout(timer);  // 타이머 정리!
  }, [value, delay]);

  return debouncedValue;
}

// 사용: 검색 API 호출 최적화
function SearchPage() {
  const [query, setQuery] = useState("");
  const debouncedQuery = useDebounce(query, 500);  // 500ms 대기 후 값 갱신

  useEffect(() => {
    if (!debouncedQuery) return;
    // 사용자가 타이핑을 멈춘 후 500ms 뒤에만 API 호출!
    // "r" → "re" → "rea" → "reac" → "react" 중 "react"만 호출됨!
    fetchSearchResults(debouncedQuery);
  }, [debouncedQuery]);

  return <input value={query} onChange={(e) => setQuery(e.target.value)} />;
}
```

### 훅 3: useLocalStorage — 브라우저 저장소 연동

```tsx
function useLocalStorage<T>(key: string, initialValue: T) {
  const [storedValue, setStoredValue] = useState<T>(() => {
    if (typeof window === "undefined") return initialValue;
    try {
      const item = window.localStorage.getItem(key);
      return item ? (JSON.parse(item) as T) : initialValue;
    } catch {
      return initialValue;
    }
  });

  const setValue = useCallback((value: T | ((prev: T) => T)) => {
    setStoredValue(prev => {
      const newValue = value instanceof Function ? value(prev) : value;
      window.localStorage.setItem(key, JSON.stringify(newValue));
      return newValue;
    });
  }, [key]);

  return [storedValue, setValue] as const;
}

// 사용: 다크 모드 설정 유지
function App() {
  const [theme, setTheme] = useLocalStorage<"light" | "dark">("theme", "light");

  return (
    <div className={theme === "dark" ? "bg-gray-900 text-white" : "bg-white"}>
      <button onClick={() => setTheme(prev => prev === "light" ? "dark" : "light")}>
        테마 변경
      </button>
    </div>
  );
}
```

### 훅 4: useMediaQuery — 반응형 감지

```tsx
function useMediaQuery(query: string): boolean {
  const [matches, setMatches] = useState(() => {
    if (typeof window === "undefined") return false;
    return window.matchMedia(query).matches;
  });

  useEffect(() => {
    const mediaQuery = window.matchMedia(query);
    const handler = (e: MediaQueryListEvent) => setMatches(e.matches);

    mediaQuery.addEventListener("change", handler);
    return () => mediaQuery.removeEventListener("change", handler);
  }, [query]);

  return matches;
}

// 사용
function Layout() {
  const isMobile = useMediaQuery("(max-width: 768px)");
  const isDark = useMediaQuery("(prefers-color-scheme: dark)");

  return (
    <div>
      {isMobile ? <MobileNav /> : <DesktopNav />}
      {isDark && <p>다크 모드 활성화됨</p>}
    </div>
  );
}
```

### 훅 5: useFetch — 데이터 페칭 (46번 복습)

```tsx
function useFetch<T>(url: string | null) {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    if (!url) return;  // url이 null이면 요청 안 함

    const controller = new AbortController();
    let cancelled = false;

    const fetchData = async () => {
      setLoading(true);
      setError(null);
      try {
        const res = await fetch(url, { signal: controller.signal });
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const json = await res.json();
        if (!cancelled) setData(json);
      } catch (err) {
        if (!cancelled && err instanceof Error && err.name !== "AbortError") {
          setError(err.message);
        }
      } finally {
        if (!cancelled) setLoading(false);
      }
    };

    fetchData();
    return () => { cancelled = true; controller.abort(); };
  }, [url]);

  return { data, loading, error };
}

// 사용
const { data: users, loading, error } = useFetch<User[]>("/api/users");
```

## 4. 커스텀 훅 설계 원칙 🟡

```
1. 단일 책임: 하나의 훅은 하나의 관심사만
   ✅ useAuth — 인증만
   ❌ useAuthAndThemeAndNotification — 너무 많은 책임!

2. 반환값 설계:
   객체 반환: 이름이 있어서 가독성 좋음 (대부분 이 방식)
     const { value, toggle } = useToggle();

   배열 반환: 이름 변경이 자유로움 (useState 패턴)
     const [count, setCount] = useCounter(0);
     const [name, setName] = useCounter(0);  // 같은 훅인데 다른 이름으로!

3. as const: 배열 반환 시 필수!
   return [value, setter] as const;  // 튜플 타입 보장

4. 인자는 최소한으로:
   useDebounce(value, delay) — 간결!
   useDebounce({ value, delay, immediate, maxWait }) — 너무 복잡!
```

## 5. 실습: 커스텀 훅 3개 직접 만들기

```
과제 1: useCounter(initialValue)
  - count, increment(), decrement(), reset() 반환
  - min/max 범위 옵션 지원

과제 2: useClickOutside(ref, callback)
  - ref 바깥을 클릭하면 callback 실행 (모달 닫기에 활용)
  - 클린업 포함

과제 3: useAsync(asyncFn)
  - 비동기 함수를 받아서 { execute, data, loading, error } 반환
  - execute()를 호출해야 실행됨 (자동 실행 아님)
```

```tsx
// 과제 1: useCounter
function useCounter(initialValue = 0, { min = -Infinity, max = Infinity } = {}) {
  const [count, setCount] = useState(initialValue);
  const increment = useCallback(() => setCount(prev => Math.min(prev + 1, max)), [max]);
  const decrement = useCallback(() => setCount(prev => Math.max(prev - 1, min)), [min]);
  const reset = useCallback(() => setCount(initialValue), [initialValue]);
  return { count, increment, decrement, reset };
}

// 과제 2: useClickOutside
function useClickOutside(ref: React.RefObject<HTMLElement>, callback: () => void) {
  useEffect(() => {
    const handler = (e: MouseEvent) => {
      if (ref.current && !ref.current.contains(e.target as Node)) {
        callback();
      }
    };
    document.addEventListener("mousedown", handler);
    return () => document.removeEventListener("mousedown", handler);
  }, [ref, callback]);
}

// 사용: 모달 바깥 클릭 시 닫기
function Modal({ onClose }: { onClose: () => void }) {
  const modalRef = useRef<HTMLDivElement>(null);
  useClickOutside(modalRef, onClose);
  return <div ref={modalRef}>모달 내용</div>;
}

// 과제 3: useAsync
function useAsync<T>(asyncFn: () => Promise<T>) {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const execute = useCallback(async () => {
    setLoading(true);
    setError(null);
    try {
      const result = await asyncFn();
      setData(result);
    } catch (err) {
      setError(err instanceof Error ? err.message : "에러 발생");
    } finally {
      setLoading(false);
    }
  }, [asyncFn]);

  return { execute, data, loading, error };
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. 커스텀 훅은 왜 만드나요?**

> 여러 컴포넌트에서 반복되는 상태 관리 로직(useState + useEffect 조합)을 재사용 가능한 함수로 추출하기 위해 만든다. UI 로직과 비즈니스 로직을 분리하여 컴포넌트를 깔끔하게 유지하고, 로직을 독립적으로 테스트할 수 있게 한다. `use` 접두사를 붙여야 React가 훅의 규칙을 적용한다.

**Q2. 훅의 규칙(Rules of Hooks)은?**

> 첫째, 최상위에서만 호출해야 한다. 조건문, 반복문, 중첩 함수 안에서 호출하면 안 된다. React가 훅을 호출 순서로 식별하기 때문에 순서가 바뀌면 State가 꼬인다. 둘째, React 함수(컴포넌트 또는 커스텀 훅) 안에서만 호출해야 한다.

**Q3. useDebounce를 설명하고 구현해주세요.**

> 값이 변경된 후 일정 시간(delay) 동안 추가 변경이 없을 때만 최종 값을 반환하는 훅이다. 검색창에서 타이핑할 때마다 API를 호출하는 대신, 타이핑이 멈춘 후 한 번만 호출하는 데 사용한다. 내부적으로 `useEffect` + `setTimeout`을 사용하며, 값이 바뀔 때마다 이전 타이머를 `clearTimeout`으로 정리한다.
