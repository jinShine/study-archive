## 1. Portal이란? 🔴

> 💡 **비유: 순간이동 포탈**
> 컴포넌트가 React 트리에서는 자식이지만, **실제 DOM에서는 완전히 다른 곳에** 렌더링된다.
> 모달이 부모의 `overflow: hidden`이나 `z-index`에 갇히지 않고 화면 최상단에 뜰 수 있다!

```
React 트리 (논리적):           실제 DOM (물리적):
App                            <div id="root">
 └─ Page                         <div>Page...</div>
     └─ Button                 </div>
     └─ Modal (Portal!) ───▶   <div id="modal-root"> ← 여기에 렌더링!
         └─ 내용                   <div class="modal">내용</div>
                                </div>
```

### 왜 Portal이 필요한가?

```
❌ Portal 없이 모달:
  <div style="overflow: hidden">  ← 부모의 overflow에 갇힘!
    <div class="modal" style="position: fixed"> ← 짤림!
  </div>

✅ Portal로 모달:
  모달이 DOM 트리 최상위에 렌더링 → overflow, z-index 영향 없음!
```

## 2. createPortal 기본 사용법 🔴

```tsx
import { createPortal } from "react-dom";

function Modal({ isOpen, onClose, children }: {
  isOpen: boolean;
  onClose: () => void;
  children: React.ReactNode;
}) {
  if (!isOpen) return null;

  return createPortal(
    // 렌더링할 JSX
    <div className="fixed inset-0 z-50 flex items-center justify-center">
      {/* 오버레이 (배경 어둡게) */}
      <div
        className="fixed inset-0 bg-black/50 backdrop-blur-sm"
        onClick={onClose}  // 배경 클릭 시 닫기
      />
      {/* 모달 본체 */}
      <div className="relative bg-white rounded-xl p-6 shadow-2xl max-w-md w-full mx-4 z-10">
        <button
          onClick={onClose}
          className="absolute top-3 right-3 text-gray-400 hover:text-gray-600"
        >
          ✕
        </button>
        {children}
      </div>
    </div>,
    // Portal 대상 DOM 노드
    document.getElementById("modal-root")!  // HTML에 <div id="modal-root"> 필요!
  );
}

// 사용
function Page() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <div>
      <button onClick={() => setIsOpen(true)}>모달 열기</button>
      <Modal isOpen={isOpen} onClose={() => setIsOpen(false)}>
        <h2 className="text-xl font-bold mb-4">확인</h2>
        <p>정말 삭제하시겠습니까?</p>
      </Modal>
    </div>
  );
}
```

## 3. ESC 키 + 포커스 트랩 (접근성) 🟡

```tsx
function Modal({ isOpen, onClose, children }: ModalProps) {
  const modalRef = useRef<HTMLDivElement>(null);

  // ESC 키로 닫기
  useEffect(() => {
    if (!isOpen) return;

    const handleEsc = (e: KeyboardEvent) => {
      if (e.key === "Escape") onClose();
    };
    document.addEventListener("keydown", handleEsc);
    return () => document.removeEventListener("keydown", handleEsc);
  }, [isOpen, onClose]);

  // 모달 열릴 때 포커스 이동
  useEffect(() => {
    if (isOpen) {
      modalRef.current?.focus();
      document.body.style.overflow = "hidden";  // 배경 스크롤 방지!
    }
    return () => {
      document.body.style.overflow = "";
    };
  }, [isOpen]);

  if (!isOpen) return null;

  return createPortal(
    <div className="fixed inset-0 z-50" role="dialog" aria-modal="true">
      <div className="fixed inset-0 bg-black/50" onClick={onClose} />
      <div ref={modalRef} tabIndex={-1} className="relative ...">
        {children}
      </div>
    </div>,
    document.getElementById("modal-root")!
  );
}
```

## 4. 토스트 시스템 (Portal 활용) 🟡

```tsx
// 간단한 토스트 훅
type Toast = { id: number; message: string; type: "success" | "error" | "info" };

const useToastStore = create<{
  toasts: Toast[];
  addToast: (message: string, type?: Toast["type"]) => void;
  removeToast: (id: number) => void;
}>((set) => ({
  toasts: [],
  addToast: (message, type = "info") => {
    const id = Date.now();
    set(state => ({ toasts: [...state.toasts, { id, message, type }] }));
    setTimeout(() => {
      set(state => ({ toasts: state.toasts.filter(t => t.id !== id) }));
    }, 3000);  // 3초 후 자동 삭제
  },
  removeToast: (id) => set(state => ({
    toasts: state.toasts.filter(t => t.id !== id),
  })),
}));

// 토스트 컨테이너 (Portal로 최상위에 렌더링)
function ToastContainer() {
  const toasts = useToastStore(state => state.toasts);

  return createPortal(
    <div className="fixed top-4 right-4 z-[9999] space-y-2">
      {toasts.map(toast => (
        <div key={toast.id} className={`px-4 py-3 rounded-lg shadow-lg text-white
          ${toast.type === "success" ? "bg-green-500" :
            toast.type === "error" ? "bg-red-500" : "bg-blue-500"}`}
        >
          {toast.message}
        </div>
      ))}
    </div>,
    document.getElementById("toast-root")!
  );
}

// 사용
function App() {
  return (
    <>
      <MyApp />
      <ToastContainer />
    </>
  );
}

// 어디서든 토스트 발생!
function SaveButton() {
  const addToast = useToastStore(state => state.addToast);

  const handleSave = async () => {
    try {
      await saveData();
      addToast("저장 완료!", "success");
    } catch {
      addToast("저장 실패!", "error");
    }
  };

  return <button onClick={handleSave}>저장</button>;
}
```

> 💡 **실무에서는** `react-hot-toast`, `sonner` 같은 토스트 라이브러리를 사용한다.
> 하지만 원리(Portal + Zustand)를 이해해야 커스터마이징이 가능!

## 5. Portal의 이벤트 버블링 🟡

> ⚠️ **핵심**: Portal은 DOM에서는 분리되어 있지만, **React 이벤트 버블링은 React 트리를 따른다!**

```tsx
function Parent() {
  return (
    // 이 onClick은 Portal 안의 버튼 클릭도 잡는다!
    <div onClick={() => console.log("Parent 클릭!")}>
      <Modal>
        <button>모달 안 버튼</button>
        {/* 클릭하면 → "Parent 클릭!" 출력됨! (React 트리 기준 버블링) */}
      </Modal>
    </div>
  );
}
// DOM 트리에서는 분리되어 있지만, React 이벤트는 논리적 트리를 따름!
```

### 🔴 필수 질문

**Q1. React Portal이란?**

> 컴포넌트를 React 트리의 논리적 위치는 유지하면서, 실제 DOM에서는 다른 위치(보통 body 직하위)에 렌더링하는 기능이다. `createPortal(JSX, DOMNode)`로 사용하며, 모달, 토스트, 툴팁처럼 부모의 `overflow: hidden`이나 `z-index`에 영향받지 않아야 하는 UI에 필수적이다. React 이벤트 버블링은 DOM이 아닌 React 트리를 기준으로 동작한다.
