## 1. shadcn/ui가 뭔지? 🔴

> 💡 **핵심**: shadcn/ui는 "라이브러리"가 아니라 **복사-붙여넣기 가능한 컴포넌트 모음**이다!

```
일반 UI 라이브러리 (MUI, Ant Design):
  npm install 하면 → node_modules에 들어감
  → 커스터마이징이 어려움 (덮어쓰기 전쟁!)
  → 라이브러리 업데이트에 종속

shadcn/ui:
  npx shadcn@latest add button → 내 프로젝트에 Button.tsx 파일이 복사됨!
  → 코드가 내 것이므로 마음대로 수정 가능!
  → node_modules에 없음! 의존성 아님!
  → Radix UI(접근성) + Tailwind CSS(스타일)로 구성
```

### 기술 스택

```
shadcn/ui = Radix UI (Headless) + Tailwind CSS (스타일) + cn() (유틸)

Radix UI: 접근성(a11y), 키보드 네비게이션, 포커스 트랩 등을 담당
          → UI는 없고 로직만! (Headless)

Tailwind CSS: 시각적 스타일 담당
              → 클래스 수정만으로 디자인 변경!

cn() = clsx + tailwind-merge: 조건부 클래스 조합
```

## 2. 설치 및 세팅 🔴

```bash
# Next.js 프로젝트에서
npx shadcn@latest init

# 물어보는 것들:
# Style: New York (추천) 또는 Default
# Base color: Slate / Gray / Zinc / Neutral / Stone
# CSS variables: Yes (테마 변경 가능!)

# 결과: components/ui/ 폴더와 lib/utils.ts가 생성됨
```

### 생성되는 파일 구조

```
src/
├── components/
│   └── ui/           ← shadcn 컴포넌트들이 여기에!
│       ├── button.tsx
│       ├── input.tsx
│       ├── dialog.tsx
│       └── ...
├── lib/
│   └── utils.ts      ← cn() 함수
└── app/
    └── globals.css    ← CSS 변수 (테마)
```

### cn() 유틸 (가장 중요한 도우미 함수!)

```ts
// lib/utils.ts
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}

// clsx: 조건부 클래스 조합
// twMerge: Tailwind 클래스 충돌 해결 (px-4 + px-8 → px-8만!)
```

### Dialog (모달) 사용 예시

```tsx
import {
  Dialog, DialogContent, DialogDescription,
  DialogHeader, DialogTitle, DialogTrigger, DialogFooter,
} from "@/components/ui/dialog";
import { Button } from "@/components/ui/button";

function DeleteConfirm({ onDelete }: { onDelete: () => void }) {
  return (
    <Dialog>
      <DialogTrigger asChild>
        <Button variant="destructive">삭제</Button>
      </DialogTrigger>
      <DialogContent>
        <DialogHeader>
          <DialogTitle>정말 삭제하시겠습니까?</DialogTitle>
          <DialogDescription>
            이 작업은 되돌릴 수 없습니다.
          </DialogDescription>
        </DialogHeader>
        <DialogFooter>
          <Button variant="outline">취소</Button>
          <Button variant="destructive" onClick={onDelete}>삭제</Button>
        </DialogFooter>
      </DialogContent>
    </Dialog>
  );
}
// → 접근성(ESC 닫기, 포커스 트랩, aria 속성) 전부 자동!
```

### Form (RHF + Zod 통합!) 사용 예시

```tsx
import { useForm } from "react-hook-form";
import { zodResolver } from "@hookform/resolvers/zod";
import { z } from "zod";
import { Form, FormControl, FormField, FormItem, FormLabel, FormMessage } from "@/components/ui/form";
import { Input } from "@/components/ui/input";
import { Button } from "@/components/ui/button";

const schema = z.object({
  name: z.string().min(2, "이름은 2자 이상"),
  email: z.string().email("올바른 이메일을 입력하세요"),
});

function UserForm() {
  const form = useForm({
    resolver: zodResolver(schema),
    defaultValues: { name: "", email: "" },
  });

  return (
    <Form {...form}>
      <form onSubmit={form.handleSubmit(console.log)} className="space-y-4">
        <FormField
          control={form.control}
          name="name"
          render={({ field }) => (
            <FormItem>
              <FormLabel>이름</FormLabel>
              <FormControl>
                <Input {...field} placeholder="이름 입력" />
              </FormControl>
              <FormMessage />  {/* Zod 에러 메시지 자동 표시! */}
            </FormItem>
          )}
        />
        <FormField
          control={form.control}
          name="email"
          render={({ field }) => (
            <FormItem>
              <FormLabel>이메일</FormLabel>
              <FormControl>
                <Input {...field} type="email" placeholder="이메일 입력" />
              </FormControl>
              <FormMessage />
            </FormItem>
          )}
        />
        <Button type="submit">제출</Button>
      </form>
    </Form>
  );
}
```

## 4. 커스터마이징 — 내 코드니까 자유롭게! 🟡

```tsx
// components/ui/button.tsx를 직접 수정!
// 예: 기본 border-radius를 더 둥글게
const buttonVariants = cva(
  "... rounded-xl ...",  // rounded-md → rounded-xl로 변경!
  {
    variants: {
      variant: {
        default: "bg-primary text-primary-foreground hover:bg-primary/90",
        // 커스텀 variant 추가!
        brand: "bg-gradient-to-r from-purple-500 to-pink-500 text-white hover:opacity-90",
      },
    },
  }
);

// 사용
<Button variant="brand">브랜드 버튼</Button>
```

## 6. 실습

```
과제: shadcn/ui로 관리자 대시보드 UI 구축

1. shadcn init + 필요한 컴포넌트 추가
   (button, card, input, dialog, table, badge, toast)
2. 상단: 네비게이션 (로고 + 메뉴 + 사용자 아바타)
3. 카드 3개: 총 사용자 / 이번 달 매출 / 신규 가입
4. 테이블: 최근 주문 목록 (이름, 상품, 가격, 상태 Badge)
5. "주문 상세" 버튼 클릭 → Dialog로 상세 정보
6. 다크 모드 토글 (CSS 변수 활용)
7. 커스텀: Button에 brand variant 추가
```

### 🟡 이해 질문

**Q1. shadcn/ui를 선택한 이유?**

> shadcn/ui는 npm 패키지가 아니라 프로젝트에 직접 복사되는 컴포넌트 모음이다. 코드가 내 것이므로 자유롭게 커스터마이징 가능하고, Radix UI로 접근성이 보장되며, Tailwind CSS로 스타일을 제어한다. MUI나 Ant Design처럼 라이브러리 버전에 종속되지 않아 장기 유지보수가 용이하다. 2026년 React/Next.js 생태계에서 가장 높은 점유율을 가진다.
