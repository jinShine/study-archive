## 1. 코드 스플리팅이란? 🔴

> 💡 **비유: 도서관에서 책 빌리기**
> ❌ 도서관의 모든 책을 한 번에 집에 가져오기 (번들 전체 다운로드)
> ✅ 지금 읽을 책만 빌려오기 (필요한 코드만 로드!)

```
❌ 코드 스플리팅 없이:
  사용자가 첫 페이지 접속 → 앱의 모든 JS를 다운로드 (2MB)
  → 관리자 페이지 코드, 차트 라이브러리, 설정 페이지... 당장 안 쓰는 것까지!
  → 초기 로딩 5초 😱

✅ 코드 스플리팅:
  사용자가 첫 페이지 접속 → 메인 페이지 JS만 다운로드 (200KB)
  관리자 페이지 클릭 → 그때 관리자 JS 다운로드 (추가 100KB)
  → 초기 로딩 1초! ⚡
```

## 2. React.lazy + Suspense 🔴

```tsx
import { lazy, Suspense } from "react";

// 동적 import: 이 컴포넌트가 필요할 때 비로소 JS 파일을 다운로드!
const AdminDashboard = lazy(() => import("./pages/AdminDashboard"));
const ChartComponent = lazy(() => import("./components/Chart"));
const SettingsPage = lazy(() => import("./pages/Settings"));

function App() {
  const [currentPage, setCurrentPage] = useState("home");

  return (
    <div>
      <nav>
        <button onClick={() => setCurrentPage("home")}>홈</button>
        <button onClick={() => setCurrentPage("admin")}>관리자</button>
        <button onClick={() => setCurrentPage("settings")}>설정</button>
      </nav>

      {/* Suspense: lazy 컴포넌트가 로딩되는 동안 fallback을 보여줌 */}
      <Suspense fallback={<div className="p-10 text-center">페이지 로딩 중...</div>}>
        {currentPage === "home" && <HomePage />}
        {currentPage === "admin" && <AdminDashboard />}  {/* 클릭 시 로드! */}
        {currentPage === "settings" && <SettingsPage />}  {/* 클릭 시 로드! */}
      </Suspense>
    </div>
  );
}
```

### 빌드 결과 비교

```
❌ 스플리팅 없이:
  dist/assets/index-abc123.js    (2.1 MB) ← 전부 하나!

✅ 스플리팅 후:
  dist/assets/index-abc123.js    (200 KB) ← 메인 번들
  dist/assets/AdminDashboard-def456.js  (300 KB) ← 관리자 (필요할 때)
  dist/assets/Settings-ghi789.js (100 KB) ← 설정 (필요할 때)
  dist/assets/Chart-jkl012.js    (400 KB) ← 차트 (필요할 때)
```

## 3. Next.js에서의 코드 스플리팅 🔴

> Next.js는 **페이지(라우트) 단위로 자동 코드 스플리팅**한다!
> `/about` 페이지의 코드는 `/about`에 접속할 때만 로드됨.

```tsx
// Next.js에서 클라이언트 전용 컴포넌트 동적 로딩
import dynamic from "next/dynamic";

// SSR 제외 (브라우저 전용 라이브러리: 차트, 에디터, 지도 등)
const Chart = dynamic(() => import("@/components/Chart"), {
  ssr: false,  // 서버에서는 렌더링하지 않음!
  loading: () => <div className="h-64 bg-gray-100 animate-pulse rounded" />,
});

// SSR 포함 (기본)
const HeavyTable = dynamic(() => import("@/components/HeavyTable"), {
  loading: () => <p>테이블 로딩 중...</p>,
});

function Dashboard() {
  return (
    <div>
      <Chart data={salesData} />      {/* 클라이언트에서만 로드 */}
      <HeavyTable data={orderData} /> {/* 필요할 때 로드 */}
    </div>
  );
}
```

### React.lazy vs next/dynamic

```
React.lazy:
  - 순수 React 프로젝트 (Vite 등)
  - SSR 미지원
  - Suspense로 감싸야 함

next/dynamic:
  - Next.js 프로젝트
  - ssr: false 옵션으로 클라이언트 전용 가능
  - loading 옵션으로 로딩 UI 내장
  - 내부적으로 React.lazy + Suspense를 감싸준 것
```

## 4. Suspense의 확장 — 데이터 페칭 🟡

```tsx
// React 19 + TanStack Query의 Suspense 모드
import { useSuspenseQuery } from "@tanstack/react-query";

function UserProfile({ userId }: { userId: number }) {
  // useSuspenseQuery: 로딩 중이면 컴포넌트를 "서스펜드"시킴!
  // → isLoading 체크 불필요! Suspense의 fallback이 자동 표시!
  const { data: user } = useSuspenseQuery({
    queryKey: ["user", userId],
    queryFn: () => fetchUser(userId),
  });

  // 여기 도달했다 = 데이터가 확실히 있다! (user는 undefined가 아님!)
  return <h1>{user.name}</h1>;
}

// 부모에서 Suspense로 감싸기
function Page() {
  return (
    <Suspense fallback={<ProfileSkeleton />}>
      <UserProfile userId={1} />
    </Suspense>
  );
}

// Suspense + ErrorBoundary 조합 (완벽한 패턴!)
function Page() {
  return (
    <ErrorBoundary FallbackComponent={ErrorFallback}>
      <Suspense fallback={<Skeleton />}>
        <UserProfile userId={1} />
      </Suspense>
    </ErrorBoundary>
  );
}
// → 로딩: Skeleton / 에러: ErrorFallback / 성공: UserProfile
```

## 5. 프리로드 전략 — 더 빠르게! 🟡

```tsx
// 마우스 hover 시 미리 로드!
const AdminPage = lazy(() => import("./pages/AdminPage"));

function NavLink() {
  const preloadAdmin = () => {
    // 마우스를 올리면 이미 다운로드 시작!
    import("./pages/AdminPage");
  };

  return (
    <button
      onMouseEnter={preloadAdmin}  // hover 시 프리로드
      onClick={() => setPage("admin")}
    >
      관리자
    </button>
  );
}
```

> 💡 **Next.js**의 `<Link>` 컴포넌트는 뷰포트에 들어오면 자동으로 프리페칭!
> 수동 프리로드가 필요 없다.

## 면접 대비

### 🔴 필수 질문

**Q1. 코드 스플리팅이란?**

> 앱의 전체 JavaScript 번들을 여러 개의 작은 청크(chunk)로 나누어, 사용자가 필요한 코드만 필요한 시점에 로드하는 기법이다. `React.lazy` + `Suspense`(또는 Next.js의 `dynamic`)로 구현하며, 초기 번들 크기를 줄여 첫 로딩 속도를 크게 개선한다. Next.js는 페이지 단위로 자동 코드 스플리팅을 수행한다.

**Q2. Suspense의 역할?**

> 자식 컴포넌트가 아직 준비되지 않은 동안(로딩 중) 대체 UI(fallback)를 보여주는 컴포넌트다. `React.lazy`의 동적 로딩, TanStack Query의 `useSuspenseQuery`, Next.js의 `loading.tsx` 등과 함께 사용한다. ErrorBoundary와 조합하면 로딩/에러/성공 3가지 상태를 선언적으로 처리할 수 있다.
