## 1. 에러 바운더리란? 🔴

> 💡 **비유: 건물의 방화문**
> 한 방에서 불이 나면 방화문이 닫혀서 **다른 방으로 번지지 않는다**.
> 에러 바운더리는 컴포넌트 트리의 "방화문"이다.
> 한 컴포넌트에서 에러가 나면 **앱 전체가 죽는 대신**, 해당 영역만 에러 UI를 보여준다.

```
❌ 에러 바운더리 없이:
  하위 컴포넌트 에러 → 앱 전체 하얀 화면(White Screen of Death)!

✅ 에러 바운더리 있으면:
  하위 컴포넌트 에러 → 해당 영역만 "문제가 발생했습니다" 표시
                     나머지 앱은 정상 작동!
```

### 에러 바운더리가 잡을 수 있는 것 / 없는 것

```
✅ 잡을 수 있음:
  - 렌더링 중 발생하는 에러 (return JSX에서)
  - 생명주기 메서드 에러
  - 자식 컴포넌트의 에러

❌ 잡을 수 없음:
  - 이벤트 핸들러 에러 (onClick 등) → try-catch로 직접 처리
  - 비동기 코드 에러 (setTimeout, fetch) → try-catch
  - 서버 사이드 렌더링 에러
  - 에러 바운더리 자체의 에러
```

## 2. react-error-boundary 라이브러리 (실무 표준) 🔴

> 에러 바운더리는 클래스 컴포넌트로만 구현 가능하지만,
> `react-error-boundary` 라이브러리가 함수형으로 감싸준다!

```bash
npm install react-error-boundary
```

### 기본 사용법

```tsx
import { ErrorBoundary } from "react-error-boundary";

// 에러 시 보여줄 Fallback 컴포넌트
function ErrorFallback({ error, resetErrorBoundary }: {
  error: Error;
  resetErrorBoundary: () => void;
}) {
  return (
    <div className="p-6 bg-red-50 border border-red-200 rounded-lg text-center">
      <h2 className="text-red-600 font-bold text-lg mb-2">문제가 발생했습니다</h2>
      <p className="text-red-500 text-sm mb-4">{error.message}</p>
      <button
        onClick={resetErrorBoundary}
        className="px-4 py-2 bg-red-600 text-white rounded hover:bg-red-700"
      >
        다시 시도
      </button>
    </div>
  );
}

// App에서 사용
function App() {
  return (
    <div>
      <header>헤더 (에러 영향 안 받음)</header>

      <ErrorBoundary
        FallbackComponent={ErrorFallback}
        onError={(error, info) => {
          // 에러 로깅 (Sentry 등)
          console.error("에러 발생:", error);
          console.error("컴포넌트 스택:", info.componentStack);
        }}
        onReset={() => {
          // 리셋 시 실행할 로직 (예: 쿼리 무효화)
        }}
      >
        <main>
          <DangerousComponent />  {/* 여기서 에러 나면 이 영역만 Fallback! */}
        </main>
      </ErrorBoundary>

      <footer>푸터 (에러 영향 안 받음)</footer>
    </div>
  );
}
```

## 3. 에러 격리 전략 — 어디에 에러 바운더리를 놓을까? 🔴

```
전략 1: 페이지 단위 (기본)
  → 한 페이지에서 에러 → 해당 페이지만 에러 UI
  → Next.js의 error.tsx가 이 역할!

전략 2: 위젯/섹션 단위 (세밀한 격리)
  → 대시보드의 차트/피드/알림을 각각 에러 바운더리로
  → 차트에서 에러 나도 피드는 정상!

전략 3: 전역 (최후 방어선)
  → App 전체를 감싸서 앱이 완전히 죽는 것을 방지
  → Next.js의 global-error.tsx
```

```tsx
// 실전: 대시보드 위젯별 격리
function Dashboard() {
  return (
    <div className="grid grid-cols-3 gap-4">
      <ErrorBoundary FallbackComponent={WidgetError}>
        <SalesChart />  {/* 차트 에러 → 차트만 에러 UI */}
      </ErrorBoundary>

      <ErrorBoundary FallbackComponent={WidgetError}>
        <RecentOrders />  {/* 주문 에러 → 주문만 에러 UI */}
      </ErrorBoundary>

      <ErrorBoundary FallbackComponent={WidgetError}>
        <UserActivity />  {/* 활동 에러 → 활동만 에러 UI */}
      </ErrorBoundary>
    </div>
  );
}
```

## 4. resetKeys — 자동 리셋 🟡

```tsx
// userId가 바뀌면 에러 바운더리를 자동으로 리셋!
<ErrorBoundary
  FallbackComponent={ErrorFallback}
  resetKeys={[userId]}  // 이 값이 바뀌면 에러 상태 자동 초기화
>
  <UserProfile userId={userId} />
</ErrorBoundary>

// 검색어가 바뀌면 리셋
<ErrorBoundary resetKeys={[searchQuery]}>
  <SearchResults query={searchQuery} />
</ErrorBoundary>
```

## 5. 이벤트 핸들러 에러는 직접 처리 🔴

```tsx
// 에러 바운더리는 이벤트 핸들러 에러를 잡지 못한다!

// ❌ 에러 바운더리가 못 잡음
function Button() {
  const handleClick = () => {
    throw new Error("클릭 에러!");  // 에러 바운더리에 안 잡힘!
  };
  return <button onClick={handleClick}>클릭</button>;
}

// ✅ try-catch + 상태로 직접 처리
function Button() {
  const [error, setError] = useState<string | null>(null);

  const handleClick = async () => {
    try {
      await dangerousOperation();
    } catch (err) {
      setError(err instanceof Error ? err.message : "에러 발생");
    }
  };

  if (error) return <p className="text-red-500">{error}</p>;
  return <button onClick={handleClick}>클릭</button>;
}

// ✅ 또는: useErrorBoundary 훅으로 에러 바운더리에게 넘기기!
import { useErrorBoundary } from "react-error-boundary";

function Button() {
  const { showBoundary } = useErrorBoundary();

  const handleClick = async () => {
    try {
      await dangerousOperation();
    } catch (err) {
      showBoundary(err);  // 에러 바운더리로 에러를 던짐!
    }
  };

  return <button onClick={handleClick}>클릭</button>;
}
```

### 🔴 필수 질문

**Q1. React에서 에러 바운더리란?**

> 자식 컴포넌트 트리에서 렌더링 중 발생하는 에러를 잡아서 앱 전체가 죽는 것을 방지하고, 에러 UI(Fallback)를 대신 보여주는 컴포넌트다. 이벤트 핸들러나 비동기 코드의 에러는 잡지 못하므로 try-catch로 별도 처리해야 한다. 실무에서는 `react-error-boundary` 라이브러리를 사용하며, Next.js의 `error.tsx`도 에러 바운더리 원리로 동작한다.
