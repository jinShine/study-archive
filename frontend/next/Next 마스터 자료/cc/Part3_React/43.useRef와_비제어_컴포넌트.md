## 1. useRef의 두 가지 역할 🔴

> 💡 **비유: 이름표와 메모장**
> 역할 1 (이름표): DOM 요소에 이름표를 붙여서 직접 접근 (`inputRef.current.focus()`)
> 역할 2 (메모장): 리렌더링 없이 값을 기억하는 비밀 메모장 (이전 값 추적, 타이머 ID 저장)

### 역할 1: DOM 요소 접근 🔴

```tsx
function SearchBar() {
  const inputRef = useRef<HTMLInputElement>(null);

  // 마운트 시 자동 포커스
  useEffect(() => {
    inputRef.current?.focus();
  }, []);

  // 버튼 클릭 시 포커스
  const handleClear = () => {
    inputRef.current!.value = "";
    inputRef.current?.focus();
  };

  return (
    <div>
      <input ref={inputRef} placeholder="검색어 입력..." />
      <button onClick={handleClear}>지우기</button>
    </div>
  );
}
```

```tsx
// 실전: 스크롤 이동
function ScrollToBottom() {
  const bottomRef = useRef<HTMLDivElement>(null);

  const scrollToBottom = () => {
    bottomRef.current?.scrollIntoView({ behavior: "smooth" });
  };

  return (
    <div>
      {/* ... 긴 콘텐츠 ... */}
      <div ref={bottomRef} />  {/* 스크롤 목표 지점 */}
      <button onClick={scrollToBottom}>맨 아래로</button>
    </div>
  );
}

// 실전: 비디오 제어
function VideoPlayer() {
  const videoRef = useRef<HTMLVideoElement>(null);

  return (
    <div>
      <video ref={videoRef} src="/video.mp4" />
      <button onClick={() => videoRef.current?.play()}>재생</button>
      <button onClick={() => videoRef.current?.pause()}>일시정지</button>
    </div>
  );
}
```

### 역할 2: 렌더링 없는 값 저장 🟡

> `useState`와 달리 `.current`를 바꿔도 **리렌더링이 발생하지 않는다!**

```tsx
function Timer() {
  const [seconds, setSeconds] = useState(0);
  const intervalRef = useRef<NodeJS.Timeout | null>(null);  // 타이머 ID 저장

  const start = () => {
    if (intervalRef.current) return; // 이미 실행 중이면 무시
    intervalRef.current = setInterval(() => {
      setSeconds(prev => prev + 1);
    }, 1000);
  };

  const stop = () => {
    if (intervalRef.current) {
      clearInterval(intervalRef.current);
      intervalRef.current = null;
    }
  };

  // 언마운트 시 정리
  useEffect(() => {
    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current);
    };
  }, []);

  return (
    <div>
      <p>{seconds}초</p>
      <button onClick={start}>시작</button>
      <button onClick={stop}>정지</button>
    </div>
  );
}
```

```tsx
// 이전 값 추적 (usePrevious 커스텀 훅)
function usePrevious<T>(value: T): T | undefined {
  const ref = useRef<T>();
  useEffect(() => {
    ref.current = value;  // 렌더링 후에 이전 값 저장
  });
  return ref.current;     // 이전 렌더링의 값 반환
}

// 사용
function Counter() {
  const [count, setCount] = useState(0);
  const prevCount = usePrevious(count);

  return (
    <p>현재: {count}, 이전: {prevCount ?? "없음"}</p>
  );
}
```

### useState vs useRef 비교

```
               useState              useRef
값 변경 시      리렌더링 ✅           리렌더링 ❌
읽기            state                ref.current
화면 반영       바로 반영             반영 안 됨
용도            UI에 보이는 데이터     DOM 접근, 내부 변수
```

> 💡 **원칙**: 화면에 보여야 하는 값 → `useState`
> 화면과 무관한 내부 값(타이머 ID, 이전 값, DOM) → `useRef`

## 2. 제어 컴포넌트 vs 비제어 컴포넌트 🔴

### 제어 컴포넌트 (Controlled) — React가 주도권 ⭐ 기본 패턴

```tsx
// React State가 input의 값을 완전히 통제
function ControlledForm() {
  const [name, setName] = useState("");
  const [email, setEmail] = useState("");

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log({ name, email });  // State에서 바로 읽기!
  };

  return (
    <form onSubmit={handleSubmit}>
      <input
        value={name}                     // React State가 값을 결정
        onChange={(e) => setName(e.target.value)}  // 타이핑마다 State 업데이트
      />
      <input
        value={email}
        onChange={(e) => setEmail(e.target.value)}
      />
      <button type="submit">제출</button>
    </form>
  );
}
```

### 비제어 컴포넌트 (Uncontrolled) — DOM이 주도권

```tsx
// DOM 자체가 값을 관리, React는 필요할 때만 읽기
function UncontrolledForm() {
  const nameRef = useRef<HTMLInputElement>(null);
  const emailRef = useRef<HTMLInputElement>(null);

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log({
      name: nameRef.current!.value,    // ref로 DOM에서 직접 읽기
      email: emailRef.current!.value,
    });
  };

  return (
    <form onSubmit={handleSubmit}>
      <input ref={nameRef} defaultValue="" />   {/* defaultValue: 초기값만! */}
      <input ref={emailRef} defaultValue="" />
      <button type="submit">제출</button>
    </form>
  );
}
```

### 언제 뭘 써야 하나?

```
제어 컴포넌트 ✅ (대부분의 경우):
  - 실시간 유효성 검사가 필요할 때
  - 입력에 따라 UI가 즉시 변해야 할 때 (검색 자동완성)
  - 입력값을 조작해야 할 때 (자동 포맷팅, 대문자 변환)
  - 조건부 제출 버튼 활성화
  → React가 "진실의 원천(Single Source of Truth)"

비제어 컴포넌트 ✅ (특수한 경우):
  - 파일 업로드 (<input type="file" />)
  - 서드파티 DOM 라이브러리 연동
  - 단순 폼 (제출 시에만 값 필요)
  - 성능 최적화 (대량 입력 시 리렌더링 방지)
  → React Hook Form이 내부적으로 비제어 방식! (성능 우위)
```

### React Hook Form은 비제어 방식! 🟡

```tsx
// React Hook Form = 비제어 컴포넌트 + ref 기반 = 리렌더링 최소화!
import { useForm } from "react-hook-form";

function Form() {
  const { register, handleSubmit } = useForm();

  const onSubmit = (data: any) => console.log(data);

  return (
    <form onSubmit={handleSubmit(onSubmit)}>
      {/* register가 내부적으로 ref를 연결! State 없이 DOM에서 직접 읽음 */}
      <input {...register("name")} />
      <input {...register("email")} />
      <button type="submit">제출</button>
    </form>
  );
}
```

> 💡 **면접 포인트**: "React Hook Form이 제어 컴포넌트 방식보다 빠른 이유는
> 비제어(ref) 기반이라 타이핑할 때마다 리렌더링이 발생하지 않기 때문입니다."

## 3. 실습: 자동 포커스 + 실시간 글자 수 카운터

```
과제:
1. 텍스트 입력 필드 (제어 컴포넌트)
2. 마운트 시 자동 포커스 (useRef)
3. 실시간 글자 수 표시 (State)
4. 최대 100자 제한 (100자 넘으면 빨간색 경고)
5. "지우기" 버튼 클릭 시 내용 삭제 + 다시 포커스 (useRef)
6. 렌더링 횟수 표시 (useRef로 카운트 — 리렌더링 없이!)
```

<details>
<summary>정답 보기</summary>

```tsx
function TextInput() {
  const [text, setText] = useState("");
  const inputRef = useRef<HTMLTextAreaElement>(null);
  const renderCount = useRef(0);

  renderCount.current++;  // 리렌더링 때마다 증가 (화면에는 영향 없음)

  useEffect(() => {
    inputRef.current?.focus();
  }, []);

  const handleClear = () => {
    setText("");
    inputRef.current?.focus();
  };

  const isOverLimit = text.length > 100;

  return (
    <div>
      <textarea
        ref={inputRef}
        value={text}
        onChange={(e) => setText(e.target.value)}
        rows={5}
        style={{ borderColor: isOverLimit ? "red" : "#ccc" }}
      />
      <div style={{ display: "flex", justifyContent: "space-between" }}>
        <span style={{ color: isOverLimit ? "red" : "#666" }}>
          {text.length}/100자 {isOverLimit && "(초과!)"}
        </span>
        <button onClick={handleClear}>지우기</button>
      </div>
      <p style={{ color: "#999", fontSize: 12 }}>
        렌더링 횟수: {renderCount.current}
      </p>
    </div>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. useRef와 useState의 차이?**

> `useState`는 값이 변경되면 리렌더링을 발생시키며, 화면에 보이는 데이터에 사용한다. `useRef`는 `.current`를 변경해도 리렌더링이 발생하지 않으며, DOM 요소에 접근하거나 렌더링과 무관한 값(타이머 ID, 이전 값)을 저장할 때 사용한다.

**Q2. 제어 컴포넌트와 비제어 컴포넌트의 차이?**

> 제어 컴포넌트는 React State(`value` + `onChange`)가 input의 값을 완전히 통제하여, 실시간 유효성 검사나 값 조작이 가능하다. 비제어 컴포넌트는 DOM 자체가 값을 관리하고 `ref`로 필요할 때만 읽으며, 리렌더링이 없어 성능이 좋다. React Hook Form은 비제어 방식으로 동작하여 대규모 폼에서 성능 우위가 있다.
