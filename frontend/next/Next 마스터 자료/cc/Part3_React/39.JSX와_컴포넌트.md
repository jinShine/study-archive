## 1. JSX란? — HTML처럼 보이는 JavaScript 🔴

> 💡 **비유: 스무디 믹서**
> HTML(구조)과 JavaScript(로직)을 하나의 파일에서 자연스럽게 섞는 문법.
> 실제로는 `React.createElement()` 함수 호출로 변환된다.

```tsx
// JSX 문법 (개발자가 작성)
const element = <h1 className="title">안녕, {name}!</h1>;

// 실제로는 이렇게 변환됨 (React 내부)
const element = React.createElement(
  "h1",
  { className: "title" },
  "안녕, ",
  name,
  "!"
);
```

### JSX 핵심 규칙 🔴

```tsx
// 규칙 1: 반드시 하나의 루트 요소로 감싸야 한다
// ❌
return (
  <h1>제목</h1>
  <p>본문</p>
);

// ✅ div로 감싸기
return (
  <div>
    <h1>제목</h1>
    <p>본문</p>
  </div>
);

// ✅ Fragment로 감싸기 (불필요한 DOM 없이!)
return (
  <>
    <h1>제목</h1>
    <p>본문</p>
  </>
);

// 규칙 2: JavaScript 표현식은 {} 중괄호로
const name = "철수";
return <h1>안녕, {name}!</h1>;
return <p>{1 + 2 + 3}</p>;          // 6
return <p>{isLoggedIn ? "환영!" : "로그인 필요"}</p>;
return <p>{items.map(i => <span key={i.id}>{i.name}</span>)}</p>;

// 규칙 3: HTML과 다른 속성명 (camelCase)
<div className="card">     // ❌ class → ✅ className
<label htmlFor="email">    // ❌ for → ✅ htmlFor
<input tabIndex={0} />     // ❌ tabindex → ✅ tabIndex
<img onClick={handleClick} /> // ❌ onclick → ✅ onClick

// 규칙 4: 셀프 클로징 태그
<img src="photo.jpg" />    // 반드시 /> 로 닫기
<input type="text" />
<br />
<MyComponent />

// 규칙 5: style은 객체로
<div style={{ color: "red", fontSize: "16px", backgroundColor: "#f0f0f0" }}>
  {/* 문자열이 아닌 객체! 속성명은 camelCase! */}
</div>
```

## 2. 함수형 컴포넌트 🔴

```tsx
// 가장 기본적인 컴포넌트
function Greeting() {
  return <h1>안녕하세요!</h1>;
}

// 화살표 함수 (실무에서 더 많이 사용)
const Greeting = () => {
  return <h1>안녕하세요!</h1>;
};

// 한 줄이면 return 생략
const Greeting = () => <h1>안녕하세요!</h1>;

// ⚠️ 컴포넌트 이름은 반드시 대문자로 시작!
// 소문자면 HTML 태그로 인식됨
function greeting() { }  // ❌ <greeting />은 HTML 태그 취급
function Greeting() { }  // ✅ <Greeting />은 React 컴포넌트 취급
```

> 💡 **React.FC 쓸까 말까?** (2026년 기준)
> `const Comp: React.FC<Props> = (props) => { }`는 쓰지 않는 것이 주류.
> 이유: children이 암묵적으로 포함되어 불명확하고, 제네릭 컴포넌트에 불편.
> 그냥 `function Comp(props: Props) { }`가 깔끔하다.

## 3. Props — 컴포넌트의 입력값 🔴

> 💡 **비유: 함수의 매개변수**
> 함수에 인자를 넘기듯, 컴포넌트에 데이터를 넘기는 것이 Props.

```tsx
// Props 타입 정의
type UserCardProps = {
  name: string;              // 필수
  email: string;             // 필수
  avatar?: string;           // 선택적
  role?: "admin" | "user";   // 선택적 + 유니온
};

// 구조 분해 + 기본값
function UserCard({ name, email, avatar, role = "user" }: UserCardProps) {
  return (
    <div className="card">
      {avatar && <img src={avatar} alt={name} />}
      <h3>{name}</h3>
      <p>{email}</p>
      <span>{role}</span>
    </div>
  );
}

// 사용
<UserCard name="철수" email="chul@test.com" />
<UserCard name="영희" email="young@test.com" avatar="/img/young.jpg" role="admin" />
```

### Props는 읽기 전용! 🔴

```tsx
// ❌ Props를 직접 수정하면 안 된다!
function Bad({ name }: { name: string }) {
  name = "변경!";  // ❌ 금지! Props는 부모의 데이터!
  return <h1>{name}</h1>;
}

// ✅ 수정이 필요하면 State를 사용
function Good({ initialName }: { initialName: string }) {
  const [name, setName] = useState(initialName);
  return <input value={name} onChange={(e) => setName(e.target.value)} />;
}
```

## 4. children — 특별한 Props 🔴

```tsx
// children: 열린 태그와 닫힌 태그 사이의 모든 것
type CardProps = {
  title: string;
  children: React.ReactNode;  // 어떤 JSX든 받음
};

function Card({ title, children }: CardProps) {
  return (
    <div className="card">
      <h2>{title}</h2>
      <div className="card-body">
        {children}  {/* 여기에 부모가 넣어준 내용이 들어감 */}
      </div>
    </div>
  );
}

// 사용: 태그 사이의 내용이 children으로 전달됨!
<Card title="공지사항">
  <p>이번 주 금요일은 휴무입니다.</p>
  <p>다음 주 월요일에 만나요!</p>
</Card>

<Card title="프로필">
  <UserCard name="철수" email="chul@test.com" />
</Card>
```

> 💡 **children 패턴**은 React 합성(Composition)의 핵심!
> Next.js의 `layout.tsx`에서 `{children}`으로 페이지를 감싸는 것도 이 원리!

## 5. 조건부 렌더링 🔴

```tsx
// 패턴 1: && (조건이 true면 렌더링)
{isLoggedIn && <UserProfile />}
{items.length > 0 && <ItemList items={items} />}
// ⚠️ 주의: {count && <p>{count}개</p>}에서 count가 0이면 "0"이 렌더링됨!
// ✅ 안전: {count > 0 && <p>{count}개</p>}

// 패턴 2: 삼항 연산자 (이것 또는 저것)
{isLoggedIn ? <Dashboard /> : <LoginForm />}
{error ? <ErrorMessage error={error} /> : <DataView data={data} />}

// 패턴 3: Early Return (복잡한 조건)
function UserProfile({ user }: { user: User | null }) {
  if (!user) return <p>로그인이 필요합니다.</p>;
  if (user.isBanned) return <p>정지된 계정입니다.</p>;

  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
    </div>
  );
}

// 패턴 4: 객체 매핑 (여러 조건)
const STATUS_MESSAGE: Record<string, string> = {
  idle: "대기 중",
  loading: "로딩 중...",
  success: "완료!",
  error: "에러 발생",
};

function StatusBadge({ status }: { status: string }) {
  return <span>{STATUS_MESSAGE[status] ?? "알 수 없음"}</span>;
}
```

## 6. 리스트 렌더링과 key 🔴

```tsx
const products = [
  { id: 1, name: "노트북", price: 1200000 },
  { id: 2, name: "키보드", price: 85000 },
  { id: 3, name: "마우스", price: 45000 },
];

function ProductList() {
  return (
    <ul>
      {products.map((product) => (
        <li key={product.id}>  {/* ⚠️ key 필수! */}
          <strong>{product.name}</strong>
          <span>{product.price.toLocaleString()}원</span>
        </li>
      ))}
    </ul>
  );
}
```

### key가 왜 필요한가? 🔴

```
key = React가 리스트 요소를 식별하는 고유 이름표

key가 없으면:
  React가 순서(index)로 비교 → 중간에 삽입/삭제 시 전체를 다시 그림!

key가 있으면:
  React가 key로 비교 → 변경된 요소만 정확히 업데이트!

예시: [A, B, C] → [A, X, B, C] (B 앞에 X 삽입)

key 없음 (index):
  index 0: A → A (변경 없음)
  index 1: B → X (변경! B를 X로 교체)
  index 2: C → B (변경! C를 B로 교체)
  index 3: (없음) → C (추가!)
  → 3개 변경 😱

key 있음 (id):
  key="A": A → A (변경 없음)
  key="X": (새로 추가!)
  key="B": B → B (변경 없음)
  key="C": C → C (변경 없음)
  → 1개 추가만! ✅
```

### key 규칙

```tsx
// ✅ 좋은 key: 데이터의 고유 ID
{users.map(user => <UserCard key={user.id} {...user} />)}

// ❌ 나쁜 key: index (삽입/삭제/정렬 시 버그!)
{users.map((user, index) => <UserCard key={index} {...user} />)}
// index를 key로 쓰면: 입력 필드의 값이 잘못된 항목에 남아있는 버그 발생!

// ❌ 나쁜 key: Math.random() (매 렌더링마다 바뀜 → 항상 재생성!)
{users.map(user => <UserCard key={Math.random()} {...user} />)}

// 🟡 index를 key로 써도 되는 예외적 상황:
// 1. 리스트가 절대 변하지 않음 (정적)
// 2. 순서가 절대 바뀌지 않음
// 3. 삽입/삭제가 없음
```

## 7. Fragment — 불필요한 DOM 없이 그룹핑 🟡

```tsx
// ❌ 의미 없는 div 추가
function UserInfo() {
  return (
    <div>  {/* 이 div는 스타일링에도 안 쓰이는 쓸모없는 태그 */}
      <h1>이름</h1>
      <p>이메일</p>
    </div>
  );
}

// ✅ Fragment: DOM에 아무것도 추가하지 않음
function UserInfo() {
  return (
    <>
      <h1>이름</h1>
      <p>이메일</p>
    </>
  );
}

// key가 필요한 Fragment (리스트에서)
function Glossary({ items }: { items: { term: string; desc: string }[] }) {
  return (
    <dl>
      {items.map(item => (
        <Fragment key={item.term}>
          <dt>{item.term}</dt>
          <dd>{item.desc}</dd>
        </Fragment>
      ))}
    </dl>
  );
}
```

// ProductCard.tsx
type ProductCardProps = {
  id: number;
  name: string;
  price: number;
  inStock: boolean;
  image?: string;
  children?: React.ReactNode;
};

function ProductCard({ name, price, inStock, image, children }: ProductCardProps) {
  return (
    <div style={{ border: "1px solid #ddd", padding: 16, borderRadius: 8 }}>
      <img
        src={image ?? "https://via.placeholder.com/200"}
        alt={name}
        style={{ width: "100%", height: 150, objectFit: "cover", borderRadius: 4 }}
      />
      <h3>{name}</h3>
      <p>{price.toLocaleString()}원</p>
      <span style={{ color: inStock ? "green" : "red", fontWeight: "bold" }}>
        {inStock ? "구매 가능" : "품절"}
      </span>
      {children && <div style={{ marginTop: 8 }}>{children}</div>}
    </div>
  );
}

// ProductList.tsx
type Product = { id: number; name: string; price: number; inStock: boolean; image?: string };

function ProductList({ products }: { products: Product[] }) {
  if (products.length === 0) {
    return <p>상품이 없습니다.</p>;

    <div style={{ display: "grid", gridTemplateColumns: "repeat(auto-fit, minmax(200px, 1fr))", gap: 16 }}>

          <button disabled={!product.inStock}>장바구니 담기</button>
        </ProductCard>
      ))}
    </div>
  );

## 면접 대비

### 🔴 필수 질문

**Q1. JSX란 무엇인가요?**

> JSX는 JavaScript 안에서 HTML과 유사한 구문을 작성할 수 있게 해주는 문법 확장이다. 실제로는 `React.createElement()` 함수 호출로 변환되며, 빌드 시 Babel이나 SWC가 이 변환을 수행한다. HTML과의 차이로 `className`(class 대신), `htmlFor`(for 대신), camelCase 이벤트 이름(onClick), 중괄호 `{}`로 JS 표현식 삽입 등이 있다.

**Q2. key가 필요한 이유? index를 key로 쓰면 안 되는 이유?**

> key는 React의 재조정 알고리즘이 리스트 요소를 고유하게 식별하는 데 사용한다. key가 있으면 요소의 삽입/삭제/이동을 정확히 감지하여 최소한의 DOM 변경만 수행한다. index를 key로 쓰면 배열의 순서가 바뀔 때(정렬, 삽입, 삭제) React가 요소를 잘못 식별하여 입력 필드의 값이 엉뚱한 항목에 남거나, 불필요한 리렌더링이 발생한다. 데이터의 고유 ID를 key로 사용해야 한다.

**Q3. Props와 State의 차이?**

> Props는 부모가 자식에게 전달하는 읽기 전용 데이터로, 자식이 직접 변경할 수 없다. State는 컴포넌트 내부에서 관리하는 가변 데이터로, `setState`를 통해 변경하면 리렌더링이 발생한다. Props는 외부 입력, State는 내부 상태라고 볼 수 있다.
