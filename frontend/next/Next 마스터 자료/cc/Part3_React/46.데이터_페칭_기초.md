## 1. 데이터 페칭의 3가지 상태 🔴

> 💡 **비유: 음식 배달 앱**
> 주문 후 3가지 상태가 있다: "배달 중(로딩)" / "배달 완료(성공)" / "배달 실패(에러)"
> 각 상태마다 다른 화면을 보여줘야 한다!

```tsx
type User = { id: number; name: string; email: string };

function UserProfile({ userId }: { userId: number }) {
  const [user, setUser] = useState<User | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const controller = new AbortController();

    const fetchUser = async () => {
      setLoading(true);
      setError(null);

      try {
        const res = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`, {
          signal: controller.signal,
        });
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const data: User = await res.json();
        setUser(data);
      } catch (err) {
        if (err instanceof Error && err.name !== "AbortError") {
          setError(err.message);
        }
      } finally {
        setLoading(false);
      }
    };

    fetchUser();

    return () => controller.abort();  // 클린업: 이전 요청 취소!
  }, [userId]);

  // 3가지 상태에 따른 UI 분기 🔴
  if (loading) return <div className="animate-pulse">로딩 중...</div>;
  if (error) return <div className="text-red-500">에러: {error}</div>;
  if (!user) return <div>사용자를 찾을 수 없습니다</div>;

  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
    </div>
  );
}
```

## 2. Race Condition 방지 🟡

> 💡 **비유: 택시 2대 동시 호출**
> userId가 1→2로 빠르게 바뀌면, 1번 요청이 2번보다 늦게 도착할 수 있다.
> 그러면 화면에 2번 유저가 아닌 1번 유저가 표시되는 버그!

```tsx
useEffect(() => {
  let isCancelled = false;  // 이 변수가 Race Condition을 막는 열쇠!
  const controller = new AbortController();

  const fetchData = async () => {
    setLoading(true);
    try {
      const res = await fetch(`/api/users/${userId}`, { signal: controller.signal });
      const data = await res.json();

      if (!isCancelled) {   // 아직 유효한 요청인지 확인!
        setUser(data);      // 취소된 요청이면 State 업데이트 안 함
      }
    } catch (err) {
      if (!isCancelled && err instanceof Error && err.name !== "AbortError") {
        setError(err.message);
      }
    } finally {
      if (!isCancelled) setLoading(false);
    }
  };

  fetchData();

  return () => {
    isCancelled = true;      // 다음 effect 시작 시 이전 요청 무효화!
    controller.abort();       // 네트워크 요청도 취소!
  };
}, [userId]);
```

## 3. 커스텀 훅으로 추출 — useFetch 🟡

> 매번 같은 보일러플레이트를 반복하지 말고, 커스텀 훅으로 추출하자!

```tsx
function useFetch<T>(url: string) {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const controller = new AbortController();
    let isCancelled = false;

    const fetchData = async () => {
      setLoading(true);
      setError(null);

      try {
        const res = await fetch(url, { signal: controller.signal });
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        const json: T = await res.json();
        if (!isCancelled) setData(json);
      } catch (err) {
        if (!isCancelled && err instanceof Error && err.name !== "AbortError") {
          setError(err.message);
        }
      } finally {
        if (!isCancelled) setLoading(false);
      }
    };

    fetchData();
    return () => { isCancelled = true; controller.abort(); };
  }, [url]);

  return { data, loading, error };
}

// 사용: 보일러플레이트가 1줄로!
function UserProfile({ userId }: { userId: number }) {
  const { data: user, loading, error } = useFetch<User>(`/api/users/${userId}`);

  if (loading) return <p>로딩 중...</p>;
  if (error) return <p>에러: {error}</p>;
  if (!user) return <p>데이터 없음</p>;

  return <h1>{user.name}</h1>;
}
```

## 4. 실무에서의 데이터 페칭 현실 🔴

```
useEffect + fetch의 한계:
  1. 캐싱 없음: 같은 데이터를 매번 다시 요청
  2. 중복 요청: 여러 컴포넌트가 같은 API 호출
  3. 백그라운드 갱신 없음: 탭 전환 후 돌아와도 옛날 데이터
  4. 낙관적 업데이트 없음: mutation 후 수동 refetch
  5. 로딩/에러 상태 보일러플레이트 반복
  6. Infinite Scroll/Pagination 직접 구현

이 모든 문제를 해결하는 것이 → TanStack Query (52번!) ⭐
```

```tsx
// 예고: TanStack Query는 이 모든 것을 한 줄로!
function UserProfile({ userId }: { userId: number }) {
  const { data: user, isLoading, error } = useQuery({
    queryKey: ["user", userId],
    queryFn: () => fetch(`/api/users/${userId}`).then(r => r.json()),
    staleTime: 5 * 60 * 1000,  // 5분 캐싱!
  });
  // 캐싱, 중복 요청 제거, 백그라운드 갱신, 에러 재시도... 전부 자동!
}
```

> 💡 **하지만 원리(useEffect + fetch)를 모르면 TanStack Query도 제대로 못 쓴다!**
> 바퀴를 발명할 필요는 없지만, 바퀴가 어떻게 굴러가는지는 알아야 한다.

## 5. 실습: 게시글 목록 + 상세 페이지

```
과제: JSONPlaceholder API를 활용한 블로그 뷰어

1. PostList: /posts에서 게시글 목록 표시 (제목만)
2. PostDetail: /posts/{id}에서 상세 표시 (제목 + 본문)
3. 게시글 클릭 시 PostDetail로 전환 (State로 selectedId 관리)
4. 로딩 중: 스켈레톤 또는 "로딩 중..." 표시
5. 에러 시: 에러 메시지 + "다시 시도" 버튼 (refetch)
6. useFetch 커스텀 훅 사용
7. AbortController로 Race Condition 방지
```

<summary>핵심 구조 정답</summary>

```tsx
function Blog() {
  const [selectedId, setSelectedId] = useState<number | null>(null);

  return (
    <div style={{ display: "flex", gap: 24 }}>
      <div style={{ flex: 1 }}>
        <h2>게시글 목록</h2>
        <PostList onSelect={setSelectedId} selectedId={selectedId} />
      </div>
      <div style={{ flex: 2 }}>
        {selectedId ? (
          <PostDetail postId={selectedId} />
        ) : (
          <p>게시글을 선택하세요</p>
        )}
      </div>
    </div>
  );
}

function PostList({ onSelect, selectedId }: {
  onSelect: (id: number) => void;
  selectedId: number | null;
}) {
  const { data: posts, loading, error } = useFetch<Post[]>(
    "https://jsonplaceholder.typicode.com/posts"
  );

  if (loading) return <p>목록 로딩 중...</p>;
  if (error) return <p>에러: {error}</p>;

  return (
    <ul>
      {posts?.slice(0, 20).map(post => (
        <li
          key={post.id}
          onClick={() => onSelect(post.id)}
          style={{
            cursor: "pointer",
            fontWeight: post.id === selectedId ? "bold" : "normal",
            padding: "8px 0",
          }}
        >
          {post.title}
        </li>
      ))}
    </ul>
  );
}

function PostDetail({ postId }: { postId: number }) {
  const { data: post, loading, error } = useFetch<Post>(
    `https://jsonplaceholder.typicode.com/posts/${postId}`
  );

  if (loading) return <p>게시글 로딩 중...</p>;
  if (error) return <p>에러: {error}</p>;
  if (!post) return null;

  return (
    <article>
      <h2>{post.title}</h2>
      <p>{post.body}</p>
    </article>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. useEffect에서 데이터를 페칭할 때 주의할 점?**

> 첫째, async 함수를 useEffect 콜백으로 직접 전달하면 안 되고 내부에서 선언 후 호출해야 한다. 둘째, AbortController로 컴포넌트 언마운트나 의존성 변경 시 이전 요청을 취소하여 Race Condition을 방지해야 한다. 셋째, 로딩/에러/성공 3가지 상태를 모두 관리하여 사용자에게 적절한 피드백을 제공해야 한다. 넷째, `res.ok`를 체크하여 HTTP 에러도 처리해야 한다.

**Q2. Race Condition이란? 어떻게 방지하나요?**

> 비동기 요청이 여러 개 동시에 진행될 때, 나중에 보낸 요청보다 먼저 보낸 요청이 늦게 도착하여 오래된 데이터가 화면에 표시되는 버그다. useEffect 클린업에서 `isCancelled` 플래그와 `AbortController.abort()`를 사용하여, 의존성이 변경되면 이전 요청을 무효화/취소하여 방지한다.
