## 1. 다중 입력 핸들링 — name 속성 활용 🔴

```tsx
// ❌ 나쁜 패턴: 입력 필드마다 개별 핸들러
const [name, setName] = useState("");
const [email, setEmail] = useState("");
const [phone, setPhone] = useState("");
// ... 필드가 10개면 useState 10개 + 핸들러 10개 😱

// ✅ 좋은 패턴: 하나의 객체 State + name 속성으로 통합
type FormData = {
  name: string;
  email: string;
  phone: string;
  message: string;
};

function ContactForm() {
  const [form, setForm] = useState<FormData>({
    name: "", email: "", phone: "", message: "",
  });

  // 하나의 핸들러로 모든 입력 처리!
  const handleChange = (e: React.ChangeEvent<HTMLInputElement | HTMLTextAreaElement>) => {
    const { name, value } = e.target;
    setForm(prev => ({ ...prev, [name]: value }));
    // [name]은 계산된 속성명: e.target.name이 "email"이면 { ...prev, email: value }
  };

  // 체크박스는 checked 사용
  const handleCheckChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const { name, checked } = e.target;
    setForm(prev => ({ ...prev, [name]: checked }));
  };

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();
    console.log(form);
  };

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" value={form.name} onChange={handleChange} placeholder="이름" />
      <input name="email" type="email" value={form.email} onChange={handleChange} placeholder="이메일" />
      <input name="phone" value={form.phone} onChange={handleChange} placeholder="전화번호" />
      <textarea name="message" value={form.message} onChange={handleChange} placeholder="메시지" />
      <button type="submit">전송</button>
    </form>
  );
}
```

## 2. 실시간 유효성 검사 🔴

```tsx
type FormErrors = Partial<Record<keyof FormData, string>>;

function SignupForm() {
  const [form, setForm] = useState({ name: "", email: "", password: "" });
  const [errors, setErrors] = useState<FormErrors>({});
  const [touched, setTouched] = useState<Partial<Record<string, boolean>>>({});

  // 유효성 검사 함수
  const validate = (field: keyof typeof form, value: string): string | undefined => {
    switch (field) {
      case "name":
        if (!value.trim()) return "이름을 입력하세요";
        if (value.length < 2) return "이름은 2자 이상이어야 합니다";
        return undefined;
      case "email":
        if (!value) return "이메일을 입력하세요";
        if (!/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value)) return "올바른 이메일 형식이 아닙니다";
        return undefined;
      case "password":
        if (!value) return "비밀번호를 입력하세요";
        if (value.length < 8) return "비밀번호는 8자 이상이어야 합니다";
        if (!/[A-Z]/.test(value)) return "대문자를 1개 이상 포함해야 합니다";
        return undefined;
    }
  };

  const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const { name, value } = e.target;
    setForm(prev => ({ ...prev, [name]: value }));

    // 이미 터치한 필드만 실시간 검증 (입력 전에 에러 보이면 UX 나쁨!)
    if (touched[name]) {
      const error = validate(name as keyof typeof form, value);
      setErrors(prev => ({ ...prev, [name]: error }));
    }
  };

  // blur(포커스 해제) 시 첫 검증 발동
  const handleBlur = (e: React.FocusEvent<HTMLInputElement>) => {
    const { name, value } = e.target;
    setTouched(prev => ({ ...prev, [name]: true }));
    const error = validate(name as keyof typeof form, value);
    setErrors(prev => ({ ...prev, [name]: error }));
  };

  const handleSubmit = (e: React.FormEvent) => {
    e.preventDefault();

    // 제출 시 모든 필드 검증
    const newErrors: FormErrors = {};
    (Object.keys(form) as (keyof typeof form)[]).forEach(field => {
      const error = validate(field, form[field]);
      if (error) newErrors[field] = error;
    });

    if (Object.keys(newErrors).length > 0) {
      setErrors(newErrors);
      setTouched({ name: true, email: true, password: true });
      return;
    }

    console.log("제출 성공:", form);
  };

  // 모든 필드가 유효할 때만 버튼 활성화
  const isValid = Object.values(errors).every(e => !e) &&
                  Object.values(form).every(v => v.trim().length > 0);

  return (
    <form onSubmit={handleSubmit}>
      <div>
        <input name="name" value={form.name} onChange={handleChange} onBlur={handleBlur} />
        {touched.name && errors.name && <p className="text-red-500 text-sm">{errors.name}</p>}
      </div>

      <div>
        <input name="email" type="email" value={form.email} onChange={handleChange} onBlur={handleBlur} />
        {touched.email && errors.email && <p className="text-red-500 text-sm">{errors.email}</p>}
      </div>

      <div>
        <input name="password" type="password" value={form.password} onChange={handleChange} onBlur={handleBlur} />
        {touched.password && errors.password && <p className="text-red-500 text-sm">{errors.password}</p>}
      </div>

      <button type="submit" disabled={!isValid}>가입</button>
    </form>
  );
}
```

### 유효성 검사 타이밍 전략

```
onChange:  타이핑할 때마다 즉시 검증 → 이미 터치한 필드만! (UX 최적)
onBlur:   포커스를 떠날 때 첫 검증 → "입력 끝났겠지?" 시점
onSubmit: 제출 시 전체 검증 → 최종 방어선

실무 정석:
  1. 처음 입력 시 → 에러 안 보임 (아직 타이핑 중이니까)
  2. 포커스 떠남(blur) → 첫 검증 발동
  3. 이후 타이핑(change) → 실시간 검증 (에러 즉시 사라짐)
  4. 제출 시 → 전체 검증 (놓친 것 잡기)
```

## 3. select, checkbox, radio 핸들링 🟡

```tsx
function MultiInputForm() {
  const [form, setForm] = useState({
    country: "",
    interests: [] as string[],
    gender: "",
    agreed: false,
  });

  // select
  <select name="country" value={form.country} onChange={handleChange}>
    <option value="">선택하세요</option>
    <option value="KR">한국</option>
    <option value="US">미국</option>
  </select>

  // checkbox (다중 선택)
  const handleInterestChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const { value, checked } = e.target;
    setForm(prev => ({
      ...prev,
      interests: checked
        ? [...prev.interests, value]
        : prev.interests.filter(i => i !== value),
    }));
  };

  <label>
    <input type="checkbox" value="react" checked={form.interests.includes("react")}
           onChange={handleInterestChange} /> React
  </label>
  <label>
    <input type="checkbox" value="next" checked={form.interests.includes("next")}
           onChange={handleInterestChange} /> Next.js
  </label>

  // radio (단일 선택)
  <label>
    <input type="radio" name="gender" value="male"
           checked={form.gender === "male"} onChange={handleChange} /> 남성
  </label>
  <label>
    <input type="radio" name="gender" value="female"
           checked={form.gender === "female"} onChange={handleChange} /> 여성
  </label>

  // checkbox (boolean 토글)
  <label>
    <input type="checkbox" name="agreed" checked={form.agreed}
           onChange={(e) => setForm(prev => ({ ...prev, agreed: e.target.checked }))} />
    약관 동의
  </label>
}
```

### 🔴 필수 질문

**Q1. React에서 폼을 다루는 기본 패턴은?**

> 제어 컴포넌트 패턴을 사용한다. 각 input의 `value`를 State로 관리하고 `onChange`로 State를 업데이트한다. 다중 입력은 하나의 객체 State에 `name` 속성을 활용한 통합 핸들러로 관리한다. 유효성 검사는 blur(첫 검증) + change(실시간) + submit(최종) 3단계 전략을 사용한다.

**Q2. 실무에서 폼 라이브러리를 쓰는 이유?**

> 제어 컴포넌트 방식은 입력 필드마다 State + 핸들러 + 검증 로직 + 에러 상태 관리가 필요하여 보일러플레이트가 매우 많다. React Hook Form은 비제어 방식(ref 기반)으로 리렌더링을 최소화하면서 Zod와 연동하여 타입 안전한 스키마 기반 검증을 제공한다. 코드량이 약 1/10로 줄어들고 성능도 향상된다.
