## 1. React 19의 핵심 변화 — "서버와 클라이언트의 융합" 🔴

> React 19는 단순한 버전 업이 아니라 **패러다임 전환**이다.
> "클라이언트에서만 돌아가던 React"가 "서버에서도 돌아가는 React"로 확장되었다.

```
React 18 이전:           React 19:
┌─────────────────┐     ┌─────────────────┐
│   브라우저       │     │    서버          │ ← NEW!
│   (클라이언트)    │     │  Server Components│
│   모든 코드가    │     │  Server Actions  │
│   여기서 실행    │     │  async 컴포넌트   │
│                 │     ├─────────────────┤
│                 │     │   브라우저        │
│                 │     │  Client Components│
│                 │     │  'use client'     │
└─────────────────┘     └─────────────────┘
```

## 2. `<form action={함수}>` — 서버 함수를 폼에 직접 바인딩 🔴

> React 19 이전: `onSubmit` + `e.preventDefault()` + `fetch` + 상태 관리
> React 19: `<form action={서버함수}>`  — 끝!

```tsx
// ❌ React 18 패턴
function OldForm() {
  const [loading, setLoading] = useState(false);

  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();
    setLoading(true);
    const formData = new FormData(e.currentTarget as HTMLFormElement);
    await fetch("/api/save", { method: "POST", body: formData });
    setLoading(false);
  };

  return <form onSubmit={handleSubmit}>...</form>;
}

// ✅ React 19 패턴 (Next.js Server Actions!)
async function saveAction(formData: FormData) {
  "use server";
  const name = formData.get("name") as string;
  await db.user.create({ data: { name } });
}

function NewForm() {
  return (
    <form action={saveAction}>
      <input name="name" />
      <button type="submit">저장</button>
    </form>
  );
}
// fetch 코드 0줄! 상태 관리 0줄! 프레임워크가 알아서!
```

## 3. useActionState — 서버 액션의 결과를 화면에 반영 🔴

> 서버 함수가 반환한 값을 자동으로 컴포넌트 State에 동기화!

```tsx
import { useActionState } from "react";

// 서버 액션 (prevState가 첫 번째 인자로 자동 주입!)
async function createPost(prevState: any, formData: FormData) {
  "use server";
  const title = formData.get("title") as string;

  if (title.length < 2) {
    return { success: false, message: "제목은 2자 이상이어야 합니다" };
  }

  await db.post.create({ data: { title } });
  return { success: true, message: "게시글이 작성되었습니다!" };
}

// 클라이언트 컴포넌트
"use client";
function PostForm() {
  const [state, formAction, isPending] = useActionState(createPost, {
    success: false,
    message: "",
  });

  return (
    <form action={formAction}>
      <input name="title" />
      <button type="submit" disabled={isPending}>
        {isPending ? "저장 중..." : "작성"}
      </button>

      {/* 서버 응답이 자동으로 state에 반영! */}
      {state.message && (
        <p className={state.success ? "text-green-500" : "text-red-500"}>
          {state.message}
        </p>
      )}
    </form>
  );
}
```

## 4. useFormStatus — 폼 전송 중 상태 추적 🔴

> 부모 `<form>`의 전송 상태를 자식 컴포넌트에서 자동 감지!

```tsx
"use client";
import { useFormStatus } from "react-dom";

// ⚠️ 반드시 <form> 내부의 자식 컴포넌트에서만 사용!
function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button type="submit" disabled={pending}>
      {pending ? "저장 중..." : "저장"}
    </button>
  );
}

// 부모 폼
function Form() {
  return (
    <form action={serverAction}>
      <input name="title" />
      <SubmitButton />  {/* 이 안에서 useFormStatus 사용! */}
    </form>
  );
}
```

## 5. useOptimistic — 서버 응답 전에 UI 먼저 변경 🟡

> "좋아요 버튼을 누르면 즉시 빨갛게! 서버 응답은 나중에"

```tsx
"use client";
import { useOptimistic } from "react";

function LikeButton({ initialLiked }: { initialLiked: boolean }) {
  const [optimisticLiked, setOptimisticLiked] = useOptimistic(initialLiked);

  const handleLike = async () => {
    const newStatus = !optimisticLiked;
    setOptimisticLiked(newStatus);  // UI 즉시 변경! (서버 기다리지 않음)

    try {
      await toggleLikeAction(newStatus);  // 서버 액션 (백그라운드)
    } catch {
      // 실패하면 자동 롤백! (initialLiked로 복구)
      alert("실패! 원래 상태로 돌아갑니다.");
    }
  };

  return (
    <form action={handleLike}>
      <button type="submit">
        {optimisticLiked ? "❤️" : "🤍"}
      </button>
    </form>
  );
}
```

## 6. use() 훅 — Promise와 Context를 직접 읽기 🟡

```tsx
// use()는 조건문 안에서도 호출 가능! (다른 훅과 다른 특별한 규칙)

// Promise 읽기
function UserProfile({ userPromise }: { userPromise: Promise<User> }) {
  const user = use(userPromise);  // Suspense와 함께 사용!
  return <h1>{user.name}</h1>;
}

// Context 읽기 (useContext 대안)
function ThemeButton() {
  const theme = use(ThemeContext);  // 조건문 안에서도 OK!
  return <button className={theme === "dark" ? "bg-gray-800" : "bg-white"}>테마</button>;
}
```

## 7. React Compiler — 자동 메모이제이션 🟢

```
React Compiler (React 19+):
  - memo, useMemo, useCallback을 자동으로 추가해주는 빌드 도구
  - 개발자가 수동 최적화할 필요가 크게 줄어듦
  - 하지만 "왜 최적화가 필요한지" 이해는 여전히 필수! (면접!)

현재 상태 (2026년):
  - Next.js 15+에서 experimental로 사용 가능
  - 점진적으로 안정화 중
  - 설정: next.config.ts에 reactCompiler: true
```

## 8. 정리: React 19 이전 vs 이후

```
폼 전송:
  이전: onSubmit + e.preventDefault() + fetch + useState × 3
  이후: <form action={serverFn}> 한 줄! ✅

전송 상태:
  이전: const [loading, setLoading] = useState(false); ...
  이후: useFormStatus() → { pending } 자동! ✅

서버 응답 반영:
  이전: useState + try-catch + setResult + setError
  이후: useActionState → [state, formAction, isPending] 자동! ✅

낙관적 업데이트:
  이전: setState(가짜) → 서버 호출 → 실패 시 setState(원래) 수동
  이후: useOptimistic → 자동 롤백! ✅

메모이제이션:
  이전: memo, useMemo, useCallback 수동
  이후: React Compiler가 자동! ✅
```

## 면접 대비

### 🟡 이해 질문

**Q1. React 19에서 바뀐 핵심은?**

> 가장 큰 변화는 서버와 클라이언트의 경계를 프레임워크 레벨에서 지원하는 것이다. `<form action={서버함수}>` 패턴으로 API 엔드포인트 없이 서버 함수를 직접 호출할 수 있고, `useActionState`로 서버 응답을 자동 동기화하며, `useFormStatus`로 전송 상태를 자식 컴포넌트에서 추적하고, `useOptimistic`으로 낙관적 업데이트와 자동 롤백을 지원한다. React Compiler의 도입으로 수동 메모이제이션의 필요성도 줄어들었다.

**Q2. useActionState와 useState로 직접 관리하는 것의 차이?**

> `useActionState`는 서버 액션의 반환값을 자동으로 컴포넌트 State에 동기화하며, `prevState`를 서버 함수에 자동 주입하여 이전 상태를 기반으로 한 누적 로직이 가능하다. useState로 직접 관리하면 try-catch, loading 상태, 에러 상태, 성공 상태를 모두 수동으로 관리해야 하는 보일러플레이트가 발생한다.
