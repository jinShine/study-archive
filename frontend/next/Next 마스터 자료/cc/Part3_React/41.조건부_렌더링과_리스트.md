## 1. 조건부 렌더링 4대 패턴 정리 🔴

### 패턴 1: `&&` — "조건이 참이면 보여줘" ⭐ 가장 많이 사용

```tsx
// 로그인했을 때만 프로필 보여주기
{isLoggedIn && <UserProfile />}

// 에러가 있을 때만 에러 메시지
{error && <p className="text-red-500">{error}</p>}

// 데이터가 있을 때만 리스트
{items.length > 0 && <ItemList items={items} />}
```

> ⚠️ **`&&` 함정: 숫자 0이 화면에 렌더링됨!**

```tsx
// ❌ count가 0이면 "0"이 화면에 그대로 출력됨!
{count && <span>{count}개</span>}
// count = 0 → 0 (JSX는 0을 렌더링함!)

// ✅ 명시적으로 boolean 변환
{count > 0 && <span>{count}개</span>}
{!!count && <span>{count}개</span>}
{Boolean(count) && <span>{count}개</span>}

// ❌ 빈 문자열도 비슷한 문제
{message && <p>{message}</p>}  // message = "" → false → OK (렌더링 안 됨)
// 하지만 의도적으로 빈 문자열을 보여줘야 할 때는 주의!
```

### 패턴 2: 삼항 연산자 — "이것 아니면 저것"

```tsx
// 로그인 상태에 따라 다른 컴포넌트
{isLoggedIn ? <Dashboard /> : <LoginForm />}

// 로딩 상태
{isLoading ? <Spinner /> : <DataView data={data} />}

// 인라인 스타일/클래스
<div className={isActive ? "bg-blue-500 text-white" : "bg-gray-100"}>
  {isActive ? "활성" : "비활성"}
</div>

// ⚠️ 중첩 삼항은 가독성 최악! Early Return 사용하라
// ❌ 나쁜 예
{status === "loading" ? <Spinner /> : status === "error" ? <Error /> : <Data />}
```

### 패턴 3: Early Return — "조건 안 맞으면 일찍 빠져나가"

```tsx
// ⭐ 복잡한 조건 분기에 가장 깔끔!
function UserProfile({ user }: { user: User | null }) {
  // 1. 비정상 케이스를 먼저 처리하고 빠져나감
  if (!user) {
    return <p>로그인이 필요합니다.</p>;
  }

  if (user.isBanned) {
    return <p>정지된 계정입니다.</p>;
  }

  if (user.isPrivate && !user.isFollowing) {
    return <p>비공개 계정입니다.</p>;
  }

  // 2. 정상 케이스만 아래에 도달
  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.bio}</p>
      <p>{user.email}</p>
    </div>
  );
}
```

### 패턴 4: 객체/Map 매핑 — "여러 상태에 따른 UI"

```tsx
// 상태별 메시지와 색상을 객체로 관리
const STATUS_CONFIG = {
  idle: { message: "대기 중", color: "text-gray-500", icon: "⏳" },
  loading: { message: "로딩 중...", color: "text-blue-500", icon: "🔄" },
  success: { message: "완료!", color: "text-green-500", icon: "✅" },
  error: { message: "에러 발생", color: "text-red-500", icon: "❌" },
} as const;

type Status = keyof typeof STATUS_CONFIG;

function StatusBadge({ status }: { status: Status }) {
  const config = STATUS_CONFIG[status];
  return (
    <span className={config.color}>
      {config.icon} {config.message}
    </span>
  );
}

// 컴포넌트 매핑도 가능!
const STEP_COMPONENTS: Record<number, React.FC> = {
  1: PersonalInfoStep,
  2: AddressStep,
  3: PaymentStep,
  4: ConfirmStep,
};

function Wizard({ step }: { step: number }) {
  const StepComponent = STEP_COMPONENTS[step];
  if (!StepComponent) return <p>잘못된 단계입니다.</p>;
  return <StepComponent />;
}
```

## 2. 리스트 렌더링 실전 패턴 🔴

### 기본 리스트 + 빈 상태 UI

```tsx
type User = { id: number; name: string; email: string; role: "admin" | "user" };

function UserList({ users }: { users: User[] }) {
  // 빈 상태 처리 (Empty State) — UX 필수!
  if (users.length === 0) {
    return (
      <div className="text-center py-12 text-gray-500">
        <p className="text-4xl mb-4">📭</p>
        <p className="text-lg font-medium">등록된 사용자가 없습니다</p>
        <p className="text-sm mt-2">새로운 사용자를 초대해보세요</p>
      </div>
    );
  }

  return (
    <ul className="divide-y">
      {users.map(user => (
        <li key={user.id} className="flex justify-between py-3">
          <div>
            <p className="font-medium">{user.name}</p>
            <p className="text-sm text-gray-500">{user.email}</p>
          </div>
          <span className={user.role === "admin" ? "text-blue-600" : "text-gray-400"}>
            {user.role}
          </span>
        </li>
      ))}
    </ul>
  );
}
```

### 필터링 + 검색 패턴 ⭐ 실무 최다 사용

```tsx
function FilterableUserList({ users }: { users: User[] }) {
  const [search, setSearch] = useState("");
  const [roleFilter, setRoleFilter] = useState<"all" | "admin" | "user">("all");

  // 파생 상태(Derived State): State를 기반으로 계산
  const filteredUsers = users
    .filter(user =>
      user.name.toLowerCase().includes(search.toLowerCase()) ||
      user.email.toLowerCase().includes(search.toLowerCase())
    )
    .filter(user =>
      roleFilter === "all" ? true : user.role === roleFilter
    );

  return (
    <div>
      <div className="flex gap-4 mb-4">
        <input
          value={search}
          onChange={(e) => setSearch(e.target.value)}
          placeholder="이름 또는 이메일 검색..."
          className="flex-1 border rounded px-3 py-2"
        />
        <select
          value={roleFilter}
          onChange={(e) => setRoleFilter(e.target.value as typeof roleFilter)}
          className="border rounded px-3 py-2"
        >
          <option value="all">전체</option>
          <option value="admin">관리자</option>
          <option value="user">일반</option>
        </select>
      </div>

      <p className="text-sm text-gray-500 mb-2">
        {filteredUsers.length}명 표시 중 (전체 {users.length}명)
      </p>

      <UserList users={filteredUsers} />
    </div>
  );
}
```

> 💡 **파생 상태(Derived State)**: `filteredUsers`는 별도 State가 아니라
> 기존 State(`search`, `roleFilter`, `users`)로부터 **계산**되는 값이다.
> 불필요하게 `const [filteredUsers, setFilteredUsers] = useState(...)` 하지 마라!

### 리스트 CRUD (추가/수정/삭제) 🔴

```tsx
function TodoList() {
  const [todos, setTodos] = useState<Todo[]>([
    { id: 1, text: "React 공부", done: false },
    { id: 2, text: "TypeScript 연습", done: true },
  ]);

  // 추가
  const addTodo = (text: string) => {
    setTodos(prev => [...prev, { id: Date.now(), text, done: false }]);
  };

  // 수정 (토글)
  const toggleTodo = (id: number) => {
    setTodos(prev => prev.map(todo =>
      todo.id === id ? { ...todo, done: !todo.done } : todo
    ));
  };

  // 수정 (텍스트)
  const updateTodo = (id: number, newText: string) => {
    setTodos(prev => prev.map(todo =>
      todo.id === id ? { ...todo, text: newText } : todo
    ));
  };

  // 삭제
  const deleteTodo = (id: number) => {
    setTodos(prev => prev.filter(todo => todo.id !== id));
  };

  // 전체 삭제 (완료된 것만)
  const clearCompleted = () => {
    setTodos(prev => prev.filter(todo => !todo.done));
  };

  // 통계 (파생 상태)
  const total = todos.length;
  const doneCount = todos.filter(t => t.done).length;
  const pendingCount = total - doneCount;

  return (
    <div>
      <h1>할 일 ({pendingCount}개 남음)</h1>
      {/* ... UI 생략 ... */}
    </div>
  );
}
```

## 3. 실습: 쇼핑몰 상품 필터링 시스템

```
과제: 상품 리스트 + 필터링 + CRUD

1. 상품 10개 하드코딩 (id, name, price, category, inStock)
2. 카테고리 필터 (전체/전자제품/의류/식품)
3. 검색 (이름으로)
4. 재고 있는 상품만 보기 체크박스
5. 가격순 정렬 (오름차순/내림차순)
6. 빈 결과 시 "검색 결과가 없습니다" + 필터 초기화 버튼
7. "X개 상품 표시 중" 카운트
```

```tsx
type Product = {
  id: number;
  name: string;
  price: number;
  category: "전자제품" | "의류" | "식품";
  inStock: boolean;
};

function ProductFilter({ products }: { products: Product[] }) {
  const [search, setSearch] = useState("");
  const [category, setCategory] = useState<string>("전체");
  const [inStockOnly, setInStockOnly] = useState(false);
  const [sortOrder, setSortOrder] = useState<"asc" | "desc" | "none">("none");

  const filtered = products
    .filter(p => p.name.includes(search))
    .filter(p => category === "전체" || p.category === category)
    .filter(p => !inStockOnly || p.inStock)
    .sort((a, b) => {
      if (sortOrder === "asc") return a.price - b.price;
      if (sortOrder === "desc") return b.price - a.price;
      return 0;
    });

  const resetFilters = () => {
    setSearch("");
    setCategory("전체");
    setInStockOnly(false);
    setSortOrder("none");
  };

  return (
    <div>
      <input value={search} onChange={e => setSearch(e.target.value)} placeholder="검색" />
      <select value={category} onChange={e => setCategory(e.target.value)}>
        <option>전체</option>
        <option>전자제품</option>
        <option>의류</option>
        <option>식품</option>
      </select>
      <label>
        <input type="checkbox" checked={inStockOnly} onChange={e => setInStockOnly(e.target.checked)} />
        재고 있는 상품만
      </label>
      <select value={sortOrder} onChange={e => setSortOrder(e.target.value as any)}>
        <option value="none">정렬 없음</option>
        <option value="asc">가격 낮은순</option>
        <option value="desc">가격 높은순</option>
      </select>

      <p>{filtered.length}개 상품 표시 중</p>

      {filtered.length === 0 ? (
        <div>
          <p>검색 결과가 없습니다</p>
          <button onClick={resetFilters}>필터 초기화</button>
        </div>
      ) : (
        <ul>
          {filtered.map(p => (
            <li key={p.id}>
              {p.name} - {p.price.toLocaleString()}원
              {!p.inStock && <span>(품절)</span>}
            </li>
          ))}
        </ul>
      )}
    </div>
  );
}
```

## 면접 대비

### 🔴 필수 질문

**Q1. React에서 `&&` 조건부 렌더링의 함정?**

> `{count && <Component />}`에서 count가 0이면, JavaScript의 `&&` 연산자가 0(Falsy)을 반환하고 React가 숫자 0을 그대로 렌더링한다. "0"이 화면에 보이는 의도치 않은 결과가 발생한다. `{count > 0 && <Component />}`처럼 명시적으로 boolean으로 변환해야 한다.

**Q2. 파생 상태(Derived State)란? 별도 State로 만들면 안 되는 이유?**

> 기존 State에서 계산으로 얻을 수 있는 값을 말한다. 예를 들어 `filteredItems`는 `items`와 `searchQuery`로부터 계산되므로 별도 State가 아닌 렌더링 시 계산하면 된다. 별도 State로 만들면 원본과 파생 값의 동기화가 꼬여 불일치 버그가 발생하고, 불필요한 `useEffect`가 필요해져 코드가 복잡해진다.

```
┌────────────────────────────────────────────────┐
│        조건부 렌더링과 리스트 핵심 요약            │
├────────────────────────────────────────────────┤
│                                                │
│  조건부 4패턴:                                   │
│    && → 간단한 "보이기/숨기기" (0 주의!)          │
│    삼항 → "이것 또는 저것"                       │
│    Early Return → 복잡한 분기 (가장 깔끔)        │
│    객체 매핑 → 상태별 설정/컴포넌트 전환          │
│                                                │
│  리스트:                                         │
│    map + key={고유ID} 필수!                      │
│    빈 상태 UI(Empty State) 꼭 만들기!            │
│    filter → 검색/카테고리 필터링                  │
│                                                │
│  파생 상태:                                      │
│    기존 State에서 계산 가능하면 State 만들지 마라! │
│    const filtered = items.filter(...)           │
│                                                │
│  리스트 CRUD:                                    │
│    추가: [...prev, new]                         │
│    수정: prev.map(x => x.id === id ? {...} : x) │
│    삭제: prev.filter(x => x.id !== id)          │
│                                                │
└────────────────────────────────────────────────┘
```
