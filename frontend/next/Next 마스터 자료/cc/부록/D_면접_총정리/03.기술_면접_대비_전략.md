## 1. 프론트엔드 면접의 3가지 유형

```
유형 1: 기술 질문 (가장 흔함!)
  → "React의 Virtual DOM을 설명해주세요"
  → "CSR, SSR, RSC의 차이는?"
  → 지식을 묻는 질문! D-01에서 준비!

유형 2: 라이브 코딩 / 과제
  → "실시간으로 TODO 앱을 만들어보세요"
  → "이 API를 호출해서 목록을 표시하세요"
  → 코딩 능력을 직접 확인!

유형 3: 경험 기반 질문
  → "프로젝트에서 겪은 가장 어려운 문제는?"
  → "기술적 의사결정을 한 경험이 있나요?"
  → 실무 능력과 소통 능력을 확인!
```

## 2. 기술 질문 답변 프레임워크 — "정의 → 원리 → 실무" 🔴

> 모든 기술 질문에 이 3단계로 답하면 깔끔!

```
Q: "Virtual DOM이 뭔가요?"

❌ 나쁜 답변:
  "가상 DOM이요... React에서 쓰는 건데... 빠르게 해주는 거예요."

✅ 좋은 답변 (3단계!):

  1단계 — 정의 (1문장!):
    "Virtual DOM은 실제 DOM의 가벼운 복사본을 메모리에 유지하는 것입니다."

  2단계 — 원리 (핵심 키워드!):
    "상태가 변경되면 새로운 Virtual DOM을 만들어
     이전 것과 비교(Diffing)하고,
     실제로 변경된 부분만 실제 DOM에 반영(Reconciliation)합니다."

  3단계 — 실무 연결:
    "덕분에 전체 DOM을 다시 그리는 비용 없이 효율적으로 업데이트되며,
     React에서는 key Props를 정확히 부여해야 이 Diffing이 올바르게 작동합니다.
     프로젝트에서 리스트의 key로 index 대신 고유 ID를 사용하여 버그를 해결한 경험이 있습니다."

이 패턴을 모든 질문에 적용!
  "정의 → 원리(키워드) → 실무 경험"
```

### 답변 길이

```
30초~1분이 이상적!

  너무 짧으면: "모르는 것 같은데..." 😰
  너무 길면: "포인트가 뭐지?" 면접관 피로!

  1문장 정의 + 2~3문장 원리 + 1문장 실무 = 30초!
  + 면접관이 꼬리 질문하면 → 더 깊게!
```

## 3. "모르겠습니다"를 말하는 법 🔴

> 모르는 걸 아는 척하면 → 꼬리 질문에서 100% 들통남!
> 솔직하게 모른다고 말하되, **학습 의지**를 보여주자!

```
❌ 최악의 답변:
  "모르겠습니다." (끝!)

❌ 나쁜 답변:
  "음... 그게... 아마... 뭔가..." (시간만 끌기!)

✅ 좋은 답변:
  "정확한 동작 원리는 모르지만,
   제가 이해하기로는 ~라고 알고 있습니다.
   면접 후에 공식 문서를 확인해보겠습니다."

✅ 더 좋은 답변:
  "해당 기술은 아직 직접 사용해보지 못했지만,
   비슷한 개념인 ~는 프로젝트에서 사용한 경험이 있습니다.
   ~와 어떤 차이가 있는지 공부해보겠습니다."

핵심: 솔직함 + 유사 경험 연결 + 학습 의지!
```

## 4. 경험 기반 질문 — STAR 프레임워크 🔴

> "프로젝트에서 겪은 어려운 문제를 설명해주세요"

```
STAR 프레임워크:

  S (Situation - 상황):
    "쇼핑몰 프로젝트에서 상품 목록 페이지의
     Lighthouse 성능 점수가 45점이었습니다."

  T (Task - 과제):
    "배포 전까지 성능을 90점 이상으로 개선해야 했습니다."

  A (Action - 행동):
    "React DevTools Profiler로 분석한 결과,
     상품 이미지가 최적화 없이 원본(5MB)으로 로드되고 있었습니다.
     next/image로 교체하여 WebP 자동 변환 + Lazy Loading을 적용했고,
     불필요한 리렌더링을 React.memo와 useMemo로 최적화했습니다."

  R (Result - 결과):
    "Lighthouse 점수가 45 → 96으로 개선되었고,
     LCP가 4.2초에서 1.1초로 단축되었습니다."

면접관이 좋아하는 것:
  ✅ 구체적인 수치! (45→96, 4.2초→1.1초!)
  ✅ 문제 해결 과정! (분석→원인 발견→해결!)
  ✅ 사용한 도구! (DevTools, Profiler, Lighthouse!)
```

## 5. 라이브 코딩 대비 🔴

### 자주 나오는 라이브 코딩 과제

```
초급:
  - 카운터 (useState)
  - TODO 리스트 (CRUD)
  - 검색 필터 (input + filter)

중급:
  - API 호출 + 목록 표시 (fetch + loading/error)
  - 무한 스크롤 (Intersection Observer)
  - 디바운스 검색 (useDebounce)
  - 폼 유효성 검사

고급:
  - 상태 관리 설계 (Context or Zustand)
  - 커스텀 훅 추출
  - 성능 최적화 (memo, useMemo, useCallback)
```

### 라이브 코딩 팁

```
1. 시작 전에 구조를 말해라! (3분!)
   "먼저 컴포넌트 구조를 설명드리겠습니다.
    TodoList(부모) → TodoItem(자식) → AddTodo(입력폼)
    상태는 TodoList에서 관리하고 Props로 내려주겠습니다."
   → 면접관에게 "이 사람은 설계부터 하는구나!" 인상!

2. 타입부터 정의해라! (TypeScript!)
   interface Todo {
     id: number;
     text: string;
     completed: boolean;
   }
   → "타입을 먼저 잡는 습관이 있구나!" 가산점!

3. 완성보다 과정이 중요!
   → 30분 안에 완벽하게 못 끝내도 OK!
   → "여기까지 구현했고, 시간이 더 있으면 ~를 추가하겠습니다."
   → 사고 과정을 보여주는 것이 핵심!

4. 입으로 설명하며 코딩!
   → 조용히 코딩하지 마! 면접관이 답답해함!
   → "지금 상태를 배열로 관리하고, map으로 순회하겠습니다."
   → "여기서 key는 index 대신 id를 사용하겠습니다."

5. 에러가 나면 당황하지 마!
   → "콘솔에 이런 에러가 나네요. 원인을 확인해보겠습니다."
   → 디버깅 과정 자체가 면접 평가 대상!
```

## 6. 역질문 — 면접의 마지막 인상! 🔴

> "혹시 저희에게 질문 있으신가요?" → 이 순간이 매우 중요!

```
✅ 좋은 역질문 (관심과 열정!):

  "팀에서 현재 사용하시는 기술 스택이 궁금합니다.
   Next.js App Router를 사용하시나요?"

  "프론트엔드 팀의 코드 리뷰 프로세스가 어떻게 되나요?"

  "신입 개발자에게 기대하는 온보딩 기간이 어느 정도인가요?"

  "현재 팀에서 가장 큰 기술적 챌린지는 무엇인가요?"

  "테스트 코드를 어느 정도 작성하시나요?"

❌ 나쁜 역질문:

  "질문 없습니다." (관심 없어 보임!)
  "연봉이 얼마인가요?" (처우는 나중에!)
  "야근이 많나요?" (일하기 싫어 보임!)
  "언제 결과가 나오나요?" (불안해 보임!)
```

## 8. 면접 후 복기

```
면접이 끝나면 바로 기록!

  1. 받았던 질문 목록!
     → 다음 면접에 대비!
     → 못 답한 질문은 공부!

  2. 내 답변 중 아쉬웠던 것!
     → "이렇게 답할 걸..." 정리!
     → 다음에는 더 좋은 답변!

  3. 회사/팀 분위기!
     → 면접관의 태도, 질문 스타일
     → "여기서 일하고 싶은가?" 판단 기준!

  4. 감사 메일/메시지 (선택!)
     → "오늘 좋은 경험이었습니다. 감사합니다."
     → 짧게! 부담 안 되게!
```

## 9. 프론트엔드 취업 로드맵

```
Phase 1: 기초 (1~2개월)
  → HTML/CSS/JS 기본기
  → cc 커리큘럼 Part 1~2 완주!

Phase 2: React + Next.js (2~3개월)
  → cc 커리큘럼 Part 3~4 완주!
  → 실습 프로젝트 1~2개!

Phase 3: 포트폴리오 (1~2개월)
  → 메인 프로젝트 1개 풀스택으로!
  → README, 트러블 슈팅, 배포!
  → GitHub 프로필 정리!

Phase 4: 면접 준비 (2~4주)
  → 면접 질문 100선 연습!
  → 라이브 코딩 연습!
  → STAR 프레임워크로 경험 정리!

Phase 5: 지원! (계속!)
  → 이력서 + 포트폴리오 제출!
  → 서류 탈락해도 멈추지 마!
  → 10곳 이상 지원이 기본!
  → 매 면접 후 복기 → 개선!

취업까지 평균: 4~8개월
  (사람마다 다름! 꾸준함이 핵심!)
```
