## 1. Vercel이 뭔데? 🔴

> 💡 **비유: 자동 주차장!**
> GitHub에 코드를 올리면(push) → Vercel이 알아서 빌드 → 배포 → URL 생성!
> 수동으로 서버를 세팅하는 것 = 직접 주차하는 것
> Vercel = 차를 맡기면 알아서 주차해주는 발렛 파킹!

```
Vercel = Next.js를 만든 회사의 배포 플랫폼!

왜 Vercel인가?
  ✅ Next.js 최적화 1등! (만든 회사니까!)
  ✅ GitHub push만 하면 자동 배포!
  ✅ PR마다 프리뷰 URL 자동 생성!
  ✅ HTTPS 자동 적용!
  ✅ 전 세계 CDN으로 빠른 로딩!
  ✅ 무료 플랜으로 충분! (개인 프로젝트!)
```

## 2. 첫 배포 Step-by-Step 🔴

### Step 1: GitHub에 프로젝트 올리기

```bash
# 아직 GitHub에 안 올렸다면:
git init
git add .
git commit -m "feat: 초기 프로젝트 셋업"
git remote add origin https://github.com/내아이디/my-next-app.git
git push -u origin main
```

### Step 2: Vercel 가입 & 연동

```
1. https://vercel.com 접속
2. "Sign Up" → "Continue with GitHub" ⭐
   → GitHub 계정으로 로그인!
3. "Add New Project" 클릭
4. GitHub 저장소 목록에서 my-next-app 선택!
5. "Import" 클릭!
```

### Step 3: 프로젝트 설정

```
Project Name: my-next-app (자동 입력됨!)
Framework Preset: Next.js (자동 감지!) ⭐
Root Directory: ./ (기본!)
Build Command: npm run build (자동!)
Output Directory: .next (자동!)

→ "Deploy" 버튼 클릭!
→ 1~2분 후 배포 완료! 🎉
→ URL: https://my-next-app-xxxxx.vercel.app
```

### Step 4: 이후 배포 (자동!)

```
이제부터는:
  1. 코드 수정!
  2. git add . && git commit -m "feat: 새 기능"
  3. git push origin main
  4. → Vercel이 자동으로 빌드 + 배포! ⚡
  5. → 1~2분 후 반영!

즉, git push가 곧 배포! 🚀
```

## 3. 환경 변수 설정 🔴

> ⚠️ `.env.local`에 있는 API 키, DB 비밀번호 등은 GitHub에 안 올림!
> → Vercel 대시보드에서 별도로 설정해야 함!

```
Vercel 대시보드 → 프로젝트 → Settings → Environment Variables

추가할 변수 예시:
  Key: DATABASE_URL
  Value: postgresql://user:password@host:5432/db

  Key: NEXTAUTH_SECRET
  Value: super-secret-key-here

  Key: NEXT_PUBLIC_API_URL
  Value: https://api.myshop.com

환경별 설정:
  Production: 실제 서비스용 값!
  Preview: PR 프리뷰용 값! (테스트 DB 등!)
  Development: 로컬 개발용! (보통 .env.local에!)
```

### .env 파일 종류 정리

```
.env                → 모든 환경 공통! (Git에 올려도 OK, 비밀 아닌 것만!)
.env.local          → 로컬 전용! (Git에 안 올림! .gitignore에 포함!)
.env.production     → 프로덕션 전용!
.env.development    → 개발 전용!

⚠️ NEXT_PUBLIC_ 접두사:
  NEXT_PUBLIC_API_URL → 브라우저에서 접근 가능! (공개!)
  DATABASE_URL        → 서버에서만 접근! (비공개!)

규칙:
  ✅ 공개해도 되는 것 → NEXT_PUBLIC_ 접두사!
  ✅ 비밀 (DB, API 키) → 접두사 없이! (서버 전용!)
  ❌ .env.local을 Git에 올리지 마! (.gitignore 확인!)
```

## 4. 프리뷰 배포 (Preview Deployment) 🟡

> 💡 **PR을 만들면 자동으로 미리보기 URL 생성!**

```
팀 협업 흐름:

  1. feature/new-design 브랜치에서 작업
  2. GitHub에 PR 생성!
  3. → Vercel이 자동으로 이 PR의 코드를 빌드!
  4. → 프리뷰 URL 생성! (예: my-app-abc123-team.vercel.app)
  5. → PR 코멘트에 프리뷰 링크 자동 추가!
  6. 팀원이 프리뷰 URL에서 직접 확인!
  7. "LGTM! 머지!" → main에 머지 → 프로덕션 자동 배포!

이점:
  → 코드 리뷰 시 실제 화면을 볼 수 있음! ⭐
  → "이거 디자인 어떤가요?" → 프리뷰 링크 공유!
  → 디자이너/기획자도 확인 가능!
```

## 5. 커스텀 도메인 연결 🟡

```
Vercel 대시보드 → 프로젝트 → Settings → Domains

1. "Add Domain" → myshop.com 입력
2. Vercel이 DNS 설정을 안내!
3. 도메인 등록 업체(가비아, Route53 등)에서:
   - A 레코드: 76.76.21.21
   - CNAME: cname.vercel-dns.com
4. 몇 분 후 HTTPS 자동 적용! 🔒

결과:
  https://myshop.com → Vercel이 서빙!
  https://www.myshop.com → 자동 리다이렉트!
```

## 6. 빌드 에러 디버깅 🔴

```
Vercel에서 빌드 실패 시:

  1. Vercel 대시보드 → Deployments → 실패한 배포 클릭!
  2. "Build Logs" 탭에서 에러 메시지 확인!

흔한 에러:

  ❌ "Module not found"
  → 로컬에선 되는데 Vercel에서 안 됨!
  → 원인: import 경로의 대소문자! (Mac은 대소문자 무시, Linux는 구분!)
  → 해결: import 경로를 정확한 대소문자로!

  ❌ "Environment variable not found"
  → Vercel에 환경 변수를 안 설정했음!
  → 해결: Vercel 대시보드 → Settings → Environment Variables!

  ❌ "Type error"
  → 로컬에서 npm run build를 안 해보고 push!
  → 해결: push 전에 반드시 npm run build 확인! ⭐

  ❌ "Build exceeded timeout"
  → 빌드가 너무 오래 걸림!
  → 해결: 불필요한 패키지 제거, 이미지 최적화!
```

### 배포 전 체크리스트

```
git push 전에 반드시:

  ✅ npm run build → 에러 없는지!
  ✅ npm run lint → 경고/에러 없는지!
  ✅ 환경 변수가 Vercel에 설정되어 있는지!
  ✅ .env.local이 .gitignore에 있는지!
  ✅ 이미지에 width/height가 있는지! (CLS 방지!)
```

## 실습

```
과제: Next.js 프로젝트를 Vercel에 배포!

1. GitHub에 프로젝트 push
2. Vercel 가입 (GitHub 연동!)
3. 프로젝트 Import → Deploy!

5. 환경 변수 1개 이상 추가! (NEXT_PUBLIC_SITE_NAME 등)
6. 코드 수정 → git push → 자동 재배포 확인!

```
