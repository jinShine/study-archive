## 1. SEO가 뭔데? 왜 중요한데? 🔴

> 💡 **비유: 가게 간판!**
> 아무리 맛있는 식당이라도 간판이 없으면 → 아무도 못 찾음!
> SEO = 구글/네이버에서 내 사이트가 잘 검색되게 하는 것!

```
SEO = Search Engine Optimization (검색 엔진 최적화)

검색 엔진(구글)이 하는 일:
  1. 크롤링: 로봇이 웹사이트를 돌아다니며 페이지를 수집!
  2. 인덱싱: 수집한 페이지를 분류하여 데이터베이스에 저장!
  3. 랭킹: 검색어에 맞는 페이지를 순위대로 보여줌!

SEO가 좋으면:
  → 구글 검색 1페이지에 노출!
  → 무료로 트래픽(방문자)이 몰림!
  → 광고비 0원으로 마케팅!

Next.js가 SEO에 유리한 이유:
  → SSR/SSG로 완성된 HTML을 제공! (검색 로봇이 읽을 수 있음!)
  → CSR(React SPA)은 빈 HTML → 로봇이 내용을 못 읽음! ❌
```

## 2. 정적 메타데이터 (metadata 객체) 🔴

```tsx
// app/layout.tsx 또는 app/page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  // 브라우저 탭 제목 + 구글 검색 결과 제목!
  title: "My Shop | 최고의 전자제품 쇼핑몰",

  // 구글 검색 결과에서 제목 아래 나오는 설명!
  description: "최신 노트북, 키보드, 마우스를 최저가로! 무료 배송!",

  // 검색에 사용할 키워드
  keywords: ["노트북", "키보드", "전자제품", "쇼핑몰"],

  // 소셜 미디어 공유 시 표시! (카카오톡, 슬랙 등!)
  openGraph: {
    title: "My Shop | 전자제품 특가!",
    description: "지금 최대 50% 할인 중!",
    url: "https://myshop.com",
    siteName: "My Shop",
    images: [
      {
        url: "https://myshop.com/og-image.jpg",
        width: 1200,
        height: 630,
        alt: "My Shop 메인 이미지",
      },
    ],
    type: "website",
  },

  // 트위터(X) 카드
  twitter: {
    card: "summary_large_image",
    title: "My Shop | 전자제품 특가!",
    description: "지금 최대 50% 할인 중!",
    images: ["https://myshop.com/og-image.jpg"],
  },

  // 검색 로봇 지시
  robots: {
    index: true,    // 검색 결과에 포함!
    follow: true,   // 페이지 내 링크를 따라가기!
  },
};
```

## 3. 동적 메타데이터 (generateMetadata) 🔴

> 상품 상세처럼 URL마다 제목/설명이 달라야 할 때!

```tsx
// app/products/[id]/page.tsx
import type { Metadata } from "next";

type Props = {
  params: Promise<{ id: string }>;
};

// generateMetadata = URL의 id에 따라 다른 메타데이터!
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const { id } = await params;

  // DB나 API에서 상품 정보 가져오기!
  const product = await fetch(`https://api.myshop.com/products/${id}`)
    .then((res) => res.json());

  return {
    title: `${product.name} | My Shop`,
    description: product.description.slice(0, 160),
    // 160자 제한! 구글이 이 정도만 표시!

    openGraph: {
      title: `${product.name} - 특가 판매 중!`,
      description: `${product.name}을 최저가로 만나보세요!`,
      images: [product.imageUrl],
    },
  };
}

// 💡 fetch 중복 걱정 NO!
// generateMetadata와 page 컴포넌트에서 같은 URL을 fetch해도
// Next.js가 자동으로 Request Memoization! (1번만 실행!)
```

## 4. Open Graph (OG) — 소셜 공유의 꽃! 🔴

```
카카오톡에 링크를 보내면 예쁜 카드가 뜨는 이유!
  → 바로 이 Open Graph 태그 덕분!

OG 태그가 없으면:
  카카오톡에 링크 전송 → 텍스트 URL만 덩그러니... 😞

OG 태그가 있으면:
  카카오톡에 링크 전송 → 썸네일 + 제목 + 설명 카드! 🎉
  → 클릭률(CTR) 10배 이상 차이!

OG 이미지 권장 크기:
  1200 x 630px (가로가 긴 직사각형!)
  → 카카오톡, 슬랙, X(트위터), 페이스북 전부 호환!
```

## 5. sitemap.xml — 검색 로봇의 지도! 🔴

```tsx
// app/sitemap.ts
import type { MetadataRoute } from "next";

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: "https://myshop.com",
      lastModified: new Date(),
      changeFrequency: "daily",
      priority: 1.0,      // 가장 중요!
    },
    {
      url: "https://myshop.com/products",
      lastModified: new Date(),
      changeFrequency: "daily",
      priority: 0.8,
    },
    {
      url: "https://myshop.com/about",
      lastModified: new Date(),
      changeFrequency: "monthly",
      priority: 0.5,
    },
  ];
}
// → https://myshop.com/sitemap.xml 으로 자동 생성!

// 동적 상품 페이지도 포함!
export default async function sitemap(): MetadataRoute.Sitemap {
  const products = await fetch("https://api.myshop.com/products").then(r => r.json());

  const productUrls = products.map((p: { id: number; updatedAt: string }) => ({
    url: `https://myshop.com/products/${p.id}`,
    lastModified: new Date(p.updatedAt),
  }));

  return [
    { url: "https://myshop.com", lastModified: new Date(), priority: 1.0 },
    ...productUrls,
  ];
}
```

## 6. robots.txt — 로봇에게 규칙 알려주기! 🟡

```tsx
// app/robots.ts
import type { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: "*",        // 모든 검색 로봇!
      allow: "/",            // 전부 크롤링 허용!
      disallow: [
        "/admin/",           // 관리자 페이지는 크롤링 금지!
        "/api/",             // API 엔드포인트도 금지!
        "/private/",         // 비공개 페이지도 금지!
      ],
    },
    sitemap: "https://myshop.com/sitemap.xml",
    // 사이트맵 위치를 로봇에게 알려줌!
  };
}
// → https://myshop.com/robots.txt 로 자동 생성!
```

## 7. 시맨틱 HTML과 SEO 🟡

```tsx
// ❌ div만 쓰면 검색 로봇이 구조를 모름!
<div>
  <div>My Shop</div>
  <div>
    <div>상품 목록</div>
    <div>노트북...</div>
  </div>
  <div>Copyright 2026</div>
</div>

// ✅ 시맨틱 태그로 구조를 알려줌!
<header>
  <nav>My Shop</nav>           {/* "이건 네비게이션!" */}
</header>
<main>                          {/* "이건 핵심 콘텐츠!" */}
  <h1>상품 목록</h1>            {/* "이건 제목!" */}
  <article>                     {/* "이건 독립된 콘텐츠!" */}
    <h2>노트북</h2>
    <p>최신 노트북...</p>
  </article>
</main>
<footer>Copyright 2026</footer> {/* "이건 하단 정보!" */}

SEO 효과:
  → 검색 로봇이 페이지 구조를 정확히 이해!
  → h1 태그 = "이 페이지의 핵심 주제!" (페이지당 1개!)
  → main 태그 = "이게 본문이야!" (광고/네비 제외!)
```

## 8. SEO 체크리스트 🔴

```
필수:
  ✅ metadata 또는 generateMetadata 설정!
  ✅ 모든 페이지에 고유한 title + description!
  ✅ Open Graph 태그! (소셜 공유!)
  ✅ sitemap.xml 생성!
  ✅ robots.txt 설정!
  ✅ h1 태그는 페이지당 1개!
  ✅ 모든 이미지에 alt 속성!
  ✅ next/image 사용! (LCP 최적화!)
  ✅ HTTPS 필수! (Vercel 자동!)

권장:
  ✅ 시맨틱 HTML 사용! (header, nav, main, footer, article)
  ✅ URL을 의미 있게! (/products/laptop ✅ / /p/123 ❌)
  ✅ Core Web Vitals 점수 90 이상!
  ✅ 모바일 친화적! (반응형!)
```

### 🔴 필수 질문

**Q1. Next.js에서 SEO를 어떻게 최적화하나요?**

> 첫째, SSR/SSG로 완성된 HTML을 제공하여 검색 로봇이 콘텐츠를 읽을 수 있게 한다. 둘째, `metadata` 객체나 `generateMetadata` 함수로 각 페이지의 title, description, Open Graph 태그를 설정한다. 셋째, `sitemap.ts`와 `robots.ts`를 생성하여 검색 로봇에게 사이트 구조를 알려준다. 넷째, `next/image`와 `next/font`로 Core Web Vitals(LCP, CLS)를 최적화하여 구글 검색 순위를 높인다.
