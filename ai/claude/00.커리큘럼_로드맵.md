# Claude Code 하네스 엔지니어링 — 마스터 로드맵

> **목표**: Claude Code를 단순한 AI 채팅이 아니라, **나만의 개발팀처럼** 세팅하고 부리는 것
> **대상**: Claude Code를 처음 쓰거나, 쓰고는 있지만 "이거 제대로 쓰고 있나?" 싶은 사람
> **두 가지 관점**: 🧑‍💻 1인 개발자 (혼자 팀을 만든다) + 👔 팀 리더 (팀원에게 환경을 깔아준다)

---

## 비유로 이해하기

```
일반 사용자:     "클로드야, 이거 만들어줘"  →  매번 처음부터 설명
                 (매번 새 알바를 고용하는 것)

하네스 엔지니어:  CLAUDE.md + Hooks + Skills + Agents + MCP
                 (전담 팀을 꾸려놓고, 각자 역할이 있는 것)

🧑‍💻 [1인 개발자 모드]                    👔 [팀 리더 모드]

┌──────────────────────────┐    ┌──────────────────────────┐
│     🎯 나 (1인 개발자)      │    │     🎯 나 (팀 리더)        │
│          │                │    │          │                │
│   ┌──────┼──────┐        │    │   ┌──────┼──────┐        │
│   ▼      ▼      ▼        │    │   ▼      ▼      ▼        │
│ Agent  Agent  Agent      │    │ 팀원A  팀원B  팀원C      │
│ (리뷰) (테스트) (보안)     │    │   │      │      │        │
│   │      │      │        │    │   ▼      ▼      ▼        │
│   ▼      ▼      ▼        │    │  각자의 Claude Code       │
│   MCP 서버들 (외부 도구)   │    │  (내가 깔아준 세팅으로)    │
└──────────────────────────┘    └──────────────────────────┘

공통 도구:
📋 CLAUDE.md    = 팀 규칙서 (이 프로젝트는 이렇게 해)
🪝 Hooks        = 자동 반응 (코드 저장하면 자동 포맷)
⚡ Skills       = 전문 기술서 (TDD는 이 절차로)
🤖 Agents       = 전문가 역할 (코드리뷰어, 테스터)
🔌 MCP          = 외부 도구 연결 (Slack, GitHub 등)
```

## Phase 7: 팀 리더의 하네스 엔지니어링 🔴 👔

> "맛집 사장이 레시피만 잘 만들면 끝이 아니다.
> 알바생 누가 와도 그 맛이 나오게 **시스템을 깔아주는 것**이 진짜 실력이다.
> 팀 리더의 하네스 엔지니어링도 똑같다."

```
❌ 나쁜 팀 리더:                        ✅ 좋은 팀 리더:

"클로드 코드 쓰세요"                     프로젝트 CLAUDE.md 세팅 완료
"알아서 잘 하세요"                       공통 Hooks로 자동 린트/포맷
"코드 리뷰는 각자"                       코드리뷰 에이전트 + 스킬 배포
                                        팀 온보딩 가이드 준비
결과: 팀원마다 다른 품질               결과: 누가 해도 같은 품질
```

| # | 주제 | 배울 것 |
|---|------|---------|
| 23 | 팀 프로젝트 CLAUDE.md 설계 | 공유 규칙 vs 개인 설정 분리, 계층 구조 전략 |
| 24 | 팀 공통 환경 세팅 | 공유 Hooks, 공유 MCP, 공유 커맨드 배포 |
| 25 | 팀 온보딩 & 가이드 | 팀원 온보딩 자동화, 학습 곡선 줄이기 |
| 26 | 팀 워크플로우 & 품질 관리 | PR 리뷰 자동화, 코드 품질 게이트, 모니터링 |

**이 Phase를 끝내면**: 팀에 Claude Code를 도입하고, 팀원 누가 와도 일정한 품질이 나오는 시스템을 만들 수 있다.

```
👔 팀 리더가 깔아주는 것들:

┌─────────────────── 프로젝트 저장소 ───────────────────┐
│                                                       │
│  📋 CLAUDE.md (프로젝트 공통 규칙)                      │
│     "이 프로젝트는 TypeScript, 테스트 필수, ..."        │
│                                                       │
│  📁 .claude/                                          │
│     ├── commands/        ← 팀 공통 슬래시 커맨드        │
│     │   ├── review.md    (코드 리뷰 절차)              │
│     │   └── deploy.md    (배포 체크리스트)              │
│     ├── agents/          ← 팀 전용 에이전트             │
│     │   └── pr-reviewer.md                            │
│     └── settings.json    ← 프로젝트 권한/MCP 설정      │
│                                                       │
│  팀원 A, B, C 누가 clone해도 → 동일한 Claude 환경!     │
└───────────────────────────────────────────────────────┘

┌─────────── 각 팀원의 개인 설정 ──────────┐
│  ~/.claude/                              │
│     ├── CLAUDE.md     (개인 스타일)       │
│     └── settings.json (개인 권한)        │
│                                          │
│  프로젝트 규칙 + 개인 설정 = 합쳐서 적용   │
└──────────────────────────────────────────┘
```

## 학습 방식

```
너 (강의 수강)  →  키워드/자료 던짐  →  나 (정리 + 예제)  →  ai/claude/에 저장
     │                                      │
     │         "이 개념 이해 안 돼"           │
     └──────────────────────────────────────┘
              추가 설명 + 예제 보강
```

**규칙:**
1. 각 주제는 하나의 `.md` 파일로 정리
2. 반드시 **실행 가능한 예제** 포함 (복사-붙여넣기로 바로 따라할 수 있는)
3. 비유 먼저 → 정의 나중에
4. 🔴/🟡/🟢 우선순위 표시
5. 강의에서 배운 내용 + 내가 보충한 내용 구분

## 부록 A: 실전 생태계 — 사람들은 뭘 쓰나?

> "요리를 배우려면 레시피도 중요하지만, 프로 셰프들이 어떤 칼과 팬을 쓰는지 아는 것도 중요하다."

### 🔌 많이 쓰는 MCP 서버 TOP 10

```
Claude가 기본으로 할 수 있는 것:  파일 읽기/쓰기, 터미널 명령, 웹 검색
MCP를 달아주면 할 수 있는 것:    Slack 메시지, GitHub PR, DB 쿼리, 브라우저 조작, ...

MCP = Claude에게 새로운 "손"을 달아주는 것
```

| MCP 서버 | 한 줄 설명 | 쓰는 상황 |
|----------|-----------|-----------|
| **GitHub** | PR/이슈/리포 관리 | 코드 리뷰, PR 자동 생성, 이슈 관리 |
| **Playwright** | 브라우저 자동 조작 | E2E 테스트, 웹 스크래핑, UI 검증 |
| **PostgreSQL** | DB 직접 쿼리 | 스키마 탐색, 데이터 확인, 쿼리 디버깅 |
| **Supabase** | DB + 인증 + 스토리지 통합 | 풀스택 개발, BaaS 연동 |
| **Context7** | 프레임워크 최신 문서 주입 | "React 19 문법 알려줘" → 최신 문서 기반 답변 |
| **Figma** | 디자인 → 코드 변환 | 디자인 토큰 추출, 레이아웃 코드 생성 |
| **Sentry** | 에러/크래시 진단 | 프로덕션 버그 추적, 스택 트레이스 분석 |
| **Slack** | 메시지 읽기/보내기 | 팀 알림, 배포 상태 공유 |
| **Jira** | 티켓 관리, JQL 검색 | 이슈 트래킹, 스프린트 관리 |
| **Leapfrog** | 15개 병렬 브라우저 | 대규모 웹 테스트, 멀티 세션 |

> 💡 MCP 서버 모음집: [awesome-mcp-servers](https://github.com/wong2/awesome-mcp-servers) (커뮤니티 큐레이션)

### ⚡ 많이 쓰는 스킬 & 커맨드

```
커맨드 = "이 절차대로 해줘" (단순 프롬프트)
스킬   = "이 전문 기술을 써줘" (더 정교한 워크플로우 + 도구 제한)
```

| 스킬/커맨드 | 설명 | 인기 이유 |
|------------|------|-----------|
| **/tdd** | Red→Green→Refactor TDD 워크플로우 | 테스트 먼저 짜는 습관 강제 |

### 🔧 하네스 프레임워크 — claude-forge

> "하네스를 하나하나 직접 만드는 건 좋지만,
> 검증된 프레임워크를 깔고 거기서 커스텀하는 게 더 빠르다.
> claude-forge는 **oh-my-zsh의 Claude Code 버전**이다."

```
❌ 처음부터 직접:                    ✅ 프레임워크 깔고 커스텀:

CLAUDE.md 작성                      git clone claude-forge
hooks 하나씩 세팅                    ./install.sh
에이전트 하나씩 만들기                 → 11개 에이전트
커맨드 하나씩 작성                    → 40개 커맨드
스킬 하나씩 작성                      → 15개 스킬
                                     → 학습하는 hooks
시간: 며칠                           → 즉시 사용, 필요한 것만 커스텀
```

**claude-forge 핵심 기능:**

| 기능 | 설명 |
|------|------|
| **원클릭 설치** | `./install.sh` → `~/.claude/`에 심링크 생성 |
| **에이전트 자기 학습** | 작업 후 `agent-memory/`에 인사이트 자동 저장 |
| **검증 강제** | 테스트/빌드 증거 없이 "완료" 주장 차단 |
| **과잉 수정 방지** | "요청한 것만 변경" 원칙 내장 |
| **워크플로우 체인** | `/plan → /tdd → /code-review → /commit-push-pr` |
| **업데이트** | `git pull`로 즉시 반영 |
