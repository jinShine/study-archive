# 12. Hooks 기초 — 자동화의 시작

> **키워드**: `Hooks` `PreToolUse` `PostToolUse` `Stop` `Notification` `/hooks` `자동 포맷`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | Hooks란, 4가지 타입, 실전 예시 | 진짜 자동화의 시작 |
| 🔴 필수 | 자동 포맷, 자동 린트 세팅 | 팀 표준화 |
| 🟡 이해 | 복잡한 조건 분기, 디버깅 | 고급 활용 |

---

## 1. Hooks란? 🔴

```
비유: 센서 + 자동반응 시스템

문 열기 센서: 사람이 들어오면 → 자동으로 불이 켜짐
                                → 자동으로 경보음
                                → 자동으로 로그 기록

Claude의 Hooks도 똑같다:
Claude가 파일 쓰기 작업 → 자동으로 Prettier 포맷
Claude가 세션 종료      → 자동으로 커밋 체크
Claude가 도구 사용 전    → 자동으로 보안 검증
```

### 왜 필요한가?

```
❌ Hook 없이:
   "CLAUDE.md에 '코드 저장 후 포맷팅 해줘'라고 써둠"
   → Claude가 가끔 까먹음
   → 팀원마다 포맷이 다름

✅ Hook 있으면:
   PostToolUse Hook: "Write 도구 쓴 후에 prettier 실행"
   → 물리적으로 100% 실행됨
   → 팀 전체 동일한 결과

💡 핵심 원칙:
"프롬프트로 품질 요청하지 말고, 메커니즘으로 강제하라"
```

---

## 2. Hooks의 4가지 타입 🔴

```
┌────────── Hook 발동 시점 ──────────┐
│                                    │
│  PreToolUse   → 도구 사용 "직전"    │
│                 (실행 전 검증 용도)  │
│                                    │
│  PostToolUse  → 도구 사용 "직후"    │
│                 (포맷, 린트 용도)    │
│                                    │
│  Stop         → Claude 응답 "종료"  │
│                 (정리, 로그 용도)    │
│                                    │
│  Notification → 알림 발생 시         │
│                 (데스크톱 알림 등)   │
│                                    │
└────────────────────────────────────┘
```

### 각 타입 실전 용도

| Hook | 언제 발동 | 쓰는 상황 |
|------|-----------|-----------|
| **PreToolUse** | Claude가 도구 쓰기 직전 | 위험한 명령 차단, 사전 검증 |
| **PostToolUse** | 도구 사용 완료 직후 | 자동 포맷, 린트, 테스트 |
| **Stop** | Claude가 응답 끝낼 때 | 세션 요약, 자동 커밋 체크 |
| **Notification** | 권한 요청 등 알림 | 데스크톱 알림, 소리 |

## 3. Hook 설정 방법 🔴

### 저장 위치

```
settings.json 의 hooks 필드에 작성:

전역:     ~/.claude/settings.json
프로젝트: .claude/settings.json
```

### 기본 구조

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$CLAUDE_FILE\""
          }
        ]
      }
    ]
  }
}
```

### 구조 해설

```
PostToolUse           ← Hook 타입 (언제 발동할지)
  matcher: "Write|Edit"  ← 어떤 도구에 반응할지 (정규식)
  hooks:
    type: "command"    ← Hook 종류 (셸 명령 실행)
    command: "..."     ← 실제 실행할 명령어
```

## 4. 실전 예시 — 가장 많이 쓰는 Hook 5가지 🔴

### 예시 1: 파일 저장 시 자동 Prettier 포맷

**파일**: `.claude/settings.json`

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$CLAUDE_FILE_PATH\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

**효과:**
- Claude가 파일을 쓸 때마다 자동으로 Prettier 실행
- 팀 전체 동일한 포맷
- Claude에게 "포맷해줘" 요청 불필요

---

### 예시 2: 파일 저장 시 자동 린트

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx eslint --fix \"$CLAUDE_FILE_PATH\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

**효과:**
- 린트 오토픽스 자동 실행
- 기본적인 스타일 문제는 자동 수정

---

### 예시 3: 위험한 명령어 사전 차단

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "if echo \"$CLAUDE_COMMAND\" | grep -E 'rm -rf /|DROP TABLE|FORCE PUSH'; then echo 'DANGEROUS COMMAND BLOCKED' && exit 1; fi"
          }
        ]
      }
    ]
  }
}
```

**효과:**
- `rm -rf /`, `DROP TABLE`, force push 등 위험 명령 차단
- settings.json의 `deny`와 이중 안전장치

---

### 예시 4: 세션 종료 시 TODO 체크

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "grep -rn 'TODO\\|FIXME' src/ | tail -5 || true"
          }
        ]
      }
    ]
  }
}
```

**효과:**
- 세션 끝날 때 미처리 TODO 리스트 표시
- 잊어버리는 것 방지

---

### 예시 5: 작업 완료 알림 (macOS)

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude 작업 완료\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}
```

**효과:**
- 긴 작업 끝나면 데스크톱 알림
- 다른 작업하고 있어도 놓치지 않음

```bash
# 현재 설정된 Hook 보기

# → 각 Hook의 matcher와 command 확인
```

## 6. 환경변수 — Hook에서 쓸 수 있는 값들 🟡

```
Hook 안에서 사용 가능한 환경변수:

$CLAUDE_FILE_PATH   파일 경로 (Write, Edit 도구)
$CLAUDE_COMMAND     실행할 명령어 (Bash 도구)
$CLAUDE_TOOL_NAME   도구 이름
$CLAUDE_SESSION_ID  현재 세션 ID
```

### 활용 예시

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo '[$(date)] $CLAUDE_TOOL_NAME on $CLAUDE_FILE_PATH' >> .claude/activity.log"
          }
        ]
      }
    ]
  }
}
```

→ 모든 파일 수정을 로그 파일에 기록!

## 7. 복잡한 조건 분기 🟡

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "case \"$CLAUDE_FILE_PATH\" in *.ts|*.tsx) npx prettier --write \"$CLAUDE_FILE_PATH\" && npx eslint --fix \"$CLAUDE_FILE_PATH\" ;; *.py) ruff format \"$CLAUDE_FILE_PATH\" && ruff check --fix \"$CLAUDE_FILE_PATH\" ;; *.go) gofmt -w \"$CLAUDE_FILE_PATH\" ;; esac"
          }
        ]
      }
    ]
  }
}
```

**효과:**
- TypeScript → Prettier + ESLint
- Python → Ruff
- Go → gofmt
- 파일 확장자별로 다른 포맷터 실행!

## 8. 팀 리더 전략 — Hook으로 품질 강제 👔

### 전략 1: PR 전 자동 검증

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "npm run lint && npm run type-check && npm test"
          }
        ]
      }
    ]
  }
}
```

→ Claude가 응답 끝낼 때 자동으로 린트/타입체크/테스트 실행!

### 전략 2: 팀 공통 Hook 배포

```
.claude/settings.json 에 Hook 정의
→ git 커밋
→ 팀원 누구나 clone하면 같은 Hook 적용

효과:
- 전원 동일한 포맷
- 전원 동일한 린트 규칙
- "네 컴퓨터에선 되는데" 사라짐
```

### 전략 3: Hook + 에이전트 조합

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "claude --agent code-reviewer --input \"마지막 변경사항 리뷰\""
          }
        ]
      }
    ]
  }
}
```

→ 응답 끝날 때마다 자동으로 code-reviewer 에이전트가 리뷰!

## 9. 디버깅 팁 🟡

### Hook이 동작 안 할 때

```bash
# 1. 설정 확인
/hooks
# → Hook이 등록됐는지 확인

# 2. 명령어 단독 실행
bash -c "실제 command 내용"
# → 명령어 자체에 문제 없는지 확인

# 3. 진단 도구
/doctor
# → 설정 오류 체크

# 4. 로그 확인
tail -f ~/.claude/logs/hooks.log
# → Hook 실행 로그
```

### 흔한 실수

```
❌ 경로에 공백 있는 파일에서 실패
→ ✅ "$CLAUDE_FILE_PATH" 처럼 따옴표로 감싸기

❌ 명령어 실패하면 Hook이 멈춤
→ ✅ 끝에 || true 추가 (실패해도 계속 진행)

❌ 너무 느린 명령어
→ ✅ 백그라운드 실행 (& 사용) 또는 비동기화
```

---

## 10. 주의사항 🔴

```
⚠️ Hook은 매번 실행된다 → 느리면 전체가 느려진다

❌ 피해야 할 Hook:
- 5초 이상 걸리는 작업 (타입체크 풀 실행 등)
- 네트워크 요청
- 확실하지 않은 명령

✅ 좋은 Hook:
- 1초 이내 완료
- 로컬에서만 실행
- 실패해도 안전 (|| true)

💡 무거운 작업은 Hook이 아니라 Stop 시점에 (세션 끝날 때)
```

## 실습: 내 첫 Hook 만들기

```bash
# 1. 간단한 로깅 Hook 추가
cat > ~/.claude/settings.json << 'EOF'
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "language": "korean",
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "echo '[Hook] $CLAUDE_FILE_PATH 수정됨' && date"
          }
        ]
      }
    ]
  }
}
EOF

# 2. Claude Code 재시작
claude

# 3. 확인
/hooks
# → Hook이 등록됐는지 확인

# 4. 테스트
test.md 파일을 만들어줘
# → Hook이 실행되면서 메시지가 출력됨

# 5. 실전 Hook으로 업그레이드 (프로젝트 폴더에서)
mkdir -p .claude
cat > .claude/settings.json << 'EOF'
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "npx prettier --write \"$CLAUDE_FILE_PATH\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
EOF
```

## 면접 대비

### 🔴 필수 질문

**Q: Claude Code의 Hooks란?**
> Claude가 도구를 사용하거나 응답을 완료할 때 자동으로 실행되는 셸 명령어입니다. PreToolUse, PostToolUse, Stop, Notification 4가지 타입이 있으며, 자동 포맷팅/린트/검증 등을 통해 "프롬프트로 요청하는 대신 메커니즘으로 품질을 강제"할 수 있습니다.

**Q: Hook과 CLAUDE.md 규칙의 차이는?**
> CLAUDE.md는 프롬프트 기반(Claude가 따르길 "바라는" 규칙)이라 가끔 무시될 수 있습니다. Hook은 settings.json에 정의된 시스템 레벨 자동화로, Claude의 행동과 무관하게 100% 실행됩니다. 중요한 규칙은 Hook으로 강제하는 것이 안전합니다.

## 핵심 요약

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  Hooks = Claude 행동에 자동으로 반응하는 시스템                │
│                                                              │
│  4가지 타입:                                                  │
│  PreToolUse   → 도구 쓰기 전 (검증, 차단)                     │
│  PostToolUse  → 도구 쓴 후 (포맷, 린트) ← 가장 많이 씀        │
│  Stop         → 응답 끝날 때 (정리, 알림)                     │
│  Notification → 알림 발생 (데스크톱 알림)                     │
│                                                              │
│  설정 위치: settings.json 의 hooks 필드                      │
│  관리 명령: /hooks                                            │
│                                                              │
│  환경변수:                                                    │
│  $CLAUDE_FILE_PATH  $CLAUDE_COMMAND  $CLAUDE_TOOL_NAME       │
│                                                              │
│  🎯 핵심 원칙:                                                │
│  "프롬프트로 요청하지 말고, Hook으로 강제하라"                 │
│                                                              │
│  주의: 1초 이내로 빨라야 함 (매번 실행됨)                     │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```
