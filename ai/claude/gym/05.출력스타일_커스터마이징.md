# 05. 출력 스타일 & 상태 표시줄 커스터마이징

> **키워드**: `/output-style` `/statusline` `커스텀 스타일` `keep-coding-instructions` `프론트매터`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 커스텀 출력 스타일 만들기, keep-coding-instructions | 매일 쓰는 Claude 톤 설정 |
| 🟡 이해 | 기본 스타일 종류, 프론트매터 옵션 | 활용 범위 넓히기 |
| 🟢 참고 | /statusline | 한 번 설정하면 끝 |

---

## 1. 출력 스타일이란? 🔴

```
비유: Claude의 "말투 설정"

같은 사람이라도:
  - 친구한테: "야, 이거 그냥 for문 돌려"
  - 면접에서: "반복 처리를 위해 for 루프를 활용합니다"
  - 선생님으로: "자, for문이 뭔지 먼저 알아보자. 비유하자면..."

Claude도 똑같이 말투를 바꿀 수 있다:
  - Concise:     짧고 핵심만
  - Explanatory: 설명 풍부하게
  - Teacher:     가르치듯이 (직접 만들기!)
```

### 기본 사용법

```bash
# 메뉴에서 선택
/output-style

# 직접 지정
/output-style concise       # 짧게
/output-style explanatory   # 설명 많이
```

## 2. 나만의 출력 스타일 만들기 🔴

### 파일 저장 위치

```
모든 프로젝트에서 사용 (전역):
~/.claude/output-styles/my-style.md

특정 프로젝트에서만 사용 (프로젝트 레벨):
프로젝트/.claude/output-styles/my-style.md
```

### 파일 구조

```markdown
---
name: 스타일 이름
description: 설명 (메뉴에 표시됨)
keep-coding-instructions: true
---

여기에 Claude한테 줄 지시사항을 쓴다
```

### 실전 예시 1: 한국어 친절 선생님 스타일

```markdown
---
name: 한국어 선생님
description: 코드를 가르치듯 설명하는 한국어 스타일
keep-coding-instructions: true
---

# 한국어 선생님 스타일

당신은 친절한 프로그래밍 선생님입니다.

## 규칙
- 모든 응답은 한국어로 작성합니다
- 새로운 개념은 일상 비유로 먼저 설명합니다
- 코드를 작성할 때 왜 이렇게 했는지 주석으로 설명합니다
- 어려운 부분은 "쉽게 말하면..." 으로 풀어줍니다
- 반말 톤을 사용합니다 ("~하자", "~이다")
```

저장: `~/.claude/output-styles/teacher-kr.md`

### 실전 예시 2: 시니어 개발자 스타일

```markdown
---
name: 시니어 리뷰어
description: 코드 리뷰하듯 날카롭지만 건설적인 피드백
keep-coding-instructions: true
---

# 시니어 리뷰어 스타일

당신은 10년차 시니어 개발자입니다.

## 규칙
- 코드를 볼 때 잠재적 버그와 성능 이슈를 먼저 짚습니다
- 좋은 점도 반드시 언급합니다
- 개선 사항은 구체적 코드 예시와 함께 제안합니다
- "왜" 그래야 하는지 이유를 항상 설명합니다
- 불필요한 칭찬이나 사과는 하지 않습니다
```

저장: `~/.claude/output-styles/senior-reviewer.md`

### 실전 예시 3: 간결 스타일 (팀용)

```markdown
---
name: 팀 간결
description: 팀 커뮤니케이션에 최적화된 간결한 스타일
keep-coding-instructions: true
---

# 팀 간결 스타일

## 규칙
- 불필요한 인사말, 사과, 요약 생략
- 코드 변경 시 변경 사항만 보여주기
- 질문에는 답만 하기
- 에러 발생 시 원인 → 해결 순서로 짧게
- 한국어로 응답
```

저장: `프로젝트/.claude/output-styles/team-concise.md`

### 사용법

```bash
# 메뉴에서 선택
/output-style

# 직접 전환
/output-style teacher-kr
/output-style senior-reviewer
/output-style team-concise
```

설정은 `.claude/settings.local.json`에 자동 저장 → 다음 세션에서도 유지!

## 3. keep-coding-instructions 완전 이해 🔴

이 옵션 하나가 **가장 중요하다.**

```
keep-coding-instructions: true  (권장!)
= Claude의 코딩 능력은 그대로 + 내 스타일만 추가

keep-coding-instructions: false (기본값, 주의!)
= Claude의 코딩 능력 전부 제거 + 내 스타일로 대체
```

### 비유로 이해

```
true  = 베테랑 개발자에게 "말투만 바꿔줘"
        → 코딩 실력은 그대로, 말투만 달라짐 ✅

false = 베테랑 개발자에게 "개발 경험 다 잊어, 새로 시작해"
        → 코딩 실력이 사라짐 ❌ (특수 용도 아니면 위험!)
```

### 언제 false를 쓰나?

```
Claude Code를 코딩이 아닌 완전히 다른 용도로 쓸 때:
  - 소설 작가 에이전트
  - 데이터 분석 전용
  - 상담 챗봇
  → 이런 경우에만 false

🎯 개발자라면 항상 true를 써라!
```

## 4. /statusline — 상태 표시줄 🟢

터미널 하단에 유용한 정보를 항상 표시하는 기능.

```bash
/statusline
# → 상태 표시줄 설정 메뉴

# 표시할 수 있는 정보:
# - 현재 프로젝트 경로
# - Git 브랜치
# - 사용 중인 모델
# - 토큰 사용량
# - MCP 서버 상태
```

> 💡 한 번 설정하면 계속 보이니까, 처음에 세팅해두면 편하다.

## 5. 팀 리더를 위한 스타일 전략 👔

```
┌────────── 팀에 출력 스타일 배포하기 ──────────┐

│  Step 1: 팀 공통 스타일 만들기                   │
│    프로젝트/.claude/output-styles/team.md       │
│    → git에 커밋                                │
│                                                │
│  Step 2: 팀원들에게 알려주기                     │
│    "/output-style team 으로 설정해주세요"        │
│                                                │
│  Step 3: 개인 스타일은 자유롭게                  │
│    각자 ~/.claude/output-styles/ 에 추가        │
│                                                │
│  효과:                                          │
│  ├── 팀 전체 Claude 응답 톤 통일                │
│  ├── 커밋 메시지, 코드 리뷰 포맷 표준화          │
│  └── 새 팀원도 바로 같은 스타일 적용             │
│                                                │
└────────────────────────────────────────────────┘
```

## 실습: 나만의 스타일 만들기

```bash
# 1. 스타일 파일 생성
mkdir -p ~/.claude/output-styles

# 2. 파일 작성 (아래 내용을 복사해서 저장)
cat > ~/.claude/output-styles/my-style.md << 'EOF'
---
name: 내 스타일
description: 한국어 간결 스타일 + 이유 설명
keep-coding-instructions: true
---

# 내 스타일

## 규칙
- 한국어로 응답
- 불필요한 인사말/사과 생략
- 코드 변경 시 "왜" 이렇게 했는지 한 줄 설명 추가
- 에러 해결 시: 원인 → 해결 → 확인 순서
- 반말 사용
EOF

# 3. Claude Code에서 적용
/output-style my-style

# 4. 테스트
JavaScript로 피보나치 함수를 짜줘
```

```
│  전역  → ~/.claude/output-styles/                             │

│                                                              │
└──────────────────────────────────────────────────────────────┘
```
