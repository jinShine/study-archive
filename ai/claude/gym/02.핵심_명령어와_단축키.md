# 02. 핵심 명령어와 단축키 총정리

> **키워드**: `슬래시 커맨드` `단축키` `@` `!` `ESC` `/help` `/model` `/doctor` `/config`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 대화형 모드 기호(@, !, ESC), 기본 명령어 | 매일 쓰는 핵심 |
| 🟡 이해 | /doctor, /config, /status | 문제 생길 때 쓰는 것 |
| 🟢 참고 | 전체 명령어 목록 | 있다는 것만 알면 됨 |

---

## 1. 대화형 모드 — 3가지 특수 기호 🔴

Claude Code 안에서 대화할 때, 일반 텍스트 외에 **3가지 특수 기호**를 쓸 수 있다.

```
일반 텍스트:  Claude에게 말하기      "로그인 기능 만들어줘"
@ 기호:       파일/폴더 가리키기     @src/app.ts 이 파일 봐줘
! 기호:       터미널 명령 직접 실행   !npm run build
ESC 키:       작업 흐름 제어         ESC 1번 = 중단, 2번 = 뒤로
```

### `@` — 파일/폴더 참조 🔴

```
비유: 대화 중에 "이거 봐" 하고 손가락으로 가리키는 것

"@src/login.ts 이 파일에 있는 버그 고쳐줘"
= src/login.ts 파일을 Claude의 컨텍스트에 넣고, 그걸 기반으로 작업해줘
```

**자주 쓰는 패턴:**

```bash
# 파일 하나 참조
@src/app.ts 이 파일 설명해줘

# 여러 파일 동시 참조
@src/login.ts @src/auth.ts 이 두 파일의 관계를 설명해줘

# 폴더 전체 참조
@src/components/ 이 폴더 구조를 분석해줘

# CLAUDE.md 확인
@CLAUDE.md 현재 프로젝트 규칙 보여줘
```

> 💡 **팁**: `@`를 치면 자동완성이 나온다. 파일명을 다 외울 필요 없음!

### `!` — Bash 명령어 실행 🔴

```
비유: Claude에게 말 안 걸고, 직접 터미널에서 명령어 치는 것

!npm run build = Claude한테 부탁하는 게 아니라, 내가 직접 실행
```

**자주 쓰는 패턴:**

```bash
# 빌드 상태 확인
!npm run build

# git 상태 보기
!git status

# 테스트 실행
!npm test

# 파일 목록 보기
!ls -la src/
```

> ⚠️ `!`는 Claude를 거치지 않고 바로 실행된다.
> Claude에게 "빌드해줘"라고 하면 Claude가 판단해서 실행하지만,
> `!npm run build`는 무조건 바로 실행.

### `ESC` — 작업 흐름 제어 🔴

```
ESC 1번 누르기 = "잠깐 멈춰!" (작업 중단, 세션은 유지)
                 Claude가 긴 작업 중일 때 중간에 끊고 싶을 때

ESC 2번 누르기 = "아까 그거 다시" (이전 프롬프트로 돌아가기)
                 입력한 내용을 수정하고 싶을 때
```

**실전 시나리오:**

```
나: "프로젝트 전체 파일을 리팩토링해줘"
Claude: (파일 하나씩 수정 중...)
나: (생각해보니 이 방향이 아닌 것 같다)
→ ESC 1번 = 중단!
→ "아니, 이 방향이 아니라 이렇게 해줘" 로 다시 시작
```

## 2. 기본 명령어 🔴

### 자주 쓰는 순서대로 정리

```
/help     →  "뭐가 있더라?" (명령어 목록)
/model    →  "다른 모델로 바꿔줘" (Sonnet ↔ Opus ↔ Haiku)
/resume   →  "아까 하던 거 이어서" (이전 세션 재개)
/status   →  "지금 상태 어때?" (MCP, 권한, 프로젝트 정보)
/doctor   →  "뭐가 문제야?" (시스템 진단)
/config   →  "설정 바꿀래" (설정 GUI)
```

### `/help` 🔴

사용 가능한 모든 명령어를 보여준다. **기억 안 날 때 가장 먼저 치는 명령어.**

```bash
/help
# → 전체 명령어 목록이 나온다
```

### `/model` 🔴

AI 모델을 전환한다. 상황에 따라 모델을 바꾸면 비용과 성능을 최적화할 수 있다.

```bash
/model
# → 모델 선택 메뉴가 나온다

# 모델별 용도:
# Haiku  = 빠르고 저렴 → 간단한 질문, 코드 생성
# Sonnet = 균형 잡힘   → 일반 개발 작업 (기본값)
# Opus   = 가장 똑똑   → 복잡한 아키텍처, 디버깅
```

> 💡 **비용 절약 팁**: 간단한 작업은 Haiku로, 복잡한 작업만 Opus로!
> 모델을 바꾸는 건 세션 중에 언제든 가능하다.

### `/resume` 🔴

이전 세션을 이어서 한다. 터미널에서는 `claude --continue`와 같다.

```bash
# Claude Code 안에서
/resume

# 터미널에서 (Claude Code 시작할 때)
claude --continue
```

### `/status` 🟡

현재 상태를 한눈에 본다.

```bash
/status
# → 활성 MCP 서버, 권한 모드, 프로젝트 경로 등 표시
```

### `/doctor` 🟡

뭔가 안 될 때 진단 도구. **"Claude가 이상하게 동작할 때 가장 먼저 치는 명령어."**

```bash
/doctor
# → 설정 오류, 권한 문제, MCP 서버 상태 등 점검
# → 문제가 있으면 해결 방법도 제시해줌
```

### `/config` 🟡

설정을 GUI로 편하게 관리한다.

```bash
/config
# → 권한, 환경 변수, MCP 서버 등 설정 화면
```

## 3. 빠른 참조 카드

```
┌───────────────────── Claude Code 치트시트 ─────────────────────┐
│                                                                │
│  📝 대화형 모드                                                 │
│  @파일명        파일을 컨텍스트에 추가                            │
│  !명령어        Bash 명령어 직접 실행                            │
│  ESC 1번       현재 작업 중단                                   │
│  ESC 2번       이전 프롬프트로 돌아가기                          │
│                                                                │
│  🔧 필수 명령어                                                 │
│  /help         명령어 목록 보기                                  │
│  /init         프로젝트 초기화 (CLAUDE.md 생성)                  │
│  /model        AI 모델 변경                                     │
│  /resume       이전 세션 이어서                                  │
│                                                                │
│  🔍 상태 & 진단                                                 │
│  /status       현재 상태 확인                                   │
│  /doctor       시스템 진단                                      │
│  /config       설정 관리                                        │
│                                                                │
│  🧹 토큰 관리 (다음 장에서 자세히)                               │
│  /clear        대화 초기화                                      │
│  /compact      대화 압축                                        │
│  /context      컨텍스트 확인                                    │
│                                                                │
│  ⌨️ 줄바꿈: Shift+Enter                                        │
│  🛑 완전 종료: Ctrl+C                                          │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

## 실습: 명령어 직접 써보기

```bash
# 1. Claude Code 시작
claude

# 2. 도움말 보기
/help

# 3. 현재 모델 확인 & 변경
/model

# 4. 파일 참조해서 질문하기
@package.json 이 프로젝트의 기술 스택이 뭐야?

# 5. 터미널 명령 직접 실행
!git log --oneline -5

# 6. 시스템 진단
/doctor
```
