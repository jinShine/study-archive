# 07. 메모리 시스템 5종 완전 정리

> **키워드**: `유저 메모리` `프로젝트 메모리` `모듈형 규칙` `로컬 메모리` `자동 메모리` `language` `/memory`

| 등급 | 섹션 | 의미 |
|------|------|------|
| 🔴 필수 | 5종 메모리 구분, 유저/프로젝트 메모리, language 설정 | 매번 쓰는 핵심 |
| 🟡 이해 | 로컬 메모리, 자동 메모리, 모듈형 규칙 | 활용 범위 넓히기 |
| 🟢 참고 | 미션 공유 템플릿 | 팀 공유용 |

---

## 1. 왜 메모리가 5종이나 필요한가? 🔴

```
비유: 회사에서 쓰는 문서들

사원증 (유저 메모리)        = 어디서든 나를 증명 → 모든 프로젝트에 적용
프로젝트 규칙서 (프로젝트)  = 이 프로젝트의 공식 규칙 → 팀원 전체 공유
세부 규정집 (모듈형 규칙)   = 부서별 상세 규칙 → 주제별 분리
내 메모장 (로컬 메모리)     = 나만 보는 메모 → 공유 안 됨
회의록 (자동 메모리)        = 자동으로 기록됨 → Claude가 알아서 저장
```

### 한눈에 보는 5종 메모리

```
┌──────────────────────────────────────────────────────────────────────┐
│ 종류           │ 파일 위치                       │ 공유  │ 누가 쓰나  │
├──────────────────────────────────────────────────────────────────────┤
│ 유저 메모리     │ ~/.claude/CLAUDE.md             │ 나만  │ 내가 직접  │
│ 프로젝트 메모리 │ ./CLAUDE.md                     │ 팀 전체│ 내가 직접  │
│ 모듈형 규칙     │ ./.claude/rules/*.md            │ 팀 전체│ 내가 직접  │
│ 로컬 메모리     │ ./CLAUDE.local.md               │ 나만  │ 내가 직접  │
│ 자동 메모리     │ ~/.claude/projects/<id>/memory/  │ 나만  │ Claude가!  │
└──────────────────────────────────────────────────────────────────────┘
```

> 💡 **핵심**: 메모리 파일은 Claude Code 시작 시 **자동으로 읽힌다.**
> 여기에 적어둔 규칙과 선호사항이 매 세션마다 적용!

## 2. 유저 메모리 — 나만의 전역 규칙 🔴

```
위치: ~/.claude/CLAUDE.md
범위: 모든 프로젝트에 적용
공유: 나만 사용
편집: /memory 명령어 또는 직접 편집
```

### 비유

```
유저 메모리 = 내가 어느 프로젝트를 하든 가져가는 "업무 습관"
             "나는 한국어로 커밋해", "들여쓰기는 2칸"
             → A 프로젝트든 B 프로젝트든 항상 적용
```

### 실전 예시 — 가장 많이 쓰는 내용

```markdown
# 코딩 스타일
- 변수명은 camelCase 사용
- 함수에는 간단한 JSDoc 주석 추가
- console.log 대신 적절한 로깅 라이브러리 사용
- 들여쓰기는 2칸 스페이스
- 함수는 30줄 이하로 유지, 길어지면 분리 제안
- 매직 넘버 사용 금지, 상수로 정의

# 커뮤니케이션
- 코드 변경 시 무엇을 왜 바꿨는지 간단히 설명
- 파일을 새로 만들 때는 먼저 계획을 말해주고 진행
- 에러 발생 시 원인과 해결 방법을 함께 제시

# 커밋 메시지
- 한국어로 작성
- 형식: feat: 기능 설명 / fix: 버그 설명

# 자주 쓰는 명령어
- 빌드: npm run build
- 테스트: npm run test
- 린트: npm run lint
```

> ⚠️ **중요**: "한국어로 응답해줘"는 여기가 아니라 `settings.json`의 `language` 옵션으로!
> (아래 6번 섹션에서 자세히 설명)

## 3. 프로젝트 메모리 — 팀이 공유하는 규칙 🔴

```
위치: ./CLAUDE.md (프로젝트 루트)
범위: 이 프로젝트에서만 적용
공유: 팀원 전체 (Git으로 공유)
편집: /init 자동 생성 또는 직접 작성
```

### 비유

```
프로젝트 메모리 = 프로젝트 "공식 매뉴얼"
                 git에 커밋되니까, clone하면 자동으로 받아짐
                 → 팀원 누구나 같은 규칙으로 Claude를 씀
```

### 실전 예시 — React 프로젝트

```markdown
# 프로젝트 개요
React + TypeScript 기반 할일 관리 앱

# 기술 스택
- React 19 + TypeScript
- Tailwind CSS
- Zustand (상태관리)

# 빌드 & 실행 명령어
- 개발 서버: npm run dev
- 빌드: npm run build
- 테스트: npm run test

# 코드 컨벤션
- 컴포넌트 파일명: PascalCase (예: TodoList.tsx)
- 훅 파일명: use로 시작 (예: useTodos.ts)
- 타입 정의는 types/ 디렉토리에 모아서 관리

# 프로젝트 구조
- src/components/ : UI 컴포넌트
- src/hooks/ : 커스텀 훅
- src/types/ : TypeScript 타입 정의
- src/stores/ : Zustand 스토어
```

### 실전 예시 — Python FastAPI 프로젝트

```markdown
# 프로젝트 개요
FastAPI 기반 REST API 서버

# 기술 스택
- Python 3.12 + FastAPI
- SQLAlchemy (ORM)
- PostgreSQL (데이터베이스)

# 실행 명령어
- 개발 서버: uvicorn main:app --reload
- 테스트: pytest
- 린트: ruff check .
- 포맷팅: ruff format .

# 코드 컨벤션
- 함수명/변수명: snake_case
- 클래스명: PascalCase
- 모든 API 엔드포인트에 Pydantic 모델로 입출력 검증
- docstring은 Google 스타일
```

## 4. 모듈형 규칙 — 주제별 분리 🟡

```
위치: ./.claude/rules/*.md
범위: 이 프로젝트에서만 적용
공유: 팀원 전체 (Git으로 공유)
```

```
언제 쓰나?
  CLAUDE.md가 60줄 넘어갈 것 같을 때!
  → 핵심만 CLAUDE.md에 남기고
  → 세부 규칙은 rules/ 폴더로 분리

your-project/
├── CLAUDE.md                   # 핵심 규칙 (간결)
├── .claude/
│   └── rules/
│       ├── code-style.md       # 코드 스타일 상세
│       ├── testing.md          # 테스트 규칙 상세
│       └── security.md         # 보안 규칙 상세
```

> 💡 rules/ 안의 파일들은 Claude가 **자동으로 읽는다.** 별도 설정 불필요!
> 자세한 예시는 [06.CLAUDE_MD_실전_작성법.md](06.CLAUDE_MD_실전_작성법.md) 섹션 7 참고.

## 5. 로컬 메모리 — 나만 보는 프로젝트 메모 🟡

```
위치: ./CLAUDE.local.md (프로젝트 루트)
범위: 이 프로젝트에서만 적용
공유: 나만 사용 (자동으로 .gitignore됨!)
```

### 비유

```
프로젝트 메모리 (CLAUDE.md)       = 팀 공유 규칙서  → git에 커밋
로컬 메모리 (CLAUDE.local.md)     = 내 포스트잇 메모 → 나만 봄

"이 프로젝트에서 내가 맡은 부분은 인증 모듈이야"
"DB 비밀번호는 1234야" ← 이런 건 절대 CLAUDE.md에 쓰면 안 됨!
→ CLAUDE.local.md에 쓰면 git에 안 올라감 ✅
```

### 실전 예시

```markdown
# 내 담당 영역
- 인증/인가 모듈 (src/auth/)
- 사용자 관리 API (src/api/users/)

# 로컬 개발 환경
- DB: localhost:5432, 계정: dev/dev1234
- Redis: localhost:6379

# 개인 메모
- UserService의 validateToken() 리팩토링 예정
- PR #42의 리뷰 코멘트 반영 중
```

> ⚠️ `.gitignore`에 자동 추가되지만, 혹시 모르니 **절대 비밀번호/API 키를 여기 쓰지 말 것!**
> 비밀 정보는 `.env`에만.

## 6. 자동 메모리 — Claude가 알아서 기록 🟡

```
위치: ~/.claude/projects/<프로젝트ID>/memory/
범위: 이 프로젝트에서만 적용
공유: 나만 사용
특징: Claude가 알아서 기록! 내가 안 써도 됨
```

### 비유

```
자동 메모리 = 비서가 알아서 작성하는 회의록

대화 중에 Claude가 판단해서 저장:
├── "아, 이 유저는 camelCase를 좋아하는구나" → 기록
├── "이 프로젝트는 Prisma를 쓰는구나" → 기록
├── "테스트 커버리지 80% 이상을 원하는구나" → 기록
└── 다음 세션에서 자동으로 불러옴
```

### 어떻게 확인하나?

```bash
# Claude Code에서
/memory

# 또는 직접 폴더 확인
ls ~/.claude/projects/
```

> 💡 자동 메모리는 수동으로 수정할 수도 있다.
> 잘못 기록된 게 있으면 파일을 직접 편집하면 됨.

## 7. `language` 설정 — 한국어 응답의 올바른 방법 🔴

### ❌ 잘못된 방법 vs ✅ 올바른 방법

```
❌ CLAUDE.md에: "한국어로 응답해줘"
   → 프롬프트 기반이라 가끔 무시될 수 있음
   → 토큰도 낭비

✅ settings.json에: "language": "korean"
   → 시스템 레벨 설정이라 100% 적용
   → v2.1.0에서 추가된 공식 옵션
```

### 설정 방법

**유저 설정 (모든 프로젝트에 적용):** `~/.claude/settings.json`

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "language": "korean",
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test *)",
      "Bash(npm run build)"
    ],
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  }
}
```

**프로젝트 설정 (팀 공유용):** `.claude/settings.json`

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "allow": [
      "Bash(npm run dev)",
      "Bash(npm run build)",
      "Bash(npx prisma *)",
      "Bash(docker compose *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Bash(git push --force *)",
      "Read(./.env.production)"
    ]
  },
  "env": {
    "NODE_ENV": "development"
  }
}
```

> 💡 **`$schema` 필드**: JSON 스키마를 지정하면 에디터에서 **자동완성과 유효성 검사**가 된다!
> 복사해서 넣어두면 편함.

### 설정 파일 우선순위

```
로컬 설정 (.claude/settings.local.json)   ← 가장 높음 (나만 쓰는 것)
프로젝트 설정 (.claude/settings.json)     ← 중간 (팀 공유)
유저 설정 (~/.claude/settings.json)       ← 기본 (모든 프로젝트)

더 구체적인 설정이 우선!
```

## 8. 전체 구조 한눈에 보기 🔴

```
┌────────────────── 내 컴퓨터 전체 ──────────────────┐
│                                                     │
│  ~/.claude/                                         │
│  ├── CLAUDE.md          ← 유저 메모리 (전역 규칙)    │
│  ├── settings.json      ← 유저 설정 (전역 권한)     │
│  └── projects/          ← 자동 메모리 (Claude 기록)  │
│      └── <project-id>/                              │
│          └── memory/                                │
│                                                     │
└─────────────────────────────────────────────────────┘

┌────────────────── 프로젝트 폴더 ───────────────────┐
│                                                     │
│  my-project/                                        │
│  ├── CLAUDE.md          ← 프로젝트 메모리 (팀 공유)  │
│  ├── CLAUDE.local.md    ← 로컬 메모리 (나만 봄)     │
│  ├── .claude/                                       │
│  │   ├── settings.json      ← 프로젝트 설정 (팀 공유)│
│  │   ├── settings.local.json← 로컬 설정 (나만)      │
│  │   └── rules/             ← 모듈형 규칙 (팀 공유)  │
│  │       ├── code-style.md                          │
│  │       ├── testing.md                             │
│  │       └── security.md                            │
│  └── src/                                           │
│      └── api/                                       │
│          └── CLAUDE.md  ← 하위 폴더 메모리 (선택)    │
│                                                     │
└─────────────────────────────────────────────────────┘

Git에 올라가는 것: CLAUDE.md, .claude/settings.json, .claude/rules/
Git에 안 올라가는 것: CLAUDE.local.md, .claude/settings.local.json
```

## 9. 팀 리더를 위한 메모리 전략 👔

```
Step 1: 프로젝트 메모리 세팅
  CLAUDE.md → 기술 스택, 빌드 명령, 핵심 규칙
  .claude/rules/ → 코드 스타일, 테스트, 보안 규칙 분리
  .claude/settings.json → 팀 공통 권한 설정
  → git commit & push

Step 2: 팀원 가이드
  "git clone하면 Claude 설정이 자동 적용됩니다"
  "개인 설정은 CLAUDE.local.md에 작성하세요"
  "한국어 설정은 ~/.claude/settings.json에 language: korean"

Step 3: 점진적 개선
  코드 리뷰에서 반복되는 피드백 → rules/에 규칙 추가
  → 다음부터 Claude가 자동으로 방지!
```

## 실습: 5종 메모리 직접 만들기

```bash
# 1. 유저 메모리 (전역)
# Claude Code에서:
/memory
# → 편집기 열림, 위 예시 참고해서 작성

# 2. 유저 설정 (전역) — language 포함!
cat > ~/.claude/settings.json << 'EOF'
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "language": "korean",
  "permissions": {
    "allow": [
      "Bash(npm run *)",
      "Bash(git status)",
      "Bash(git diff)"
    ],
    "deny": [
      "Read(./.env*)",
      "Bash(rm -rf *)"
    ]
  }
}
EOF

# 3. 프로젝트 메모리
cd my-project
# Claude Code에서:
/init

# 4. 프로젝트 설정
mkdir -p .claude
cat > .claude/settings.json << 'EOF'
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "allow": ["Bash(npm run *)"],
    "deny": ["Bash(rm -rf *)", "Read(./.env.production)"]
  }
}
EOF

# 5. 로컬 메모리 (나만 봄)
cat > CLAUDE.local.md << 'EOF'
# 내 담당
- 인증 모듈 개발 중
- PR #42 리뷰 반영 중
EOF

# 6. 확인
/memory   # 메모리 상태 확인
/status   # 전체 상태 확인
```

## 면접 대비

### 🔴 필수 질문

**Q: Claude Code의 메모리 종류를 설명하라.**
> 유저 메모리(전역 개인 규칙), 프로젝트 메모리(팀 공유 규칙), 모듈형 규칙(주제별 분리), 로컬 메모리(개인 프로젝트 메모), 자동 메모리(Claude 자동 기록) 5종입니다. 시작 시 자동으로 읽히며, 더 구체적인(가까운) 설정이 우선합니다.

**Q: 한국어 응답을 설정하는 올바른 방법은?**
> CLAUDE.md에 "한국어로 답해줘"보다, settings.json의 `"language": "korean"` 옵션이 시스템 레벨에서 적용되어 더 정확하고 안정적입니다. v2.1.0에서 추가된 공식 옵션입니다.

### 🟡 개념 질문

**Q: CLAUDE.md와 CLAUDE.local.md의 차이는?**
> CLAUDE.md는 git에 커밋되어 팀 전체가 공유하는 규칙이고, CLAUDE.local.md는 자동으로 gitignore되어 나만 보는 메모입니다. 개인 담당 영역이나 로컬 개발 환경 정보를 여기에 씁니다.

## 핵심 요약

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  5종 메모리:                                                  │
│  ├── 유저 메모리     ~/.claude/CLAUDE.md       나만, 전역     │
│  ├── 프로젝트 메모리 ./CLAUDE.md               팀, Git 공유   │
│  ├── 모듈형 규칙     .claude/rules/*.md        팀, Git 공유   │
│  ├── 로컬 메모리     ./CLAUDE.local.md         나만, gitignore│
│  └── 자동 메모리     ~/.claude/projects/.../   나만, 자동     │
│                                                              │
│  한국어 설정:                                                 │
│  ❌ CLAUDE.md에 "한국어로"  → 가끔 무시됨                     │
│  ✅ settings.json에 "language": "korean" → 100% 적용         │
│                                                              │
│  우선순위: 로컬 > 프로젝트 > 유저 (구체적인 게 우선)           │
│  Git 공유: CLAUDE.md, settings.json, rules/                  │
│  Git 제외: CLAUDE.local.md, settings.local.json              │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```
