## 1. `claude mcp add` — 설치 명령어 패턴 🔴

Claude Code는 **CLI 한 줄로 MCP를 설치**할 수 있다.

### 기본 문법

```bash
claude mcp add <이름> <실행 방식> [옵션]

--scope project   ← 이 프로젝트에서만 사용 (.mcp.json 에 저장)
--scope user      ← 전역 사용 (~/.claude/settings.json)
```

### 스코프 차이 이해

```
project 스코프:
  .mcp.json 파일에 저장
  git에 커밋 가능 → 팀 전체 공유
  clone 하면 자동 적용 ✅

user 스코프:
  ~/.claude/settings.json 에 저장
  나만 사용
  모든 프로젝트에서 쓸 수 있음
```

> 💡 **팀 리더 팁 👔**: 프로젝트에 필요한 MCP는 `--scope project`로 설치해서 git에 커밋!

## 2. 필수 MCP 4종 — 강의에서 소개된 것들 🔴

### MCP 1: Context7 — 최신 문서 주입

```
비유: Claude에게 "최신 개발 가이드북"을 쥐여주는 것

문제:
  "React 19의 새 훅 사용법 알려줘"
  → Claude: "제 학습 시점 기준으로는..."
  → 옛날 정보일 수 있음

Context7 연결 후:
  "React 19의 새 훅 사용법 알려줘. use context7"
  → Context7이 실시간으로 최신 문서 가져와서 답변
  → 정확한 최신 정보!
```

**설치:**

```bash
claude mcp add --transport http context7 https://mcp.context7.com/mcp --scope project
```

**활용 예시:**

```bash
# 프롬프트에 "use context7" 추가하면 최신 문서 참조
"Next.js 15의 서버 액션 예제 보여줘. use context7"

"Prisma 6의 새로운 기능 설명해줘. use context7"

"Tailwind CSS v4 마이그레이션 방법 알려줘. use context7"
```

**쓰는 상황:**
- 최신 프레임워크 버전 사용 시
- 문서가 자주 업데이트되는 라이브러리
- AI의 학습 시점 이후 나온 기능

---

### MCP 2: Playwright — 브라우저 자동화

```
비유: Claude에게 "눈과 손"을 달아주는 것

Claude가 웹 페이지를 직접 열고, 스크린샷 찍고,
클릭하고, 폼 채우고, 테스트할 수 있게 됨.
```

**설치:**

```bash
claude mcp add playwright npx @playwright/mcp@latest --scope project
```

**활용 예시:**

```bash
# UI 개발 반복
"로그인 페이지 만든 다음, Playwright로 스크린샷 찍어서
내가 준 디자인이랑 비교해줘."

# E2E 테스트
"회원가입 플로우 전체를 Playwright로 테스트해줘.
이메일 입력 → 인증번호 → 비밀번호 → 완료까지."

# 웹 스크래핑
"GitHub 트렌딩 저장소 Top 10을 Playwright로 가져와서
README에 정리해줘."
```

**쓰는 상황:**
- UI 개발 (스크린샷 반복)
- E2E 테스트 작성/실행
- 웹 스크래핑
- 버그 재현 자동화

### MCP 3: Sequential Thinking — 단계별 사고

```
비유: Claude의 "생각 정리 노트"

복잡한 문제를 풀 때:
  ❌ 그냥 답하기: 생각이 뒤죽박죽
  ✅ Sequential Thinking: 단계별로 차근차근 생각

각 단계를 기록하면서 논리적으로 진행
→ 복잡한 문제에서 정확도 급상승
```

**설치:**

```bash
claude mcp add sequential-thinking --scope project -- npx -y @modelcontextprotocol/server-sequential-thinking
```

**활용 예시:**

```bash
# 복잡한 버그 디버깅
"이 간헐적 401 에러의 원인을 sequential thinking으로 단계별 분석해줘."

# 아키텍처 결정
"모노리스에서 MSA로 전환하는 방안을
sequential thinking으로 트레이드오프를 하나씩 검토하며 결정해줘."

# 알고리즘 설계
"이 추천 시스템 로직을 sequential thinking으로 설계해줘.
각 단계의 이유를 명시하면서."
```

**쓰는 상황:**
- 복잡한 디버깅
- 아키텍처 의사결정
- 알고리즘 설계
- "왜" 라는 질문이 많은 문제

### MCP 4: ShadcnUI MCP — 컴포넌트 라이브러리

```
비유: Claude에게 "shadcn/ui 카탈로그" 주는 것

shadcn/ui 컴포넌트를 사용할 때:
  ❌ 수동: 문서 찾아서 → 설치 → 복붙 → 수정
  ✅ MCP: "Button 컴포넌트 추가해줘" → 자동!
```

**설치:**

```bash
npx shadcn@latest mcp init --client claude
```

**활용 예시:**

```bash
# 컴포넌트 추가
"shadcn의 Dialog 컴포넌트를 추가하고,
로그인 모달로 활용하는 예제를 만들어줘."

# 테마 커스터마이징
"shadcn 테마를 다크모드 + 보라 계열로 커스터마이징해줘."

# 전체 페이지 구성
"shadcn 컴포넌트만 써서 대시보드 페이지를 만들어줘.
Card, Table, Chart, Dropdown Menu 활용."
```

**쓰는 상황:**
- Next.js + Tailwind + shadcn/ui 프로젝트
- 빠른 프로토타입
- 일관된 디자인 시스템

## 3. 추가 추천 MCP 🟡

강의에 없지만 실전에서 많이 쓰는 것들.

### GitHub MCP — PR, 이슈 관리

```bash
claude mcp add github --scope user -- npx -y @modelcontextprotocol/server-github
# 환경변수 GITHUB_PERSONAL_ACCESS_TOKEN 필요
```

**활용:**
```bash
"이번 스프린트에 할당된 이슈 목록 보여줘"
"PR #42에 달린 리뷰 코멘트 반영해줘"
"수정 완료된 이슈는 닫아줘"
```

### PostgreSQL MCP — DB 직접 조회

```bash
claude mcp add postgres --scope project -- npx -y @modelcontextprotocol/server-postgres postgresql://localhost:5432/mydb
```

**활용:**
```bash
"users 테이블 스키마 보여줘"
"최근 가입한 유저 10명 조회해줘"
"이 쿼리 성능 분석해줘"
```

### Supabase MCP — BaaS 통합

```bash
claude mcp add supabase --scope project -- npx -y @supabase/mcp-server-supabase
```

**활용:**
```bash
"Supabase에 todos 테이블 만들고, RLS 정책 설정해줘"
"Auth 사용자 목록 조회해줘"
"Storage에 이미지 업로드하는 함수 만들어줘"
```

### Sentry MCP — 에러 추적

```bash
claude mcp add sentry --scope project -- npx -y @sentry/mcp-server
```

**활용:**
```bash
"오늘 가장 많이 발생한 에러 Top 5 보여줘"
"이 에러의 스택 트레이스를 분석하고 수정안 제시해줘"
```

### superpowers — 스킬 시스템 (⚠️ MCP가 아닌 스킬 시스템)

```
superpowers는 "MCP 서버"가 아니라 Claude Code의 스킬 플러그인 시스템이다.
(obra/superpowers 프로젝트)

설치하면:
- TDD 스킬
- 디버깅 스킬
- 플래닝 스킬
- 코드 리뷰 스킬
등 수십 개의 스킬이 /skill-name 형태로 활성화됨
```

설치 방법은 프로젝트마다 다르니 공식 레포 확인:
- https://github.com/obra/superpowers

## 5. 실전 조합 — 풀스택 프로젝트 MCP 세팅 🔴

### 시나리오: Next.js + Supabase + shadcn/ui 프로젝트

```bash
# 프로젝트 폴더에서
cd my-project

# 1. Context7 (최신 문서)
claude mcp add --transport http context7 \
  https://mcp.context7.com/mcp --scope project

# 2. Playwright (UI 테스트)
claude mcp add playwright \
  npx @playwright/mcp@latest --scope project

# 3. shadcn/ui (컴포넌트)
npx shadcn@latest mcp init --client claude

# 4. Supabase (백엔드)
claude mcp add supabase --scope project \
  -- npx -y @supabase/mcp-server-supabase

# 5. GitHub (PR, 이슈)
claude mcp add github --scope user \
  -- npx -y @modelcontextprotocol/server-github

# 확인
claude
/status
```

**결과: 하나의 프롬프트로 풀스택 작업 가능!**

```bash
"Next.js 15의 서버 액션으로 (use context7)
Supabase todos 테이블에 할일을 추가하는 기능을
shadcn Button 컴포넌트로 구현하고,
Playwright로 테스트해줘."
```

## 6. 팀 리더 전략 — MCP 배포하기 👔

### 전략 1: 프로젝트 필수 MCP는 git에 커밋

```
프로젝트/.mcp.json (git 커밋):
{
  "mcpServers": {
    "context7": {...},
    "playwright": {...},
    "supabase": {...}
  }
}

→ 팀원이 clone하면 바로 같은 MCP 환경
```

### 전략 2: 개인 토큰은 환경변수로

```
# .mcp.json (공유)
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "${GITHUB_TOKEN}"
      }
    }
  }
}

# 각 팀원의 셸 환경 (.zshrc, .bashrc)
export GITHUB_TOKEN="ghp_..."

→ 토큰은 각자, 설정은 공유!
```

### 전략 3: 온보딩 문서에 MCP 설치 가이드

```markdown
# README.md - 개발 환경 세팅

## Claude Code MCP 설치

1. `git clone` (이 단계에서 .mcp.json 자동으로 받아짐)
2. 환경변수 설정:
   ```bash
   export GITHUB_TOKEN="your-github-token"
   export SUPABASE_URL="your-supabase-url"
   ```
3. Claude Code 실행:
   ```bash
   claude
   /status   # MCP 연결 확인
   ```
```

## 실습: MCP 직접 설치해보기

```bash
# 아무 프로젝트에서
cd my-project

# 1. Context7 설치 (가장 유용한 MCP)
claude mcp add --transport http context7 \
  https://mcp.context7.com/mcp --scope project

# 2. Playwright 설치
claude mcp add playwright \
  npx @playwright/mcp@latest --scope project

# 3. 확인
claude
/status
# → context7, playwright 표시되어야 함

# 4. 테스트
# Context7 테스트:
"Next.js 15의 새 App Router 기능 설명해줘. use context7"

# Playwright 테스트:
"example.com 을 열어서 스크린샷 찍어줘"

# 5. .mcp.json 확인
cat .mcp.json
# → 설치된 서버 설정이 보임

# 6. git에 커밋 (팀 공유)
git add .mcp.json
git commit -m "chore: MCP 서버 설정 추가 (context7, playwright)"
```

## 면접 대비

### 🔴 필수 질문

**Q: MCP 스코프(scope)의 차이는?**
> `--scope project`는 `.mcp.json`에 저장되어 git으로 팀 공유가 가능하고, `--scope user`는 `~/.claude/settings.json`에 저장되어 나만 모든 프로젝트에서 사용합니다. 팀 프로젝트에 필수적인 MCP는 project 스코프로 설치해 공유하는 것이 좋습니다.

**Q: Context7 MCP의 용도는?**
> 프레임워크와 라이브러리의 실시간 최신 문서를 Claude에게 주입하는 MCP입니다. Claude의 학습 시점 이후 나온 기능이나 자주 업데이트되는 문서를 참조할 때 프롬프트에 `use context7`을 추가하면 정확한 최신 정보를 바탕으로 답변받을 수 있습니다.
