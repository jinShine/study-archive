## 1. Claude Code란 무엇인가? 🔴

### 비유부터

```
웹 ChatGPT/Claude =  카페에서 바리스타에게 주문하는 것
                     "아메리카노 하나요" → 받아서 마심 → 끝

Claude Code (CLI) =  바리스타가 내 주방에 와서 일하는 것
                     내 냉장고(파일)를 열고, 내 도구(터미널)를 쓰고,
                     내 레시피(CLAUDE.md)를 따라서 직접 요리함
```

### 정의

**Claude Code** = Anthropic이 만든 **CLI(Command Line Interface) 기반 AI 코딩 도구**.
내 컴퓨터의 파일을 직접 읽고, 쓰고, 터미널 명령어를 실행할 수 있다.

### 웹 vs CLI 차이

| | 웹 (claude.ai) | CLI (Claude Code) |
|---|---|---|
| 파일 접근 | ❌ 직접 불가 (복붙 필요) | ✅ 내 프로젝트 파일 직접 읽기/쓰기 |
| 터미널 명령 | ❌ 불가 | ✅ `npm run build`, `git commit` 등 직접 실행 |
| 프로젝트 이해 | ❌ 매번 설명 필요 | ✅ CLAUDE.md로 프로젝트 규칙 자동 이해 |
| 작업 연속성 | ❌ 대화마다 리셋 | ✅ 세션/메모리로 이어서 작업 |
| 도구 연결 | ❌ 불가 | ✅ MCP로 GitHub, Slack, DB 등 연결 |

> 💡 **왜 CLI인가?**
> 웹은 "대화"만 가능하지만, CLI는 "행동"이 가능하다.
> 코드를 읽고 → 수정하고 → 테스트하고 → 커밋까지 Claude가 직접 한다.

## 2. 설치하기 🔴

### Step 1: Node.js 확인

```bash
# 터미널에서 실행
node --version
# v18.0.0 이상이면 OK
```

### Step 2: Claude Code 설치

```bash
# npm으로 전역 설치
npm install -g @anthropic-ai/claude-code
```

### Step 3: 프로젝트 폴더에서 실행

```bash
# 원하는 프로젝트 폴더로 이동
cd my-project

# Claude Code 시작
claude
```

### Step 4: 최초 로그인

처음 실행하면 Anthropic 계정 로그인을 요구한다. 브라우저가 열리고 인증하면 끝.

> 📖 공식 문서: https://code.claude.com/docs/ko/overview

## 3. 줄바꿈 설정 🟢

Claude Code에서 **여러 줄 입력**을 하려면 `Shift+Enter`가 필요하다.

```
✅ 기본 지원:      iTerm2, WezTerm, Ghostty, Kitty
❌ 추가 설정 필요:  VS Code, Cursor, Alacritty, Zed, Warp
```

**VS Code / Cursor 사용자라면:**

```
Claude Code 안에서 /terminal-setup 실행
→ Shift+Enter 자동 설정됨
```

> 💡 한 번만 하면 되니까 처음에 해두자.

## 4. 핵심 개념 3가지 🔴

### 컨텍스트(Context)란?

```
비유: Claude의 "작업 기억(Working Memory)"

사람이 요리할 때 머릿속에 떠올리는 것들:
├── 지금 만드는 요리 (현재 작업)
├── 레시피 내용 (CLAUDE.md)
├── 냉장고에 뭐가 있는지 (프로젝트 파일들)
└── 아까 소금 넣었다는 기억 (이전 대화)

Claude의 컨텍스트도 똑같다:
├── 현재 대화 내용
├── CLAUDE.md 규칙
├── 읽은 파일들
└── 실행한 명령어 결과들
```

**핵심 포인트:**
- 컨텍스트에는 **용량 제한**이 있다 (= 컨텍스트 윈도우)
- 너무 많은 파일을 읽거나 대화가 길어지면 → 오래된 내용을 잊는다
- 그래서 **컨텍스트 관리**가 중요하다 (나중에 자세히 배움)

```
⚠️ AI 모델이 더 똑똑해지고, 컨텍스트 윈도우가 더 커지더라도
   결국 우리가 해야 할 일은 → 컨텍스트를 효과적으로 관리하는 것
```

### 세션(Session)이란?

```
비유: 카페에서의 "한 번의 방문"

카페 방문 1 (= 세션 1):
  "아메리카노 주세요" → "여기요" → "감사합니다" → 퇴장

카페 방문 2 (= 세션 2):
  "지난번에 먹었던 그거요" → "???" (바리스타가 기억 못 함)

Claude Code 세션도 비슷:
  세션 1: "로그인 기능 만들어줘" → 작업 → 끝
  세션 2: "아까 만든 거 수정해줘" → 이전 세션 내용 모름!
          (하지만 /resume이나 --continue로 이어갈 수 있다)
```

**핵심 포인트:**
- **세션** = Claude Code를 시작해서 끝낼 때까지의 한 번의 대화
- 세션이 끝나면 컨텍스트가 사라진다
- 이어서 하려면: `/resume` 또는 `claude --continue`

### `/init` 명령어란?

```
비유: 새 알바가 출근했을 때 사수가 주는 "업무 매뉴얼"

/init = 이 프로젝트의 업무 매뉴얼(CLAUDE.md)을 자동으로 만들어주는 명령어

실행하면:
1. 프로젝트 폴더를 쭉 훑어본다
2. 기술 스택, 구조, 빌드 방법 등을 파악한다
3. CLAUDE.md 파일을 자동 생성한다
```

**실전 사용법:**

```bash
# 프로젝트 폴더에서 Claude Code 실행 후
/init

# 한국어로 초기화하고 싶다면
/init 프로젝트 초기화를 한국어로 진행해주세요.
```

> 💡 **팀 리더 팁 👔**: 새 프로젝트를 시작할 때 `/init`으로 기본 CLAUDE.md를 만들고,
> 팀 규칙을 추가해서 git에 커밋하면 → 팀원 전원이 같은 규칙으로 Claude를 쓸 수 있다.

## 5. AI 네이티브 개발자 🟡

```
일반 개발자:         AI를 "도구"로 사용
                    "이 코드 설명해줘" → 답변 복사 → 붙여넣기

AI 네이티브 개발자:  AI를 "워크플로우에 통합"
                    CLAUDE.md로 규칙 세팅 → Hooks로 자동화 →
                    에이전트로 리뷰/테스트 → 커밋까지 자동
```

> AI 도구를 단순히 사용하는 것을 넘어,
> **AI의 동작 원리를 이해하고 이를 개발 워크플로우에 자연스럽게 통합**하는 개발자

이 커리큘럼의 궁극적 목표가 바로 이것이다.

## 실습: 첫 Claude Code 세션

```bash
# 1. 아무 프로젝트 폴더로 이동 (없으면 하나 만들기)
mkdir my-first-claude && cd my-first-claude

# 2. Claude Code 시작
claude

# 3. 프로젝트 초기화 (한국어)
/init 프로젝트 초기화를 한국어로 진행해주세요.

# 4. 생성된 CLAUDE.md 확인
@CLAUDE.md 이 파일 내용을 보여줘

# 5. 간단한 작업 시켜보기
index.html 파일을 만들어줘. "Hello, Claude Code!" 라고 나오게.

# 6. 세션 종료
/clear 또는 Ctrl+C
```

## 면접 대비

### 🔴 필수 질문

**Q: Claude Code와 웹 Claude의 차이점은?**
> Claude Code는 CLI 기반으로 로컬 파일 시스템에 직접 접근하여 파일 읽기/쓰기, 터미널 명령 실행이 가능합니다. 웹 Claude는 대화만 가능하지만, Claude Code는 실제 코드 수정과 실행까지 할 수 있습니다.

**Q: 컨텍스트(Context)란 무엇인가?**
> Claude가 현재 대화에서 참조할 수 있는 모든 정보의 총합입니다. 대화 내용, CLAUDE.md 규칙, 읽은 파일, 실행 결과 등이 포함되며, 용량 제한(컨텍스트 윈도우)이 있어 효과적인 관리가 필요합니다.

### 🟡 개념 질문

**Q: AI 네이티브 개발자란?**
> AI를 단순 도구로 쓰는 것을 넘어, AI의 동작 원리를 이해하고 개발 워크플로우에 자연스럽게 통합하는 개발자입니다. CLAUDE.md, Hooks, 에이전트 등을 활용해 AI와 협업 시스템을 구축합니다.
