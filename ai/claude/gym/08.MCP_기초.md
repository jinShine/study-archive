## 1. MCP란 무엇인가? 🔴

```
비유: 스마트폰의 "앱 설치"

스마트폰 기본 기능:      전화, 문자, 카메라
앱을 설치하면:           배달 주문, 은행 업무, 내비게이션, ...

Claude Code 기본 기능:   파일 읽기/쓰기, 터미널 명령, 웹 검색
MCP를 연결하면:          GitHub PR 관리, DB 쿼리, Slack 메시지, Figma 디자인, ...
```

### 정의

**MCP (Model Context Protocol)** = Claude에게 **외부 도구를 연결**해주는 표준 프로토콜.
Claude가 기본적으로 할 수 없는 일 (DB 조회, Slack 전송 등)을 가능하게 해준다.

```
┌────────── Claude Code ──────────┐
│                                  │
│   기본 도구:                      │
│   ├── 파일 읽기/쓰기             │
│   ├── 터미널 명령 실행            │
│   ├── 웹 검색/페이지 읽기         │
│   └── 코드 분석                  │
│                                  │
│   MCP로 추가:                     │
│   ├── 🐙 GitHub  → PR, 이슈 관리  │
│   ├── 💬 Slack   → 메시지 읽기/쓰기│
│   ├── 🗄️ PostgreSQL → DB 쿼리     │
│   ├── 🎨 Figma   → 디자인 추출    │
│   ├── 🌐 Playwright → 브라우저 조작│
│   └── ... (수백 개의 MCP 서버)    │
│                                  │
└──────────────────────────────────┘
```

## 2. MCP가 왜 필요한가? 🔴

### ❌ MCP 없이

```
나: "이번 PR에 달린 리뷰 코멘트 보여줘"
Claude: "저는 GitHub에 직접 접근할 수 없습니다.
        브라우저에서 확인해서 복사해주세요."
나: (브라우저 열고, GitHub 가서, 코멘트 복사해서 붙여넣기...)
```

### ✅ MCP 연결 후

```
나: "이번 PR에 달린 리뷰 코멘트 보여줘"
Claude: (GitHub MCP로 직접 조회)
        "PR #42에 3개의 코멘트가 있습니다:
         1. 김팀장: 'validateToken 함수에 에러 핸들링 추가 필요'
         2. 이개발: '테스트 케이스 보완 부탁'
         3. ..."
나: "1번 코멘트 반영해서 코드 수정해줘"
Claude: (바로 코드 수정 + 커밋)
```

> 💡 **MCP = Claude의 행동 반경을 넓혀주는 것**

## 3. 가장 많이 쓰는 MCP 서버 🟡

| MCP 서버 | 하는 일 | 쓰는 상황 |
|----------|---------|-----------|
| **GitHub** | PR, 이슈, 리포 관리 | 코드 리뷰, PR 생성, 이슈 관리 |
| **Playwright** | 브라우저 자동 조작 | E2E 테스트, UI 검증, 스크래핑 |
| **PostgreSQL** | DB 직접 쿼리 | 스키마 확인, 데이터 조회, 디버깅 |
| **Supabase** | DB + 인증 + 스토리지 | 풀스택 개발, BaaS 연동 |
| **Context7** | 프레임워크 최신 문서 | "React 19 최신 문법 알려줘" |
| **Figma** | 디자인 → 코드 | 디자인 토큰 추출, 레이아웃 변환 |
| **Sentry** | 에러 추적 | 프로덕션 버그 진단 |
| **Slack** | 메시지 읽기/보내기 | 팀 알림, 배포 상태 공유 |

## 5. MCP 서버 연결하기 🔴

### 설정 파일 위치

```
전역 (모든 프로젝트):   ~/.claude/settings.json → mcpServers
프로젝트별:             .claude/settings.json → mcpServers
                       또는 .mcp.json (전용 파일)
```

### 기본 구조

```json
{
  "mcpServers": {
    "서버이름": {
      "command": "실행 명령어",
      "args": ["인수1", "인수2"],
      "env": {
        "환경변수": "값"
      }
    }
  }
}
```

### 실전 예시: GitHub MCP 연결

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<your-token>"
      }
    }
  }
}
```

### 실전 예시: PostgreSQL MCP 연결

```json
{
  "mcpServers": {
    "postgres": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-postgres",
        "postgresql://user:password@localhost:5432/mydb"
      ]
    }
  }
}
```

### 실전 예시: 여러 MCP 서버 동시 연결

```json
{
  "mcpServers": {
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": {
        "GITHUB_PERSONAL_ACCESS_TOKEN": "<token>"
      }
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres",
               "postgresql://localhost:5432/mydb"]
    },
    "slack": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-server-slack"],
      "env": {
        "SLACK_BOT_TOKEN": "<bot-token>"
      }
    }
  }
}
```

### 연결 확인

```bash
# Claude Code에서
/status
# → 연결된 MCP 서버 목록이 표시됨

# 문제 있으면
/doctor
# → MCP 서버 상태 진단
```

## 6. 팀 리더를 위한 MCP 전략 👔

```
┌────────── 팀에 MCP 배포하기 ──────────┐
│                                        │
│  프로젝트 MCP 설정: .mcp.json          │
│  → git에 커밋 (토큰은 빼고!)           │
│                                        │
│  개인 토큰: .claude/settings.local.json│
│  → gitignore (각자 자기 토큰 사용)     │
│                                        │
│  팀 공유:                              │
│  {                                     │
│    "mcpServers": {                     │
│      "github": {                       │
│        "command": "npx",               │
│        "args": [...],                  │
│        "env": {                        │
│          "GITHUB_PERSONAL_ACCESS_TOKEN":│
│            "${GITHUB_TOKEN}"  ← 환경변수│
│        }                               │
│      }                                 │
│    }                                   │
│  }                                     │
│                                        │
│  → 각 팀원이 자기 GITHUB_TOKEN만       │
│    환경변수로 설정하면 됨!              │
│                                        │
└────────────────────────────────────────┘
```

> ⚠️ **절대 토큰/비밀번호를 git에 커밋하지 말 것!**
> 환경변수(`${VAR_NAME}`) 문법을 사용하자.
