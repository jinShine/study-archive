## 왜 중요한가

사람이 가장 많이 낭비하는 시간 중 하나는 "다른 도구의 내용을 Claude에게 복붙하는 시간"이다. 이슈 트래커 내용, 에러 대시보드, 문서 플랫폼, DB 결과를 계속 복사해 오면 느릴 뿐 아니라 빠뜨리는 것도 많다.

MCP는 이 과정을 줄여 준다. Claude Code가 필요한 시스템을 직접 읽고, 경우에 따라 쓸 수도 있게 해 주는 연결 레이어다.

## 핵심 개념

### 1. MCP란 무엇인가

공식 문서 기준으로 MCP는 외부 도구와 AI를 연결하는 개방형 표준이다. Claude Code는 MCP 서버를 통해 이슈, 문서, 데이터베이스, 내부 API 같은 시스템에 접근할 수 있다.

### 2. 언제 붙여야 하나

아래 상황이면 MCP 도입 가치가 높다.

- 매일 같은 시스템 내용을 복붙한다
- 문맥 누락 때문에 판단이 자주 틀린다
- Claude가 읽으면 좋을 메타데이터가 있다
- 사람이 직접 조회하느라 전환 비용이 크다

### 3. 언제 아직 붙이지 말아야 하나

- 해당 도구를 거의 안 쓴다
- 팀이 접근 권한 구조를 아직 정리하지 않았다
- 붙여도 결국 사람이 수동 판단해야 한다
- 운영 주체가 없다

### 4. 실무 가치가 큰 연결 후보

- 이슈 트래커
- 문서/위키
- 에러 모니터링
- 배포/관측 대시보드
- 데이터베이스 조회

이 중에서도 보통은 이슈 트래커와 문서 연결의 체감 가치가 가장 빨리 나온다.

### 5. MCP와 슬래시 커맨드

공식 문서 기준으로 MCP 서버는 자체 프롬프트를 슬래시 커맨드처럼 노출할 수 있다. 즉, 외부 시스템 사용법을 매번 설명하지 않고 `/mcp__서버__프롬프트` 형태로 부를 수 있는 경우가 있다.

### 6. MCP는 편의성만이 아니라 정확성 도구다

복붙된 정보는 누락되기 쉽다. MCP로 직접 읽게 하면 최신 이슈 상태, 문서 원문, 에러 맥락을 더 정확하게 가져올 수 있다.

## 실무에서 잘 쓰는 사람의 방식

- 가장 자주 복붙하는 시스템부터 붙인다.
- 읽기 전용 연결부터 시작한다.
- 연결 목적을 명확히 한다. 예를 들면 "이슈를 읽기 위해"인지 "티켓 상태까지 수정하기 위해"인지.
- MCP가 있다고 무조건 넓게 열지 않는다.
- Claude에게 외부 도구를 읽게 한 뒤에도 최종 의사결정은 사람이 한다.

## 자주 망하는 방식

- 멋있어 보여서 도구를 한꺼번에 다 붙인다.
- 운영 책임자 없이 팀 공통 MCP를 연다.
- 읽기와 쓰기 권한을 분리하지 않는다.
- 어떤 질문을 MCP로 해결할지 정의하지 않는다.
- 실제로는 거의 쓰지 않는 도구를 계속 유지한다.

## 바로 써보는 예시

### MCP 도입 타당성 판단 요청

```text
우리 팀이 Claude Code에 MCP를 붙일 후보를 정하고 싶어.
아래 기준으로 우선순위를 매겨줘.
1. 복붙 빈도
2. 정보 누락 위험
3. 읽기 전용으로 시작 가능한지
4. 실무 체감 가치
```

### 이슈 트래커 연동 후 요청 예시

```text
ENG-241 이슈 내용을 읽고,
현재 저장소에서 관련 코드 위치를 먼저 찾아줘.
구현 전에 위험 요소와 필요한 테스트를 정리해줘.
```

### 문서 시스템 연동 후 요청 예시

```text
결제 정책 문서를 읽고,
현재 코드가 문서와 어긋나는 부분이 있는지 확인해줘.
추측은 하지 말고 문서와 코드에 근거해서만 말해줘.
```

## 1인개발 적용

혼자 개발할 때 MCP는 "내가 자주 드나드는 두세 개 시스템의 출입문" 정도로 생각하면 충분하다. 처음부터 대형 자동화 아키텍처를 만들 필요는 없다.

추천 순서는 이렇다.

1. 이슈/할 일 관리
2. 문서 저장소
3. 에러 대시보드

이 셋만 잘 연결해도 복붙 시간이 크게 줄어든다.

## 팀 적용

팀에서는 MCP가 곧 운영 이슈다. 누가 설정하고, 누가 토큰을 관리하고, 읽기/쓰기 범위를 어디까지 줄 것인지 명확해야 한다. 팀 공통 연결은 기술 문제보다 책임 분담 문제가 더 중요하다.

권장 원칙은 아래와 같다.

- 공통 연결은 읽기 전용부터
- 팀 문서에 사용 예시 포함
- 어떤 질문에 이 연결을 써야 하는지 명시
- 분기별로 실제 사용량과 효용 재검토

## 체크리스트

- MCP의 목적을 "복붙 감소 + 정확성 향상"으로 설명할 수 있다.
- 붙이면 좋은 도구와 아직 이른 도구를 구분할 수 있다.
- 읽기 전용으로 시작하는 이유를 이해한다.
- MCP가 노출하는 슬래시 커맨드 개념을 안다.
- 내 작업 흐름에서 가장 먼저 붙일 후보를 정할 수 있다.
