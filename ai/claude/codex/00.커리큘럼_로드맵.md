# Claude Code 마스터 커리큘럼 로드맵

> 목표: Claude Code를 "신기한 AI"가 아니라, 일하고 검증하고 반복하는 개발 파트너로 다룰 수 있게 만드는 것

이 커리큘럼은 완전 입문자를 기준으로 설계했다. 순서는 `개념 이해 -> 안전한 사용 -> 반복 자동화 -> 1인개발 실무 -> 팀 확장`이다. 각 문서는 따로 읽어도 되지만, 처음 한 바퀴는 반드시 순서대로 도는 것을 권장한다.

## 이 묶음의 원칙

- Claude Code 기능을 중심에 둔다.
- 기존 학습 자료의 표현을 답습하지 않고, 바로 실무에 연결되는 방식으로 설명한다.
- 모든 모듈은 `왜 중요한가`, `핵심 개념`, `실무에서 잘 쓰는 사람의 방식`, `자주 망하는 방식`, `바로 써보는 예시`, `1인개발 적용`, `팀 적용`, `체크리스트`를 같은 구조로 가진다.
- 설명은 입문자 친화적으로 쓰되, 문서 끝에 도달하면 혼자 프로젝트를 굴리거나 팀 표준을 만들 수 있도록 설계한다.

## 권장 학습 루프

1. 문서를 읽는다.
2. 바로 아래 예시를 Claude Code에서 실제로 실행해 본다.
3. 실패한 로그와 결과를 메모한다.
4. 내 프로젝트에 맞게 문장을 한 번 더 바꿔 본다.
5. 마지막 체크리스트로 "아는 것"이 아니라 "할 수 있는 것"을 점검한다.

이 커리큘럼은 읽기만 하면 반밖에 못 배운다. 최소한 각 모듈의 예시는 손으로 한 번 쳐봐야 한다.

## 선행 관계

- `01 -> 02 -> 03`은 필수 선행이다.
- `04`는 `03` 다음에 보는 것이 좋다. 요청을 잘해도 안전과 비용을 모르면 오래 못 간다.
- `05`와 `06`은 함께 보면 좋다. 규칙을 정리하고 자동화해야 반복 설명이 줄어든다.
- `07`과 `08`은 중급 진입 구간이다. 기초가 약한 상태에서 먼저 들어가면 멋있어 보이는 설정만 늘어난다.
- `09`, `10`, `11`은 실무 적용 파트다. 여기서는 기능이 아니라 운영 습관을 만든다.

## 학습 중 꼭 지킬 원칙

- 처음부터 모든 기능을 켜지 않는다.
- 권한 모드는 공격적으로 올리기보다 천천히 넓힌다.
- 길고 모호한 요청보다 짧고 구조화된 요청이 낫다.
- 자동화는 성공 경험이 반복된 후에 붙인다.
- 팀 도입은 개인 성공 루프가 검증된 뒤에 확장한다.

## 이 커리큘럼을 다 끝내면

- Claude Code에게 일을 "대신" 시키는 수준이 아니라, 일을 "설계해서 맡기는" 수준에 도달한다.
- 개인 작업에서는 탐색, 구현, 검증, 문서화를 짧은 루프로 돌릴 수 있다.
- 팀 작업에서는 규칙, 권한, 재현 가능한 워크플로를 설계할 수 있다.
- `하네스`를 무언가 신비한 제품으로 보지 않고, Claude Code가 일하는 환경 전체를 조율하는 개념으로 이해하게 된다.
