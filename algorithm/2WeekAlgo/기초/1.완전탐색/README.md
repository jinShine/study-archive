## 문제 1. 비밀번호 (#1816)

확인하고자 하는 수가 **적절한 비밀번호**인지 판별하는 프로그램.

- 적절하면 `YES`, 아니면 `NO` 출력
- **적절한 비밀번호**: 모든 소인수가 1,000,000보다 큰 수
- **소인수**: 1을 제외한 약수

입력 / 출력 예시:

```text
3
1000036000099
1500035500153
20000000000002
```

```text
YES
NO
NO
```

풀이 포인트:

- N의 소인수를 직접 다 구할 필요 없음
- **2부터 1,000,000까지의 수로 나눠봐서**, 하나라도 나누어떨어지면 `NO`
- N이 매우 크므로 JavaScript에서는 `BigInt` 사용

## 문제 2. 사탕 (#14568)

친구 A, B, C에게 사탕 N개를 나누어 주는 경우의 수를 구하시오.

조건:

1. 남는 사탕이 없어야 함
2. A는 B보다 2개 이상 많은 사탕을 가져야 함
3. 셋 중 하나도 못 받는 친구가 없어야 함 (모두 1개 이상)
4. C가 받는 사탕의 수는 짝수

입력 / 출력 예시:

```text
6
```

```text
1
```

풀이 포인트:

- A, B, C 모두 1 이상이고 합이 N
- 3중 for문으로 (a, b, c) 모든 조합 시도
- 조건 4가지를 모두 만족하는 경우만 카운트

## 문제 3. 연립방정식 (#19532)

다음 연립방정식의 해 (x, y)를 구하시오.

```text
A·x + B·y = C
D·x + E·y = F
```

- x, y는 -10,000 이상 10,000 이하 정수

입력 / 출력 예시:

```text
1 3 -1 4 1 7
```

```text
2 -1
```

풀이 포인트:

- x, y의 범위가 좁음 (각 20,001가지)
- **완전탐색**: -10000 ~ 10000 두 번 돌면서 두 식 모두 만족하는 (x, y) 찾기
- (수학적 풀이: 크래머 공식으로도 가능)

## 문제 4. 숫자야구 (#2503)

A는 **세 자리 정답**(서로 다른 숫자)을 정한다.
B가 던진 숫자에 대해 A는 Strike, Ball을 알려준다.

- **Strike**: 숫자가 같고 자리도 같음
- **Ball**: 숫자는 같지만 자리는 다름
- 0은 사용하지 않음 (1~9)

질문 결과들이 주어졌을 때, **가능한 정답 후보 수**를 출력하시오.

입력 / 출력 예시:

```text
4
123 1 1
356 1 0
327 2 0
489 0 1
```

```text
2
```

풀이 포인트:

- 가능한 모든 세 자리 수(1~9, 서로 다른 숫자)를 만들어봄 → 9×8×7 = 504개
- 각 후보마다 모든 질문에 대해 Strike/Ball을 계산
- 모든 질문 조건을 만족하면 정답 후보로 카운트

풀이 단계:

```text
(1) 세 자리 숫자 만들기   → 3중 for문 (a, b, c)
(2) 서로 다른 숫자 검사   → a≠b, b≠c, c≠a
(3) 질문마다 Strike/Ball 계산
(4) 모든 질문 조건과 일치하면 +1
```

## 문제 5. 모이기 (#1090)

N명의 학생이 한 곳에 모이려고 한다.
모든 사람의 이동 거리 합이 **최소**가 되는 위치에 모인다.

학생들의 집 위치(Y, X)가 주어질 때, **k명(k=1..N)이 모일 수 있는 최소 거리**를 각각 구하시오.

거리는 **맨해튼 거리** = `|x1-x2| + |y1-y2|`

조건:

- N ≤ 50
- 좌표 ≤ 1,000,000

입력 / 출력 예시:

```text
4
15 14
15 16
14 15
16 15
```

```text
0 2 3 4
```

풀이 포인트:

- **모이는 위치 후보**: 각 학생의 X 좌표와 Y 좌표 조합 (`arr_x × arr_y`)
- 후보 위치마다:
  1. 각 학생까지의 거리 계산
  2. 가까운 순으로 정렬
  3. k=1,2,...,N에 대해 누적합으로 "k명이 모일 때의 거리"를 갱신
- 모든 후보 위치에 대해 최솟값 유지
