## 1. 그래프란?

### 한 줄 정의

**그래프(Graph)** 는 객체 간의 **연결 관계**를 표현하는 자료구조다. 객체는 **정점(Vertex/Node)**, 연결은 **간선(Edge)**으로 표현한다.

### 현실 세계의 그래프

그래프는 추상적인 개념이 아니라 우리 주변에 가득하다.

```
🗺️ 지도/네비게이션
   정점 = 도시, 교차로
   간선 = 도로
   가중치 = 거리, 시간

👥 SNS (인스타그램)
   정점 = 사용자
   간선 = 팔로우 관계
   방향 = 내가 팔로우 ≠ 상대가 팔로우 (방향(단방향) 그래프)

👥 SNS (카카오톡 친구)
   정점 = 사용자
   간선 = 친구 관계
   양방향 = 내가 친구면 상대도 친구 (무방향(양방향) 그래프)

🌐 웹페이지
   정점 = 웹페이지
   간선 = 하이퍼링크
   방향 = A에서 B로 링크 ≠ B에서 A로 링크
```

### 왜 그래프를 배우는가?

코딩테스트에서 그래프 문제는 **매우 자주** 출제된다.

| 문제 유형 | 설명 | 예시 |
|----------|------|------|
| 최단 경로 | A에서 B까지 가장 빠른 길 | 미로 탈출, 지도 경로 |
| 연결 요소 | 서로 연결된 그룹이 몇 개? | 섬 개수 세기, 네트워크 |
| 경로 존재 여부 | A에서 B로 갈 수 있는가? | 네트워크 연결 확인 |
| 사이클 검출 | 순환이 존재하는가? | 작업 순서의 모순 찾기 |

## 2. 그래프 용어 정리

그래프를 공부하기 전에 용어부터 확실히 잡자. 코드와 문제에서 계속 나온다.

```
그래프 시각화:

    (1) ───── (2)
     |  \      |
     |    \    |
    (3)    (4) |
     |         |
    (5) ───── (6)
```

### 필수 용어

| 용어 | 영어 | 의미 | 위 그래프 예시 |
|------|------|------|--------------|
| **정점 (노드)** | Vertex / Node | 그래프의 각 객체 | 1, 2, 3, 4, 5, 6 |
| **간선** | Edge | 정점 사이의 연결선 | (1,2), (1,3), (1,4), ... |
| **인접** | Adjacent | 간선으로 직접 연결된 관계 | 1과 2는 인접, 1과 5는 비인접 |
| **차수** | Degree | 한 정점에 연결된 간선의 수 | 정점 1의 차수 = 3 (2,3,4와 연결) |
| **경로** | Path | 정점 A에서 B까지 간선을 따라 가는 길 | 1→3→5→6 은 하나의 경로 |
| **경로 길이** | Path Length | 경로에 포함된 간선의 수 | 1→3→5→6 의 길이 = 3 |
| **사이클** | Cycle | 시작 정점으로 되돌아오는 경로 | 1→2→6→5→3→1 |

### 방향(단방향) 그래프에서의 추가 용어

```
방향 그래프:

    (1) ──→ (2)
     ↑        |
     |        ↓
    (3) ←── (4)
```

| 용어 | 의미 | 예시 |
|------|------|------|
| **진입 차수 (in-degree)** | 들어오는 간선의 수 | 정점 3의 진입 차수 = 1 (4→3) |
| **진출 차수 (out-degree)** | 나가는 간선의 수 | 정점 1의 진출 차수 = 1 (1→2) |

> 💡 **코딩테스트에서**: 문제에서 "노드", "정점", "꼭짓점"은 전부 같은 뜻이다. "간선", "변", "엣지"도 같은 뜻이다.

## 3. 그래프의 종류

### 방향 유무에 따라

```
무방향(양방향) 그래프 (Undirected)     방향(단방향) 그래프 (Directed)

    (1) ── (2)                      (1) ──→ (2)
     |       |                       ↑        |
    (3) ── (4)                      (3) ←── (4)

  1↔2 서로 오갈 수 있음             1→2 한쪽으로만 감
  카카오톡 친구                     인스타그램 팔로우
```

| 종류 | 간선 방향 | 예시 | 코드에서의 차이 |
|------|----------|------|---------------|
| **무방향(양방향) 그래프** | 양방향 (A↔B) | 카카오톡 친구, 도로 | 간선 추가 시 **양쪽 다** 추가 |
| **방향(단방향) 그래프** | 단방향 (A→B) | 인스타 팔로우, 웹 링크 | 간선 추가 시 **한쪽만** 추가 |

```java
// 무방향(양방향): 양쪽 다 추가
adj.get(1).add(2);
adj.get(2).add(1);   // ← 이것도 추가해야 함!

// 방향(단방향): 한쪽만 추가
adj.get(1).add(2);   // 1→2만 존재
```

> ⚠️ **자주 하는 실수**: 무방향(양방향) 그래프인데 한쪽만 추가하는 것! 문제에서 "양방향" 또는 "무방향"이라고 하면 **반드시 양쪽** 추가.

### 가중치 유무에 따라

```
비가중치 그래프                      가중치 그래프

    (1) ── (2)                      (1) ──3── (2)
     |       |                       |          |
    (3) ── (4)                      (3) ──7── (4)
                                          2
  간선의 비용이 모두 동일            간선마다 비용(거리, 시간 등)이 다름
  → BFS로 최단 경로 구할 수 있음     → 다익스트라 등 별도 알고리즘 필요
```

### 사이클 유무에 따라

```
순환 그래프 (Cyclic)                 비순환 그래프 (Acyclic)

    (1) → (2)                        (1) → (2)
     ↑      ↓                               ↓
    (4) ← (3)                        (3) → (4)

  1→2→3→4→1 사이클 존재              돌아오는 경로가 없음
                                    DAG(방향 비순환 그래프)
                                    → 위상 정렬에 사용
```

### 단순 그래프

- 정점 간에 **하나의 간선만** 존재하며, **self-loop**(자기 자신을 가리키는 간선)이 없는 그래프
- 코딩테스트에서 다루는 대부분의 그래프는 단순 그래프다

```
단순 그래프 ✅                       단순 그래프 ❌

    (1) ── (2)                      (1) ══ (2)    ← 간선 2개 (다중 간선)
     |       |                       ↺
    (3) ── (4)                      self-loop     ← 자기 자신으로의 간선
```

## 4. 그래프 구현 방법 1 — 인접 행렬

### 📌 예시 그래프 (이후 모든 구현 방법에서 이 그래프를 사용)

```
  방향 그래프 (5개 노드) — 양방향/단방향 혼합

    (1) ←──→ (2)
     │  ╲      │
     ↕   ╲     ↓
    (3)   ↘  (4)
     │
     ↓
    (5)

  간선 목록:
    양방향: 1↔2, 1↔3       (서로 오갈 수 있음)
    단방향: 1→4, 2→4, 3→5  (한쪽으로만 감)

  → 총 간선 7개: 1→2, 2→1, 1→3, 3→1, 1→4, 2→4, 3→5
```

### 인접 행렬이란?

**2차원 배열(N×N)** 로 정점 간의 연결 관계를 나타낸다. 연결되어 있으면 `1`, 아니면 `0`.

```
쉽게 말해: "표(Table)로 연결 관계를 기록하는 것"

  행(row) = 출발 노드
  열(col) = 도착 노드
  값 = 연결 여부 (1 또는 0)
```

### 인접 행렬 도표

|       | **1** | **2** | **3** | **4** | **5** |
|:-----:|:-----:|:-----:|:-----:|:-----:|:-----:|
| **1** |   0   |   1   |   1   |   1   |   0   |
| **2** |   1   |   0   |   0   |   1   |   0   |
| **3** |   1   |   0   |   0   |   0   |   1   |
| **4** |   0   |   0   |   0   |   0   |   0   |
| **5** |   0   |   0   |   0   |   0   |   0   |

### 읽는 법

```
matrix[1][2] = 1  →  1에서 2로 갈 수 있다 ✅
matrix[2][1] = 1  →  2에서 1로 갈 수 있다 ✅  (양방향이니까)

matrix[1][4] = 1  →  1에서 4로 갈 수 있다 ✅
matrix[4][1] = 0  →  4에서 1로 갈 수 없다 ❌  (단방향이니까)

matrix[4][5] = 0  →  4에서 5로 갈 수 없다 ❌
```

> 💡 양방향 간선(1↔2)이면 `matrix[1][2]=1, matrix[2][1]=1` 양쪽 다 1이다.
> 단방향 간선(1→4)이면 `matrix[1][4]=1`이지만 `matrix[4][1]=0`이다.

### 코드

```java
int N = 5;
int[][] matrix = new int[N + 1][N + 1]; // 1-indexed (0번은 안 씀)

// 단방향 간선 추가 (e[0] → e[1])
int[][] oneWay = {{1,4}, {2,4}, {3,5}};
for (int[] e : oneWay) {
    matrix[e[0]][e[1]] = 1;              // 한쪽만!
}

// 양방향 간선 추가 (서로 오갈 수 있음)
int[][] twoWay = {{1,2}, {1,3}};
for (int[] e : twoWay) {
    matrix[e[0]][e[1]] = 1;              // A → B
    matrix[e[1]][e[0]] = 1;              // B → A  ← 이거 빼먹으면 안 됨!
}

// 노드 1에서 갈 수 있는 곳 확인
for (int j = 1; j <= N; j++) {
    if (matrix[1][j] == 1) {
        System.out.println("1 → " + j + " 간선 존재");
    }
}
// 출력: 1 → 2, 1 → 3, 1 → 4 간선 존재
```

### 가중치 그래프라면?

```java
// 0/1 대신 가중치(비용)를 저장
matrix[1][2] = 5;   // 1→2 비용 5
matrix[1][4] = 10;  // 1→4 비용 10

// 연결 안 된 곳은 0 또는 INF(무한대)로 표시
int INF = Integer.MAX_VALUE;
```

### 장단점

| | 설명 |
|---|---|
| ✅ 장점 | 두 노드 간 연결 여부를 **O(1)** 로 바로 확인 가능 (`matrix[i][j]`) |
| ❌ 단점 | 공간 복잡도 **O(V²)**, 노드가 10만 개면 100억 칸 → 메모리 초과! |
| ❌ 단점 | 간선이 적은 **희소 그래프**에서 메모리 낭비가 심함 |

### 언제 쓰는가?

```
✅ 노드 수가 적을 때 (V ≤ 500 정도)
✅ 두 노드의 연결 여부를 자주 확인해야 할 때
✅ 플로이드-워셜 알고리즘 (모든 쌍 최단 경로)
```

## 5. 그래프 구현 방법 2 — 인접 리스트

### 인접 리스트란?

각 정점마다 **연결된 정점들의 리스트**를 저장한다. 실제 간선만 저장하므로 메모리 효율적이다.

```
쉽게 말해: "각 노드별로 친구 목록을 따로 만드는 것"
```

### 인접 리스트 도표

```
노드 →  나가는 간선의 도착 노드 리스트
──────────────────────────────────
  1  →  [ 2, 3, 4 ]     ← 양방향(2,3) + 단방향(4)
  2  →  [ 1, 4 ]         ← 양방향(1) + 단방향(4)
  3  →  [ 1, 5 ]         ← 양방향(1) + 단방향(5)
  4  →  [ ]              ← 들어오기만 하는 노드
  5  →  [ ]              ← 들어오기만 하는 노드
```

### 코드

```java
int N = 5;

// 인접 리스트 생성 (0번~N번까지, 1-indexed이므로 0번은 안 씀)
List<List<Integer>> adj = new ArrayList<>();
for (int i = 0; i <= N; i++) {
    adj.add(new ArrayList<>());
}

// 단방향 간선 추가
int[][] oneWay = {{1,4}, {2,4}, {3,5}};
for (int[] e : oneWay) {
    adj.get(e[0]).add(e[1]);             // 한쪽만!
}

// 양방향 간선 추가
int[][] twoWay = {{1,2}, {1,3}};
for (int[] e : twoWay) {
    adj.get(e[0]).add(e[1]);             // A → B
    adj.get(e[1]).add(e[0]);             // B → A
}

// 노드 1에서 갈 수 있는 곳 확인
for (int neighbor : adj.get(1)) {
    System.out.println("1 → " + neighbor + " 간선 존재");
}
// 출력: 1 → 2, 1 → 3, 1 → 4 간선 존재
```

### 인접 행렬 vs 인접 리스트 — 같은 정보를 다르게 저장하는 것

```
같은 그래프를 두 가지로 표현:

인접 행렬 (표 방식):              인접 리스트 (목록 방식):
    1  2  3  4  5                 1 → [2, 3, 4]
1 [ 0  1  1  1  0 ]              2 → [1, 4]
2 [ 1  0  0  1  0 ]              3 → [1, 5]
3 [ 1  0  0  0  1 ]              4 → []
4 [ 0  0  0  0  0 ]              5 → []
5 [ 0  0  0  0  0 ]

행렬은 "빈 칸(0)"도 다 저장       리스트는 실제 연결만 저장
→ 메모리 낭비                     → 메모리 효율적
```

### 장단점

| | 설명 |
|---|---|
| ✅ 장점 | 공간 복잡도 **O(V + E)**, 간선이 적으면 매우 효율적 |
| ✅ 장점 | 인접 노드 순회가 빠름 (실제 연결된 것만 순회) |
| ❌ 단점 | 두 노드 간 연결 확인이 **O(degree)** (리스트를 순회해야 함) |

### 언제 쓰는가?

```
✅ 노드 수가 많을 때 (V > 500)
✅ 간선이 상대적으로 적을 때 (희소 그래프)
✅ BFS, DFS, 다익스트라 등 대부분의 그래프 알고리즘
✅ 코딩테스트에서 가장 많이 사용하는 방법!
```

### 🔄 간선 리스트 → 인접 리스트 변환

#### 왜 변환이 필요한가?

코딩테스트 문제에서는 그래프를 **간선 리스트(Edge List)** 형태로 주는 경우가 매우 많다.

```
문제 입력 예시:
  "노드 수 n = 5, 간선: [[0,1], [0,2], [1,3], [2,4], [3,4]]"

이때 입력은 int[][] edges = {{0,1}, {0,2}, {1,3}, {2,4}, {3,4}};
→ 이걸 그대로 쓰면 "노드 0에서 갈 수 있는 곳이 어디?"를 찾으려면
  매번 edges 전체를 순회해야 함 → O(E)

→ 인접 리스트로 변환하면
  graph.get(0) = [1, 2] 바로 접근 → O(1)
```

#### 간선 리스트 vs 인접 리스트

| | 간선 리스트 `int[][] edges` | 인접 리스트 `List<List<Integer>>` |
|---|---|---|
| **형태** | `[[0,1], [0,2], [1,3]]` | `0→[1,2], 1→[3], ...` |
| **특정 노드의 이웃 찾기** | 전체 순회 O(E) ❌ | 바로 접근 O(1) ✅ |
| **BFS/DFS에 사용** | 불편 ❌ | 바로 사용 가능 ✅ |
| **코테에서의 역할** | **입력** 형태 | **탐색용** 자료구조 |

> 💡 **핵심**: 간선 리스트는 "문제가 주는 입력 형태"이고, 인접 리스트는 "우리가 탐색에 쓰는 자료구조"다. 변환은 거의 모든 그래프 문제의 **첫 번째 단계**이다!

#### 변환 코드 — 무방향(양방향) 그래프

```java
int n = 5;  // 노드 수
int[][] edges = {{0,1}, {0,2}, {1,3}, {2,4}, {3,4}};

// 1. 빈 인접 리스트 생성 (노드 개수만큼)
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}

// 2. 간선 리스트를 순회하면서 인접 리스트에 추가
for (int[] edge : edges) {
    graph.get(edge[0]).add(edge[1]);  // A → B
    graph.get(edge[1]).add(edge[0]);  // B → A  ← 무방향(양방향)이니까 양쪽!
}

// 결과:
// 0 → [1, 2]
// 1 → [0, 3]
// 2 → [0, 4]
// 3 → [1, 4]
// 4 → [2, 3]
```

#### 변환 코드 — 방향(단방향) 그래프

```java
int n = 5;
int[][] edges = {{0,1}, {0,2}, {1,3}, {2,4}, {3,4}};

List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}

// 방향(단방향) 그래프: 한쪽만 추가!
for (int[] edge : edges) {
    graph.get(edge[0]).add(edge[1]);  // A → B 만!
}

// 결과:
// 0 → [1, 2]
// 1 → [3]
// 2 → [4]
// 3 → [4]
// 4 → []
```

#### ⚠️ 0-indexed vs 1-indexed 주의!

문제에 따라 노드 번호가 **0부터** 시작할 수도 있고, **1부터** 시작할 수도 있다.

```java
// 📌 0-indexed (노드 번호: 0, 1, 2, ..., n-1)
// → 배열 크기 = n
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}
// graph.get(0) ~ graph.get(n-1) 사용

// 📌 1-indexed (노드 번호: 1, 2, 3, ..., n)
// → 배열 크기 = n+1 (0번은 안 씀)
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i <= n; i++) {    // ← n+1개 생성!
    graph.add(new ArrayList<>());
}
// graph.get(1) ~ graph.get(n) 사용
```

> ⚠️ **자주 하는 실수**: 1-indexed 문제인데 `n`개만 생성하면, `graph.get(n)`에서 `IndexOutOfBoundsException` 발생! 항상 문제의 노드 번호 범위를 확인하자.

#### 변환 후 바로 BFS/DFS 실행하는 전체 흐름

```java
// ✅ 코딩테스트 전형적인 흐름
int n = 5;
int[][] edges = {{0,1}, {0,2}, {1,3}, {2,4}, {3,4}};

// STEP 1: 간선 리스트 → 인접 리스트 변환
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}
for (int[] edge : edges) {
    graph.get(edge[0]).add(edge[1]);
    graph.get(edge[1]).add(edge[0]);  // 무방향(양방향)
}

// STEP 2: BFS 실행
boolean[] visited = new boolean[n];
bfs(graph, 0, visited);  // 0번 노드부터 탐색 시작
```

> 💡 **패턴 정리**: 코테 그래프 문제는 거의 항상 `입력 파싱 → 인접 리스트 변환 → BFS/DFS 탐색` 이 3단계를 따른다. 이 흐름을 외워두면 문제의 뼈대를 빠르게 잡을 수 있다!

## 6. 인접 행렬 vs 인접 리스트 비교

| 비교 항목 | 인접 행렬 | 인접 리스트 |
|:---------:|:---------:|:-----------:|
| **공간 복잡도** | O(V²) | O(V + E) |
| **간선 확인** | O(1) ✅ | O(degree) |
| **인접 노드 순회** | O(V) | O(degree) ✅ |
| **적합한 경우** | 밀집 그래프, 작은 V | 희소 그래프, 큰 V |
| **대표 알고리즘** | 플로이드-워셜 | 다익스트라, BFS/DFS |

### 실전에서 뭘 쓰나?

```
1순위: 인접 리스트  → 대부분의 그래프 문제 (코딩테스트 기본!)
2순위: 인접 행렬   → 노드 수가 적고, 플로이드-워셜을 쓸 때
```

## 7. 그래프 순회란?

**그래프 순회(Graph Traversal)** 는 그래프의 모든 노드를 **체계적으로 방문**하는 과정이다.

### 왜 순회가 필요한가?

```
배열 순회:  for (int i = 0; i < n; i++)  ← 순서가 정해져 있음

그래프 순회: ???  ← 순서가 정해져 있지 않음!
                   한 노드에서 여러 곳으로 갈 수 있고,
                   이미 방문한 곳을 다시 갈 수도 있음.
                   → 체계적인 방법이 필요하다!
```

### 두 가지 방법

| 방법 | 비유 | 핵심 |
|------|------|------|
| **BFS** (너비 우선) | 물이 퍼지듯이 | 가까운 것부터 순서대로 |
| **DFS** (깊이 우선) | 미로에서 한 길로 끝까지 | 한 방향으로 깊이 파고듦 |

### visited 배열 — 왜 필요한가?

```
그래프에는 사이클이 있을 수 있다!

    (1) ── (2)
     |       |
    (3) ── (4)

1 → 2 → 4 → 3 → 1 → 2 → 4 → 3 → 1 → ...  무한 루프!

→ "이미 방문한 노드"를 기록해서 다시 방문하지 않아야 한다.
→ 이것이 visited[] 배열의 역할!
```

```java
boolean[] visited = new boolean[N + 1];  // false로 초기화됨

// 노드 방문 시:
visited[node] = true;  // "여기 왔다!" 표시

// 다음 노드로 갈 때:
if (!visited[neighbor]) {  // 아직 안 갔으면 갈 수 있음
    // 방문!
}
```

> 💡 **visited는 BFS/DFS 모두에서 필수**이다. 없으면 무한 루프에 빠진다.

## 8. BFS — 너비 우선 탐색

### 한 줄 정의

**BFS(Breadth-First Search)** 는 시작 노드에서 **가까운 노드부터 차례대로** 방문하는 탐색 방법이다.

### 비유: 돌을 물에 던지면

```
돌을 던지면 파문이 동심원으로 퍼진다.
BFS도 마찬가지로 시작점에서 "한 칸 거리" → "두 칸 거리" → "세 칸 거리" 순서로 퍼져나간다.

Level 0:  시작점 자체
Level 1:  시작점에서 1칸 거리인 노드들
Level 2:  시작점에서 2칸 거리인 노드들
...
```

### 사용 자료구조: 큐 (Queue)

```
BFS는 큐를 사용한다. 왜?

큐 = FIFO (먼저 넣은 것이 먼저 나옴)
→ "먼저 발견한 (= 가까운) 노드를 먼저 처리"
→ 자연스럽게 가까운 것부터 방문하게 된다!
```

### BFS 탐색 과정 (시작 노드: 1)

위 예시 그래프(1↔2, 1↔3, 1→4, 2→4, 3→5)를 기준으로:

```
단계별 탐색:

  Level 0:  (1)             ← 시작
              ↓
  Level 1:  (2) (3) (4)     ← 1과 인접한 노드들
              ↓   ↓
  Level 2:  (4) (5)         ← 2,3과 인접한 미방문 노드
                              (4는 이미 방문했으므로 스킵)

  ✅ 방문 순서: 1 → 2 → 3 → 4 → 5
```

### 큐 상태 변화 (하나하나 따라가보자)

| 단계 | 큐에서 꺼냄 | 큐 상태 | 방문 완료 | 설명 |
|:---:|:---:|:---:|:---:|:---|
| 1 | - | [**1**] | - | 시작 노드 1을 큐에 넣음 |
| 2 | **1** | [2, 3, 4] | 1 | 1을 꺼내고, 1의 이웃(2,3,4)을 큐에 넣음 |
| 3 | **2** | [3, 4] | 1, 2 | 2를 꺼내고, 2의 이웃 중 미방문(4)은 이미 큐에 있음 |
| 4 | **3** | [4, 5] | 1, 2, 3 | 3을 꺼내고, 3의 이웃 중 미방문(5)을 큐에 넣음 |
| 5 | **4** | [5] | 1, 2, 3, 4 | 4를 꺼냄, 이웃 없음 |
| 6 | **5** | [] | 1, 2, 3, 4, 5 ✅ | 5를 꺼냄, 큐 비어있음 → 종료 |

### BFS 코드

BFS는 **3단계 구조**를 외우면 어떤 문제든 응용할 수 있다.

```
BFS 3단계 구조:

1. 시작 노드 예약   → 큐에 넣고 visited 표시
2. 방문             → 큐에서 꺼냄 (poll)
3. 다음 노드 예약   → 이웃 중 미방문을 큐에 넣고 visited 표시

이 세 단계가 while문 안에서 반복된다!
```

```java
void bfs(List<List<Integer>> graph, int start, boolean[] visited) {
    // 1. 시작 노드 예약
    Queue<Integer> queue = new ArrayDeque<>();
    queue.offer(start);
    visited[start] = true;

    while (!queue.isEmpty()) {
        // 2. 방문
        int cur = queue.poll();

        // 3. 다음 노드 예약
        for (int next : graph.get(cur)) {
            if (!visited[next]) {
                queue.offer(next);
                visited[next] = true;
            }
        }
    }
}

// 사용:
// boolean[] visited = new boolean[graph.size()];
// bfs(graph, 1, visited);
// → 방문 순서: 1 2 3 4 5
```

### 3단계를 큐 상태로 따라가보기

```
graph: 1↔2, 1↔3, 1→4, 2→4, 3→5

[1단계: 시작 노드 예약]
  큐: [1]        visited: {1}

[while 반복 시작]

  [2단계: 방문] cur = 1 (큐에서 꺼냄)
  [3단계: 예약] 1의 이웃(2,3,4) → 큐에 넣고 visited 표시
  큐: [2, 3, 4]  visited: {1, 2, 3, 4}

  [2단계: 방문] cur = 2
  [3단계: 예약] 2의 이웃(1,4) → 모두 visited → 아무것도 안 넣음
  큐: [3, 4]     visited: {1, 2, 3, 4}

  [2단계: 방문] cur = 3
  [3단계: 예약] 3의 이웃(1,5) → 5만 미방문 → 5 예약
  큐: [4, 5]     visited: {1, 2, 3, 4, 5}

  [2단계: 방문] cur = 4
  [3단계: 예약] 이웃 없음
  큐: [5]

  [2단계: 방문] cur = 5
  [3단계: 예약] 이웃 없음
  큐: []          → 종료!
```

### BFS 핵심 포인트

```
⚠️ "예약"할 때 (큐에 넣을 때) visited = true 를 한다!

// ✅ 올바른 방법: 예약(큐에 넣을 때) 시점에 방문 표시
if (!visited[next]) {
    queue.offer(next);
    visited[next] = true;    // ← 예약과 동시에!
}

// ❌ 잘못된 방법: 방문(큐에서 꺼낼 때) 시점에 방문 표시
int cur = queue.poll();
visited[cur] = true;         // ← 너무 늦음! 같은 노드가 여러 번 큐에 들어감
```

```
왜 "예약할 때"여야 하나?

노드 A와 B가 모두 노드 C와 인접한 경우:
  A를 처리하면서 C를 예약(큐에 넣음)
  B를 처리하면서 C를 또 예약할 수 있음! → 중복!

→ 예약할 때 바로 visited = true 하면 중복 방지됨
```

### BFS가 최단 경로를 보장하는 이유

```
BFS는 Level 순서대로 방문한다:
  Level 0 (거리 0) → Level 1 (거리 1) → Level 2 (거리 2) → ...

어떤 노드를 처음 방문했을 때의 거리 = 최단 거리
(비가중치 그래프에서만! 가중치가 있으면 다익스트라 필요)
```

### BFS로 최단 거리 구하기

같은 3단계 구조에 `dist[]` 배열만 추가하면 된다. `dist[]`가 visited 역할도 겸한다.

```java
int[] bfsDistance(List<List<Integer>> graph, int start) {
    int[] dist = new int[graph.size()];
    Arrays.fill(dist, -1);                // -1 = 미방문 (visited 역할)

    // 1. 시작 노드 예약
    Queue<Integer> queue = new ArrayDeque<>();
    queue.offer(start);
    dist[start] = 0;                      // 시작점 거리 = 0

    while (!queue.isEmpty()) {
        // 2. 방문
        int cur = queue.poll();

        // 3. 다음 노드 예약
        for (int next : graph.get(cur)) {
            if (dist[next] == -1) {            // 미방문이면
                queue.offer(next);
                dist[next] = dist[cur] + 1;    // 거리 = 이전 노드 + 1
            }
        }
    }

    return dist;
    // dist[i] = 시작점에서 노드 i까지의 최단 거리
    // dist[i] == -1 이면 도달 불가
}
```

## 9. DFS — 깊이 우선 탐색

### 한 줄 정의

**DFS(Depth-First Search)** 는 시작 노드에서 **한 방향으로 끝까지 깊이 들어간 후**, 더 이상 갈 곳이 없으면 되돌아오는 탐색 방법이다.

### 비유: 미로 탈출

```
미로에서 길을 찾을 때:
1. 갈림길에서 하나를 골라 끝까지 간다
2. 막다른 길이면 갈림길로 되돌아온다 (백트래킹)
3. 아직 안 가본 다른 길을 선택한다
4. 출구를 찾을 때까지 반복

이것이 바로 DFS의 동작 방식이다!
```

### 사용 자료구조: 스택 (Stack) 또는 재귀 (Recursion)

```
DFS는 스택을 사용한다. 왜?

스택 = LIFO (나중에 넣은 것이 먼저 나옴)
→ "가장 최근에 발견한 (= 가장 깊은) 노드를 먼저 처리"
→ 자연스럽게 깊이 우선으로 탐색하게 된다!

재귀도 내부적으로 콜 스택을 사용하므로 같은 원리다.
```

### DFS 탐색 과정 (시작 노드: 1)

```
단계별 탐색 (깊이 우선):

  (1) → (2) → (4)           ← 4에서 막힘! 되돌아감
                  ↩
        (2) ← 인접 노드 다 방문함, 되돌아감
                  ↩
  (1) → (3) → (5)           ← 5에서 막힘! 되돌아감
                  ↩
        (3) ← 다 방문함, 되돌아감
                  ↩
  (1) → (4) ← 이미 방문함, 스킵
  탐색 종료!

  ✅ 방문 순서: 1 → 2 → 4 → 3 → 5
```

### 스택(재귀 호출) 상태 변화

| 단계 | 현재 노드 | 호출 스택 | 행동 | 방문 완료 |
|:---:|:---:|:---:|:---|:---|
| 1 | 1 | [1] | 1 방문, 인접 노드 2로 이동 | 1 |
| 2 | 2 | [1, 2] | 2 방문, 인접 노드 4로 이동 | 1, 2 |
| 3 | 4 | [1, 2, 4] | 4 방문, 인접 노드 없음 → 되돌아감 | 1, 2, 4 |
| 4 | 2 | [1, 2] | 2의 남은 인접 없음 → 되돌아감 | 1, 2, 4 |
| 5 | 1 | [1] | 인접 노드 3으로 이동 | 1, 2, 4 |
| 6 | 3 | [1, 3] | 3 방문, 인접 노드 5로 이동 | 1, 2, 4, 3 |
| 7 | 5 | [1, 3, 5] | 5 방문, 인접 없음 → 되돌아감 | 1, 2, 4, 3, 5 |
| 8 | 1 | [1] | 인접 노드 4 → 이미 방문, 종료 | 1, 2, 4, 3, 5 ✅ |

### DFS 코드 — 재귀 방식 (가장 많이 쓰는 방식)

```java
void dfs(List<List<Integer>> graph, int cur, boolean[] visited) {
    // 1. 방문
    visited[cur] = true;                 
    System.out.print(cur + " ");         // 방문 처리

    // 2. 방문안한 다음 노드 찾기
    for (int next : graph.get(cur)) {
        if (!visited[next]) {            // 아직 방문 안 한 이웃이면
            dfs(graph, next, visited);   // 3. 다음 노드 DFS 실행 (재귀 호출 → 더 깊이 들어감)
        }
    }
    // 여기까지 오면 모든 이웃을 다 방문함 → 자동으로 되돌아감 (return)
}

// 사용:
// boolean[] visited = new boolean[graph.size()];
// dfs(graph, 1, visited);
// → 출력: 1 2 4 3 5
```

### 재귀 DFS vs 스택 DFS

| | 재귀 DFS | 스택 DFS |
|---|---|---|
| 코드 길이 | 짧고 깔끔 ✅ | 길고 복잡 |
| 방문 표시 | 함수 진입 시 | 꺼낼 때 |
| 스택 오버플로우 | 깊이 크면 위험 ❌ | 안전 ✅ |
| 코딩테스트 | 대부분 이걸 씀 ✅ | 깊이가 매우 클 때만 |

> 💡 **코딩테스트에서는 재귀 DFS를 기본으로 쓰자.** 코드가 훨씬 간결하다. 재귀 깊이가 10만 이상이면 스택 방식을 고려.

## 10. BFS vs DFS 비교

### 탐색 순서 비교

```
같은 그래프에서:

    (1) ── (2) ── (5)
     |       |
    (3) ── (4)

BFS (1에서 시작):  1 → 2 → 3 → 5 → 4      (가까운 것부터)
DFS (1에서 시작):  1 → 2 → 5 → 4 → 3      (깊은 것부터)

BFS:                          DFS:
 Level 0: (1)                  (1)→(2)→(5) 끝까지 감
 Level 1: (2)(3)                    ↩ 되돌아옴
 Level 2: (5)(4)               (1)→(2)→(4) ...
```

### 비교표

| 비교 항목 | BFS (너비 우선) | DFS (깊이 우선) |
|:---------:|:---------------:|:---------------:|
| **자료구조** | 큐 (Queue) | 스택 (Stack) / 재귀 |
| **탐색 방식** | 가까운 노드부터 | 깊은 노드부터 |
| **최단 경로** | ✅ 보장 (비가중치) | ❌ 보장하지 않음 |
| **메모리** | 넓은 그래프에서 큼 | 깊은 그래프에서 큼 |
| **방문 표시 시점** | 큐에 **넣을 때** | 스택에서 **꺼낼 때** / 재귀 진입 시 |
| **시간 복잡도** | O(V + E) | O(V + E) |
| **구현 난이도** | 보통 | 쉬움 (재귀) |

### 언제 뭘 쓰나?

| 문제 유형 | 추천 | 이유 |
|----------|------|------|
| **최단 거리** (비가중치) | BFS ✅ | 레벨 순서 = 최단 거리 |
| **미로 최단 경로** | BFS ✅ | 가장 빨리 도착하는 경로 |
| **경로 존재 여부** | 둘 다 가능 | BFS든 DFS든 도달하면 됨 |
| **연결 요소 개수** | 둘 다 가능 | 미방문 노드에서 탐색 시작 |
| **섬 개수 세기** | 둘 다 가능 | 격자에서 연결된 1 묶음 개수 |
| **모든 경로 탐색** | DFS ✅ | 경로를 기록하며 깊이 탐색 |
| **사이클 검출** | DFS ✅ | 방문 중인 노드를 다시 만나면 사이클 |

> 💡 **판단 기준**: "최단"이라는 단어가 보이면 → BFS. 그 외에는 DFS가 더 쓰기 쉬움.

## 11. 연결된 영역 — BFS/DFS 대표 문제 유형

BFS/DFS를 배웠으면, 실전 코딩테스트에서 **가장 자주 나오는 유형**이 바로 "연결된 영역" 문제다.

### 연결된 영역이란?

```
그래프에서 서로 이어진 노드들의 묶음 = 연결된 영역 (Connected Component)

예시:
    (1) ── (2)      (4) ── (5)      (7)
     |                |
    (3)              (6)

→ 연결된 영역 3개:
  영역 1: {1, 2, 3}     ← 서로 이어져 있음
  영역 2: {4, 5, 6}     ← 서로 이어져 있음
  영역 3: {7}           ← 혼자
```

> 💡 **핵심 아이디어**: 한 노드에서 BFS/DFS를 실행하면, 그 노드와 **연결된 모든 노드**를 방문한다. 즉 BFS/DFS **한 번 = 연결된 영역 하나**를 탐색하는 것이다!

### 대표 문제 3가지

코테에서 "연결된 영역" 문제는 크게 3가지 유형으로 나온다.

| 유형 | 질문 | 예시 문제 |
|------|------|----------|
| **1. 네트워크 개수** | 연결된 그룹이 **몇 개**인가? | "네트워크 수", "친구 그룹 수" |
| **2. 영역의 크기** | 각 영역에 포함된 **요소가 몇 개**인가? | "가장 큰 네트워크", "연결 요소 크기" |
| **3. 두 요소 간 연결 여부** | A와 B가 **같은 영역**에 속하는가? | "두 사람이 같은 네트워크?", "연결 확인" |

---

### 유형 1: 네트워크 개수 (연결된 영역이 몇 개?)

```
접근법:
  모든 노드를 순회하면서
  → 아직 방문 안 한 노드를 발견하면
  → 거기서 BFS/DFS 실행 (= 연결된 영역 하나를 전부 방문)
  → 카운트 +1
  → 모든 노드를 방문할 때까지 반복
```

```java
int countNetworks(List<List<Integer>> graph, int n) {
    boolean[] visited = new boolean[n];
    int count = 0;

    for (int i = 0; i < n; i++) {
        if (!visited[i]) {          // 아직 방문 안 한 노드 발견!
            dfs(graph, i, visited); // 이 노드와 연결된 모든 노드 방문
            count++;                // 영역 하나 완료 → 카운트 +1
        }
    }

    return count;
}

// 기존 DFS 그대로 사용
void dfs(List<List<Integer>> graph, int cur, boolean[] visited) {
    visited[cur] = true;
    for (int next : graph.get(cur)) {
        if (!visited[next]) {
            dfs(graph, next, visited);
        }
    }
}
```

```
위 예시 그래프로 실행:

i=0: visited[0]=false → dfs(0) → {0,1,2} 방문 → count=1
i=1: visited[1]=true  → 스킵
i=2: visited[2]=true  → 스킵
i=3: visited[3]=false → dfs(3) → {3,4,5} 방문 → count=2
i=4: visited[4]=true  → 스킵
i=5: visited[5]=true  → 스킵
i=6: visited[6]=false → dfs(6) → {6} 방문   → count=3

→ 결과: 네트워크 3개
```

---

### 유형 2: 영역의 크기 (각 영역에 요소가 몇 개?)

DFS/BFS 실행 시 **방문한 노드 수를 세면** 된다.

```java
// 방법 1: DFS가 방문한 노드 수를 리턴하도록 수정
int dfsCount(List<List<Integer>> graph, int cur, boolean[] visited) {
    visited[cur] = true;
    int count = 1;  // 현재 노드 포함

    for (int next : graph.get(cur)) {
        if (!visited[next]) {
            count += dfsCount(graph, next, visited);  // 재귀로 방문한 수 합산
        }
    }

    return count;
}

// 사용 예: 각 영역의 크기 구하기
boolean[] visited = new boolean[n];
for (int i = 0; i < n; i++) {
    if (!visited[i]) {
        int size = dfsCount(graph, i, visited);
        System.out.println("노드 " + i + "가 속한 영역 크기: " + size);
    }
}
// 출력:
// 노드 0가 속한 영역 크기: 3
// 노드 3가 속한 영역 크기: 3
// 노드 6가 속한 영역 크기: 1
```

> 💡 **응용**: "가장 큰 네트워크의 크기를 구하시오" → 각 영역 크기를 구한 뒤 `Math.max()`로 최대값 찾기

### 유형 3: 두 요소 간 연결 여부 (A와 B가 연결되어 있는가?)

A에서 BFS/DFS를 실행한 후, B가 방문되었는지 확인하면 된다.

```java
boolean isConnected(List<List<Integer>> graph, int a, int b, int n) {
    boolean[] visited = new boolean[n];
    dfs(graph, a, visited);     // a에서 시작해서 연결된 모든 노드 방문

    return visited[b];          // b가 방문되었으면 연결된 것!
}
```

```
예시: isConnected(graph, 0, 2, 7)
  → dfs(0) → {0, 1, 2} 방문
  → visited[2] = true → 연결됨! ✅

예시: isConnected(graph, 0, 4, 7)
  → dfs(0) → {0, 1, 2} 방문
  → visited[4] = false → 연결 안 됨! ❌
```

---

### 3가지 유형 정리

```
연결된 영역 문제 = BFS/DFS의 가장 기본적인 활용!

유형 1. 네트워크 개수  → 모든 노드 순회, 미방문이면 DFS + count++
유형 2. 영역의 크기    → DFS 안에서 방문 노드 수 카운트
유형 3. 연결 여부      → A에서 DFS 후, visited[B] 확인

공통 패턴:
  1. 인접 리스트 만들기
  2. visited 배열 선언
  3. BFS/DFS 실행
  4. visited 상태로 결과 판단
```

> 💡 **이 유형들이 중요한 이유**: 그래프 문제의 절반 이상이 "연결된 영역"의 변형이다. 이 패턴만 확실히 익혀도 많은 문제를 풀 수 있다!

### 최단 거리 문제 유형 (BFS)

"A에서 B까지 최소 몇 칸?" 같은 문제는 **BFS의 또 다른 대표 유형**이다.

#### 왜 BFS가 최단 거리를 보장하는가?

```
BFS는 Level 순서대로 방문한다:
  Level 0 (거리 0) → Level 1 (거리 1) → Level 2 (거리 2) → ...

→ 어떤 노드를 처음 방문했을 때의 거리 = 최단 거리!
  (비가중치 그래프에서만! 가중치가 있으면 다익스트라 필요)
```

#### 핵심 아이디어: 큐에 {노드, 거리}를 함께 저장

기존 BFS에서 큐에 노드만 넣었다면, 최단 거리 문제에서는 **{노드, 거리}를 함께** 넣는다.

```
기존 BFS:     큐에 넣는 것 = 노드 번호 (int)
최단 거리 BFS: 큐에 넣는 것 = {노드 번호, 거리} (int[])
```

#### 코드

```java
int bfs(List<List<Integer>> graph, int start, int end, boolean[] visited) {
    // 1. 시작 노드 예약 — {노드, 거리}
    Queue<int[]> queue = new ArrayDeque<>();
    queue.offer(new int[]{start, 0});   // 시작점 거리 = 0
    visited[start] = true;

    while (!queue.isEmpty()) {
        // 2. 방문
        int[] cur = queue.poll();
        //  cur[0] = 현재 노드
        //  cur[1] = 시작점에서 현재 노드까지의 거리

        // 도착했으면 거리 리턴
        if (cur[0] == end) {
            return cur[1];
        }

        // 3. 다음 노드 예약
        for (int next : graph.get(cur[0])) {
            if (!visited[next]) {
                queue.offer(new int[]{next, cur[1] + 1});  // 거리 +1
                visited[next] = true;
            }
        }
    }

    return -1;  // 도달 불가
}
```

#### 기존 BFS와 달라진 점

```
기존 BFS                          최단 거리 BFS
─────────────────────────────────────────────────────────
Queue<Integer>                    Queue<int[]>
queue.offer(start)                queue.offer(new int[]{start, 0})
int cur = queue.poll()            int[] cur = queue.poll()
graph.get(cur)                    graph.get(cur[0])
queue.offer(next)                 queue.offer(new int[]{next, cur[1] + 1})

→ 3단계 구조(예약→방문→다음 예약)는 완전히 동일!
→ 큐에 넣는 데이터에 "거리"만 추가한 것
```

#### 큐 상태로 따라가보기

```
graph: 0↔1, 0↔2, 1↔3, 2↔4, 3↔4
start=0, end=4

[1단계: 시작 노드 예약]
  큐: [{0,0}]         visited: {0}

[2단계: 방문] cur = {0, 0}  →  end(4)가 아님
[3단계: 예약] 0의 이웃(1,2)
  큐: [{1,1}, {2,1}]  visited: {0, 1, 2}

[2단계: 방문] cur = {1, 1}  →  end(4)가 아님
[3단계: 예약] 1의 이웃(3)
  큐: [{2,1}, {3,2}]  visited: {0, 1, 2, 3}

[2단계: 방문] cur = {2, 1}  →  end(4)가 아님
[3단계: 예약] 2의 이웃(4)
  큐: [{3,2}, {4,2}]  visited: {0, 1, 2, 3, 4}

[2단계: 방문] cur = {3, 2}  →  end(4)가 아님
  큐: [{4,2}]

[2단계: 방문] cur = {4, 2}  →  end(4)와 같음! → return 2

→ 결과: 0에서 4까지 최단 거리 = 2
```

#### 전체 흐름 예시 (간선 리스트 → BFS 최단 거리)

```java
// ✅ 코테 전형적인 최단 거리 문제 흐름
int n = 5;
int[][] edges = {{0,1}, {0,2}, {1,3}, {2,4}, {3,4}};

// STEP 1: 인접 리스트 변환
List<List<Integer>> graph = new ArrayList<>();
for (int i = 0; i < n; i++) {
    graph.add(new ArrayList<>());
}
for (int[] edge : edges) {
    graph.get(edge[0]).add(edge[1]);
    graph.get(edge[1]).add(edge[0]);  // 무방향(양방향)
}

// STEP 2: BFS 최단 거리
boolean[] visited = new boolean[n];
int dist = bfs(graph, 0, 4, visited);
System.out.println("최단 거리: " + dist);  // 출력: 2
```

> 💡 **판단 기준**: 문제에서 "최소", "최단", "가장 빠른"이 보이고 **가중치가 없으면** → BFS + `int[]{노드, 거리}` 패턴으로 바로 풀 수 있다!

## 12. 실전 팁 & 자주 하는 실수

### 그래프 문제 풀이 순서

```
1. 그래프 유형 파악
   → 방향(단방향)/무방향(양방향)? 가중치 있음/없음?

2. 구현 방법 결정
   → 인접 리스트 (대부분의 경우)

3. 탐색 방법 결정
   → "최단 거리" → BFS  /  그 외 → DFS

4. 코드 작성
   → visited 배열 선언 → 탐색 함수 작성
```

### 자주 하는 실수

| 실수 | 결과 | 해결 |
|------|------|------|
| visited 배열 안 만듦 | 무한 루프 | 반드시 `boolean[] visited` 선언 |
| 무방향(양방향)인데 한쪽만 간선 추가 | 노드 누락 | `adj.get(a).add(b); adj.get(b).add(a);` 양쪽! |
| BFS에서 꺼낼 때 방문 표시 | 같은 노드 중복 방문 | **넣을 때** `visited = true` |
| 0-indexed / 1-indexed 혼동 | 오답 | 문제 확인 후 배열 크기 `N+1`로 맞추기 |
| 인접 리스트 초기화 안 함 | NullPointerException | for문으로 `new ArrayList<>()` 넣기 |

### 코딩테스트 BFS 템플릿

```java
// 그래프 BFS 기본 틀 (3단계: 예약 → 방문 → 다음 예약)
void bfs(List<List<Integer>> graph, int start, boolean[] visited) {
    // 1. 시작 노드 예약
    Queue<Integer> queue = new ArrayDeque<>();
    queue.offer(start);
    visited[start] = true;

    while (!queue.isEmpty()) {
        // 2. 방문
        int cur = queue.poll();
        // 여기서 cur 처리

        // 3. 다음 노드 예약
        for (int next : graph.get(cur)) {
            if (!visited[next]) {
                queue.offer(next);
                visited[next] = true;
            }
        }
    }
}
```

### 코딩테스트 DFS 템플릿

```java
// 그래프 DFS 기본 틀 (재귀)
void dfs(List<List<Integer>> graph, int cur, boolean[] visited) {
    // 1. 방문
    visited[cur] = true;
    // 여기서 cur 처리

    // 2. 방문안한 다음 노드 찾기
    for (int next : graph.get(cur)) {
        if (!visited[next]) {
            dfs(graph, next, visited);
        }
    }
}
```

### 📋 QUIZ 4 — 이 코드의 문제점은?

```java
void bfs(List<List<Integer>> adj, int start) {
    Queue<Integer> queue = new LinkedList<>();
    queue.offer(start);

    while (!queue.isEmpty()) {
        int node = queue.poll();
        System.out.print(node + " ");

        for (int neighbor : adj.get(node)) {
            queue.offer(neighbor);
        }
    }
}
```

---

<details>
<summary><b>정답 보기 (클릭)</b></summary>

### 문제점 2가지

**1. visited 배열이 없다!**

```java
// ❌ visited가 없으면 같은 노드를 계속 방문 → 무한 루프!
// 사이클이 있으면: 1 → 2 → 1 → 2 → 1 → 2 → ...

// ✅ 수정
boolean[] visited = new boolean[adj.size()];
visited[start] = true;   // ← 추가
```

**2. 큐에 넣을 때 방문 체크를 안 한다!**

```java
// ❌ 모든 이웃을 무조건 큐에 넣음
for (int neighbor : adj.get(node)) {
    queue.offer(neighbor);
}

// ✅ 미방문 이웃만 넣어야 함
for (int neighbor : adj.get(node)) {
    if (!visited[neighbor]) {
        visited[neighbor] = true;
        queue.offer(neighbor);
    }
}
```

> 💡 **교훈**: BFS/DFS에서 visited는 선택이 아니라 **필수**다!

</details>

## 📝 Part 4 핵심 정리

```
1. 그래프 = 정점(노드) + 간선(엣지)으로 연결 관계를 표현
2. 무방향(양방향) = 양쪽 추가, 방향(단방향) = 한쪽만 추가 (실수 주의!)
3. 인접 행렬 = O(V²) 공간, O(1) 연결 확인 → 노드 적을 때
4. 인접 리스트 = O(V+E) 공간, 효율적 → 코딩테스트 기본!
5. 간선 리스트(입력) → 인접 리스트(탐색용)로 변환하는 것이 코테 첫 단계
6. visited 배열은 BFS/DFS 모두 필수! 없으면 무한 루프
7. BFS = 큐, 가까운 것부터, 최단 거리 보장 (비가중치)
8. DFS = 재귀, 깊은 것부터, 경로 탐색에 유리
9. BFS 방문 표시 = 큐에 "넣을 때" / DFS 방문 표시 = 함수 "진입 시"
10. 연결된 영역 = 네트워크 개수, 영역 크기, 연결 여부 (BFS/DFS 대표 유형)
11. 최단 거리 = BFS + int[]{노드, 거리} 패턴 (비가중치 그래프)
12. "최단" → BFS / 그 외 → DFS가 코드 간결
```
