  • 디자인, 기획, 개발을 모두 경험해 서비스 전체를 함께 보는 시각이 있습니다.
  • 사용자 흐름, 운영 구조, 실제 구현 방식을 분리해서 보지 않고 하나의 제품 경험으로 연결하려고 합니다.
  • 복잡한 요구사항도 기능 단위로만 보지 않고, 무엇이 먼저 필요한지 정리해 실행 가능한 구조로 바꾸는 데 익숙합니다.
  • 그래서 새로운 서비스를 만들 때도 아이디어 수준에 머무르지 않고, 빠르게 프로토타입과 MVP로 구체화할 수 있습니다.

Execution

  • 리소스가 부족하거나 역할이 명확히 나뉘지 않은 환경에서도 제품을 끝까지 만들어내는 편입니다.
  • 프론트엔드, 백엔드, 모바일, 디자인, 기획 중 꼭 내 전문 영역이 아니더라도 필요하다면 적극적으로 탐색하고 구현합니다.
  • “이가 없으면 잇몸으로”라는 표현처럼, 제한된 조건 안에서도 대안을 찾고 결국 결과물을 만드는 실행력을 중요하게 생각합니다.
  • 작게 만들고 빠르게 확인하면서, 실제로 동작하는 결과로 다음 판단을 이어가는 방식에 익숙합니다.

0 to 1

  • 기존 기능을 유지보수하는 것도 강점이지만, 아직 형태가 없는 서비스를 처음부터 구조화하고 만들어가는 과정에 더 강점이 있습니다.
  • MVP 범위를 정하고, 핵심 사용자 흐름을 먼저 만들고, 이후 검증하면서 확장하는 방식으로 많이 일해왔습니다.
  • 그래서 새로운 서비스나 프로토타입, 초기 제품 개발 환경에서 빠르게 방향을 잡고 실행하는 데 자신이 있습니다.
  • 처음부터 완벽한 구조를 고집하기보다, 지금 필요한 수준의 구조를 만들고 실제 사용을 통해 발전시키는 방식을 선호합니다.

Vibe Coding

  • Claude Code, Codex 같은 AI 코딩 도구를 활용한 개발에 익숙합니다.
  • 도구를 100% 신뢰하거나 의존하기보다, 빠른 초안 작성과 반복 속도를 높이는 보조 수단으로 활용합니다.
  • 생성된 결과는 직접 검증하고 수정하며, 최종 구조와 품질은 스스로 판단하는 방식으로 개발합니다.
  • 그래서 프로토타입, MVP, 실험성 기능 개발에서 특히 빠르게 결과를 만드는 데 강점이 있습니다.

Communication

  • 직무나 포지션에 관계없이 필요한 의견은 분명하게 공유하는 편입니다.
  • 디자인, 기획, 개발을 모두 경험했기 때문에 서로 다른 관점을 번역하듯 연결하며 대화할 수 있습니다.
  • 단순히 요청을 전달받는 역할보다, 왜 필요한지와 어떻게 구현할지를 함께 맞추는 커뮤니케이션을 중요하게 생각합니다.
  • 좋은 결과를 위해서는 커뮤니케이션이 많아야 한다고 믿고, 혼자 판단하기보다 맥락을 열어두고 협업하는 방식을 선호합니다.

Design Background

  • 디자이너 출신이라 화면의 완성도와 사용자 흐름에 자연스럽게 민감한 편입니다.
  • 기능이 많아도 사용자가 헷갈리면 좋은 제품이 아니라고 생각하기 때문에, 구현 단계에서도 흐름과 인터랙션을 계속 점검합니다.
  • 작은 애니메이션이나 전환 디테일이 서비스 인상을 크게 바꾼다고 믿으며, 이런 감각을 개발 과정 안에서도 살리려고 합니다.
  • 그래서 개발자로 일할 때도 단순 구현보다, 실제 사용 경험이 어떻게 느껴질지까지 함께 보게 됩니다.

Mobile

  • iOS를 중심으로 모바일 개발을 해왔습니다.
  • 서비스 초기 구조 설계부터 핵심 화면 구현, 출시와 운영까지 iOS 앱 개발 전반에 익숙합니다.
  • Android와 Flutter도 프로젝트에 따라 직접 구현해왔고, 필요한 수준의 앱 기능을 개발하고 서비스 흐름을 연결하는 데 익숙합니다.
  • 모바일 환경에서 사용자 흐름, 화면 구조, 상태 처리, 인터랙션을 함께 고려하며 개발하는 편입니다.

Web

  • Next.js를 사용해 어드민 페이지와 랜딩 페이지를 직접 개발하고 배포한 경험들이 있습니다.
  • 서비스 운영에 필요한 관리용 화면이나 빠르게 검증해야 하는 웹 페이지를 구현하는 데 익숙합니다.
  • 제품 구조상 모바일 앱만으로 해결되지 않는 영역은 웹으로 함께 연결해 서비스 전체 흐름을 완성하는 방식으로 자주 작업해왔습니다.

Backend

  • AWS의 주요 서비스들을 활용해 서비스 운영에 필요한 기본적인 백엔드 환경을 구성할 수 있습니다.
  • Node.js, FastAPI를 사용해 API 서버를 만들고, MySQL, PostgreSQL을 활용한 데이터 구조 설계와 저장 흐름 구현 경험이 있습니다.
  • 모바일이나 웹 개발을 하면서 필요한 범위의 서버 작업을 직접 연결할 수 있으며, 초기 서비스나 MVP 단계에서 백엔드까지 함께 구축하는 데 익숙합니다.
