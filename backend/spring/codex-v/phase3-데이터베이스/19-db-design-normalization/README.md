# 19. DB 모델링과 정규화

> 오늘의 목표: 테이블을 그냥 만드는 것이 아니라, 어떤 기준으로 나누고 연결해야 하는지 설계 감각을 익힌다.

---

## 오늘 끝나면 되는 것

- ERD가 왜 필요한지 설명할 수 있다.
- 엔티티와 관계를 식별할 수 있다.
- 1NF, 2NF, 3NF의 목적을 이해할 수 있다.
- 정규화가 왜 중복과 이상 현상을 줄이는지 설명할 수 있다.
- 반정규화는 "나쁜 것"이 아니라 상황에 따라 선택하는 것임을 안다.

---

## 머릿속 그림

DB 설계는 "코드 쓰기 전 구조 설계"입니다.

```text
요구사항
 -> 어떤 데이터가 필요한가?
 -> 어떤 것들이 서로 연결되는가?
 -> 어떤 테이블로 나눌 것인가?
 -> 어떤 컬럼이 PK/FK가 되는가?
```

이 작업을 잘해야 나중에 JPA Entity도 깔끔해집니다.

---

## ERD는 왜 먼저 그릴까

건물을 짓기 전에 설계도가 필요하듯,
DB를 만들기 전에 ERD가 필요합니다.

예: 학습 일지 서비스

```text
Member 1 --- N StudyLog
StudyLog N --- N Tag
```

여기서 생각해야 할 것:

- 회원은 여러 학습 일지를 가질 수 있다
- 학습 일지는 여러 태그를 가질 수 있다

즉, "무엇이 있고", "어떻게 연결되는가"를 먼저 그림으로 정리해야 합니다.

---

## 엔티티 찾는 법

요구사항에서 명사를 먼저 뽑아보면 도움이 됩니다.

예:

- 회원
- 학습 일지
- 댓글
- 태그

이 후보 중에서 실제로 관리해야 할 데이터가 엔티티가 됩니다.

---

## 나쁜 설계 예시

모든 걸 한 테이블에 넣으면 처음엔 쉬워 보입니다.

```text
order_id | product_name | customer_name | customer_email | customer_address
```

문제:

- 고객 정보가 계속 중복됨
- 이메일 변경 시 여러 행 수정 필요
- 일부 행만 수정하면 데이터 불일치 발생

이걸 이상 현상이라고 합니다.

- 삽입 이상
- 수정 이상
- 삭제 이상

---

## 정규화의 목적

정규화는 어려운 수학이 아니라,

> "중복을 줄이고, 데이터가 꼬이지 않게 테이블을 나누는 작업"

이라고 이해하면 됩니다.

---

## 1NF, 2NF, 3NF 아주 쉽게

### 1NF

한 칸에는 하나의 값만 넣는다.

나쁜 예:

```text
tags = "spring,jpa,database"
```

좋은 예:

- 태그를 별도 테이블로 분리
- 또는 중간 테이블 사용

### 2NF

복합키 일부에만 종속되는 컬럼을 분리한다.

초급 단계에서는:

> "키 전체가 아니라 일부에만 의존하는 컬럼이 있으면 분리"

정도로 이해해도 괜찮습니다.

### 3NF

기본키가 아닌 컬럼이 또 다른 일반 컬럼을 결정하면 분리합니다.

예:

```text
member_id -> zipcode -> city
```

`city`가 `zipcode`로 결정된다면 분리 여지가 있습니다.

---

## 관계와 카디널리티

### 1:1

한 회원 - 한 프로필

### 1:N

한 회원 - 여러 학습 일지

### N:M

여러 학습 일지 - 여러 태그

N:M은 보통 바로 테이블로 두지 않고,
중간 테이블로 풀어냅니다.

```text
study_log_tag
- study_log_id
- tag_id
```

---

## 반정규화는 언제 하나

정규화가 항상 끝은 아닙니다.

조회 성능 때문에 일부 중복을 감수하고 컬럼을 추가하기도 합니다.

예:

- 주문 총액 저장
- 게시글 댓글 수 캐싱

핵심은:

- 정규화가 기본
- 성능상 필요가 검증되면 반정규화 검토

---

## 실무 포인트

- 테이블 이름, 컬럼 이름 규칙을 초기에 정하기
- PK는 단순하고 일관되게
- FK로 관계를 명확히 드러내기
- "당장 편해 보이는 중복"을 경계하기

---

## 자주 하는 실수

- 엔티티를 못 찾고 기능명으로 테이블을 만드는 것
- 배열/콤마 문자열을 한 칸에 넣는 것
- N:M을 바로 표현하려는 것
- 정규화 이유 없이 중복 컬럼을 남기는 것
