# AI 시대 핵심 용어 정리

---

## 1. X(Transformation) 시리즈 — "무엇을 바꾸는가"

X = Transformation(전환, 변혁)의 약자. 어떤 분야를 근본적으로 바꾸겠다는 뜻.

| 용어 | 풀이 | 핵심 | 예시 |
|------|------|------|------|
| **DX** | Digital Transformation | 아날로그 → 디지털로 업무/서비스 전환 | 종이 문서 → 전자 결재, 오프라인 매장 → 이커머스 |
| **AX** | AI Transformation | AI를 중심에 놓고 비즈니스 자체를 재설계 | 고객센터를 AI 챗봇 중심으로 재편, AI 기반 수요 예측 |
| **CX** | Customer Experience (Transformation) | 고객 경험 전체를 혁신 | 개인화 추천, AI 챗봇 응대, UX 개선 |
| **GX** | Green Transformation | 탄소 중립·친환경으로 전환 | AI로 에너지 사용 최적화, ESG 경영 |

> **DX vs AX 차이**
> - **DX**: "종이를 없애고 컴퓨터로 하자" (도구를 디지털로 교체)
> - **AX**: "AI가 판단·실행까지 하게 하자" (의사결정 자체를 AI에 위임)
> - DX가 1단계라면, AX는 그 위에 올라가는 2단계. DX 없이 AX는 불가능.

## 2. AI 활용 패러다임의 진화

| 단계 | 이름 | 핵심 | 비유 |
|------|------|------|------|
| 1단계 | **Prompt Engineering** | 질문을 잘 하는 법 | "피자 주문을 정확하게" |
| 2단계 | **Context Engineering** | 맥락(배경정보)을 잘 주는 법 | "내 알레르기, 취향까지 알려주기" |
| 3단계 | **Harness Engineering** | AI에게 도구·규칙·환경을 세팅해서 자율적으로 일하게 만드는 법 | "셰프에게 주방 세팅해주기" |

## 3. AI 에이전트 관련 용어

### 에이전트 (Agent)
- **스스로 판단하고 행동하는 AI**. 채팅 AI는 물으면 답하지만, 에이전트는 목표를 주면 알아서 계획→실행→검증까지 한다.
- 예: Claude Code, ChatGPT Operator, Devin

### 서브에이전트 (Sub-agent)
- 메인 에이전트가 **하위 작업을 위임하는 보조 에이전트**
- 메인이 지시 → 서브가 실행 → 결과를 메인에게 반환
- 서브에이전트끼리는 직접 소통 불가

### 에이전트 팀 (Agent Team)
- 여러 에이전트가 **서로 대화하며 협업**하는 구조
- 리드 에이전트가 작업 분배, 팀원 에이전트끼리 직접 의견 교환 가능
- 아직 실험적 기능 (2026년 기준)

### 에이전틱 루프 (Agentic Loop)
- 에이전트가 **계획→실행→검증→수정을 스스로 반복**하는 순환 구조
- Claude Code의 `/loop` 명령으로 활성화 (최대 3일)

## 4. AI 아키텍처 용어

### PGE 구조 (Plan → Generate → Evaluate)
- Anthropic이 제안한 3-에이전트 파이프라인
- **Planner**: 작업 분석 및 계획 수립
- **Generator**: 실제 코드/결과물 생성
- **Evaluator**: 결과물 검증 및 피드백
- 이 3단계가 반복되면서 품질이 올라감

### RAG (Retrieval-Augmented Generation)
- AI가 답변할 때 **외부 문서/DB를 검색해서 참고**하는 방식
- LLM의 환각(Hallucination)을 줄이는 핵심 기법
- 예: 사내 문서를 검색해서 정확한 답변 생성

### Fine-tuning (파인튜닝)
- 이미 학습된 AI 모델을 **특정 분야 데이터로 추가 학습**시키는 것
- 비유: 의대 졸업한 의사(기본 모델)가 피부과 전문의 수련(파인튜닝)을 받는 것

### 토큰 (Token)
- AI가 텍스트를 처리하는 **최소 단위**
- 대략 한글 1글자 ≈ 2~3 토큰, 영어 1단어 ≈ 1~2 토큰
- 토큰이 많을수록 비용 ↑, 처리 시간 ↑

### 컨텍스트 윈도우 (Context Window)
- AI가 **한 번에 읽을 수 있는 텍스트의 최대 길이** (토큰 수로 측정)
- Claude: 200K 토큰 (≈ 한국어 약 7~10만 자)
- 윈도우를 넘으면 앞부분부터 잊어버림 → `/compact`로 정리 필요

### 환각 (Hallucination)
- AI가 **없는 사실을 그럴듯하게 지어내는 현상**
- 예: 존재하지 않는 논문을 인용, 가짜 API 만들어내기
- 대응: RAG, 검증 단계 추가, Evaluator 활용

## 5. 연결·확장 관련 용어

### API (Application Programming Interface)
- 서비스 간 **요청을 주고받는 규격화된 통로**
- 비유: 식당의 주문서. 정해진 양식으로 요청하면 결과를 돌려줌
- API Key: 이 통로를 쓸 수 있는 **디지털 출입증**

### MCP (Model Context Protocol)
- AI가 **외부 도구를 표준화된 방식으로 연결**하는 프로토콜
- 비유: USB-C처럼 하나의 규격으로 여러 기기 연결
- 노션, 슬랙, GitHub 등을 같은 방식으로 붙일 수 있게 해줌
- API와의 차이: API는 개별 연결, MCP는 통합 규격

### Webhook
- 이벤트가 발생하면 **자동으로 알려주는 메커니즘**
- 비유: "피자 나오면 전화해줘" (내가 계속 확인하지 않아도 됨)
- API는 내가 물어봐야 답하지만, Webhook은 알아서 알려줌

### SDK (Software Development Kit)
- 특정 서비스를 **쉽게 쓸 수 있도록 묶어놓은 도구 모음**
- 비유: 이케아 가구 조립 키트 (설명서 + 부품 + 공구)

## 6. Claude Code 생태계 용어

| 용어 | 정의 | 비유 |
|------|------|------|
| **CLAUDE.md** | 프로젝트 규칙·맥락을 적은 설정 파일. 매 세션마다 자동으로 읽힘 | 알림장, 헌법 |
| **Skill** | 반복 작업의 절차를 적은 `.md` 매뉴얼 | 레시피북 |
| **Hook** | 특정 동작 전/후에 자동 실행되는 트리거 | 집 자동화 (현관문 열면 불 켜짐) |
| **Plugin** | MCP + Skill + Agent 등을 통합한 확장 패키지 | 에스프레소 머신 추가 장착 |
| **Command** | 자주 쓰는 요청을 `/명령어`로 저장한 단축 명령 | 음성비서의 "잠자기 모드" |

## 7. 개발·배포 관련 용어

| 용어 | 정의 | 비유 |
|------|------|------|
| **Git** | 코드 변경 이력을 추적하는 **버전 관리 시스템** | 문서의 "변경 내용 추적" |
| **GitHub** | Git 기반 **코드 저장소 + 협업 플랫폼** | Google Drive for 코드 |
| **PR (Pull Request)** | 내 코드를 메인에 합쳐달라는 **검토 요청** | "이거 확인하고 합쳐줘" |
| **CI/CD** | 코드 → 테스트 → 배포를 **자동화**하는 파이프라인 | 공장 컨베이어 벨트 |
| **Vercel** | 웹사이트를 인터넷에 **자동 배포**해주는 서비스 | 인쇄소 (원고 → 출판) |
| **npm** | Node.js 패키지(프로그램)를 **설치·관리**하는 도구 | 앱스토어 |
| **PRD** | Product Requirements Document. **뭘 만들 것인가** 정의하는 문서 | 건축 설계도의 요구사항서 |

## 8. 프롬프트·대화 관련 용어

| 용어 | 정의 |
|------|------|
| **프롬프트 (Prompt)** | AI에게 보내는 입력 텍스트. 질문, 지시, 맥락 모두 포함 |
| **시스템 프롬프트** | AI의 역할·규칙을 미리 설정하는 숨겨진 지시문 (사용자에겐 안 보임) |
| **Few-shot** | 예시를 몇 개 보여주고 "이런 식으로 해줘"라고 가르치는 기법 |
| **Zero-shot** | 예시 없이 바로 지시하는 방식 |
| **Chain of Thought (CoT)** | "단계별로 생각해봐"라고 시켜서 추론 정확도를 높이는 기법 |
| **Temperature** | AI 답변의 창의성 조절값. 낮으면 정확, 높으면 창의적 (0.0~1.0) |

## 9. 비즈니스·전략 용어

| 용어 | 정의 |
|------|------|
| **SaaS** | Software as a Service. 설치 없이 **웹에서 쓰는 소프트웨어** (예: Notion, Slack) |
| **B2B / B2C** | Business to Business / Business to Consumer. 기업 대상 / 소비자 대상 |
| **PMF** | Product-Market Fit. 제품이 시장 수요에 **딱 맞는 상태** |
| **MVP** | Minimum Viable Product. 핵심 기능만 넣은 **최소 실행 가능 제품** |
| **ROI** | Return on Investment. **투자 대비 수익률** |
| **ESG** | Environment, Social, Governance. 환경·사회·지배구조 책임 경영 |
| **CTA** | Call to Action. 사용자에게 **행동을 유도하는 요소** ("구매하기" 버튼 등) |

## 10. 자주 헷갈리는 개념 비교

### AI vs ML vs DL
```
AI (인공지능)
 └─ ML (머신러닝) — 데이터에서 패턴을 학습
      └─ DL (딥러닝) — 신경망으로 학습 (GPT, Claude 등)
```
- AI가 가장 넓은 개념, ML은 그 안의 한 방법, DL은 ML의 한 종류

### LLM vs SLM
| | LLM (Large Language Model) | SLM (Small Language Model) |
|---|---|---|
| 크기 | 수십~수천억 파라미터 | 수억~수십억 파라미터 |
| 예시 | GPT-4, Claude, Gemini | Phi-3, Gemma, Llama 3 8B |
| 장점 | 범용 능력 우수 | 가볍고 빠르고 저렴 |
| 용도 | 복잡한 추론, 코드 생성 | 특정 업무, 온디바이스(스마트폰) |

### 온프레미스 vs 클라우드 vs 로컬
| | 온프레미스 | 클라우드 | 로컬 |
|---|---|---|---|
| 위치 | 우리 회사 서버실 | AWS, Azure 등 남의 서버 | 내 컴퓨터 |
| 비유 | 자가 주택 | 임대 아파트 | 내 방 |
| AI 예시 | 사내 LLM 서버 | ChatGPT API | Claude Code (Local 모드) |
